0.8.3 Slice-20 CE-rerank TUNING probe (`eval/rerank_tune_probe.py`, artifact
`dev/plans/runs/0.8.3-rerank-tune.json` + `-FINDINGS.md`, $0/n=606) found:

- The production blend is `0.3·ce_norm + 0.7·minmax(base)`. **α is the dominant rerank knob.**
  Raising α concentrates gold into TOP ranks (pool_n=50: r@1 0.036→0.140 ×3.9,
  MRR 0.347→0.589 at α=1.0) while **Recall@10/20 stay flat** — the exact gap-decomp signature
  (recall@K blind to reorder; answerer sees the cleaner top). See [[0.8.3-ce-rerank-accuracy-pass-marginal]].
- **✅ UPDATE (verified 2026-06-26): α/`pool_n` are NO LONGER hardcoded — the knob LANDED in the
  engine** (0.8.5 EXP-0/D4). Exposed on BOTH bindings: `Engine.search(..., alpha, pool_n)` and
  standalone `rerank(query, passages, rerank_depth, alpha, pool_n)` (→ `ce_score` out); impl
  `fathomdb-engine::search_reranked` (`src/rust/crates/fathomdb-engine/src/lib.rs:2901`) +
  `rerank_fused`/`rerank_passages` (`:4882`/`:4938`), α clamped `[0,1]` (`:5003`), wired via
  `fathomdb-py` + `fathomdb-napi`. **BUT the DEFAULT is still α=0.3** → shipping the Mem0-parity
  config (`α=1.0, pool_n=10`) is now a **default-value PRODUCT decision, not an engineering task**
  (change the default vs keep caller-opt-in). Do NOT re-flag this as "hardcoded 0.3 / not built".
  Inventory restated in `dev/design/0.8.x-parity-portfolio-strategy.md` §3.
- Tradeoff: pure CE (α=1.0) at a WIDE pool drops r@10 (0.548→0.498 at pn=50) — CE-confident
  distractors displace gold. Fixed by a NARROW pool: **α=1.0, pool_n=10** preserves r@10 (0.540)
  AND maximizes concentration (MRR 0.587, full-gold-rank 6.97 vs 19.7 prod). Best balanced config.
- **Cheap-extraction trick (reusable):** call `fathomdb.rerank` with every passage `score=0.0`
  ⇒ rrf_norm=1 ⇒ blended=0.3·ce+0.7 ⇒ recover pure `ce_norm=(blended-0.7)/0.3`; then re-blend at
  ANY α/pool_n offline — sweep the hardcoded-α reranker with NO Rust rebuild.
