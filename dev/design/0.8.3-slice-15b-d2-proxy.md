# 0.8.3 Slice-15b — D2 content-at-scale proxy (design + TDD contract)

> **Orchestrator draft for implementer (TDD) + codex review.** Grounds the §5-frozen 15b probe into an
> executable, deterministic, $0 measurement. Gate semantics are frozen in
> `decision_rule_083.probe_15b_pass` — this slice CONSUMES it and must not reimplement or relax it.

## 1. Purpose, gate, and the D0b reprioritization

**Purpose.** Decide — cheaply, before any schema migration — whether **fielded enrichment content**
(entity/alias/fact/temporal keys appended to a doc's FTS content) carries *retrieval* value at scale,
separated from the length artifact. **Three-way verdict (codex Q6 — INCONCLUSIVE overrides every
"FAIL⇒defer" elsewhere in this doc):**

- **PASS** (`probe_15b_pass` True) ⇒ Slice 25 (D2 engine) is **eligible**.
- **FAIL at power** (`probe_15b_pass` False AND 15b paired MDE ≤ `δ_d2`) ⇒ Slice 25 **defers/drops**
  (record the negative; no migration on a validated-negative lever).
- **INCONCLUSIVE** (15b paired MDE `> δ_d2`, i.e. underpowered to detect a migration-worthy lift) ⇒ **do
  NOT drop D2**; escalate (mirror `decide_083`'s underpowered handling).

**Frozen pass criterion (`probe_15b_pass`, design §5).** Enrichment *passes* iff BOTH:

1. `margin_ci_lo > 0` — paired (enriched − length-matched placebo) recall margin, 95% bootstrap CI
   **lower bound > 0** (content beats placebo *at power*, not just on the point); AND
2. `removes_length_norm_penalty == True` — fielding/`b`-tuning removes the FTS length-norm penalty the
   placebo exposes.
The harness builds `enriched = {recall, margin_ci_lo, removes_length_norm_penalty}` and
`placebo = {recall}` and calls `probe_15b_pass(enriched, placebo)`. **No raw-point gate** (mirrors the
Slice-0 fix-1 discipline — CI lower bound only).

**Eligibility framing (codex review BLOCK #6 — corrected).** A **15b pass is NECESSARY for D2 / Slice-25
*eligibility*, NOT necessary for 0.8.3 parity** (Slice 20 could close the gap; D2 is conditional on 15b
passing AND a *remaining* Mem0 gap — prereg `mem0-parity.md` §6). 15b decides only whether the D2-engine
lever is worth building.

**D0b reprioritization (un-landed; cite with provenance).** The D0b priced run (n606) — **observed in
the un-landed phaseB output, `runs/0.8.3-d0b-findings-observed-by-s15a-20260622.md`; pending primary
landing** — shows FathomDB trailing Mem0 by a large margin with the **accuracy gap ≫ the recall gap**,
pointing at **answer-formation from raw bodies**, not only retrieval recall. (Do NOT cite a specific pp
number here until D0b is landed + reported on main.) Implication: 15b sizes a *retrieval-recall* lever,
so even a 15b pass may not move parity if the gap is answer-formation — the orchestrator runs the **$0
gap-decomposition probe** alongside 15b to size retrieval-vs-answer-formation before over-investing.
This split is recorded in the output.

## 2. Method (reuse `r6_index_key_enrichment.py`, extend to the frozen probe)

- **Corpus/arms:** the LME retrieval corpus + the three FTS arms r6 already builds — `fathomdb_fts`
  (plain), `fathomdb_fts_enriched` (entity/fact keys appended to each doc's own FTS content — one row
  per doc, NOT a graph arm), `fathomdb_fts_placebo` (length-matched, **foreign-only** vocab, excludes
  the doc's own entity tokens; r6 `placebo_doc`). All LLM-free at measure time, deterministic.
- **Enrichment keys (OFFLINE-BUILD, $0) — pin the endpoint, fail-closed (codex concern #5):** reuse
  cached 0.8.2 extractions where applicable
  (`data/corpus-data/graph-cache/0.8.2-m1-v1/extractions.json`); extract LME (and, if cheaply derivable,
  the IR corpus) session keys with the **local Qwen** model. **Pin the exact local endpoint + served
  model name in the output, and assert-available-or-FAIL-CLOSED** — never silently fall back to a priced
  reader (that would break $0/determinism). GPU only at **<5% util after sibling coordination** (user
  rule). LME-only proxy is primary; IR-corpus enrichment is a reserved follow-on.
- **Key-set ↔ placebo invariant (codex concern #1):** the enriched arm uses a **frozen key set**
  (r6's `entities` + `relations`/facts; if alias/temporal keys are added they are listed explicitly).
  The length-matched placebo MUST be token-matched to the **full** enriched addition (whatever key types
  are included) and foreign-only/own-entity-excluded — extended, not just asserted in prose; a RED test
  pins the match for every key type used.
- **"At scale":** run on the **power-sized** LME gold (per-class N from Slice 5; reuse the same
  re-pinned gold so the proxy is comparable), not the n=10 smoke.
- **Metrics:** strict Recall@K per class + pooled; the **paired (enriched − placebo)** per-query margin
  with a fixed-seed bootstrap CI (same helper/shape as Slice 15a's `paired_bootstrap_ci`); **also emit
  the 15b paired MDE** for the enriched−placebo delta (codex concern #4).
- **`removes_length_norm_penalty` — PINNED, post-selection-free, length-isolated (codex BLOCK #2 v2):**
  The flaw in v1 was argmax-`b*` post-selection + that "enriched@b* > enriched@0.75" could be generic
  BM25 miscalibration, not an *enrichment length* effect. v2 isolates the **length-norm penalty using the
  placebo itself** (the placebo is *pure added length, zero content* by construction, so any placebo
  deficit IS length-norm), at a **pre-registered** lower-`b` (no selection):
  - **b is pre-registered, not selected:** `b_hi = 0.75` (production), `b_lo = 0.00` (length-norm OFF).
    Report `b=0.25` as a secondary point only (non-gating).
  - **penalty present** iff `(plain_recall − placebo_recall)` paired CI_lo `> 0` at `b_hi` (pure added
    length significantly hurts recall = the length-norm penalty exists).
  - **penalty neutralized** — *equivalence, not absence-of-evidence (codex round-3 Q2):* the residual
    pure-length effect at `b_lo` must be demonstrably **small**, not merely non-significant. Pinned TOST-style:
    neutralized iff the **paired CI of `(plain_recall − placebo_recall)@b_lo` is fully contained within
    `[−δ_d2, +δ_d2]`** (δ_d2 = 0.03). If that CI is too **wide** to fit the equivalence margin (the
    neutralization contrast is itself underpowered), the leg is **INCONCLUSIVE** — it does **NOT** count as
    neutralized (turning "not significant" into "removed" is forbidden). Report this contrast's MDE too.
  - `removes_length_norm_penalty := (NOT penalty_present_at_b_hi) OR (penalty_present_at_b_hi AND
    neutralized_at_b_lo)` — i.e. **True iff the enriched arm is not held back by an *unremoved* length-norm
    penalty**; an underpowered neutralization contrast ⇒ the 15b verdict is **INCONCLUSIVE** (§1), never a
    silent True. Mechanical, reproducible, pooled (per-class reported), pre-registered `b` (no argmax), and
    **isolated to length** (uses placebo = pure length), **distinct from criterion 1** (enriched vs placebo).
    Same fixed-seed bootstrap helper.
- **Power guard (codex concern #4) — separate 15b tolerance, NOT external ε:** the external near-parity
  band ε=0.05 is for FathomDB−Mem0; the internal D2 lever uses its **own pinned lift-of-interest
  `δ_d2 = 0.03`** (the smallest enriched−placebo recall lift that would justify a schema migration). If
  the 15b paired MDE for enriched−placebo `> δ_d2`, the verdict is **INCONCLUSIVE** (not FAIL).

## 6. Risks / open items

1. **The gap may be answer-formation, not recall** (D0b accuracy≫recall) → a 15b pass still may not
   move parity; that's why the gap-decomposition probe runs alongside (§1).
2. **Extraction footprint:** LME/IR enrichment may need a Qwen pass (GPU/OFFLINE-BUILD); reuse cached
   first; coordinate GPU; LME-only is the primary, IR is a reserved follow-on.
3. **`removes_length_norm_penalty` must stay mechanical** — pin it to r6 placebo/tune-b outputs, not a
   judgment call (codex to verify).
4. Graph-arm framing is OUT (M1 NO-GO) — enrichment is **per-doc FTS content append**, never a graph arm.
