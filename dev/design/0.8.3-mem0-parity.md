# 0.8.3 Slice-0 — Mem0-parity resolution design + pre-registration

> **Orchestrator draft.** This is the substantive design the Slice-0 implementer freezes into code
> (`eval/decision_rule_083.py`) + a falsifiable AC list, then the HITL signs. Per the three-role model
> (`dev/design/orchestration.md` §1) the orchestrator authors the contract; the implementer makes it
> executable + reviewable (TDD), the reviewer (codex §9) checks it, the HITL gates it.

## 1. Purpose — the resolution, not a lever bake-off

0.8.3 succeeds iff **FathomDB reaches near-parity-or-better with Mem0 on agentic memory**, end-to-end,
under an identical-answerer protocol, with **Graphiti/Zep** as a second comparator on the same axis. The
non-graph levers M1 surfaced (D1 dense/embedder, D2 fielded enrichment) are **means to close a measured
competitor gap**, not ends. History: M1 closed NO-GO (graph multi-hop refuted, `runs/0.8.2-m1-FINDINGS.md`);
the 2026-06-21 capability report found the product's defining number — identical-answerer accuracy vs
Mem0/Zep — was **never measured**, with the four memory classes at **N=0 gold** since 0.8.1 Slice 25.
0.8.3 makes that number exist (Phase A) and drives it to parity (Phase B).

## 2. Two-level measurement (frozen)

| Level | Quantity | Role |
|---|---|---|
| **External** | per-class `FathomDB − {Mem0, Graphiti/Zep, naive-RAG}` (strict full-gold Recall@K **and** identical-answerer accuracy) on the re-pinned, power-sized gold | **THE GATE** — decides *done* |
| **Internal** | each lever vs the corrected-fused baseline (the M1 comparator), MDE-feasible materiality | **attribution only** — explains *which* lever closed the gap; never, by itself, "done" |

The external Mem0 delta is the resolution gate. Internal lever deltas are reported per lever so gap-closure
is explainable, but an internal win is **not** a resolution.

## 3. Datasets, arms, adapters

- **Corpus:** LongMemEval-derived gold, **re-pinned + power-sized** at Slice 5 so each of the four memory
  classes (`factoid`, `knowledge_update`, `multi_session`, `temporal`) carries **≥ N_min gold** (see §5).
  New `corpus_hash` recorded in `runs/0.8.3-d0a-corpus-manifest.json`.
- **Arms (identical-answerer):** `fathomdb` · `mem0` (Mem0-OSS, local) · `graphiti_zep` · `naive_rag` (BM25).
  Adapters expose only `retrieve(question, k)`; **one answerer / one prompt / one context budget** across all
  arms (the R2 invariant) so any delta is retrieval, not prompt divergence. Reader = `gpt-5.4` (temp 0,
  seed 0), the single priced EVAL-ONLY seam; cheap-validate = `gemini-2.5-flash-lite`.
- **Harness:** reuse `eval/r2_parity_eval.py` (identical-answerer, shared adapter/metric contract).

## 4. Endpoints

- **External GATE (resolution):** per memory class, `FathomDB − Mem0` on (a) strict full-gold Recall@K and
  (b) identical-answerer accuracy, with a paired 95% CI. Graphiti/Zep reported alongside (2nd comparator).
- **Internal attribution (lever signals):** corrected-fused baseline deltas on eu8 relevance recall@10, the
  ~596-q hard subset, and MuSiQue co-primaries **all-bridges@K (presence)** and **F1-given-all-bridges
  (composition)** (ADJ-10) — a lever that wins presence but loses composition is not progress.
- **Hard gates (BLOCK):** eu7 ANN-quantization fidelity recall@10 **≥ 0.90**; AC012/AC013/AC020 latency
  budgets intact on read-path slices (ADJ-9).

## 5. Pre-registration — frozen fields (dated 2026-06-21)

> Frozen here in prose; the **implementer freezes the executable copy in `eval/decision_rule_083.py`** and a
> lint test asserts this doc carries every field below. No downstream slice may redefine these.

- **frozen-field: near-parity-band ε = 0.05** (frozen 2026-06-21) (5 percentage points). "Near-parity-or-better" on a class =
  the external `FathomDB − Mem0` paired-CI **lower bound ≥ −ε**. "Better" = CI lower bound **> 0**.
- **frozen-field: power-sizing rule.** (frozen 2026-06-21) Per-class N is the smallest N whose paired **MDE ≤ ε** at the
  **Slice-5-measured** per-class variance + pairing ρ. The *rule* is frozen now; the *number* is computed
  deterministically from the frozen variance at Slice 5 (provisional target per-class **N ≥ 150**, pooled
  ≥ 600 — to be confirmed/escalated at Slice 5; base-retrieval MDE was ≈21pp at n=160, far too coarse, so
  N must rise). Slice-10 power-check **asserts MDE ≤ ε per class** or **escalates to HITL** (no parity
  claim on an under-powered class).
- **frozen-field: decision-rule** (frozen 2026-06-21) (`decide_083`, §9): **REACHED** iff *every* class clears the near-parity
  band (external CI lower ≥ −ε) **and** eu7 ≥ 0.90 **and** latency intact ⇒ present the surpass-option
  package; HITL signs "ship at parity" or "pursue surpass". **NOT_REACHED** ⇒ record the **residual per-class
  gap + the named binding constraint** (most-likely: embedder ceiling / reader / a class-specific lever) +
  the explicit fork: (a) carry a stronger embedder, (b) the caller-side reader axis (out of library
  footprint), (c) a memory-class-specific lever. **This replaces the retired "both-null fallback": 0.8.3
  always ends in a decision, never an open redirect.**
- **frozen-field: $0-probe pass/fail.** (frozen 2026-06-21)
  - **15a embedder-ceiling (PRIMARY):** a candidate embedder *clears* iff it beats CLS-corrected `bge-small`
    on eu8 + the hard subset by a margin whose CI lower > 0 **and** is CPU-feasible **and** 1-bit-survivable
    (projected eu7 ≥ 0.90). Output = the embedder projected to close the most Mem0 gap; if none clears,
    Slice 20 keeps CLS-corrected `bge-small`.
  - **15b D2 content-at-scale proxy:** enrichment *passes* iff fielded content beats the **length-matched
    placebo** at power (CI lower > 0) **and** fielding removes the FTS length-norm penalty. FAIL ⇒ Slice 25
    (D2 engine) defers/drops.
- **frozen-field: eu7-break fork.** (frozen 2026-06-21) Any embedder/pooling change rewrites stored vectors ⇒ re-clear eu7 ≥0.90
  **once** on the chosen embedder. Breach = **BLOCK → HITL** + the fidelity-recovery fork (rotation/whitening
  before 1-bit sign-quant; raise ANN fan-out K>192; a 2-bit option) **and** a **0.937→0.896 bisect** (is CLS
  the cause, or the pre-existing unbisected vector-path regression?). Today's point eu7 = 0.896 (CI-hi 0.925).
- **frozen-field: surpass-option protocol.** (frozen 2026-06-21) At every Phase-B gate the orchestrator records whether a
  credible surpass-Mem0 lever exists (a 15a embedder with headroom; a class-specific lever; the caller-side
  reader axis). The **Slice-30 sign-off package presents it** with cost + expected lift, so HITL chooses
  "ship at parity" vs "spend to surpass" — never a silent stop at the parity line.

## 9. `eval/decision_rule_083.py` — module spec (implementer freezes this)

Extend `eval/m1_decision_rule.py`'s style: pure stdlib (no `fathomdb`/`scipy`/`networkx`/RNG/I/O),
deterministic, consumes already-computed summary stats.

- `EPS_NEAR_PARITY: float = 0.05`; `MEMORY_CLASSES = ("factoid","knowledge_update","multi_session","temporal")`.
- `decide_083(external_per_class: Mapping[str, ClassDelta], eu7_recall: float, latency_ok: bool) -> Resolution`
  where `ClassDelta` carries `point`, `ci_lo`, `ci_hi`, `mde`, `n`. Returns
  `{"verdict": "REACHED"|"NOT_REACHED", "per_class": {...}, "binding_constraint": str|None,
  "surpass_candidates": [...], "blocked_by": "eu7"|"latency"|None}`.
  - **BLOCK** first: `eu7_recall < 0.90` → `blocked_by="eu7"`; `not latency_ok` → `blocked_by="latency"`.
  - **Power guard:** any class with `mde > EPS_NEAR_PARITY` → `NOT_REACHED`, `binding_constraint="underpowered:<class>"`.
  - **REACHED** iff every class `ci_lo >= -EPS_NEAR_PARITY` (and not blocked/underpowered).
  - **surpass_candidates:** classes with `ci_lo > 0`.
- `probe_15a_pass(cand, base) -> bool`, `probe_15b_pass(enriched, placebo) -> bool` (the §5 criteria).
- `lint_preregistration_083(design_md_text: str) -> None` — assert this doc carries each `frozen-field:`
  tag + a date (mirrors `m1_decision_rule.lint_preregistration`).
- Tests (RED→GREEN, pure-Python, **no extension build**): rule truth-table (REACHED / NOT_REACHED /
  blocked / underpowered / surpass), probe criteria, and the lint over this file.
