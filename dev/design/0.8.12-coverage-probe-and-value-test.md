## A. Coverage probe (Slice 5) — pre-registered

### A.1 Purpose & gate

The probe is the **cheap-validate-before-spend** gate (R-COV-1): it measures extraction coverage `$0`
(no new LLM calls) on a fixed corpus and **must run before** the Slice-10 priced extraction. A failing
probe (coverage already at/near the reachable ceiling, or a garbage-precision lever) **blocks** the
priced run and records the negative. It simultaneously discharges the parked **OPP-6 EXP-COV Phase-A**
academic/`$0` arms (HITL 2026-07-01 fold-in).

### A.2 Frozen corpus (the `$0` gold slice)

- **Primary:** `data/corpus-data/external/memex-elps/personal.gold.jsonl` — **60 docs, 246 gold
  entities, 163 gold edges**, across 6 kinds (`chat, email, note, text_message, todo, voice_memo`),
  domain `personal`. This is a curated (oracle-grade) fact set that already exists — it is the
  `C4-oracle` gold reference of OPP-6 §4 **without any new frontier spend**.
- **Current-extractor witness (`$0`):** `personal.baseline_outputs.jsonl` — the **pre-computed**
  `anthropic/claude-haiku-4-5` extractions (prompt `elps-prompt-2`), 60 rows keyed by `doc_id` and
  `gold_sha256`. Scoring these against gold measures the **current ELPS extractor's coverage** for `$0`.
- EVAL-ONLY inputs (LOCOMO/AP-News) stay gitignored; **only derived metrics are persisted**, never
  payloads.

### A.3 Arms actually run in Slice 5 (all `$0`/local)

| Arm | Extractor | What it establishes |
|-----|-----------|---------------------|
| **C0-floor** | heuristic extractor (this repo) — capitalized-span entity spotter + intra-sentence co-occurrence edges, deterministic, CPU | the low anchor; grounds "coverage is ~1% starving" for a rule-only baseline |
| **ELPS-baseline** | pre-computed `claude-haiku-4-5` outputs, scored `$0` | the **current** operating-point coverage — the real number the "~1%" claim must be checked against |
| **C1-gliner** | GLiNER NER (entity-only), CPU/GPU, IF `gliner` installs cleanly | a cheap real-gold lever; entity-coverage only (no relations) |
| **EXP-COV-0 ceiling** | per-corpus relevance ceiling re-measure (`expm4_embedder_ceiling_run.py` basis) | the embedder ceiling the coverage lift is read against |

**HELD (NOT run — separate explicit HITL go required):** C2 (cheap LLM), C3 (frontier LLM), C4 (oracle
saturation). Any priced EXP-COV arm is out of scope for this run.

### A.4 Coverage metric (pre-registered, per OPP-6 §4)

**Fact-coverage = recall of gold facts**, reported separately for entities and edges:

- **Entity coverage** = |extracted_entities ∩ gold_entities| / |gold_entities|, matched on a
  **canonicalized key** (casefold + strip + accent-fold of `name`, unioned with `aliases`) so
  surface-form variants do not falsely penalize.
- **Edge coverage** = |extracted_edges ∩ gold_edges| / |gold_edges|, matched on the canonicalized
  triple `(from_entity, relation, to_entity)` with the relation lightly normalized (casefold, spaces→`_`).
  A relation-synonym allowance is documented but **off by default** (report both strict and lenient).
- **Precision guard (pre-registered, per OPP-6 §4 / the 0.8.3 caution):** report extraction
  **precision** = |extracted ∩ gold| / |extracted| alongside recall, per arm, for entities and edges.
  A coverage gain that arrives with collapsing precision is flagged as "garbage edges," not a win.
- **Per-class breakdown:** all metrics are also reported **per doc `kind`** (the 6 classes) — the
  per-class half of OPP-6 D-3. A class with too few gold facts to be powered is reported as
  **under-powered, no claim** (R-COV-2 "no claim on an under-powered class").

### A.5 Decision rule (pre-registered — OPP-6 §7, applied to the `$0` arms)

The `$0` arms cannot by themselves flip OPP-6 (that needs the priced coverage→outcome sweep + real
gold). What they CAN do — and what gates Slice 10 — is:

- **GATE-OPEN (priced extraction is worth proposing to HITL)** iff there is a **coverage gap with
  headroom**: the current ELPS-baseline edge-coverage is materially below the oracle-gold ceiling
  (i.e. gold facts are being missed) on ≥1 powered class, AND the C0-floor→ELPS-baseline delta shows
  extraction quality is the lever (not the embedder ceiling). Report the gap per class.
- **GATE-CLOSED (block / de-prioritize the priced run, record the negative)** iff the current
  ELPS-baseline already recovers ~all gold facts (no headroom) OR precision collapses when coverage
  rises (garbage-edge regime). Per OPP-6 §7 a flat/at-ceiling result **resolves** OPP-6 by redirecting
  to embedder/recall — a legitimate closed outcome, not a pause.

The probe's output is a **recommendation to the HARD-STOP #1 HITL decision**, not an autonomous spend.

### A.6 Determinism & discipline

- C0-floor and all scoring are deterministic (seeded, pure-Python + numpy).
- `$0`: no new LLM calls; ELPS-baseline is scored from the committed pre-computed outputs.
- Results persisted to `dev/plans/runs/EXP-COV-results.md` (per-class table, per-corpus ceiling,
  derived data). **No raw licensed payloads committed.**

## B. Consolidation value-test (Slice 20) — pre-registered

### B.1 Purpose & gate

R-CON-2: measure that **applying** the consolidation verdict (supersession/recency, per ADR-0.8.12
§2.1) buys retrieval/answer accuracy **worth its latency + lossiness cost**, BEFORE shipping the
provider ON by default. A failing test ⇒ the provider stays built-but-opt-off; the negative is
recorded. Pre-registering it now prevents post-hoc goalpost-moving.

### B.2 Design (paired, pre-registered)

- **Independent variable:** consolidation OFF (accumulate all facts) vs ON (apply the merge/supersede
  verdict as recency metadata; superseded facts down-weighted/hidden at retrieval).
- **Corpus:** a temporal/update-heavy slice where facts genuinely change over time — the
  `multi_session`/`temporal` classes (LOCOMO for real multi-session; the `personal.gold` slice carries
  `supersedes_prior`/`t_valid` structure for a `$0` functional check). EVAL-ONLY corpora stay
  gitignored.
- **Dependent variables:**
  - **Accuracy:** answer EM/F1 (or gold-in-pool / MRR after CE-rerank) where updated facts are the
    correct answer and the superseded ones are distractors — consolidation should raise the score by
    demoting stale contradictions.
  - **Lossiness:** the rate at which consolidation demotes/hides a fact that was actually still
    correct (a false-supersede) — the accuracy-negative failure mode the 0.8.3 blind-merge caution
    warns about.
  - **Latency:** the added ingest cost (cluster assembly + the caller's consolidation LLM pass), and
    any query-path cost (must be ~0 — consolidation is applied at ingest as metadata, query stays
    CPU-only).

### B.3 Decision rule (pre-registered)

- **SHIP-ON** iff the accuracy gain has a **paired-bootstrap CI lower bound > +0.04** on ≥1 powered
  temporal/update class, **net of** an acceptable false-supersede (lossiness) rate and at an acceptable
  ingest latency — confirmed with the precision/lossiness guard so the gain is not stale-fact removal
  masking as accuracy.
- **STAY-OFF (record the negative)** iff the gain's CI spans ≤ the noise floor, OR lossiness exceeds
  tolerance (consolidation drops still-correct facts), OR the latency cost is unacceptable. The seam
  ships built but default-off (build ≠ adopt).

### B.4 Footprint

Consolidation LLM is caller-side BYO-LLM; the value-test's ON arm uses a caller/stub harness. The
library query path is unchanged/CPU-only in both arms (R-CON-3). The test itself is `$0` if run on a
local/stub consolidation harness; any frontier consolidation pass is priced and HITL-gated (same
discipline as Slice 10).
