# 0.8.8 Slice 15 — Telemetry capture (OPP-9)

> Schema is **already ratified** (ADR `0.8.8-explain-and-telemetry-adr.md` §B +
> ratification artifact §3d). This doc designs the **mechanism** the schema
> didn't pin: enable/opt-in, capture point, `query_id` surfacing, the
> feedback-attach API, and the sink. Proposed defaults below; HITL-amendable.

## Locked by ratification (do not relitigate)

- `TelemetryEvent {schema_version, ts_monotonic_ms, query_id, query_chars,
  result_ids:[u64], arm_of:{id→branch}, feedback?:{relevant_ids, irrelevant_ids,
  label_source}}`.
- **Ids are the stable `logical_id`** (== `SearchHit.id`/`PerHitExplain.id`), never
  `write_cursor`. **Never capture `source_id`** (leak vector). `query_chars` =
  length only, never the query text.
- **Off by default**; **no socket/egress** (footprint test); deterministic
  `ts_monotonic_ms` + `query_id`.

## Proposed mechanism

### M1 — enable / opt-in (off by default → zero cost)

`Engine::enable_telemetry(sink_path: &str) -> Result<(), EngineError>` installs a
`TelemetrySink` (a `Mutex<Option<TelemetrySink>>` engine field, `None` by
default). When `None`: search does **no** telemetry work — no allocation, no file
handle, no branch beyond a single `is_some` check (the `if explain` discipline
from Slice 5). Reuses the `Subscriber`/`Subscription` precedent conceptually but
telemetry needs the `SearchResult` (ids + arms), which lifecycle `Event`s don't
carry — so capture is a dedicated, gated call in the search body, not a
lifecycle subscriber.

### M2 — capture point

In `search_reranked_with_explain` (the shared search body, after a successful
`SearchResult`): if telemetry enabled, build a `TelemetryEvent` from
`result.results` (`result_ids` + `arm_of`) and append it to the sink.

### M3 — `query_id` surfacing (the real decision)

`query_id = "q{open_nonce}-{seq}"`, `seq` an `AtomicU64` bumped per captured
query, `open_nonce` a deterministic per-open value (so fixtures reproduce).
**Decision D1:** how the caller learns the `query_id` to attach feedback.

- **(proposed) D1a — `Engine::last_telemetry_query_id() -> Option<String>`**:
  returns the most-recent captured query's id; `None` when telemetry off.
  Minimal; does NOT reshape `SearchResult` (keeps the byte-stable shape). Single
  opt-in caller assumption (telemetry is a dev/eval feature, not concurrent).
- D1b — add `query_id` to `SearchResult` (rejected: reshapes the result on the
  default path; the sidecar precedent argues against touching the core shape).
- D1c — `record_feedback` keyed by the result-id set (rejected: ambiguous across
  repeated identical queries).

### M4 — feedback-attach API

`Engine::record_feedback(query_id: &str, relevant_ids: &[u64],
irrelevant_ids: &[u64], label_source: &str) -> Result<(), EngineError>` appends a
feedback record to the sink (a second JSONL line correlated by `query_id`, rather
than rewriting the event — append-only sink). `label_source` is the only
exogenous string (caller-declared, e.g. `"agent:hermes"`).

### M5 — sink format

**JSONL** (one JSON object per line: an `event` row, then optional `feedback`
rows correlated by `query_id`), append-only, to `sink_path`. Local file only — a
footprint test asserts no socket/network handle is opened. serde_json (already a
dep).

## Acceptance (R-TEL-1..3) — engine

- **R-TEL-1**: enable → search → an `event` JSONL row with the right
  `result_ids`/`arm_of`/`query_chars`; `record_feedback` → a correlated
  `feedback` row. Deterministic `query_id`/`ts_monotonic_ms` (fixture-stable).
- **R-TEL-1 (off-by-default)**: no `enable_telemetry` → no file created, no rows,
  no allocation; `SearchResult` byte-identical.
- **R-TEL-3 (footprint)**: a test asserts the sink is a local file and no socket
  is opened during capture (no egress).
- **Privacy**: assert no query text and no `source_id` ever appear in the sink.

## Out of scope (later)

- SDK surfaces for `enable_telemetry`/`record_feedback` (Slice 10-style wiring,
  follow-on).
- Slice 20: telemetry → `GoldRecord` (+`query_id`/`+id_space`) + the offline
  frozen-candidate scorer.
