# 0.8.3 Slice-15a — embedder-ceiling probe (design + TDD contract)

> **Orchestrator draft for implementer (TDD) + codex review.** This grounds the §5-frozen 15a
> probe into an executable, deterministic, $0/CPU/offline measurement. The implementer makes it
> executable + reviewable under TDD; codex reviews; the orchestrator gates. **The gate semantics
> are frozen in `decision_rule_083.probe_15a_pass` — this slice CONSUMES that function and must not
> reimplement or relax it.**

## 1. Purpose & non-goals

**Purpose.** Pick the Slice-20 embedder by measuring which CPU/ONNX-class embedder is projected to
close the most Mem0 gap, subject to **CPU-feasibility** and **1-bit (Hamming) survivability**. This
is the **PRIMARY** $0 lever (ADJ-6: the dense axis is embedder-bound; fused only *ties* BM25, eu8
ceiling ≈0.571). FathomDB's FTS recall trails plain BM25 on every memory class
(`runs/0.8.3-d0b-powered-recall.json` / corpus-adequacy note), so the embedder is the lever this
probe sizes.

**Non-goals / hard constraints (footprint invariant).**
- **$0, LLM-free, NO airlock.** No gpt-5.4 / qwen / `localhost:4000`. Pure offline embedding+recall.
- **CPU only — NO GPU.** The sibling D0b run owns the GPU. All embedding on CPU
  (`CUDA_VISIBLE_DEVICES=""`); CPU is the library footprint anyway.
- **No engine / Rust change.** Pure-Python eval harness. The engine embedder swap is **Slice 20**.
- **Do NOT modify** `src/python/eval/decision_rule_083.py` or the gold artifacts. Consume only.
- **Do NOT merge to main.** Commit on `0.8.3-slice-15a`, write closure `output.json`, run codex §9,
  STOP. The human/primary lands it after D0b.
- Footprint tag: this entire slice is **OFFLINE-BUILD / EVAL-ONLY**.

## 2. Frozen pass criterion (consumed verbatim from `probe_15a_pass`)

A candidate **clears** iff ALL hold (design §5 / `decision_rule_083.probe_15a_pass`):
1. `eu8_margin_ci_lo > 0` — paired (cand − base) eu8 recall@10 margin, 95% bootstrap CI **lower bound > 0**;
2. `hard_margin_ci_lo > 0` — same for the hard-subset recall@10 margin;
3. `cpu_feasible == True`;
4. `projected_eu7 ≥ EU7_FLOOR (0.90)` (1-bit survivability).

The harness builds `cand = {eu8, hard, eu8_margin_ci_lo, hard_margin_ci_lo, projected_eu7,
cpu_feasible}` and `base = {eu8, hard}` per candidate and calls `probe_15a_pass(cand, base)`. It
**must not** add extra gates or a raw-point check (the Slice-0 fix-1 review explicitly dropped
raw-point gates; gate on the CI lower bound only).

**Decision output (frozen gate semantics).** The Slice-20 embedder = the candidate that **passes**
`probe_15a_pass` with the **largest headroom** (ranked by the binding margin = `min(eu8_margin_ci_lo,
hard_margin_ci_lo)`, with projected_eu7 headroom as tiebreak) **AND is `in_library_feasible`** (§3 —
has a candle/ONNX encoder path so Slice 20 can actually adopt it); if **none** qualifies, the pick is
the **CLS-corrected `bge-small`** — *no swap*. A candidate that passes the probe metrics but is
`in_library_feasible == False` is **NOT selectable** as the Slice-20 embedder — it is recorded as an
"infeasible-without-model-work" lever (a possible future surpass option needing a new candle encoder),
never silently chosen. A clean negative (no qualifying candidate beats the corrected base at power) is
reported as such, not papered over (the prior nomic diagnostic, §11, makes "no swap" a real possible
outcome).

## 3. Reference (base) and candidates

**Base (reference) = CLS-corrected `bge-small-en-v1.5`.** The known Slice-20 bug is mean pooling for
a CLS-trained BGE model; the reference here MUST use **CLS pooling** (`Pooling::Cls` in
`fathomdb-embedder`'s `CandleBgeEmbedder`) + the BGE query prefix. Mirroring the wrong (mean)
pooling would invalidate the entire A/B — the base must be the *corrected* model.

**Candidates** (4, per plan §4 + one memory/retrieval-tuned of our choice). The
`in_library_feasible` column records whether the model has a **candle/ONNX encoder** so Slice 20 can
adopt it in-library (`fathomdb-embedder`): `CandleBgeEmbedder` is a BERT encoder with a `Pooling`
enum (Cls/Mean) — it covers bge-small/bge-base and **e5-base-v2** (BERT arch, mean pooling); `NomicEmbedder`
(candle `nomic_bert`) covers nomic. **`gte-base-en-v1.5` (the `Alibaba-NLP/new-impl` GTE arch) has NO
candle-native encoder** and was previously excluded as requiring custom model work
(`dev/notes/IR-C-embedder-options-research.md` §~115/165) — it is **measured for attribution but flagged
`in_library_feasible=False`** and is therefore not selectable as the Slice-20 embedder (§2).

| name | HF id | dim | pooling | query prefix | passage prefix | in_library_feasible |
|---|---|---|---|---|---|---|
| **base** bge-small | `BAAI/bge-small-en-v1.5` | 384 | **CLS** | `Represent this sentence for searching relevant passages: ` | — | yes (candle BERT) |
| bge-base | `BAAI/bge-base-en-v1.5` | 768 | **CLS** | same BGE query prefix | — | yes (candle BERT) |
| gte-base | `Alibaba-NLP/gte-base-en-v1.5` | 768 | **CLS**¹ | — | — | **no** (no candle GTE encoder) |
| e5-base-v2 | `intfloat/e5-base-v2` | 768 | **mean** | `query: ` | `passage: ` | yes (candle BERT)² |
| **nomic** (memory-tuned) | `nomic-ai/nomic-embed-text-v1.5` | 768 | **mean** | `search_query: ` | `search_document: ` | yes (candle nomic_bert) |

¹ `gte-base-en-v1.5` is trained with **CLS** pooling; confirm against the model card at pin time. Since
it is measure-only/infeasible its pooling does not gate selection, but apply the documented pooling and
flag. ² e5-base-v2 is a BERT-base checkpoint loadable via the same candle BERT path with mean pooling +
the e5 `query:`/`passage:` prefixes; **verify it loads via `CandleBgeEmbedder`'s BERT path at pin time**
(if not, downgrade its `in_library_feasible`). The exact correct pooling/prefix per model is a
**RED-test fixture** (§8): a fixture catches a mean-vs-CLS or missing-prefix mistake.

**Why nomic-embed-text-v1.5 as the memory/retrieval-tuned candidate.** It is explicitly
retrieval/instruction-tuned (task-prefixed `search_query:`/`search_document:`), Matryoshka, 768-d,
CPU-friendly, and **already has a first-class engine path** (`NomicEmbedder` referenced in
`tests/ir_c_gold_diagnostics.rs`), so a Slice-20 adoption is low-friction. It is the strongest
in-footprint "memory-tuned" option that does not require a larger-than-base model.

All candidates are sentence-transformer/ONNX-class, CPU-runnable, ≤768-d (1-bit-index-friendly).
**Pinned HF revisions** are recorded per model in the output (commit SHA / revision hash) for
determinism.

## 4. Corpus, gold, and the hard subset (frozen, model-independent)

- **Corpus:** the frozen IR snapshot `tests/corpus/snapshot.json` (`corpus_hash` =
  `fe973fcd49fbbda083158f69fe720f17858ab8528e171fa2188eec84131c7d4e`); doc bodies in
  `data/corpus-data/raw/*.jsonl`. The probe records the resolved doc count + re-verifies the corpus
  hash before embedding (the pinning principle, `ir-recall-measure.md` §f — a relevance number across
  a different corpus is meaningless). Reuse the existing corpus loader/membership the IR harness uses
  (`tests/.../support/corpus_subset.rs` membership; Python side resolves doc_id→body from the same
  raw jsonl + snapshot per-source sha256).
- **eu8 gold (qrels):** `data/corpus-data/eval/ir_gold/all.gold.json` (`qrels_version`
  `ir-c-reused-v1`, 4597 queries). Each query carries `expected_top_k_doc_ids` (the doc-id qrels).
  **eu8 relevance recall@10** = the legacy doc-id reduction of Evidence Recall (mirror
  `support/ir_eval.rs::evidence_recall_at_k`): per query, **strict all-of** — `1` iff every
  `expected_top_k_doc_ids` doc is in the f32-exact top-10, else `0`; averaged over non-`negative`
  queries. Negative-class queries are excluded from the recall mean (abstention-correctness, §d of
  the measure doc).
- **Hard subset (= deep-exploratory recall).** Frozen as a **model-INDEPENDENT** query-id set so the
  A/B is fair: the queries lexical BM25 **cannot** solve within a pinned cap — exactly where dense/the
  embedder is the only lever. Pinned definition (removes the §11-Q2 drift the review flagged):
  - **cap = 50** (the deep lexical horizon; matches the recall@50 retriever-health K).
  - **Tokenizer (pinned, ONE choice):** lowercase + regex `[a-z0-9]+`; **Okapi BM25 k1=1.2, b=0.75**,
    pure-Python, deterministic. (Not FTS5 `bm25()` — a single documented tokenizer avoids the
    FTS5-vs-rank-BM25 ambiguity; the FTS5-tokenizer-mismatch caveat is recorded, and the engine FTS5
    path is *not* used so no engine dependency.)
  - **Multi-doc reduction (pinned, strict all-of — matches eu8's denominator):** a query is
    "lexical-solved" iff **every** one of its required docs (`expected_top_k_doc_ids`) is within the
    BM25 top-cap. `hard_qids = { qid (non-negative) : NOT all required docs within BM25 top-50 }`.
    This fixes the singular-"gold doc" loophole (a query with multiple required docs is hard unless
    BM25 surfaces them ALL).
  - The implementer reproduces this deterministically over the frozen corpus and records the **cap,
    tokenizer, k1/b, and the actual `hard_qids` count**. The design references a **~596-q** subset;
    the **actual reproduced frozen count governs** and is reported transparently — do NOT force 596
    (honesty prior; a forced count is a silent fudge). Metrics on the hard subset: **recall@10** (the
    gated one), **recall@50**, and **median gold rank**.
- **Memory-class strict recall (where cheaply derivable, $0).** On the re-pinned LME gold
  (`gold_repin.py` output, `repin_hash 2916cace`, per-class factoid/knowledge_update/multi_session/
  temporal), compute per-class strict recall@10 offline if the LME passages embed cheaply within the
  same harness. This is **report-only attribution** (it is NOT a `probe_15a_pass` input); if it is
  not cheaply derivable on CPU within the slice, it is recorded as "not computed — deferred to Slice
  20 re-embed" rather than faked.

## 5. Metrics computed per embedder (deterministic, CPU)

For each model (base + 4 candidates), with **pinned HF revision + fixed seed**, on CPU:

1. **eu8 recall@10** (§4) — f32-exact cosine top-10 over the full corpus, strict all-of.
2. **hard-subset** recall@10 (gated), recall@50, median gold rank (§4).
3. **memory-class strict recall@10** (report-only, where cheap; §4).
4. **projected_eu7 (1-bit survivability):** quantize the model's f32 vectors to **1-bit sign**
   (`vec >= 0`), Hamming top-**K=192** (the production fanout, `ADR-0.7.0`) → fetch f32 → cosine
   rerank → top-10; **recall@10 of this 1-bit path vs the model's own f32-exact top-10** (the same
   SUT/GT shape as `eu7_real_corpus_ac.rs` / `dev/research/eu-0/run_sweep.py`). **Mean-centering ON**
   (pinned, matches production `ADR-0.6.0`): subtract the per-corpus mean f32 vector before sign-quant
   on BOTH doc and query vectors; the f32 rerank uses the **un-centered** f32 vectors (centering is a
   sign-quant bias correction only). Record that mean-centering was applied so the proxy is comparable
   to the real eu7. **GATE = the frozen point** `projected_eu7 ≥ EU7_FLOOR (0.90)` exactly as
   `probe_15a_pass` consumes a point float; ALSO report a bootstrap CI on projected_eu7 for
   transparency (eu7 AC-075 is now CI-based, `ADR-0.7.0:173`), but the CI does **not** change the
   frozen point gate.
5. **cpu_feasible (pinned numeric rule, removes the §11-Q6 judgment-call):** measure CPU embed latency
   **ms/query** and **ms/doc** (median over a fixed sample; **`torch.set_num_threads` fixed and
   recorded**). `cpu_feasible = (ms_per_query ≤ 3.0 × base_ms_per_query)` — i.e. not pathologically
   slower than the production-accepted base (bge-small) on the synchronous caller-thread query-embed.
   ALSO report the **absolutes vs the read-path budgets** (AC012 p50≤20 ms / p99≤150 ms; AC013
   p50≤80 ms / p99≤300 ms, `perf_gates.rs:170-178`) for context — the **authoritative** absolute
   AC012/AC013/AC020 re-measure is Slice 20's engine latency gate, not this $0 probe. Report the
   numbers **and** the bool.

### Paired margin CIs (the gated quantity)

`eu8_margin_ci_lo` / `hard_margin_ci_lo` = the **paired (candidate − base) per-query delta**, 95%
**bootstrap** CI **lower bound**, fixed bootstrap seed, ≥1000 resamples, percentile method (mirror
`eu8_ir_validation.rs:283-304`). Paired = resample query indices, compute mean(cand_hit − base_hit)
on the same resampled queries (the same query set for eu8; the frozen `hard_qids` for hard). This is
the only quantity `probe_15a_pass` gates on.

## 6. Two-stage split (explicit, per the prompt)

- **Stage 1 (this slice, mechanical):** emit the per-candidate metrics, the paired margin CIs, and
  the mechanical `probe_15a_pass` verdict + the ranking + projected lifts. **The chosen Slice-20
  embedder (or "no swap") is decided here** by the frozen gate.
- **Stage 2 (after D0b lands, orchestrator/HITL):** the gap-relative "is the projected lift *enough*
  to close the Mem0 gap / does it *overshoot* (surpass)" interpretation. D0b's final per-class gap is
  not yet available; this slice records the **surpass flag as provisional** (a candidate whose
  projected eu8/hard lift is large is flagged a *candidate* surpass lever) and defers the
  gap-relative finalization. The output makes this split explicit.

## 7. Determinism & environment

- **Determinism (pinned runtime, removes the §11-Q6 over-claim):** pinned HF revisions; fixed seeds;
  fixed bootstrap seed; fixed `corpus_hash` + `hard_qids`; **`torch.set_num_threads(1)` for the
  determinism assertion** (multi-thread BLAS reduction order can perturb the low bits); record
  torch/transformers/numpy versions + BLAS. The claim is **byte-identical under the pinned runtime +
  fixed thread count** (asserted, §8a) — NOT byte-identical across arbitrary machines/BLAS. The
  measurement run may use the full recorded thread count for speed (only the determinism *assertion*
  pins threads=1).
- **Offline/$0 (pinned, removes the §11-Q6 ambiguity):** a **SETUP step** pre-fetches ALL model
  weights into the HF cache (network allowed — this is OFFLINE-BUILD model provisioning, $0, never the
  library path). The **measured run then sets `HF_HUB_OFFLINE=1` + `TRANSFORMERS_OFFLINE=1`** so it is
  strictly offline + reproducible (a missing weight fails loudly instead of silently re-downloading).
- **Environment:** the canonical `.venv` lacks torch/transformers; reuse `dev/research/eu-0/.venv`
  (torch 2.12+cpu, transformers 5.9) with `CUDA_VISIBLE_DEVICES=""`, `HF_HOME` pointed at
  `dev/research/eu-0/hf_cache` (bge-small/base + e5-small cached; gte-base, e5-base-v2, nomic-v1.5
  fetched once in SETUP). Do NOT install into the canonical `.venv`, do NOT `maturin develop`/`pip
  install -e` (protect the sibling run). ONNXRuntime is present in the canonical venv; torch-CPU and
  ONNX-CPU produce numerically equivalent f32 vectors for these models, so torch-CPU is a valid
  ceiling measurement (the ceiling is a model-weights property, runtime-independent to f32 tolerance)
  — recorded as a caveat.
- **Resilience ([[priced-runs-need-resilience-before-spend]]):** the runner checkpoints per-model
  doc/query f32 vectors to `.npy` (atomic write) and supports `--resume`; a multi-hour CPU embed
  must survive a restart. `--smoke` (small corpus subset + 2 models) proves the pipeline green fast;
  `--full` runs the real measurement.

## 8. TDD plan (RED → GREEN, backend-light)

**RED (commit first; tiny fixed vectors / fixtures where possible, no model download in unit tests):**
- (a) **embedding determinism:** same input + pinned revision → byte-identical vectors (a small
  fixture or a marked/integration test that loads the smallest model once).
- (b) **pooling/prefix correctness:** a fixture that FAILS on a mean-vs-CLS swap or a missing
  `query:`/`search_query:` prefix (e.g. assert the documented pooling + prefix string per model from
  a config table; a known-vector fixture for one model).
- (c) **probe wiring:** the harness builds the `cand`/`base` dicts and calls
  `probe_15a_pass`, emitting a mechanical PASS/FAIL + the per-candidate margin CIs; assert against a
  synthetic case where the margins/projected_eu7 are known (truth-table: passes / fails-on-eu8 /
  fails-on-hard / fails-on-cpu / fails-on-eu7). **Must call the real `probe_15a_pass`, not a copy.**
- (d) **1-bit eu7-proxy computation:** on tiny fixed f32 vectors, assert the sign-quant→Hamming→f32-
  rerank recall@10-vs-f32 equals a hand-computed value (and that identical f32 ⇒ proxy 1.0).
- (e) **paired bootstrap CI:** deterministic (fixed seed) lower bound on a tiny fixed paired array;
  assert a known value + that a strictly-positive paired delta yields ci_lo > 0 and a zero delta does
  not.

**GREEN:** the probe runner producing the outputs (§9), `--smoke` green end-to-end.

Tests run in the eu-0 venv (or canonical venv for the pure-logic ones). `ruff` + `pyright`/typecheck
green on new files. `mkdocs build` green if docs touched.

## 11. Risks / open items

1. **Hard-subset reproduction.** The frozen `hard_qids` count may not be exactly 596; the pinned
   definition (§4: cap=50, regex tokenizer, BM25 k1=1.2/b=0.75, strict all-of) governs and the actual
   count is reported. Residual caveat: this pure-Python BM25 tokenizer is not byte-identical to the
   engine's FTS5 tokenizer → a slightly different bucket than a production-FTS bucket. Recorded, not
   gated (the subset only needs to be deterministic + model-independent for a fair A/B).
2. **In-library feasibility (codex Q5).** `gte-base-en-v1.5` has no candle encoder; it is measured for
   attribution but flagged `in_library_feasible=False` and is **not selectable** as the Slice-20 pick
   (§2/§3). Verify `e5-base-v2` actually loads via the candle BERT path at pin time; if not, downgrade
   its feasibility.
3. **Prior nomic evidence (raises the "no swap" prior).** A prior full-corpus diagnostic (codex
   consult, `runs/0.8.3-s15a-design-consult-*.md`) found nomic *better on exact_fact but worse on
   exploratory*, and argued the exploratory weakness is a **granularity/task-structure** problem, not
   model-capacity — so a stronger model may not clear the *hard* (exploratory-heavy) subset. That prior
   used mean-pooled bge + 128-word chunks; this probe uses CLS-corrected bge + whole-doc, so it is a
   genuinely different measurement — but "no swap" is a real, pre-registered possible outcome (§2).
4. **projected_eu7 point-vs-CI (codex Q3).** The frozen `probe_15a_pass` gates a POINT
   `projected_eu7 ≥ 0.90`; eu7 AC-075 is CI-based. We report both but gate on the frozen point (§5) —
   a methodology caveat, not a consume-rule violation.
5. **CPU wall-clock** for 5×(7.7k–10.5k docs): mitigated by checkpointing + `--resume`; run `--full`
   as a resilient background job.
