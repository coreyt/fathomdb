## Why batch (and why it would unblock this specific run)

The interactive `/v1/chat/completions` path failed to complete the α=1.0 arm because the large-context
multi_session/temporal cells hit a **low OpenAI usage-tier TPM (tokens-per-minute) ceiling** under sustained
load — surfacing as `insufficient_quota`, which arms the airlock provider-protection quarantine
([[airlock-batch-and-provider-protection]]). The OpenAI **Batch API**:

- **Bypasses the airlock quarantine entirely** — `airlock/fast/guardian.py::async_pre_call_hook` explicitly
  skips batch calls (`if not mcp and not batch:`), so the per-minute interactive quarantine never applies.
- Is processed **server-side by OpenAI under separate (much higher) batch enqueue limits**, not the
  interactive TPM ceiling — so the large-context cells that walled interactively can complete.
- Is **~50% cheaper** and runs async (**~24h** completion window).

It is OpenAI-compatible and **already configured** on the airlock (`config.yaml` `files_settings:
custom_llm_provider: openai`); `/v1/batches` responds. Docs: `~/projects/airlock/docs/guide/batch.md`.

## Caveats (read before running)

1. **Batch still needs genuine OpenAI quota.** Batch sidesteps a *velocity/TPM* limit, **not** an exhausted
   credit balance or a hard monthly **usage limit**. If the account is hard-capped, batch rows fail too.
   Confirm the project's Settings→Limits monthly budget is raised first.
2. **Use the upstream model id** `gpt-5.4` with `?custom_llm_provider=openai` — the file uploads straight to
   OpenAI and bypasses the proxy alias map (and the PII/keyword guardrails; fine here — LongMemEval/LOCOMO
   are public benchmark text).
3. **Reader consistency is preserved.** Batch uses the same `gpt-5.4`, `temperature=0`, `seed=0` as the
   interactive cells → the 354 already-banked interactive answers and the batched remainder are the same
   reader, so the merged arm and the comparison vs the gpt-5.4 baseline/Mem0 stay valid. (gpt-5.4 default
   reasoning = 0 tokens, so there is **no** thinking-budget/`max_tokens` empty-truncation risk —
   [[airlock-batch-and-provider-protection]].)
4. **Needs a proxy restart only if `files_settings` were absent** — they are present, so no restart needed.

## Plan: two phases (the LLM cost is only phase 2)

### Phase 1 — build the request JSONL (`$0`, deterministic, no LLM)

Reuse the existing harness retrieval/prompt pieces. For each query to complete (either the **252 absent**
cells, or **all 606** for a clean single-reader artifact — both valid since same gpt-5.4/temp0/seed0; all-606
costs ~$2.6 at the 50% batch discount and is cleanest):

```python
# sketch — eval/rerank_accuracy_batch.py (build step); NOT run here
from eval.rerank_tune_probe import ReblendRerankAdapter
from eval.d0b_parity_run import build_documents_from_lme, build_live_adapters, fit_context
from eval.r2_parity_eval import load_repin_gold
from eval.gap_decomposition_run import CONTEXT_CHAR_BUDGET
import fathomdb, json

_, _, queries = load_repin_gold("dev/plans/runs/0.8.3-d0a-memory-gold.json")
docs = build_documents_from_lme(queries)
base = build_live_adapters(docs, want_mem0=False, want_graphiti=False, db_path="/tmp/rr-a1.sqlite")[0]["fathomdb"]
rr = ReblendRerankAdapter(base=base, rerank_fn=fathomdb.rerank, alpha=1.0, pool_n=10)

with open("rerank_a1_batch.jsonl", "w") as f:
    for q in queries:                      # or: for q in absent-subset
        if not q.answers: continue
        ctx = fit_context([h.body for h in rr.retrieve(q.question, 10)[:10] if h.body], CONTEXT_CHAR_BUDGET)
        prompt = build_prompt(q.question, ctx)   # the same answerer build_prompt template
        f.write(json.dumps({
            "custom_id": q.query_id,             # keys the output + idempotency/resume
            "method": "POST", "url": "/v1/chat/completions",
            "body": {"model": "gpt-5.4",
                     "messages": [{"role": "user", "content": prompt}],
                     "temperature": 0, "seed": 0},
        }) + "\n")
```

`custom_id` MUST be unique per row (it keys the output and the gateway's resume).

### Phase 2 — upload / create / poll / download (the priced async step)

```bash
# upload (purpose=batch, openai provider)
curl -s http://localhost:4000/v1/files \
  -H "Authorization: Bearer $AIRLOCK_MASTER_KEY" \
  -F purpose=batch -F custom_llm_provider=openai -F file=@rerank_a1_batch.jsonl
# -> {"id":"file-...", ...}

# create the batch
curl -s "http://localhost:4000/v1/batches?custom_llm_provider=openai" \
  -H "Authorization: Bearer $AIRLOCK_MASTER_KEY" -H "Content-Type: application/json" \
  -d '{"input_file_id":"file-...","endpoint":"/v1/chat/completions","completion_window":"24h"}'
# -> {"id":"batch-...","status":"validating"}

# poll until status == completed (validating -> in_progress -> completed)
curl -s "http://localhost:4000/v1/batches/BATCH_ID?custom_llm_provider=openai" \
  -H "Authorization: Bearer $AIRLOCK_MASTER_KEY"

# download the output JSONL (no provider param needed on content GET)
curl -s http://localhost:4000/v1/files/OUTPUT_FILE_ID/content \
  -H "Authorization: Bearer $AIRLOCK_MASTER_KEY" > rerank_a1_batch_out.jsonl
```

Output rows are OpenAI-shaped:
`{"custom_id":"<qid>","response":{"status_code":200,"body":{<chat.completion>}},"error":null}`
(a per-row failure is `"response":null,"error":{...}` — does not fail the whole batch).

## Implementation note

Factor a small `eval/rerank_accuracy_batch.py` that imports the retrieval/context/scoring from
`rerank_accuracy_run` and swaps the per-cell sync answerer loop for build-JSONL → submit → poll → parse.
The merge can reuse the existing checkpoint (the 354 banked gpt-5.4 answers) by treating the batch output as
the resume source for the absent rows. The gateway is idempotent on `(input_file_id, model, endpoint,
params)` and resumes only missing rows, so a re-submit is safe.
