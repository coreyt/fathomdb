# Slice 10 (D0b) — measure the FathomDB − {Mem0, Graphiti/Zep, naive-RAG} parity gap

PHASE A only (this spawn): build the resilient runner, prove it with backend-free
TDD, cheap-validate the FULL pipeline at ~$0, run a ≤$1 pilot to measure
cost/question, then STOP at the §PHASE-GATE. **No full priced pass here.**

## 1. The target (design §2/§4)

Per memory class `c ∈ {factoid, knowledge_update, multi_session, temporal}`, the
external gap `FathomDB − {mem0_oss, graphiti_zep, naive_rag}` on:
- **(a) strict full-gold Recall@K** — LLM-free, $0. A query scores 1 iff *every*
  `gold_doc_id` is in the arm's top-K (the existing `R2Harness.run` recall rule).
- **(b) identical-answerer accuracy** — PRICED. One answerer / one prompt / one
  context budget across all arms (the R2 invariant); reader `gpt-5.4` (temp0/seed0),
  cheap-validate `gemini-flash-lite`.

Each delta carries a **paired 95% CI** (per-question paired bootstrap, reusing the
m1 percentile helpers) and a per-class paired **MDE** (normal-approx
`(z.975+z.80)·sd(dₚ)/√n`). The accuracy deltas feed `decide_083` → REACHED /
NOT_REACHED + per-class gap + power verdict.

## 2. Runner (`eval/d0b_parity_run.py`) — resilience first, before any spend

Reuses `m1_verdict_run`'s proven mechanisms (do NOT hand-roll weaker ones):
- **Atomic checkpoint** (`_atomic_write_json`: temp-file + `os.replace`) at least
  every `checkpoint_every` (≤10) questions; the live checkpoint is never half-written.
- **Auto-resume on by default** (`_resolve_resume`): the `<output>.checkpoint.json`
  sidecar is detected automatically; a relaunch reuses every persisted `(qid,arm)`
  cell (key-present incl. `None` = a real abstention, reused; absent = re-called) →
  zero re-spend. Proven by a kill/restart unit test before any priced call.
- **429/5xx backoff-and-recover**: the answerer is `CostTrackingAnswerer` (exponential
  backoff over transient 429/5xx; a cell fails only after `max_retries`).
- **failure ≠ abstention**: a retry-exhausted cell is MISSING (absent key), counted in
  `n_errors`, never persisted as a scored `None`/0.
- **Completeness validity guard**: a pass is citable only at completeness ≥ the M1
  floor (0.97); below it the run is flagged INVALID (relaunch auto-resumes the missing
  cells).

The unit of work is a `(qid, arm)` cell. Recall is recomputed from retrieval ($0);
the answerer (priced) is the only seam. Records persist `{qid, reporting_class,
gold, recall:{arm:0/1}, answers:{arm:str|None}}`.

## 3. Arms / adapters (design §3)

- `naive_rag` — `NaiveRAGAdapter` (BM25, pure-Python, always available).
- `fathomdb` — `FathomDBAdapter` over a local SQLite ingest of the LME sessions.
- `mem0_oss` — `Mem0OSSAdapter.try_build()` → `build_local_mem0_config` (airlock
  qwen3.6-27b LLM + local bge-small + on-disk chroma, per-run isolated; NO cloud).
  Installed into an **isolated env** (`--target` dir on PYTHONPATH), never the
  canonical `.venv`.
- `graphiti_zep` — stood up locally behind the R2 adapter contract if tractable;
  else a **clean recorded blocker** and the gap runs with {mem0, naive_rag}
  (Graphiti is the 2nd comparator, not a blocker — design §3).

Corpus = LongMemEval-cleaned `oracle` sessions (the gold's `doc_source`), restricted
to the gold-relevant sessions for footprint + Mem0 per-`add()` cost. Gold via
`load_repin_gold` (runtime corpus-validity guard) on `repin_hash 2916cace`.

## 4. CI / MDE / decide_083 wiring

- `paired_class_delta(treatment, comparator, class, metric)` → per-question paired
  deltas `dₚ`; `point=mean`, `ci_lo/ci_hi` = m1 percentile bootstrap (seed 0,
  n_boot 2000), `mde = (1.96+0.84)·sd(dₚ)/√n`, `n=len`.
- `external_per_class_for_decide` builds the `decide_083` input from the **accuracy**
  `fathomdb − mem0_oss` deltas (the priced gate); recall deltas reported alongside.
- `decide_083(external, eu7_recall, latency_ok)` is imported and consumed verbatim.
  Expected at N≈150: `NOT_REACHED` `underpowered:<class>` (manifest implied-N proxy
  shows every class needs N≫150). The gap numbers LAND regardless (design §5) and the
  under-power triggers the **HITL escalation** (grow N vs accept), not suppression.

## 5. Cost model (the phase-gate deliverable)

`cost_per_question` = pilot answerer USD / pilot answerer calls (one call per
`(qid,arm)`; arm count = priced arms that ran). Projected full =
`cost_per_question × 606 × n_priced_arms`. State whether ≤ $20 (aggregate program
budget ≤ $30, design §7). Pilot is HARD-capped at $1; stop if it would exceed.
