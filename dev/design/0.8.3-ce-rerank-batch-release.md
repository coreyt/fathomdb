# Batched CE scoring + release build

## 1. Scope (one call site)
`rerank_passages` (the standalone `fathomdb.rerank`) **delegates to** `rerank_fused`
(`fathomdb-engine/src/lib.rs:4830`), so BOTH the standalone and fusion paths share the single per-pair
loop at `lib.rs:4943` (`top.iter().map(|h| model.score(_query, &h.body))`). Batch that one site.

## 2. Change
### a. `fathomdb-embedder/src/candle_reranker.rs` — add `score_batch`
`pub fn score_batch(&self, query: &str, passages: &[&str]) -> Result<Vec<f32>, candle_core::Error>`:
- Tokenize each `(query, passage)` pair (reuse the same `encode((query, passage), true)` as `score`).
- **Left-aligned pad** every pair to `L_max = max(len)` in the batch: build `(N, L_max)` `input_ids`
  (pad with the tokenizer's pad id), `token_type_ids` (pad 0), `attention_mask` (1 for real, **0 for pad**).
- ONE `self.model.forward(input_ids, token_type_ids, Some(attn_mask))` → `(N, L_max, H)`.
- `[CLS]` is position 0 (never padded): `narrow(1,0,1).squeeze(1)` → `(N, H)` → pooler→tanh→classifier
  → `(N,1)` → squeeze → `Vec<f32>` length N, order preserved.
- **Correctness invariant (the load-bearing test):** because CLS is at position 0 and the attention mask
  zeroes pad positions, `score_batch(q,[p0,p1,..])[i] == score(q,pi)` within tight tolerance. Keep the
  single `score()` for the fallback path / equivalence test.
- Empty `passages` → `Ok(vec![])` (no forward). Tokenize/forward error → propagate `Err` (caller maps to
  neutral per current contract).

### b. `fathomdb-engine/src/lib.rs` — wrapper + wire the call site
- Add `CandleCrossEncoder::score_batch(&self, query, passages: &[&str]) -> Vec<f64>` mirroring `score`'s
  neutral-on-error contract, BUT preserve per-element behavior: on a batch error, fall back to per-pair
  `score()` (so one bad pair can't neutralize the whole pool) — i.e. try batch, on `Err` map each pair
  through the existing single `score`. Document this fallback.
- In `rerank_fused` (~4943): collect `bodies: Vec<&str> = top.iter().map(|h| h.body.as_str())`,
  call `model.score_batch(_query, &bodies)` ONCE, then `zip` logits with the RRF-norm loop. The blended
  math (ALPHA=0.3, sigmoid, sort) is byte-unchanged — only the scoring is batched.

## 3. TDD (Rust, `cargo test -p fathomdb-embedder --features default-reranker`)
- `score_batch` == per-pair `score` within tol (e.g. 1e-3) on a 3-pair fixture of DIFFERENT lengths
  (proves the pad/mask is correct). THE key test (also guarantees a clean from-0 rerun is consistent).
- Determinism: same input twice → identical Vec.
- Varying lengths + a very short and a 512-cap passage in one batch → still matches per-pair.
- Empty batch → `Ok(vec![])`, no forward.
- Engine-level (if feasible without network): `rerank_fused` output order unchanged vs the pre-batch
  per-pair path on a fixture (same blended ranking).
