# CE-rerank precision probe — how much of the +0.39 oracle headroom does a REAL reranker capture?

## 1. Why (the gating question for Slice 20)
The gap-decomp showed **retrieval precision** is the lever: perfect raw-gold retrieval recovers ~+0.39 pp
(an UPPER bound — the oracle also strips distractors a reranker cannot). The realizable precision lever is
the **CE reranker** FathomDB already ships (`fathomdb.rerank`, TinyBERT; the fusion-path `rerank_fused` is
an identity stub today). This probe measures how much of that headroom a real CE-rerank captures **before**
committing the Slice-20 engine build (cheap-validate-before-engine, ADJ-7).

## 2. Method ($0, LLM-free, CPU, deterministic)
Reuse the LME powered-recall harness (`d0b_powered_recall`) + the frozen gold + corpus_hash. Arms, strict
Recall@K (K=10 headline; +@50, median rank):
- **fathomdb** — current fused/FTS retrieval (no rerank) — the baseline.
- **fathomdb_rerank** — take FathomDB's fused top-N pool (N ≫ K, e.g. 50/100), `fathomdb.rerank(query,
  pool, rerank_depth=N)`, take top-K. The realizable precision lever.
- **naive_rag** — BM25 floor (reference).
- **oracle_raw** ceiling — from the gap-decomp (reference only; the +0.39 upper bound).
Pinned: rerank_depth/pool-N frozen; deterministic CE (no RNG); record the CE model identity + pool-N.

## 3. Frozen pass criterion (mirror the probe pattern; pure rule module)
`probe_rerank_pass`: the reranker clears iff the **paired (fathomdb_rerank − fathomdb) Recall@10 margin**
has 95% bootstrap CI **lower bound > 0** at power (per-class MDE ≤ ε=0.05; underpowered ⇒ INCONCLUSIVE,
mirror decide_083). Report the **fraction of the oracle headroom captured** =
`(rerank − fathomdb) / (oracle_raw − fathomdb)` per class (diagnostic, not gated).

## 4. Decision it produces
- **PASS (rerank lifts recall at power):** the precision lever is realizable → **Slice 20 wires the
  reranker** (`Engine.search(rerank_depth>0)` made non-identity in the fused path) + re-measures the
  identical-answerer accuracy vs Mem0 (priced). Highest-leverage in-footprint lever per R=+0.39.
- **FAIL/INCONCLUSIVE:** the CE reranker doesn't capture the oracle headroom → the realizable precision
  gap is the distractor-removal / ranking the CE can't do → reconsider (stronger reranker / PRF /
  candidate-set quality), and the oracle ceiling is partly unrealizable.
