# Gap-decomposition probe v2 — retrieval vs context-form vs reader-residual

> **Orchestrator contract for implementer (TDD) + codex review. v2 folds codex's BLOCKs:** the naive
> 2-way (R)/(A) split is **causally confounded** — removing retrieval also removes distractors AND changes
> context *form* (raw long bodies vs Mem0's short, query-shaped, LLM-extracted memory units). v2 separates
> **retrieval**, **context-form**, and a **reader residual** with explicit controls, and frames every
> output as a **bounded diagnostic, not causal attribution**.

## 1. Question + honest framing (codex BLOCK: not "decisive", bounded)
Decompose the D0b FathomDB−Mem0 accuracy gap to choose the next lever. Each arm is a **bound**, not a
clean causal term: `acc_oracle` is an *upper bound* on what perfect raw-gold retrieval buys (it also
strips distractors PRF/D2 could not). We therefore report a **3-way bounded decomposition** + the
confounds, and the verdict selects a lever *direction*, not a proven cause.

## 2. Arms — identical reader (R2 invariant), one question set, per-question cells
Same reader (gpt-5.4, temp 0/seed 0), one prompt, **same 32 000-char budget + the SAME context fitter as
D0b** (`d0b_parity_run` truncation, so form differences are not a fitter artifact), same
`0.8.3-d0a-memory-gold.json` + corpus_hash + repin_hash. **Per-question cells required** (codex CONCERN):
reuse D0b's **checkpoint** per-question records for `fathomdb`+`mem0` (`0.8.3-d0b-parity-v2.checkpoint.json`
— already paid; verify it carries per-question cells, else re-run those arms), and run the new arms on the
**identical question ids** so all paired CIs are per-question.

Five arms (the middle three are the controls that de-confound retrieval vs form):
- **`acc_fathomdb`** — FathomDB retrieval → reader (D0b cell).
- **`acc_mem0`** — Mem0 retrieval (LLM-extracted memory bodies) → reader (D0b cell).
- **`acc_oracle_raw`** *(new)* — the query's **gold doc(s)**, raw, same fitter/budget → reader. **Gold-fit
  audit (codex v2 BLOCK Q1/Q6):** emit a per-question `oracle_fit_complete` flag = *all* required gold
  bodies are included **untruncated** by the 32k fitter. Questions where the oracle context is **unfit**
  (gold truncated) are **excluded from the decomposition + stratified-reported separately** — an unfit
  oracle is a budget/packaging limit, NOT an answer-formation failure, and must not inflate (Resid)/(F).
  **Fit-coverage floor (codex v3 BLOCK — exclusion changes the estimand):** pre-register a per-class
  coverage floor `FIT_COVERAGE_MIN = 0.80` (fraction of the class's questions with `oracle_fit_complete`).
  A class below the floor is forced **`INCONCLUSIVE`** (selective exclusion of long/multi-doc cases would
  mean the surviving subset no longer decomposes the D0b gap); report the per-class fit-coverage with every
  verdict.
- **`acc_oracle_distilled`** *(new)* — a **query-blind, answer-blind** one-line distillation of each gold
  doc → reader (perfect retrieval, *distilled form* — the like-for-like control vs Mem0's form).
- **`acc_fathomdb_distilled`** *(new, form control on the RETRIEVED set)* — distill FathomDB's retrieved
  bodies (same blind distiller) → reader; isolates whether *form alone* lifts FathomDB's own retrieval.

### Bounded 3-way decomposition (report all with paired 95% CIs + confounds)
- **Retrieval bound** `(R) = acc_oracle_raw − acc_fathomdb` — upper bound on perfect raw-gold retrieval
  (confound: also removes distractors; PRF/D2 cannot fully reproduce → treat as a *ceiling*, not the PRF lift).
- **Form bound** `(F) = acc_oracle_distilled − acc_oracle_raw` — value of *distilled form* at perfect
  retrieval (and cross-check `acc_fathomdb_distilled − acc_fathomdb` = form lift on the *retrieved* set).
- **Residual** `(Resid) = acc_mem0 − acc_oracle_distilled` — **Mem0 form+retrieval residual (NOT pure
  reader answer-formation; codex CONCERN Q2):** it bundles Mem0's memory consolidation/dedup/cross-session
  state AND Mem0 retrieval/indexing choices AND gold-vs-Mem0-store mismatch. Algebraic identity holds:
  `R + F + Resid = acc_mem0 − acc_fathomdb`. A dominant `Resid` therefore points at the **Mem0-form family
  but is NOT proof of memory-formation** — it triggers a *disambiguation follow-up* (§7), not a direct build.

## 3. FROZEN interpretation rule — pinned, mechanical, power-safe (codex v2 BLOCK Q3)
New pure module `src/python/eval/gap_decomposition_rule.py` (stdlib only; mirrors `decision_rule_083`'s
style; **does not edit** the frozen module). Reuses **EPS = 0.05** as the underpowered/materiality band.
Consumes already-computed paired `{point, ci_lo, ci_hi, mde, n}` for (R), (F), (Resid) per class + pooled.
**Dominance requires the winner to beat EVERY rival's uncertainty, not just point-order** (codex Q3 — a
point-largest winner must not be declared while a rival is underpowered and could actually be larger):
- **`X_DOMINANT`** (X ∈ {RETRIEVAL, DISTILLED_FORM, MEM0_RESIDUAL}) iff `X.ci_lo > 0` **AND** `X.ci_lo ≥
  max(other.ci_hi)` over the other two components **AND** `X.mde ≤ EPS` (X itself powered). I.e. even at the
  rivals' optimistic CI-upper, X still wins → no underpowered rival can hide a larger effect.
- **`INCONCLUSIVE`** otherwise (incl. any case where a rival's `ci_hi` exceeds the leader's `ci_lo`, or
  the leader is underpowered) — escalate, do not act. Deterministic tie-breaks pinned.

## 4. Budget + resilience ($30 HITL-approved; D0b spent $10.75 → ledger continues)
- **Pin `gpt-5.4` pricing — hard cap, not a default fallback (codex v2 BLOCK Q5):** the cost tracker has
  no `gpt-5.4` entry; **add explicit price metadata `gpt-5.4 = $1.25 / 1M input, $5.00 / 1M output`** (the
  exact rates the D0b n606 run recorded) and **fail-closed if a priced model lacks pinned pricing** (never
  silently use a default). Carry the **D0b $10.7479** as the ledger's opening balance; the pre-call
  **projected** cost (input+max-output tokens) must keep `ledger + projected ≤ $30` for BOTH reader and
  distiller, checked **before** each call.
- **Per-question cells — checkpoint or hard-stop (codex CONCERN):** load D0b's per-question
  fathomdb+mem0 cells from `0.8.3-d0b-parity-v2.checkpoint.json`; if the checkpoint lacks per-question
  records, **HARD-STOP and surface a paid-re-run decision** — never fall back to D0b aggregate class-means
  (paired CIs against oracle require per-question cells).
- **Cheap-validate FIRST** (`gemini-2.5-flash-lite`, class-balanced n≈40, ~$0.01): wire the 3 new arms +
  the blind distiller + the rule; confirm a directional split before priced spend.
- **Priced**: run **only** `oracle_raw`, `oracle_distilled`, `fathomdb_distilled` on the identical D0b
  question ids ≈ **3×606 ≈ 1818 reader calls** (oracle contexts are SHORTER → cheaper/call). Est.
  **~$6–10**; with the D0b $10.75 opening balance the projected total ≈ $17–21 < $30.
- **Distiller — corpus-level, query-/answer-BLIND, capped (codex CONCERN Q4 + BLOCK Q5):** distill **every
  candidate corpus doc ONCE** with a generic prompt (no query, no answers, no gold labels), persist
  `{doc_id→distilled, prompt, model, hash}`, then **select** gold/retrieved ids from that frozen cache at
  eval time (no label-selected distillation). Distiller = **local Qwen (OFFLINE-BUILD/$0)** or a
  `--max-usd`-capped cheap model, **fail-closed** if unavailable. A test asserts the distiller call sees
  neither `answers` nor `query`.
- Resilient harness (atomic checkpoint, verified `--resume`, 429/5xx backoff, failure≠abstention,
  completeness guard). Determinism: pinned hashes + temp 0/seed 0 + fixed bootstrap seed.

## 7. The decision it informs (a lever DIRECTION, with caveats)
- **RETRIEVAL_DOMINANT** (powered) ⇒ retrieval levers worth it → run 15b `--full` + Slice-20 PRF / D2.
  Caveat: oracle strips distractors PRF/D2 can't, so this is the optimistic case — confirm the PRF lift
  separately.
- **DISTILLED_FORM_DOMINANT** (query-BLIND) ⇒ a short *query-blind distilled form* helps the reader → scope the Mem0-style
  **memory-formation** experiment (offline distill/consolidate → index distilled units → re-measure);
  PRF/D2 deprioritized. HITL footprint-scope decision (fork b).
- **MEM0_RESIDUAL_DOMINANT** ⇒ Mem0's units add beyond distilled gold, but `Resid` bundles Mem0
  retrieval+indexing+consolidation (codex Q2) ⇒ **disambiguation follow-up REQUIRED** before building
  (e.g. ablate Mem0 retrieval vs its memory units); do NOT jump straight to a memory-formation build.
- **INCONCLUSIVE** ⇒ escalate (grow N or accept) — do not act on an underpowered split.
Feeds the Slice-30 resolution + the 0.8.4 (GraphRAG) / HippoRAG sequencing. **Not** a causal proof —
a bounded diagnostic that picks the highest-expected-value next experiment.
