# Gap-decomposition probe — where does the 20-pt FathomDB−Mem0 gap live?

> **Orchestrator contract for implementer (TDD) + codex review.** This is the decisive parity experiment:
> it sizes how much of the D0b accuracy gap is **(R) retrieval** vs **(A) answer-formation**, which
> selects the next lever (PRF/D2 vs Mem0-style memory-formation). Frozen interpretation rule below.

## 1. The question + why it gates everything
D0b: FathomDB −20…27 pp identical-answerer accuracy vs Mem0, with **accuracy gap ≫ recall gap** ⇒
the deficit is *likely* **(A) answer-formation** (FathomDB feeds raw whole bodies into the 32k budget;
Mem0 feeds distilled memories), not just **(R) first-stage retrieval**. The in-footprint recall levers
(embedder ruled out by 15a; PRF; D2/15b) only move (R). **If the gap is mostly (A), building PRF/D2 is
the wrong investment** — the lever is a caller-side/offline memory-formation layer. This probe decides.

## 2. Method — one new arm, identical-answerer (R2 invariant)
Reuse `r2_parity_eval.py` (`LLMAnswerer`, one prompt, one context budget) + `0.8.3-d0a-memory-gold.json`,
the **same corpus_hash + gold repin_hash + reader (gpt-5.4, temp 0, seed 0) + 32 000-char budget as D0b**
(directly comparable). Arms scored on identical-answerer accuracy (the D0b `_match`/abstention metric):
- **`acc_fathomdb`** — FathomDB retrieval → reader (from D0b; re-use or re-run for self-consistency).
- **`acc_mem0`** — Mem0 retrieval → reader (from D0b).
- **`acc_oracle`** *(new)* — feed the query's **gold doc(s) directly** as context (no retrieval) → reader.
- **`acc_distilled_oracle`** *(new, 2nd)* — feed an LLM **one-line distillation** of each gold doc → reader
  (directly tests the Mem0-form hypothesis: does *distilling* the gold help beyond raw gold?).

Per class, paired bootstrap CIs (same fixed-seed helper as the slice probes). Decomposition:
- **(R) retrieval gap** = `acc_oracle − acc_fathomdb` (what perfect retrieval of the raw gold recovers).
- **(A) answer-formation gap** = `acc_mem0 − acc_oracle` (what Mem0's *form* adds beyond raw-gold-in-context).
- **(form lift)** = `acc_distilled_oracle − acc_oracle` (does distillation alone close (A)?).

## 3. FROZEN interpretation rule (pre-registered as code BEFORE the run)
Freeze a pure module `src/python/eval/gap_decomposition_rule.py` (stdlib only, mirrors
`decision_rule_083`'s style; do NOT edit the frozen `decision_rule_083`). Consumes already-computed
paired stats; returns a mechanical verdict per class + pooled:
- `RETRIEVAL_DOMINANT` iff `(R) paired CI_lo > 0` AND `(R) point ≥ (A) point` — perfect retrieval
  recovers most of the gap ⇒ **PRF/D2 are the levers** (15b/Slice-25 worth building).
- `ANSWER_FORMATION_DOMINANT` iff `(A) paired CI_lo > 0` AND `(A) point > (R) point` — Mem0's form adds
  accuracy raw gold does not ⇒ **memory-formation lever** (caller-side/offline), PRF/D2 won't close it.
- `INCONCLUSIVE` iff neither CI_lo > 0 at the per-class paired MDE (underpowered — escalate, don't act).
- `form_lift_material` (separate bool) iff `(form lift) CI_lo > 0` — distillation alone is a credible lever.
Pin the tolerance + MDE handling exactly as the program (ε-class; underpowered ⇒ INCONCLUSIVE).

## 4. Budget + resilience (HITL $30 approved; D0b already spent $10.75)
- **Cheap-validate FIRST** (`gemini-2.5-flash-lite`, class-balanced n≈40, ~$0.01): prove the oracle +
  distilled arms wire up + a directional split, BEFORE the priced pass.
- **Priced pass** (`gpt-5.4`, temp 0/seed 0): oracle arm (≈606 calls) + distilled-oracle (≈606 + a cheap
  distill pass). Est. **~$8–12** — within the **$30** approved; **keep a running ledger** in the output;
  **hard `--max-usd` cap** so it cannot exceed budget. Resilient harness mandatory (atomic checkpoint,
  verified `--resume`, 429/5xx backoff, failure≠abstention, completeness guard) per
  [[priced-runs-need-resilience-before-spend]].
- Determinism: pinned corpus_hash + gold repin_hash + reader temp 0/seed 0 + fixed bootstrap seed.

## 5. Footprint
EVAL-ONLY (the reader is the one approved priced seam; the distillation pass is OFFLINE-BUILD/EVAL-ONLY —
local Qwen or the cheap reader, pinned + fail-closed). No in-library LLM. No engine/Rust change.

## 6. TDD + outputs
- RED: the frozen `gap_decomposition_rule` truth-table (RETRIEVAL / ANSWER_FORMATION / INCONCLUSIVE /
  form_lift); the oracle-arm builds gold-doc context correctly (a fixture catching a wrong-doc/empty-context
  bug); paired bootstrap CI determinism; the $-ledger/`--max-usd` cap halts before overspend.
- GREEN: the runner (extends `r2_parity_eval`); cheap-validate green end-to-end.
- Outputs → `runs/0.8.3-gap-decomposition-n*.json` (per-class acc_{fathomdb,mem0,oracle,distilled},
  (R)/(A)/form-lift + paired CIs + the frozen verdict + $ ledger), `runs/0.8.3-gap-decomposition-report.md`.

## 7. The decision it produces (the parity fork)
- **RETRIEVAL_DOMINANT** ⇒ run 15b `--full` + build Slice-20 PRF / Slice-25 D2 (recall levers justified).
- **ANSWER_FORMATION_DOMINANT** ⇒ scope the **Mem0-style memory-formation** experiment (caller-side/offline
  LLM distillation → index distilled units → re-measure); PRF/D2 deprioritized. HITL scope decision (fork b).
- **form_lift_material** ⇒ strong corroboration for the memory-formation lever.
This is the input to the Slice-30 resolution + the 0.8.4 (GraphRAG) / HippoRAG sequencing.
