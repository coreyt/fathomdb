## 1. Context

Codex pass-2 surfaced two judgment-call findings on EU-6 that the
implementing agent flagged for review and the user directed to fix
without deferral:

- **Concern #1 (Python).** `src/python/fathomdb/_fathomdb.pyi:48` declares
  `embedder_events: list[dict[str, Any]]`. The runtime shape is a tagged
  dict (`{"kind": "...", <variant payload>}`) but the stub erases the
  tag ŌåÆ consumers get no IDE completion, pyright cannot narrow on
  `event["kind"] == "..."`, and the variant payload set is invisible to
  static analysis. The Rust emitter (`fathomdb-py/src/lib.rs:414-446`)
  already discriminates correctly; only the declared type lags.

- **Concern #2 (TypeScript).** `src/ts/src/index.ts:73-87` declares
  `EmbedderEvent` as a single wide interface with `kind:
  "DefaultEmbedderDownload" | "DefaultEmbedderCacheHit" | "MeanVecPinned"
  | string` and every payload field as `Option<T>`-collapsed
  (`bytes?: number | null`). The `| string` opens the discriminant; the
  optional-collapse means `if (event.kind === "MeanVecPinned")` does NOT
  narrow `event.docCount` from `number | null | undefined` to `number`.
  This mirrors the wide `#[napi(object)] struct EmbedderEvent` at
  `fathomdb-napi/src/lib.rs:382-393`, which the napi-rs codegen lands
  into `src/ts/src/binding.ts` as `NativeEmbedderEvent`.

FIX-2 makes both surfaces narrowable WITHOUT changing the runtime shape
the Rust emitter produces. The Python wire payload stays a `dict`; the
RustŌåÆnapi struct stays wide. Only the declared types refine ŌĆö a strict
type-level refinement so existing consumers (the EU-6 GREEN tests at
`src/python/tests/test_use_default_embedder.py:60-61` that string-index
events, the TS tests at `src/ts/tests/use-default-embedder.test.ts:58-59`
that read `ev.kind`) keep passing. The result is per-variant IDE
completion, exhaustive pyright/tsc narrowing in `if event["kind"] ==
"..."` / `if (event.kind === "...")` branches, and a single explicit
seam where the wide napi struct is mapped to the narrow union with a
runtime sanity check.

## 5. Test architecture

### 5.1 Pyright narrowing test (AC-FIX2-3)

`src/python/tests/test_pyright_narrowing.py` invokes pyright as a
subprocess against a fixture file containing the narrowing assertions,
then asserts the expected `reveal_type` output lines are present.

**Fixture file** (`src/python/tests/_pyright_narrowing_fixture.py`,
new): contains a function that exercises every variant of the
discriminated union. Uses `typing.reveal_type` (stdlib on 3.11+) or
`typing_extensions.reveal_type` (3.10 fallback) ŌĆö but because the
fixture is read by pyright, not executed, the import line can be
`from typing import reveal_type  # type: ignore[attr-defined]` on
3.10 without runtime cost (the test never runs it). Pseudocode:

```python
# _pyright_narrowing_fixture.py ŌĆö fed to pyright; not executed at test time.
from typing import reveal_type  # noqa: F401  (analysis-time only)

from fathomdb.types import EmbedderEvent, OpenReport

def consume(report: OpenReport) -> None:
    for event in report.embedder_events:
        if event["kind"] == "DefaultEmbedderDownload":
            reveal_type(event["bytes"])     # expect: int
            reveal_type(event["url"])       # expect: str
            reveal_type(event["sha256"])    # expect: str
        elif event["kind"] == "DefaultEmbedderCacheHit":
            reveal_type(event["cache_path"])  # expect: str
        elif event["kind"] == "MeanVecPinned":
            reveal_type(event["doc_count"])   # expect: int
            reveal_type(event["dim"])         # expect: int
```

**Test driver** (`test_pyright_narrowing.py`): runs `pyright
--outputjson <fixture path>` (or plain text) via `subprocess.run` and
parses the JSON for the `informationDiagnostic` entries pyright emits
for each `reveal_type` call. Asserts each expected string
(`Type of "event[\"bytes\"]" is "int"` etc.) is present.

- Pyright invocation: locate the binary via `shutil.which("pyright")`;
  skip with a clear log message if absent (pyright is in
  `[project.optional-dependencies] typecheck` per `pyproject.toml:17`,
  which CI's dev install pulls in).
- Negative-case strategy: for AC-FIX2-3's exact requirement
  ("must reveal `int`, not `int | None`"), the positive assertion
  (`Type of "event[\"bytes\"]" is "int"`) is sufficient ŌĆö if pyright
  weren't narrowing, the type would be the union of all variant payloads
  (or the `dict[str, Any]` fall-back) and the assertion would fail. A
  separate negative assertion (no narrowing ŌåÆ wider type) is not
  required by the AC and would couple the test to pyright's exact
  un-narrowed output format.
- Latency budget: ~1-3s on a warm pyright cache; one test file. Test
  marked `pytest.mark.slow` if the suite grows a marker convention;
  otherwise it stays in the default invocation per AC-FIX2-3 ("checked
  in CI").

### 5.2 TS compile-time narrowing test (AC-FIX2-5)

`src/ts/tests/embedder-event-narrowing.test.ts` (new) uses
`// @ts-expect-error` to assert that un-narrowed access to a variant
field is a type error, AND that inside an `if (event.kind === "...")`
branch the access becomes type-safe (no `@ts-expect-error` needed).

Structure:

```typescript
import { describe, it } from "node:test";
import type { EmbedderEvent } from "../src/index.js";

declare const event: EmbedderEvent;

// Negative: without narrowing, event.docCount is not on the union.
// @ts-expect-error ŌĆö docCount only exists on MeanVecPinned
const _docCountUnnarrowed: number = event.docCount;

// Positive: inside the narrowed branch, the field is required + number.
if (event.kind === "MeanVecPinned") {
  const docCount: number = event.docCount;  // must compile
  void docCount;
}
if (event.kind === "DefaultEmbedderDownload") {
  const bytes: number = event.bytes;        // must compile
  const url: string = event.url;            // must compile
  void bytes;
  void url;
}
if (event.kind === "DefaultEmbedderCacheHit") {
  const cachePath: string = event.cachePath;  // must compile
  void cachePath;
}
```

The actual check is `tsc --strict --noEmit` (or whatever the existing
`tsconfig.json`'s strict equivalent is) over the test file ŌĆö if the
`@ts-expect-error` directive is wrong (i.e. the line is NOT a type
error), tsc fails. If any positive line fails to compile, tsc fails.
The file's runtime body is empty; `node --test` invocation is a
no-op satisfying the test runner.

B3 should verify that `src/ts/tsconfig.json` already enables `strict`
(it does for EU-6 GREEN ŌĆö see existing test files). If `tsc` is not
already part of `npm test`, B3 adds a compile step (`tsc --noEmit`)
to the test script.

## 6. Mechanism details

### 6.1 Python `_fathomdb.pyi` structure

```python
from typing import Any, Iterable, Literal, TypedDict, Union

class DefaultEmbedderDownloadEvent(TypedDict):
    kind: Literal["DefaultEmbedderDownload"]
    file: str
    url: str
    bytes: int
    sha256: str
    cache_path: str
    duration_ms: int

class DefaultEmbedderCacheHitEvent(TypedDict):
    kind: Literal["DefaultEmbedderCacheHit"]
    file: str
    sha256: str
    cache_path: str

class MeanVecPinnedEvent(TypedDict):
    kind: Literal["MeanVecPinned"]
    dim: int
    doc_count: int

EmbedderEvent = Union[
    DefaultEmbedderDownloadEvent,
    DefaultEmbedderCacheHitEvent,
    MeanVecPinnedEvent,
]

class OpenReport:
    # ... existing fields unchanged ...
    embedder_events: list[EmbedderEvent]
    # ...
```

### 6.2 Python `fathomdb/types.py` structure

Re-export the variant TypedDicts and union from `_fathomdb` (or
re-declare them in `types.py` for the public-API import path ŌĆö
`fathomdb.types.DefaultEmbedderDownloadEvent` is the more discoverable

PyO3 module and conceptually private; `types.py` is the canonical

Change `embedder_events: list[dict] = field(default_factory=list)` ŌåÆ
`embedder_events: list[EmbedderEvent] = field(default_factory=list)`.
Add the three variant names + `EmbedderEvent` to `__all__`.

### 6.3 TS `index.ts` structure

```typescript
export interface DefaultEmbedderDownloadEvent {
  readonly kind: "DefaultEmbedderDownload";
  readonly file: string;
  readonly url: string;
  readonly bytes: number;
  readonly sha256: string;
  readonly cachePath: string;
  readonly durationMs: number;
}

export interface DefaultEmbedderCacheHitEvent {
  readonly kind: "DefaultEmbedderCacheHit";
  readonly file: string;
  readonly sha256: string;
  readonly cachePath: string;
}

export interface MeanVecPinnedEvent {
  readonly kind: "MeanVecPinned";
  readonly dim: number;
  readonly docCount: number;
}

/** Forward-compat fallback for `kind` values not known to this build. */
export interface UnknownEmbedderEvent {
  readonly kind: string;
  readonly [field: string]: unknown;
}

export type EmbedderEvent =
  | DefaultEmbedderDownloadEvent
  | DefaultEmbedderCacheHitEvent
  | MeanVecPinnedEvent
  | UnknownEmbedderEvent;
```

The `UnknownEmbedderEvent` member of the union exists to make the
mapper's fallback path type-sound (D4). Discrimination on `kind ===
"..."` still narrows correctly for the three known variants because
TypeScript's union narrowing prefers more-specific members first.

Mapper (private to `index.ts`, exported as `/** @internal */` for the
unit test):

```typescript
/** @internal */
export function mapEmbedderEvent(n: NativeEmbedderEvent): EmbedderEvent {
  switch (n.kind) {
    case "DefaultEmbedderDownload":
      return {
        kind: "DefaultEmbedderDownload",
        file: n.file!, url: n.url!, bytes: n.bytes!,
        sha256: n.sha256!, cachePath: n.cachePath!, durationMs: n.durationMs!,
      };
    case "DefaultEmbedderCacheHit":
      return {
        kind: "DefaultEmbedderCacheHit",
        file: n.file!, sha256: n.sha256!, cachePath: n.cachePath!,
      };
    case "MeanVecPinned":
      return {
        kind: "MeanVecPinned",
        dim: n.dim!, docCount: n.docCount!,
      };
    default:
      return { ...n } as UnknownEmbedderEvent;
  }
}
```

The non-null assertions (`!`) are sound under AC-FIX2-6's runtime
invariant. If that invariant ever breaks (Rust emitter omits a
required field), the assertion produces a `number`/`string` instead of
`undefined` at runtime ŌĆö the resulting test failure is precisely what
AC-FIX2-6 is designed to catch.

`Engine.openReport()` threads the mapper through:

```typescript
openReport(): OpenReport {
  return interceptSync(() => {
    const r = this.#native.openReport();
    return {
      ...r,
      embedderEvents: r.embedderEvents.map(mapEmbedderEvent),
    };
  });
}
```

(The full spread + override pattern preserves every other field; only
`embedderEvents` is re-mapped.)

## 7. Risks & rollback

### 7.1 Pyright invocation latency

The subprocess pyright invocation in ┬¦5.1 costs ~1-3s warm-cache. One
test file, single fixture; negligible against the rest of the suite.
If it shows up on the CI dashboard, B3 can mark the test
`pytest.mark.slow` and add a separate `make typecheck-tests` job that
runs it (deferred ŌĆö only if the baseline test runtime regresses
visibly).

### 7.2 TS mapper per-open overhead

Mapper is O(events). The event list is small (~0-3 entries per open:
at most three file downloads + cache hits + at most one mean-vec
pinned). Negligible.

### 7.3 Non-null assertions in the mapper

The mapper's `n.file!` etc. assume the Rust emitter populates the
variant-appropriate fields. AC-FIX2-6's runtime test enforces this
invariant in CI. If a future Rust change breaks the invariant, the TS
test fails fast and the bug is caught at PR time rather than at
runtime in user code. Mitigation: if B3 wants defence in depth, the
mapper could use `?? throw ...` patterns instead of `!`, but that adds
noise for a property the test already guards. Recommend `!`; flag if
codex pushes back.

### 7.4 Rollback

FIX-2 is two commits (RED + GREEN, plus optional codex follow-ups).
Revert path:

- `git revert <FIX-2-GREEN-sha>` (and any follow-up SHAs).
- The runtime shape is unchanged across the revert, so any user code
  written against the narrow union still runs (it just loses static
  narrowing). No data-layer implications.

### 7.5 `typing_extensions` dependency drift (Python 3.10)

Per D2, the design avoids importing `Required`/`NotRequired` from
`typing_extensions` because all three variants have all-required
keys. If B3 ends up needing one (e.g. forward-compat reservation),
adding `typing_extensions>=4.0` to
`[project.optional-dependencies] dev` and `typecheck` extras is the
mitigation. Flag this in the GREEN PR description if it lands.

## 8. Alternatives considered

- **Per-variant `#[pyclass]` in PyO3.** Rejected: breaks the
  `event["kind"]` dict-style indexing that existing user code +
  EU-6 GREEN tests use. The dict surface was a deliberate choice in
  EU-6 GREEN per the comment at `fathomdb-py/src/lib.rs:408-413`;
  preserving it is the whole point of choosing TypedDict over
  dataclass.
- **Generate TS discriminated union from Rust enum via napi-rs.**
  Rejected: napi-rs has no first-class tagged-union support. Codegen
  would require forking `napi-derive` ŌĆö out of scope. The
  `index.ts`-side hand-written union (D3) is the canonical pattern
  EU-6 GREEN already uses for every other public type that wraps a
  napi-generated wide shape.
- **Python `@dataclass(frozen=True)` for each variant.** Rejected per
  D1: forces `event.kind` attribute access, breaking the dict
  surface. Would also require the PyO3 emitter to build dataclasses
  instead of dicts ŌĆö a runtime shape change, against D5.
- **Single wide `TypedDict` with all keys `NotRequired`.** Rejected:
  doesn't narrow. `Literal[...]` on `kind` would be a `Union[Literal[...],
  Literal[...], Literal[...]]` but the payload keys would all stay
  optional ŌĆö same problem as the current `dict[str, Any]` for narrowing
  purposes.
- **Open enum on `kind` (the current `| string` escape hatch).**
  Rejected: that's exactly what concern #2 flagged. Closing the
  discriminant (no `| string` in the narrow union) is what enables
  exhaustiveness checking in `switch (event.kind)` consumers. Forward-
  compat is preserved via the `UnknownEmbedderEvent` fallback member of
  the union (┬¦6.3), which has a closed structural shape (no payload-
  field constraints) but does NOT widen the `kind` literal on the known
  variants.
