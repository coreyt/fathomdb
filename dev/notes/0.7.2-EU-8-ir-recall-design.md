## Goal

Add an **IR (information-retrieval) recall** signal to evaluate embedder /
retrieval performance **alongside** the existing **ANN recall** signal ‚Äî they
measure different things and both are wanted:

- **ANN recall@k** (already operational, e.g. `eu7_real_corpus_ac.rs`): overlap of
  the quantized retrieval's top-k with the EXACT f32 nearest neighbors of the SAME
  model. Measures quantization/index *fidelity*; ground truth is self-referential
  to the embedder. (Subject to the quirks PR-2c found: target-self-retrieval
  exclusion, duplicate-body GT.)
- **IR recall@k** (this design, new): fraction of externally-labeled RELEVANT docs
  retrieved in top-k. Measures retrieval *relevance* against ground-truth labels
  independent of the embedding model. Immune to the ANN-harness measurement quirks.

**Hard constraint:** MUST NOT alter ANN recall generation or engine behavior. The
two signals are independent (separate harness, data paths, engine instances).

## Design

### New file: `src/rust/crates/fathomdb-engine/tests/eu8_ir_validation.rs` (~350‚Äì480 lines)

Independent of `eu7_real_corpus_ac.rs`. No imports from it.

```rust
struct IRQuery { text: String, expected_doc_ids: HashSet<String>, relation_type: String, chain_id: String }
struct IRQueryResult { retrieved_doc_ids: Vec<String>, hits: usize, recall_at_k: f64,
                       precision_at_k: f64, reciprocal_rank: f64, ndcg_at_k: f64 }
struct IRRecallResult { mean_recall_at_10, ci_lo, ci_hi, sigma, mean_precision_at_10,
                        mean_mrr, mean_ndcg, per_relation_type: HashMap<..>, per_chain_shape: HashMap<..> }
```

Core functions:
- `load_ir_queries(max_chains) -> Option<Vec<IRQuery>>` ‚Äî graceful skip if chains absent.
- `build_body_to_doc_id_map(&[Doc]) -> (HashMap<String,String>, usize)` ‚Äî first-occurrence
  rule for duplicate bodies; returns dup count for diagnostics.
- `map_bodies_to_doc_ids(&[String], &map) -> Vec<String>` ‚Äî engine returns bodies, not
  ids; map back (unmapped ‚Üí counted as miss).
- `compute_ir_metrics(expected, retrieved, k=10) -> IRQueryResult` ‚Äî recall@k = hits/k;
  precision@k = hits/min(k,|retrieved|); MRR = 1/rank_of_first_hit; NDCG@k (binary rel).
- `compute_ndcg(expected, retrieved, k)` ‚Äî DCG/iDCG, binary relevance.
- `bootstrap_ci(values, resamples=1000)` ‚Äî percentile CI; replicate EU-7's method.
- `measure_ir_recall(engine, &body_to_doc_id, &ir_queries, resamples) -> IRRecallResult`.
- `write_ir_measurements_json(...)` -> `dev/plans/runs/0.7.1-EU-8-measurements.json`.
- `#[test] fn eu8_ir_validation()` ‚Äî gated on `AGENT_LONG`; skip if chains/corpus absent;
  loose sanity assert (e.g. recall@10 >= 0.25 ‚Äî scouting, not a production floor).

### Measurement specifics
- **No target exclusion** (unlike ANN): chain queries reference multiple specific
  relevant docs, not a single self-source ‚Äî all expected ids are valid.
- **No embedder touching**: IR eval calls `engine.search()` (read-only on the
  embedding/quant pipeline) and scores against labels; never calls `embedder.embed()`,
  mean-centering, or quantization.
- k locked at 10. Bootstrap 1000 resamples.
- Report aggregate + per-relation-type + per-chain-shape + diagnostics
  (duplicate_bodies, unmapped_retrieval_results, queries_with_zero_hits).

### Only additive support change
`tests/support/corpus_subset.rs`: add `extract_ground_truth_queries(&[Chain]) -> Vec<IRQuery>`
(new public fn; uses the already-public `Chain` struct; no signature changes).

## ANN-safety / isolation (the hard constraint), verified by the subagent's review

| Aspect | EU-7 (ANN) | EU-8 (IR) | Isolation |
|---|---|---|---|
| Test fn / file | `eu7_real_corpus_ac.rs` | NEW `eu8_ir_validation.rs` | separate; zero cross-imports |
| Embedder | `CandleBgeEmbedder` | not initialized / not called | EU-8 never touches embedder |
| Query set | doc-derived, ~100 | chain-provided, ~400‚Äì600 | disjoint |
| Ground truth | brute-force f32 KNN (computed) | `expected_top_k_doc_ids` (labeled) | independent |
| Mean-centering / quant | applied | not touched | EU-8 read-only on pipeline |
| Engine instance | own TempDir/WAL | own TempDir/WAL | `search()` is `&self` (immutable) |
| JSON output | `0.7.1-EU-7-measurements.json` | `0.7.1-EU-8-measurements.json` | no collision |

Engine APIs used by EU-8 (`open_*_for_test`, `configure_vector_kind_for_test`,
`write`, `drain`, `search`) are all already exercised by EU-7 ‚Äî **no engine API or
implementation changes required**. No new external deps.

## Success criteria
- Compiles + passes `cargo test eu8_ir_validation`; valid JSON; bootstrap CI narrower
  than point estimate.
- EU-7 ANN recall **bit-identical** before/after (isolation proof).
- EU-8 does NOT require `--features default-embedder`.
- Sanity: recall@10 ‚àà [0.20,0.60], MRR ‚â• 0.40, NDCG ‚â• 0.30, <5% zero-hit queries,
  <50 duplicate bodies (transparently reported).

## Risks / mitigations
- Duplicate bodies ‚Üí body‚Üíid ambiguity: first-occurrence rule + report count + conservative
  (count as miss).
- Sparse/odd chains: graceful skip + per-type aggregation (>10 queries/type).
- Runtime: gated on AGENT_LONG; ~5‚Äì10 min dev-box.

## How this complements the PR-2c root-cause finding
The PR-2c investigation showed the EU-7 *ANN* harness understates recall ~6pp via
conservative methodology (target-exclude-after, body-string GT w/ dup bodies). An IR
signal is orthogonal: it measures relevance against labels and is immune to those
quirks, giving a second, decision-relevant view of embedder/retrieval quality. See
`dev/plans/runs/0.7.2-PR-2c-recall-rootcause.md`.
