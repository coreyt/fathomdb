# FathomDB 0.6.0 roadmap

Captures the limited Cypher compatibility layer with enough design detail that
implementation can start on the parser and translator immediately, before 0.5.1 ships.

## Scope

0.6.0 is the **Cypher compatibility** release. One headline item:

- **Limited Cypher-compatible query interface** (`db.cypher(query, params)`).

Full implementation design in `dev/pathway-to-basic-cypher-2026-04-17.md`.
That document supersedes `dev/pathway-to-basic-cypher-2026-03-28.md`.

---

## Dependency on 0.5.1

0.6.0 has a hard dependency on 0.5.1 for the full v1 conformance matrix. The
parser and translator can be built and tested in parallel with 0.5.1 development;
six Cypher constructs are stubbed as `UnsupportedCypherFeature` until 0.5.1 ships:

| Construct | Blocked on |
|---|---|
| `WHERE r.prop = $v` | 0.5.1 item 1 (`EdgePropertyEq` AST variant) |
| `WHERE r.prop > $v` | 0.5.1 item 1 (`EdgePropertyCompare` AST variant) |
| `WHERE r.active = true` | 0.5.1 item 1 (`EdgePropertyEq` with `Bool` value) |
| `RETURN r` / `RETURN r.prop` | 0.5.1 item 1 (`EdgeRow` result type) |
| `WHERE n.prop IN [...]` | 0.5.1 items 3 + unfused `JsonPathIn` |
| `WHERE n.active = false` | 0.5.1 item 4 (`JsonPathFusedBoolEq`) |

The engine surface (`execute_cypher`), Python binding, and full conformance test
suite all depend on 0.5.1 being released before 0.6.0 ships. Parallelism is safe
only at the parser/translator unit-test level.

## Item 11: Limited Cypher query interface

### Problem

FathomDB's native query surface (Python `QueryBuilder`, TypeScript fluent API) is
idiomatic for SDK-first use, but requires learning a FathomDB-specific API. Adoption
from graph-database-familiar developers is gated on a Cypher-compatible entry point.
Adding `db.cypher(query, params)` lowers the adoption barrier without replacing the
native API or adding a second query engine.

### Design reference

Full design, conformance matrix, AST types, translator rules, and build order:
`dev/pathway-to-basic-cypher-2026-04-17.md`.

## Implementation scope

### New code: `crates/fathomdb-query/src/cypher/`

Five files, gated on a `cypher` feature flag:

- `mod.rs` — public re-exports: `parse_cypher`, `translate_cypher`, `CypherError`
- `ast.rs` — `CypherQuery`, `MatchPattern`, `NodePattern`, `EdgePattern`, `WhereExpr`,
  `ReturnClause`
- `parser.rs` — hand-rolled recursive descent; no combinator library; fail-fast on
  first syntax error with character offset
- `translate.rs` — `CypherQuery + params → QueryAst`; stubs blocked items with
  `TranslateError::UnsupportedFeature`
- `error.rs` — `ParseError`, `TranslateError`, `CypherError`

### Engine surface

```rust
// Engine::execute_cypher
pub fn execute_cypher(
    &self,
    query: &str,
    params: HashMap<String, ScalarValue>,
) -> Result<QueryRows, CypherError>
```

### Python surface

```python
def cypher(
    self,
    query: str,
    params: dict[str, str | int | float | bool] | None = None,
) -> QueryRows: ...
```

New exception class `CypherError(FathomDBError)` with subclass
`UnsupportedCypherFeature(CypherError)`.

### TypeScript surface

```typescript
cypher(
    query: string,
    params?: Record<string, string | number | boolean>,
): Promise<QueryRows>
```

## v1 conformance matrix (supported constructs)

See `dev/pathway-to-basic-cypher-2026-04-17.md` for the full matrix. Summary:

**Supported at 0.6.0 ship:**
- `MATCH (n:Kind) RETURN n [LIMIT n]`
- `MATCH (n:Kind) WHERE n.logical_id = $id RETURN n`
- `MATCH (n:Kind) WHERE n.prop = $val RETURN n`
- `MATCH (n:Kind) WHERE n.prop > $val RETURN n` (and `>=`, `<`, `<=`)
- `MATCH (a:K1)-[:TYPE]->(b) WHERE a.logical_id = $id RETURN b`
- `MATCH (a:K1)-[:TYPE*1..N]->(b) WHERE a.logical_id = $id RETURN b`
- `WHERE p1 = $v AND p2 > $v2` (AND chain; no OR)
- `WHERE r.prop = $v`, `WHERE n.prop IN [...]` (after 0.5.1 wired)
- `RETURN r`, `RETURN r.prop` (after 0.5.1 `EdgeRow` available)

**Deferred (raise `UnsupportedCypherFeature`):**
Multi-hop different edge types, `OPTIONAL MATCH`, `WITH`, aggregation, writes,
`OR` predicates, `NOT`, undirected edges, path variables, `RETURN n.prop` projection,
Bolt protocol.

## What 0.6.0 does not touch

- Bolt protocol (no demand signal; expensive)
- `CALL fathom.vector_search(...)` procedure extension
- Multi-hop different edge types (two-query chain; deferred to 0.7.x)
- `OR` predicates (requires `Predicate::Or` in `QueryAst`)
- `ORDER BY` (requires `QueryAst.order_by`)
- `RETURN n.prop AS alias` projection
- Aggregation, `WITH`, `UNWIND`, write operations
- Per-kind FTS table wiring for Cypher (query uses Nodes driver for prop lookup)
- User-picked tokenizer / embedder projections (see `dev/design-user-picked-projections.md`; candidate for 0.7.x)

## Dev-infra follow-ons (parallel track, not blocking 0.6.0 ship)

- **Shared Python lockfile via `uv`** — migrate CI and local dev to install
  from a shared `uv.lock`. Currently dev and CI both resolve `click>=8.1`
  independently, which is what let click 8.2's `CliRunner(mix_stderr=...)`
  removal produce a green local pytest and a red CI pytest simultaneously.
  Scope: rewrite `python.yml` workflow to use `uv sync`; update
  `scripts/setup_dev.sh` to install `uv`; document `uv run pytest` as the
  canonical dev command; commit `uv.lock`. Effort: ~1 hour. Tradeoff:
  contributors must learn `uv`; zero ongoing dep-drift maintenance.
  Mitigation in place today: pre-push runs `pytest python/tests/test_cli.py`
  to catch dep-API breakage locally before CI (workflow audit option 3A).

## Critical path

1. **0.5.1 ships** — unblocks edge property predicates, `JsonPathIn`, `EdgeRow`
2. **0.6.0 development** (can start before 0.5.1 completes):
   - Cypher AST types + parser + translator (stub blocked items)
   - `Engine::execute_cypher` wired to stub translator
   - Python `db.cypher()` binding
   - Conformance matrix tests (both supported and `UnsupportedFeature` cases)
3. After 0.5.1 merges:
   - Wire edge property predicates in translator
   - Wire `JsonPathIn` (unfused) in translator
   - Wire `EdgeRow` in RETURN handling
   - TypeScript `db.cypher()` binding
4. Conformance matrix docs
5. 0.6.0 release
