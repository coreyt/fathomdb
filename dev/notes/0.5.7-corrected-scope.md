# 0.5.7 Corrected Scope

Status: draft 2026-04-24
Supersedes: `0.5.7-scope.md`, `0.5.7-design.md` (both insufficient — they patch 5 Memex symptoms without addressing the class that produced them)
Trigger: two independent architecture reviews (proposal + current implementation), 2026-04-24.

## Principle

1. **Soak is not engineering.** We do not outsource defect discovery to users. If a defect would only be caught by soak, it must be caught by a test instead.
2. **Delete before add.** fathomdb's reliability problem is surface area. Every 0.5.7 change either removes surface or is justified against a test that would have caught the specific Memex breakage.
3. **No punt to 0.6.0/0.7.0 for problems that have already burned a client.** If the retry wrapper papers over a connection-role bug, the retry wrapper is wrong scope — fix the connection role, or eliminate the DDL that triggers it.

## Diagnosis (condensed from both reviews)

**Current implementation is severely over-designed.**

- ~45k LoC hand-written. `admin/mod.rs` 7,150; `coordinator.rs` 6,014; `admin/vector.rs` 1,648; `_admin.py` 1,254; TS `admin.ts` 764.
- 25 schema migrations.
- 45 Python admin methods; 16 vector verbs in one file.
- **Three parallel vector-identity tables live simultaneously**: `vector_profiles` (≤v19), `projection_profiles('*','vec')` (v20), `vector_embedding_profiles` (v25). `ensure_vec_kind_profile` (`bootstrap.rs:1348-1359`) writes to all three to keep `BootstrapReport::vector_profile_enabled` from lying. Three sources of truth for one boolean.
- `configure_vec(kind_or_embedder)` dispatches on `isinstance(str)` (`_admin.py:613-661`) — one verb, two incompatible shapes across two schema generations.
- `EmbedderChoice::Builtin` resolves silently to `None` without `default-embedder` feature. Wheel ships without the feature (`python/pyproject.toml:47`). Error surfaces downstream with wrong message.
- Per-kind `vec_<Kind>` virtual tables (`bootstrap.rs:1318-1347`) force runtime DDL on every `configure_vec_kind`. DDL invalidates `prepare_cached` on shared pool → `SQLITE_SCHEMA` (code 17) floods query path. 12+ coordinator sites affected.

**Test suite is structurally incapable of catching integration regressions.**

- Zero test files (Python OR Rust) exercise the full Memex flow: `register_fts_property_schema` → `configure_fts` → `configure_embedding` → `configure_vec_kinds` → write → `drain_vector_projection` → `semantic_search`.
- `python/tests/test_auto_drain_vector.py:4-14` explicitly comments that an end-to-end test "is only meaningful in the Rust integration suite" — which also doesn't have it.
- `test_profile_management.py` mocks Rust out entirely (lines 112, 419). JSON assertions, not behavior.
- Per-verb Rust tests pass against fresh DBs; nothing exercises cached-reader + admin-DDL interaction, which is the exact `SQLITE_SCHEMA` trigger.

**Product hiding underneath the 45-verb admin:**

```python
Engine.open(path, embedder=...)
engine.register_kind("Note", text_paths=["$.body"], vector=True)
engine.write(nodes)
engine.search("cats").semantic().execute()
```

Four verbs. SQLite exposes 3. Meilisearch ~4. Typesense ~5. fathomdb Python admin: 45. The 41 extras are scar tissue from 4 days of Packs A–H without deletion.

## 0.5.7 correct scope

Deletes first. Fixes second. One test that proves it.

### D1 — Delete dead vector surface

- Delete `vector_profiles` table entirely. Compute `BootstrapReport::vector_profile_enabled` from `vector_embedding_profiles WHERE active=1`. Removes triple-write in `ensure_vec_kind_profile`. Schema migration v26 drops the table.
- Delete `set_vec_profile` (`admin/vector.rs:213-260`), `set_vec_profile_json` FFI (`admin_ffi.rs:199-231`), `SetVecProfileRequest`.
- Delete `configure_vec(embedder)` Python legacy shim (`_admin.py:613-661`). Raise `TypeError` with migration text instead.
- Delete `configureVec(identity)` TS shim.
- Delete `drain_vector_projection_single_tick`. One drain verb.
- Delete `preview_projection_impact` from SDK surface. Fold impact info into the configure call's response where material. (Method stays in engine as an internal helper.)
- Delete `EmbedderChoice::Builtin` variant entirely. Users who want the builtin install a separate `fathomdb-embedder` package (scaffolded in 0.5.7, populated in 0.5.8; for 0.5.7 the variant just goes away). No feature-gated silent-None surface remains.

### D2 — Move recovery tools from SDK to CLI

`rebuild_projections`, `rebuild_missing_projections`, `restore_vector_profiles`, `regenerate_vector_embeddings`, `check_integrity`, `check_semantics`, `trace_source`, `excise_source`, `restore_logical_id`, `purge_logical_id`, `safe_export` — all CLI-only. Users run `fathomdb doctor <cmd>`. Drops ~10 Python methods and equivalent TS/Rust FFI. Recovery is a human-operator surface, not an application runtime surface.

### D3 — Collapse FTS verbs

Replace `configure_fts`, `register_fts_property_schema`, `register_fts_property_schema_with_entries`, `register_fts_property_schema_async` with a single:

```python
admin.configure_fts(kind, tokenizer="unicode61", schema={"paths": [...], "separator": " ", "exclude": [...]})
```

Schema defaults to auto-detected string paths if omitted (closes Memex Issue 1). Async rebuild is the default; sync-wait is a kwarg. Four verbs → one. Also: Rust-side composite so Python and TS are pure thin wrappers (addresses the duplicated pre-flight logic across `_admin.py` and `admin.ts`).

### D4 — Drop TypeScript admin surface

Keep TS for `Engine.open`, `engine.write`, `engine.search`. Delete TS admin (`typescript/packages/fathomdb/src/admin.ts`, 764 LoC). Memex is Python-only; TS admin is 3× release cost on an unstable API with no production consumer. Restore in 0.6.x when Python is stable and the admin surface has collapsed to the declarative shape. Changelog: breaking, TS 0.5.x admin removed, use Python or Rust.

### F1 — Kill `SQLITE_SCHEMA` at the root, not the leaves

Reject the retry-wrapper band-aid. Pick one:

**Option A (preferred — eliminates the class):** flatten per-kind `vec_<Kind>` virtual tables into a single shared `vec_nodes` table with `kind TEXT` column and composite `(kind, row_id)` index. `sqlite-vec` supports this; per-kind partitioning was never justified in the design docs, and query-time `WHERE kind = ?` is a sub-microsecond filter. Runtime DDL on `configure_vec_kind` is replaced by a row insert into `vector_index_schemas`. `SQLITE_SCHEMA` goes away because admin no longer issues DDL at runtime. Schema migration v27 consolidates. One-time backfill on upgrade.

**Option B (fallback if v27 too risky for 0.5.7):** separate admin-DDL connection from read pool. Admin ops go through the writer-actor's serialized write connection, which has no cached-statement cache for reads. Read pool never sees `SQLITE_SCHEMA` because it never sees DDL. Requires auditing `AdminService` connection acquisition — currently mixed. Smaller blast radius than v27 but leaves future DDL-adjacent admin ops exposed.

**Decision:** default to Option A. It is a one-week piece of work (write migration, backfill, update 12 `prepare_cached` sites to drop the per-kind table suffix), and it deletes a schema dimension rather than adding a retry. Option B only if the migration cannot be de-risked inside 0.5.7.

### F2 — Loud builtin gate (if `EmbedderChoice::Builtin` still exists at ship)

If D1's variant-deletion slips past 0.5.7 for any reason: hard-error at `Engine::open` (not silent-None). Error text names the feature flag. `EngineError` splits `EmbedderNotWired` (open-time) from `EmbedderNotConfigured` (no profile row). **Preference is to delete the variant entirely per D1 and skip this fix.**

### T1 — One integration test that actually gates release

`python/tests/test_memex_flow_integration.py`. Requirements:

1. Runs against the **built wheel** installed from `dist/*.whl` into a fresh venv. Not `pip install -e`.
2. Uses a real `FakeEmbedder` (deterministic 16-dim vectors; no mocks of Rust).
3. Performs the full flow: `Engine.open` → `configure_fts("K1", ...)` → `configure_fts("K2", ...)` → `configure_embedding(FakeEmbedder())` → `configure_vec_kinds([(K1, "chunks"), (K2, "chunks")])` → write 50 nodes of each kind → `drain_vector_projection(timeout_ms=5000)` → `semantic_search("query", K1)` asserting ≥1 hit → `close()` → process exits within 5s.
4. **Concurrent read during admin DDL.** A second thread issues read queries during `configure_vec_kinds` and `drain_vector_projection`. Asserts no `SQLITE_SCHEMA` warnings in tracing log. This is the test that would have caught Issue 4. Without Option A above, this test fails; that failure is the release gate, forcing the root-cause fix.
5. **Feature-gate test.** Separate `Engine.open(embedder=EmbedderChoice.InProcess(None_equivalent))` assertion that no embedder → hard error at open, not deferred to drain.
6. Runs on Linux x86_64, Linux aarch64, macOS, Windows. All gate the publish.

This test is the only release gate. No soak. If it passes on all platforms, ship. If it fails, fix. No "soak 48h to see if anything else surfaces" language anywhere.

### P1 — Process (not product, but it's what actually broke)

1. **No more atomic Rust+Python+TS pack merges.** Rust feature ships first with a Rust-level integration test covering the feature's full surface. Python follow-on is a separate PR that only wraps. TS (when reinstated) is a third PR.
2. **Pack is not merged until T1 passes on its branch.** The integration test is authored alongside the feature, not after.
3. **Every 0.5.x release must show net-negative LoC.** Bug fixes may add small amounts; surface additions are only allowed if paired with removals of equal or greater LoC. This is the only backstop against the Pack A–H pattern.
4. **Deletion changelog required.** Every release notes what was deleted. Forces the habit.

## Out of scope for 0.5.7

- Declarative `admin.configure(schema)` (deferred to 0.6.0). D3 buys time; full declarative collapse is a bigger API shape change than we should attempt in a recovery release.
- `admin.configure(schema)` as **the only** public admin call, with primitives demoted to private (0.6.0 correct shape; write the RFC during 0.5.7 so 0.6.0 ships clean).
- Operational-collections surface (17 methods, 1,683 LoC `admin/operational.rs`). Separate product. Either namespace under `db.ops.*` or remove from 0.5.x line entirely. Decide in 0.6.0 planning.

| D3 FTS collapse | −400 |

This is the correct shape of a reliability release: delete 5k lines, add one test. No "soak."

## Risk

- **F1 Option A (vec-table consolidation)** has the largest blast radius. Mitigation: v27 migration is forward-only, backfill is idempotent, written with a kill-switch (config flag that reverts to per-kind tables during rollout; deleted before tag).
- **D4 (TS admin drop)** breaks any TS admin caller. Mitigation: verify none exist; if any, freeze their TS version until they migrate. No known production TS admin users per TypeScript SDK milestone memory (2026-04-07 first working slice, not yet Python-parity).
- **D1 variant deletion (`EmbedderChoice::Builtin`)** breaks wheels that passed `embedder='builtin'`. Mitigation: that code path was already silently broken on the shipped wheel; hard error at open is a strict improvement. CHANGELOG breaking-first.

## Release gate

Pass T1 on all four platforms. That's it. No soak. No "monitor main for N days." If T1 is passing and green CI, tag 0.5.7, publish, done.

## 0.6.0 RFC (written during 0.5.7)

Single-call declarative config:

```python
admin.configure({
    "kinds": {
        "KnowledgeItem": {
            "fts": {"paths": ["$.body", "$.title"], "tokenizer": "unicode61"},
            "vec": {"source": "chunks"},
        },
        "ConversationTurn": {
            "fts": {"paths": ["$.text"]},
            "vec": {"source": "chunks"},
        },
    },
    "embedder": embedder,
})
```

Engine computes diff against current state, applies idempotently, returns outcome (added/unchanged/replaced/rebuild-required per kind per facet). Primitives (`configure_fts`, `configure_embedding`, `configure_vec_kinds`) become private-by-convention in 0.6.0, deleted in 0.7.0. The declarative call is the only documented public admin verb.

The product is then: `Engine.open`, `admin.configure`, `write`, `search`, `close`. Five verbs.
