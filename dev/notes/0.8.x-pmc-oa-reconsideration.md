# 0.8.x PMC OA Reconsideration

Decision: **defer PMC OA for this corpus slice**. QASPER now fills the
previously-empty `paper` source type with 1,585 paper documents and
answer/evidence supervision, so PMC OA is no longer needed to reach the
10K-doc lower bound in 0.8.x.

Current source status, checked on 2026-06-02:

- The PMC Open Access Subset still separates **Commercial Use
  Allowed** content from non-commercial and other terms. The allowed
  bucket is described as `CC0`, `CC BY`, `CC BY-SA`, and `CC BY-ND`.
- PMC states that automated retrieval must use PMC Cloud Service, PMC
  OAI-PMH Service, PMC FTP Service, E-Utilities, or BioC API. Bulk or
  systematic retrieval through other automated paths is prohibited.
- Per-article license terms still control reuse. A future script must
  filter every selected article by its machine-readable article license
  and record PMCID, DOI, license, source package, and retrieval channel.

Expected future scope if included later:

- Select 500-1,000 papers from the Commercial Use Allowed bucket.
- Prefer PMC FTP or OAI-PMH over web scraping.
- Pin the selected PMCID list in-tree.
- Extract title, abstract, section text, authors, publication date, DOI,
  PMCID, license, journal, and references where present.
