# 0.5.2 Scope

## Theme

Patch release: known-issue cleanup + user-reported bug fixes. No new
features; no breaking changes. Scope is intentionally tight so the release
can ship quickly on top of 0.5.1.

## Status (2026-04-18)

Scope cut. Five items confirmed: #1 scale.rs race, #2 TS `configureFts`
parity, #3 FTS5 metacharacter escape (GH #31), #6 TS feedback options-bag

## Items

### 1. scale.rs race: surface actionable assertions instead of thread panics

**Status**: Confirmed (deferred from 0.5.1)
**Breaking**: No

`crates/fathomdb/tests/scale.rs:1494` calls
`handle.join().expect("heavy writer joins")` on spawned heavy-writer
threads. Under WAL lock contention during parallel test runs, a writer
thread can panic; `.expect()` propagates that as the test's failure
without preserving the panic payload or the thread identity. The test
currently flakes under sustained `cargo test --workspace` load and passes
reliably in isolation.

**Work:**

- Replace the raw `.join().expect(...)` with a helper that captures
  `thread::Result<()>`. On `Err(payload)`, downcast `payload` to `&str`
  and `String`, format with the writer index, and emit a `panic!` or
  `assert!` with useful context.
- Add a minimal reproduction test that verifies the helper surfaces the
  panic message (not the flake itself — just the error-surface contract).
- If the root-cause WAL contention is tractable, also bound the join with
  a timeout and emit a diagnostic before the main thread panics.

**Reference**: 0.5.1 CHANGELOG "Known issues", investigation transcript
available via the 2026-04-18 session's `scale.rs race investigation`
agent output.

### 2. TypeScript `configureFts` parity: raise on missing FTS schema

**Status**: Confirmed (post-0.5.1 follow-up)
**Breaking**: Yes (minor — TypeScript callers)

Python `configure_fts` (as of 0.5.1) raises `ValueError` when no FTS
property schema is registered for the target kind. TypeScript
`configureFts` still silently skips re-registration in that case. The
scope decision for 0.5.1 was to accept the divergence and ship; 0.5.2
unifies the contract.

**Work:**

- Update `typescript/packages/fathomdb/src/admin.ts::configureFts`: after
  `setFtsProfile` succeeds, fetch the schema via
  `describeFtsPropertySchema`; if null, throw a typed error matching the
  Python `ValueError` contract (message naming the kind).
- Add a TypeScript test parallel to
  `python/tests/test_profile_management.py::test_configure_fts_raises_when_no_schema_registered`.
- Document the breaking change in the 0.5.2 CHANGELOG Breaking section.

**Reference**: 2026-04-18 ARCH-005 review notes ("Parity Gap" section).

### 3. FTS5 query crashes on unescaped metacharacters (GH #31)

**Status**: Candidate (user-reported bug)
**Breaking**: No

`text_search()` passes user input raw into the FTS5 MATCH expression, so
queries containing `'`, `"`, or other FTS5 metacharacters raise
`SqliteError: fts5: syntax error near "'"`. Hit in production against
Memex's retrieval layer on `User's name`.

**Work:**

- Escape FTS5 metacharacters in the query renderer (`render_text_query_fts5`
  or the SDK-facing wrapper). At minimum: double embedded `"` and wrap
  each whitespace-separated token in double quotes.
- Tests: Rust + Python + TypeScript round-trip tests for queries
  containing `'`, `"`, `(`, `)`, `*`, `:`, `-` at start of token.

**Reference**: `gh issue view 31`.

### 6. TypeScript feedback fast-path options-bag allocation (GH #33)

**Status**: Candidate (perf nit)
**Breaking**: No

TypeScript SDK allocates an options bag on every feedback fast-path call.
Measurable under load. Small scope — rework the type to accept inline
parameters. Pairs with GH #32's options-bag consistency question; decide
whether to also tackle #32 or defer it.

**Reference**: `gh issue view 33`, `gh issue view 32`.

## Release mechanics

Standard 0.5.x release flow:

- CHANGELOG.md entry with Breaking / Added / Fixed sections (#2 in
  Breaking; rest in Fixed).
- Version bump (root `Cargo.toml`, `crates/fathomdb/Cargo.toml`,
  `python/pyproject.toml`, `typescript/packages/fathomdb/package.json`).
- Regenerate `Cargo.lock`.
- Run `scripts/preflight.sh --release` — all three release gates must
  pass (clippy tracing/python, nextest tracing).
- Run `scripts/preflight-CI.sh` for the full CI-equivalent set before
  tagging.
- Tag `v0.5.2` on HEAD. Release workflow handles publishing (now gated
  on the `all-builds-passed` cross-ecosystem aggregator added in 0.5.1).
