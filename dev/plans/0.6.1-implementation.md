# 0.6.1 Implementation Plan

This is the **single source of truth** for 0.6.1 sequencing and execution
posture. 0.6.1 is a **patch release** per
`dev/adr/ADR-0.6.0-no-shims-policy.md` § 54: no API breaks, bugfix-only,
no new verbs, no schema additions. Scope-creep into new features blocks
the release; each slice below verifies against that rule.

0.6.0 GA shipped 2026-05-20 at `edf00f7` (Phase 12 fully CLOSED — see
`dev/plans/0.6.0-implementation.md` § "Phase 12 scoreboard" L805-818).
0.6.1 closes four items that were explicitly deferred at 0.6.0 GA:

1. **AC-012 canonical-runner re-measurement** — text-query p50/p99
   numbers on canonical x86_64 tier-1 CI under `AC_FULL_SCALE=1`. No
   engine code change; numbers + verdict only.
2. **12-TX-OPENREPORT** — Python (PyO3) and TypeScript (napi) bindings
   currently drop `OpenedEngine.report`. Native Rust populates it;
   bindings surface the engine handle only. 0.6.1 surfaces the full
   `(engine, report)` tuple/object on both bindings. Symmetric closure
   (parity matrix § 1 row 2; not a TS-vs-Python gap).
3. **TS↔Python parity matrix close** — `dev/notes/12-TX-parity-matrix.md`
   § 8 lists three gaps; two closed in 12-TX (`CounterSnapshot` fields,
   `test_errors.py LEAF_CLASSES`, `test_scaffold.py`). The remaining
   ❌ row is the OpenReport row from item 2 — so item 2 is the parity
   close. **No additional TXPARITY-N slices needed**; the ❌ inventory
   is already exhausted by OPENREPORT. (Verified 2026-05-22 by re-read
   of parity matrix L29-37 and § 8 summary L137-150.)
4. **Axis-E independence demo** — per Wake decision `d-001`, axis-E
   (`fathomdb-embedder-api`) is structurally independent of axis-W
   (workspace lockstep). The 0.6.0 → 0.6.1 axis-W bump is the first
   post-GA opportunity to demonstrate the discipline holds. No trait-
   surface change in 0.6.1 → axis-E stays at `0.6.0`. Discipline
   check, not a feature.

Plus **Dependabot 4 open advisories** (pyo3 low ×2, js-yaml medium,
idna medium) triaged below.

## Contract inputs

Implementation rule inherited from 0.6.0: **TDD is mandatory** for any
behavior-changing slice (OPENREPORT-py, OPENREPORT-ts, AXIS-E-PIN).
Each starts with a failing test citing at least one `AC-*` id and one
relevant ADR or interface spec. Mechanical slices (DEPENDABOT, BUMP,
RC1, V, GA) use the existing test suite as the gate.

Cherry-pick discipline per `dev/design/orchestration.md` § 5: slices
land on a `0.6.1-rewrite` branch cut from `main`, get reviewed under
`dev/design/orchestration.md` §§ 2-9, then cherry-pick to mainline at
GA. (Per 2026-05-22 HITL: 0.6.0-rewrite is closed; 0.6.1 work happens
directly on `main` via the per-slice worktree pattern — orchestrator
manages branch shape.)

### Ordering rules

  Pack 7 un-defer trigger evaluation has fresh numbers before BUMP.
- `OPENREPORT-py` and `OPENREPORT-ts` share the engine→binding
  contract; sequence py-first per Phase 11 precedent
  (PyO3 contract locked, napi mirrors). May be parallelized once the
  py contract is reviewed.
- `DEPENDABOT` runs any time before `BUMP`; isolated from the other
  slices.
- `AXIS-E-PIN` lands before `BUMP` (BUMP is the demonstration).
- `BUMP → RC1 → V → GA` strictly serial.

### Proposed new ACs (HITL confirms before `dev/acceptance.md` append)

- **AC-068c (proposed).** Python `Engine.open` returns the structured
  open report alongside the engine handle. Caller can access every
  field of the native `OpenReport` struct
  (`src/rust/crates/fathomdb-engine/src/lib.rs:541-548`) under the
  same snake_case identifiers: `schema_version_before: int`,
  `schema_version_after: int`, `migration_steps: list`,
  `embedder_warmup_ms: int`, `query_backend: str`, and
  `default_embedder` (embedder-identity payload). Verified by a
  `tests/test_openreport.py` exercise opening a fresh DB and
  asserting every field populated, AND a legacy-callsite regression
  test asserting `engine = Engine.open(path)` (single-name binding)
  still works bit-for-bit identically to 0.6.0. Cites
  `dev/design/engine.md` § "`Engine.open` success result"
  (spec-locked field subset: `schema_version_before`,
  `schema_version_after`, `migration_steps`, `embedder_warmup_ms`)
  and `dev/interfaces/python.md` (spec post-caveat-removal).
- **AC-068d (proposed).** TypeScript `Engine.open` returns a Promise
  resolving to an object exposing `engine` plus a `report` member
  with the camelCase mirror of the native fields:
  `schemaVersionBefore: number`, `schemaVersionAfter: number`,
  `migrationSteps: ReadonlyArray<MigrationStepReport>`,
  `embedderWarmupMs: number`, `queryBackend: string`,
  `defaultEmbedder` (embedder-identity payload). Verified by
  `tests/openreport.test.ts` AND a legacy-callsite regression test
  asserting `const engine = await Engine.open(path)` still works
  bit-for-bit identically to 0.6.0. Cites
  `src/rust/crates/fathomdb-engine/src/lib.rs:541-548` and
  `dev/interfaces/typescript.md`.
