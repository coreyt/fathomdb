## Verdict: BLOCK

### 1. [high] AC-050a's 0.5.x verb-reroute check is not implemented

Refs:

- `dev/acceptance.md:794`
- `dev/plans/0.6.0-implementation.md:941`
- `scripts/security/ast_scan.py:31`
- `dev/plans/runs/12-S-security-fixtures-output.json:93`

`V05_VERBS` is empty, so the scanner's Rust/Python/TS "0.5.x verb
re-route stub" branches never fire. That leaves one explicit AC-050a
clause unenforced: "zero re-route stubs from 0.5.x verb names." The
closure artifact already documents this as a no-op, so this is not
just missing test coverage; it is missing enforcement.

### 2. [medium] The Rust scanner does not actually exclude block comments/docs, and the self-test does not cover that requirement

Refs:

- `dev/acceptance.md:798`
- `scripts/security/ast_scan.py:77`
- `scripts/tests/test_ast_scan.sh:13`
- `scripts/tests/fixtures/ast-shim/rust/dirty/legacy_admin.rs:1`

`scan_rust()` only skips lines whose trimmed text starts with `//`.
It does not skip `/* ... */` / `/** ... */` bodies, even though
AC-050a explicitly says comments and docs are excluded. The Rust
self-test also misses the crate-root `#![allow(deprecated)]` rule:
the negative fixture places that attribute in `legacy_admin.rs`,
but the scanner only checks `lib.rs` / `main.rs`, so the test
passes on the filename/prefix violations without exercising the
crate-root-specific branch.

### 3. [high] The aggregate AC-036/037 gate is still not demonstrated as a hard CI gate

Refs:

- `scripts/agent-security.sh:20`
- `dev/plans/runs/12-S-security-fixtures-output.json:68`
- `scripts/bootstrap.sh:43`

`agent-security.sh` treats rc=2 as non-fatal unless `STRICT=1`, and
the closure artifact records a passing aggregate run with two
blockers. No in-repo test or workflow wiring proves `STRICT=1` is
what CI will use. Real violations fail loudly, but the blocker path
for AC-036/037 is still policy-by-convention, not a demonstrated
gate.

On the bootstrap side, Linux/macOS gating is directionally correct
(GitHub docs: `ubuntu-latest` = Ubuntu 24.04; passwordless sudo;
`sudo apt-get install strace` plausible). But GitHub's runner
customization docs explicitly recommend `sudo apt-get update`
before install, and the script omits that, so even the proposed CI
remediation is not fully hardened.
