## Setup (what was actually run)

- **Corpus:** AP-News BenchmarkQED, 200 docs (entity-rich: the gemini index extracted 2,620 entities /
  602 community reports across 4 Leiden levels). **N_Q = 200** AutoQ questions = **150 global + 50 linked**
  (linked = multi-hop entity-relationship). EVAL-ONLY, gitignored.
- **Full-strength GraphRAG:** Microsoft GraphRAG 3.1.0 **re-indexed with gemini-3.5-flash** (reasoning off),
  queried `global --community-level 1 --no-dynamic-selection` (**307 level-≤1 reports**, vs the prior run's
  root-only 59). Dynamic selection was the flagged-cost upgrade and was **not** enabled (we STOPped before it).
- **Equivalent model use on every arm:** **gemini-3.5-flash, `reasoning_effort` disabled** for ALL answer
  synthesis (GraphRAG global-query, C, D2) **and** D2 cluster summaries **and** GraphRAG's index-time
  community reports. (Matched index-time + query-time model — the like-for-like the prior run lacked.)
- **Judge:** **claude-haiku** (anthropic) — cross-family vs the google answerers. **5 runs, order-swapped**,
  bootstrap clustered by question, length corroboration. **n = 2000 per metric** (200 q × 5 runs × 2 orders).
- **Arms:** **D2** (product: depth-1 coverage index, 17 nodes, $0.012-class build, cheap CPU query) +
  **C** (fallback: map-reduce QFS over all 278 chunks, no index).

## Registered verdict (`decide_084`, frozen rule)

**Both arms = `NOT_REACHED`**, binding constraint **`underpowered:comprehensiveness`** — comprehensiveness
mde = 0.058 (C) / 0.062 (D2), just over ε = 0.05. This was the **predicted** borderline metric (it was the
widest at the 50-q pilot; N=200 halved it to the edge of the bar). **N_Q=200 is the corpus maximum**
(all 150 global + 50 linked); because the bootstrap clusters by question, more *runs* cannot tighten it —
only more *questions* would, and the corpus is exhausted. So the power technicality is not resolvable on this
corpus, but **the substantive directions are unambiguous on every cell** (C's CIs sit well above 0.5; D2's
comprehensiveness CI sits entirely below it).

## What this means (the honest read)

1. **An almost-graph-free approach CAN match-or-beat full-strength GraphRAG — but only the expensive one.**
   C (read-everything map-reduce) wins decisively and cleanly. The graph/community machinery is not required
   to *win* global sensemaking on this corpus.
2. **The cheap product (D2) does NOT hold up.** Compressing the corpus into 17 depth-1 cluster summaries
   loses too much: D2 loses comprehensiveness and diversity to GraphRAG's richer (level-1) community
   hierarchy. **The "D2 = product, Leiden unneeded" thesis is not supported against a full-strength
   competitor.** D2's prior all-three surpass was, as suspected, substantially a community-level-0 artifact.
3. **Fork E (entity/Leiden graph) re-enters as a live option for the product path.** Per plan §7.1's gate —
   "if the gap reappears [vs full-strength GraphRAG], Fork E re-enters" — the gap **reappeared for D2**. The
   cheap-index product either needs a richer structure (a hierarchy beyond depth-1, i.e. toward Leiden) or a
   different compression. C remains the quality ceiling but is per-query expensive (reads all chunks).

## Honest caveats (load-bearing)

1. **Power:** `decide_084 = NOT_REACHED` on comprehensiveness for both arms (mde 0.058–0.062 > 0.05). Corpus
   is capped at 200 questions; question-clustered bootstrap means more runs won't help. Directions are
   nonetheless unambiguous.
2. **GraphRAG at level-1, no dynamic selection.** Full-strength relative to the prior level-0 run (307 vs 59
   reports), but dynamic community selection (rate every community per query) was not run — it was the
   flagged extra-cost upgrade we stopped before. A dynamic-selection GraphRAG could be stronger still.
3. **Both sides gemini-3.5-flash, reasoning off.** Matched, so the *relative* result is what's measured; a
   frontier or reasoning-on model could shift absolute quality on both sides.
4. **gemini-3.5-flash was flaky on the airlock** (degraded to 17–31s/hang post-restart; run finished via the
   AI Studio path at low concurrency). No degenerate answers resulted (all 600 verified non-empty), but the
   answer model ran at reduced reliability.
5. **Spend figure is a LOWER BOUND.** A harness bug spawned up to 4 concurrent resume-wrappers racing on one
   checkpoint for part of the run; each had a separate in-memory meter overwriting the shared checkpoint, so
   the recorded **$42.34 undercounts true airlock spend** (duplicate calls during the overlap). Answer DATA
   is unaffected (qid-keyed, atomic writes, all 600 valid). Fixed mid-run → finished as a single process.
