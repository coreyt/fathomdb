## Verdict: CONCERN

### 1. [medium] Hardcoded GA tag will go stale after 0.6.1

Refs: `scripts/security/check_removal_changelog.py:172`,
`dev/acceptance.md:814-815`,
`dev/plans/0.6.1-implementation.md:245`

Using `v0.6.0` as the default base is correct for the 0.6.1 patch
train, but it becomes the wrong release boundary once 0.6.1 ships.
AC-050c is defined as checking removals against the release being
cut, so 0.6.2+ should compare from `v0.6.1`, not keep accumulating
from `v0.6.0`. An automatic "latest GA tag on this line" default
would be safer; if that is intentionally deferred, the future
bump/release slice needs explicit ownership of advancing this
constant.

BLOCKER) and short (~minutes); the next 0.6.x patch will
inherit the same pattern.
