rewrite's two primary governance guarantees (allowlist-membership P1, cross-binding parity P2)

### [P1] #1 — the "live command surface" enumeration is hard-coded, so P1 passes vacuously
`src/python/tests/test_surface.py:77-84` (and the TypeScript mirror). `_live_python_command_surface()`
iterates only the hard-coded tuple `("open", "write", "search", "close")` for the `Engine` half.
A newly exposed public command (`Engine.delete`, or any live `read.*` callable) never enters the
`live` set, so `extra = live - GOVERNED_SURFACE_ALLOWLIST` stays empty and `test_public_surface_is_allowlist`
passes — even though the test's stated purpose is to **catch** an unauthorized new command. The
docstring's claim "built from actual symbol presence so the membership check (P1) is honest" is
false for the Engine half. (The `admin.__all__` half *is* properly introspected.) **Fix:** introspect
the real public command surface of `Engine` (e.g. `dir(Engine)` public names) and subtract the
documented non-command surface (instrumentation/control methods `_INSTRUMENTATION`, plus
data/config/error accessors), mapping `open` → `Engine.open`; do the same on the TS side.

### [P1] #2 — the parity test compares against a same-file duplicate, so drift is undetectable
`src/ts/tests/surface.test.ts:93-106` (and `src/python/tests/test_surface.py:101-119`). The "parity"
test compares each binding's `GOVERNED_SURFACE_ALLOWLIST` against a **duplicate literal embedded in
the same file** (Python compares to a local `ts_governed_surface_allowlist` literal; TS compares to a
local mirror). If TS adds `read.list` while Python does not, both suites still pass — cross-binding
drift is **not** detected, so guarantee P2 (SDK parity) is cosmetic. **Fix:** source both bindings'
allowlist from **one shared contract** (a single fixture file both pytest and node:test read), and
assert each binding's real declared/live surface against that single source of truth, so a divergence
between Python and TypeScript fails at least one suite.
