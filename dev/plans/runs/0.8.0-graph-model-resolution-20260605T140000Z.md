# Slice 32 Resolution — FathomDB's intended graph model (model shape + edge addressing)

> Produced by a `claude` xhigh (max-depth) research cycle, 2026-06-05; orchestrator-reviewed (sound;
> corrects the codex Graphiti=fact-as-node conflation). Inputs: FathomDB docs + on-disk schema/engine +
> v0.5.6 (git) + primary external sources (Graphiti/Zep, GraphRAG, HyperGraphRAG, Neo4j, Mem0). This is
> the resolution that feeds the eventual `ADR-0.8.0-graph-model-and-edge-addressing.md` + the HITL call.

**Verdict: (B) + (C).** Edge *identity* is settled (`logical_id`-alone, signed Slice 31). Edge
*addressing* and *model shape* are HITL product decisions, but the technically-determined parts reduce the
human's call to a crisp set. **The status-quo binary substrate is correct for 0.8.0;** the world-class
end-state is a **fact-as-EDGE temporal graph** (Graphiti-shaped) with a node-based provenance/episode
tier — and **almost none of it forces a substrate change now.**

## 0. Disentanglement
- **(a) Multigraph representability — ALREADY YES, dismissed.** Active uniqueness is
  `ON canonical_edges(logical_id) WHERE superseded_at IS NULL` (schema step-12); each typed edge has its
  own `logical_id`; `kind` is never identity (Decision 5).
- **(b) Natural-key addressing / upsert ergonomics — the real, open crux** (supersede today requires the
  caller to supply the prior opaque `logical_id`).
- Decision 5's "edge `kind` buys no real capability" is **correct about identity** (untouched) but is
  *silent* on traversal/addressing, where `kind` carries weight (it is the G5 `edge_type` filter).

## 1. The model-shape taxonomy (corrects the codex conflation)
| Shape | A "fact" IS | Real systems | n-ary? |
|---|---|---|---|
| **(1) Binary edges** (status quo) | a typed directed edge row; no body/confidence/valid-time | Mem0ᵍ, Neo4j default | no |
| **(2) Temporal fact-EDGES** | **the edge**, enriched (`fact` text + valid_at/invalid_at/created/expired) | **Graphiti/Zep** (fact = an edge property; provenance = separate Episodic NODES) | binary |
| **(3) Reified fact-NODES** | a **node** w/ text/confidence/valid-time/source/embedding/**n-ary roles** + SUBJECT/OBJECT/ROLE edges | **GraphRAG claims/covariates** (off by default); HyperGraphRAG (hyperedges) | yes |

**Load-bearing correction:** *Graphiti is fact-as-EDGE, not fact-as-node.* Graphiti reifies its
**provenance** (episodes) as nodes, not its facts. Only GraphRAG-claims / HyperGraphRAG / the research
note reify the *fact* as a node — specifically for **n-ary** facts + per-fact embeddings, which binary
edges genuinely struggle with. **Best fit for an embedded SQLite FTS5+vector single-writer store =
fact-as-EDGE** (G5 is a recursive CTE over `from_id`/`to_id`; fact-as-edge keeps the relationship in the
join columns the CTE already walks: 1 hop = 1 fact; fact-as-node doubles hops). Fact-as-node earns its
keep only for n-ary, which no named 0.8.0 consumer needs — keep it a representable-today escape hatch.

## 2a. Model-shape trade-offs (summary)
- **(1) Binary:** zero substrate risk, matches signed model; but can't express valid-time/bitemporal.
- **(2) Temporal fact-EDGE (RECOMMENDED end-state):** matches the docs' Option 2A + Graphiti + v0.5.6
  (`confidence`/`properties` on edges); valid-time lives where traversal walks; **additive later**
  (ADD COLUMN valid-time/body/confidence), not a reshape. Still binary (no n-ary).
- **(3) Reified fact-NODE:** n-ary + per-fact embedding; **representable today, zero DDL**; but 2× hops,
  ontology departure docs don't endorse, no consumer needs it.

## 2b. Addressing trade-offs (summary)
- **Opaque-`logical_id`-only (status quo, 0.8.0):** matches signed model; but "the A–friend–B edge
  changed" requires finding the prior id first.
- **Natural-`(from,to,kind)`:** MERGE ergonomics; but **collapses distinct facts sharing a tuple** — a
  footgun *if mistaken for identity*; never make it identity.
- **Hybrid (RECOMMENDED future):** keep `logical_id` identity; engine *optionally* derives/dedups the
  active edge from `(from,to,kind)` for the relation-state case + stores a `logical_id`. **Zero substrate
  change; pure write-API affordance; G4–G7 still assume `logical_id` identity.** This is the Neo4j
  MERGE affordance without surrendering explicit identity.

## 3. Coherent combined stances (not a cartesian mess)
- **α — status-quo frozen** (binary + opaque-id): coherent but under-ambitious vs Option 2A.
- **β — doc-aligned fact-as-edge end-state (RECOMMENDED):** temporal fact-EDGES built additively on the
  binary substrate + **Hybrid addressing** as a future write layer + provenance graduating from scalar
  `source_id` to optional traversable edges/episode tier (ontology, representable today). Only stance
  fully consistent with Option 2A + `agent-memory-fit:383-391` + Graphiti.
- **γ — dual graph w/ reified fact-nodes:** coherent, representable today, but unmotivated (no n-ary
  consumer; doubles hops). Keep as escape hatch.
- *Avoid:* fact-as-edge + natural-key-as-identity (forks/collapses, contradicts Slice 31); fact-as-node +
  edge-tuple addressing (category error).

## 4. Recommendation — adopt Stance β
- **Model shape:** intended model = **temporal fact-EDGES on the existing binary substrate** (not a
  reshape to fact-as-node). Fact-as-node = documented escape hatch for n-ary. Provenance edges =
  ontology recommendation layered on the scalar `source_id` that exists today.
- **Addressing:** **opaque-`logical_id`-only for 0.8.0**; document **Hybrid `(from,to,kind)` upsert as the
  intended future write-ergonomics ADR**, never identity.

### Decide / reserve NOW (hard to change post-release) — minimal
1. **Nothing in the schema is forced.** Both enrichments are additive-later → **0.8.0 ships the binary
   substrate unchanged.**
2. **The ONE thing worth reserving now (zero migration):** document in the substrate ADR that the edge
   enrichment columns (`t_valid`/`t_invalid`/`body`/`confidence`) are **reserved-additive** and
   supersession is **invalidate-not-delete** (already true via `superseded_at`). Prose, not a column —
   commits the design so the later valid-time slice is `ADD COLUMN`, not a reshape (Option 2A's intent).
3. **Read Decision 5 as identity-scoped only** (eval-memo note; no edit to the signed decision).

### Defer to 0.8.x
Edge enrichment columns (with G11); Hybrid upsert (future HITL ADR); traversable provenance + episode
tier (ontology, ship on a lineage consumer); reified fact-nodes (escape hatch, on n-ary demand); the
G4–G7 verbs themselves.

### What deferred G4–G7 assume
Identity = `logical_id`-alone; addressing = opaque-id (G7 tuple-history does a lookup-first; Hybrid may
later add MERGE without changing the verbs); `kind` = the G5 `edge_type` filter on the **existing**
`canonical_edges(from_id)/(to_id)` indexes (no new index now; a `(from_id, kind, superseded_at)` covering
index is an additive follow-up if profiling shows it hot). **G7 history scope (currently open):** recommend
**nodes first; edge-history bundled with valid-time (G11)** — "the friend edge changed" IS an edge
supersession, so G7-over-edges and G11 are the same increment.

## 5. HITL product-decision points (the crux for the human)
**Technically determined (no call needed):** edge identity = `logical_id`-alone (signed); multigraph =
already yes; fact-as-node representable today zero-DDL; `kind` already serves G5 traversal on landed
indexes; 0.8.0 substrate ships unchanged.

| # | Decision | Options | Recommended default |
|---|---|---|---|
| **H1** | Intended fact reification | binary / temporal fact-EDGE / reified fact-NODE | **temporal fact-EDGE** end-state; fact-NODE as escape hatch for n-ary |

## ADDENDUM — dual-ontology + performance analysis (claude xhigh, 2026-06-05; orchestrator-reviewed)

Triggered by HITL probing "should FathomDB *leverage both* GraphRAG + Graphiti?" and "does a dual-graph
consumer **force** native support for two graph structures?" (the only thing that would justify a
substrate change NOW). Conclusions (sound; verified against the physical primitives):

- **Category framing:** the dual-system "run both + route" pattern is an **application/consumer**
  architecture, not a substrate one. FathomDB is the substrate the consumers' routers run **on**; it must
  **not** build two graph engines or bake the router's four policy rules (precedence, "user believes X",
  reconciliation, never-overwrite-static) — those are consumer policy. The two GraphRAG/Graphiti
  ontologies **reduce to the same substrate primitives** (entity-nodes + typed-edges + community/episode
  nodes + the same recursive-CTE walk + the same filtered-KNN + `logical_id` supersession).
- **Where the "same primitives" claim cracks — exactly 3 places, all fact-on-edge ENRICHMENT:** edges have
  no `body` (fact text + embedding homeless), no `valid_at`/`invalid_at`/`confidence`, and per-fact
  semantic search is impossible (vectors key to **nodes**, not edges). Structure + traversal reduce to one
  substrate; only enrichment doesn't — and enrichment is **additive**, not a reshape.
- **Performance does NOT force native dual-engines.** Within FathomDB's embedded, single-writer,
  ≤100k–1M envelope, no router workload (global community map-reduce / point-in-time temporal / multi-hop
  CTE / per-fact semantic search) reaches a regime where two native engines beat one well-indexed SQLite
  substrate. Native engines win only at deep/large/distributed traversal — **out of envelope**, and would
  forfeit FathomDB's single-writer/replay/projection invariants. The two graphs differ in **ontology +
  policy, not physical access pattern** (the only thing that would justify divergence).

**Refinements to the recommendation (do NOT change the verdict; sharpen it):**
1. **H1 → ontology-neutral substrate** that **first-classes BOTH** the Graphiti (memory; fact-on-edge
   recommended) AND GraphRAG (corpus/sense-making: entity + relationship + community-summary) ontologies —
   committing the engine to neither. Neutrality is the load-bearing property that lets one engine serve a
   dual-graph router. (Promotes the GraphRAG corpus ontology from "deferred" to "equally-supported
   overlay" — one ADR paragraph, no substrate change.)
2. **NEW — edge-projectability** as a first-class *intended* capability: design the projection seam to
   **admit an edge source** (fact-edges embeddable/FTS-able for per-fact semantic recall), even if 0.8.0
   ships node-only.
3. **H5 sharpens → G7 history is EDGE-INCLUSIVE** for the memory ontology (the temporal-comparison
   workload needs it), sequenced with valid-time (G11) — not "nodes-first, edges-open."
4. **Fact-on-NODE stays the documented n-ary / heavy-per-fact-embedding escape hatch** (Workload 4 — per-
   fact semantic search — is exactly where reification earns its keep; representable today, no DDL).
5. **Substrate-now footprint unchanged in KIND** (still no 0.8.0 schema/code change): the reserve-now set
   is **prose/design intent** — (a) edge-enrichment columns reserved-additive (`body`/valid-time/
   `confidence`; v0.5.6 had `properties`+`confidence`, proven portable), (b) the projection seam designed
   to admit an edge source, (c) G7 declared edge-inclusive-intended. All additive/deferrable; the point is
   to not paint the load-bearing tables into a corner.

**Net:** the world-class move is **one ontology-neutral substrate first-classing both ontologies**, with
**fact-on-edge enrichment + edge-projectability + edge-inclusive G7 promoted from "deferred/open" to
"reserved-additive intended capability,"** and **fact-on-node as the n-ary escape hatch.** No native dual
engine. Lock this structure; release-sequence each piece later.
