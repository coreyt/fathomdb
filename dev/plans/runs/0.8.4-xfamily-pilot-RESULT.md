| comparison (win-rate >0.5 = GraphRAG-style better) | same-family (Qwen judge) | **cross-family (claude-haiku judge)** |
|---|---|---|

ε=0.05; 8 questions × 5 runs × 2 orders = n=80 judgments but only 8 question-clusters).

**The 0.750 "GraphRAG-style wins" signal was self-preference bias** — Qwen judging Qwen-generated
answers. Under an **unbiased cross-family judge, the minimal map-reduce arm LOSES to both
baselines**, most clearly to `long_context` (0.25). This is exactly the LLM-judge failure mode the
four bias controls exist to catch (arXiv:2410.21819), and it fired hard (~0.75 → ~0.25–0.44). The
controls are load-bearing — not ceremony.

## What this means (honest, decision-grade)

1. **Methodology validated, cautionary tale recorded.** Every prior $0 Qwen-judged smoke this
   session (vector_rag 0.833; graphrag 0.750) is now **suspect for self-preference inflation** and
   must not be cited as evidence S1 wins. Cross-family judging is mandatory — confirmed empirically.
2. **The premise looks WEAKER, not stronger, under an unbiased judge.** A minimal GraphRAG-style
   map-reduce synthesis does **not** beat strong baselines here; `long_context` (stuff-the-window)
   wins — consistent with the Samsung "VectorRAG is almost enough" prior the design named as the bar
   to beat. This is a **weak-premise signal for funding the expensive full S1 build.**
3. **NOT yet decisive — the honest caveats:**
   - **Underpowered:** only 8 questions (mde≈0.31). A real verdict needs the powered N (mde ≤ 0.05).
   - **Minimal arm, not real GraphRAG:** this is a 40-article *subset* map-reduce, NOT a full-corpus
     S1 with offline **community summaries** (the hierarchy is the hypothesized value-add; a flat
     subset map-reduce under-represents real GraphRAG). The negative could be the *minimal arm's*
     weakness, not the paradigm's.
   - **Weak answerer:** local Qwen, not gpt-5.4 — a stronger answerer might change the synthesis quality.

## Recommendation (for the HITL fund/no-fund gate)

The cross-family pilot **leans AGAINST** funding the full S1 build on current evidence — but it is
underpowered and uses a minimal arm, so it is not a kill. The cheap, decisive next step before any
large build:
- **Power up this exact pilot** (more questions → mde ≤ 0.05) with the cross-family judge — still
  ~single-dollars (claude-haiku judge is ~$0.001/call). If a properly-sized map-reduce still loses
  to `long_context`, that is a strong, registered weak-premise result → settle toward the publishable
  null without the big build.
- Only if a *powered* minimal arm is competitive (or the loss is plausibly the flat-vs-hierarchy gap)
  does the expensive full-S1 + Microsoft-GraphRAG build earn its funding.

## What this is NOT

Still not the S1-vs-running-Microsoft-GraphRAG resolution: both arms here are FathomDB-side
(map-reduce vs vector/long-context), the GraphRAG-style arm is minimal+subset, and the answerer is
Qwen. But it IS a real, cross-family-judged, `decide_084`-scored measurement — and its main finding
(self-preference overturn + a weak-premise lean) is robust to those caveats.
