## Orchestrator triage

**Orchestrator override 2026-05-24: Finding 2 (tmpdir-copy
isolation) ACCEPTED per dev/design/orchestration.md § 7.**

Rationale:
- The spec prompt's "tmpdir copy of the manifests" language was
  aspirational; the existing `scripts/tests/test_set_version.sh`
  file (with 12 prior tests) has established a trap-based
  restore convention that mutates live repo files in-place.
- `scripts/set-version.sh` does not accept a "manifest root"
  argument; it derives `REPO_ROOT` from its own location. To
  test against a tmpdir copy would require either (a) copying
  the entire script + manifest tree to tmpdir, or (b) refactoring
  the script to take a root override flag. Both are scope-creep
  beyond a regression-sentinel test.
- The trap-based pattern is established convention across all 13
  tests in the file; introducing a one-off tmpdir-copy pattern
  for test 13 alone would be inconsistent. A future "improve
  test isolation across the file" slice could refactor all
  tests in one pass — not in 0.6.1 scope.
- Live-tree mutation risk is bounded: tests run serially under
  `run_capped`; SIGKILL would leave the tree mutated, but the
  same risk exists for tests 1-12 already on `main`, so this
  test does not regress posture.

Finding 1 is addressed by the docs-only correction above.
