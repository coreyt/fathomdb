## Verdict arc (no BLOCK overridden)

**pass-1 BLOCK** — three findings, all legitimate:

1. **(Focus 4) `DEVLOOP_NUMBERS` schema not stable across ACs.** AC-019 emitted
   a different shape (no `samples`/`p50_ms`/`p99_ms`/`recall_at_10`) than
   AC-013/013b, but the doc claimed those keyed fields are the PR-7 contract.
   **Fixed:** AC-019 now emits the identical keyed schema via the single
   `emit_numbers()` helper (percentiles = stress tail); AC-019-specific extras
   (baseline, thread shape, wall time) moved to a separate
   `DEVLOOP_AC019_DETAIL` line. Doc example + schema note updated.
2. **(Focus 3) Real-embedder "requires warm cache" documented but not enforced.**
   The doc asserted a hard warm-cache requirement the code didn't enforce (and
   the handoff actually allows cold — it warms the cache). **Fixed by
   reconciliation:** cold-cache real is explicitly *allowed* (warms PR-5's
   doc-body cache, slower); the doc no longer claims a requirement; real-path
   **latency is now report-only** (so no spurious gate on the candle-influenced
   path). Cache state is surfaced via the `cache=` field.
3. **(Focus 2) Inaccurate real-path comment.** The comment claimed the measured
   real pass is "live candle inference" for queries; in fact the harness caches
   every `embed()` (incl. queries) in memory and the warmup pass primes them, so
   the *measure* pass is embed-cache-warm. **Fixed:** comment corrected;
   **latency gates are now synthetic-only** (synthetic isolates retrieval — the
   canonical-AC-013 rationale), real path carries the **recall floor**. The two
   embedders now carry complementary signals (synthetic→latency, real→recall),
   each path's off-signal report-only.
