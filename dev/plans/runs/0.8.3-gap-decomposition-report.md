## Finding (overturns the D0b answer-formation hypothesis)
The gap is **RETRIEVAL PRECISION**, not answer-formation: `oracle_raw ≈ mem0 + 0.19`, so perfect
retrieval would put FathomDB **above** Mem0 → **parity is reachable IN-FOOTPRINT** via precision levers
(reranker / vector PRF / D2 fielded enrichment). The embedder is still out (Slice 15a NO-SWAP); the
Mem0-memory-formation pivot is NOT supported (the "form" signal is a distiller confound).
