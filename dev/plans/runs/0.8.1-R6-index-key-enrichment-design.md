# R6 — Index-Key Enrichment — design (TDD-first)

> Orchestrated slice following the 2026-06-16 pivot (graph arm does NOT beat BM25 —
> `0.8.1-beat-bm25-report.md`). R6 is the new top recall lever in `IR-C-roadmap.md` §2.
> **Pure eval-layer** — no engine/schema/binding change.

## A. Hypothesis
Appending a session's **extracted entities + fact triples to its OWN doc's FTS content**
(keys *on the doc*, one row per doc) adds lexical-bridge vocabulary that lets a query match a
session via its normalized facts — improving recall@K over BM25/FTS on LongMemEval — **without**
the graph arm (adds 0 / degrades multi_session) and **without** the document-length-normalization
bias of co-mingling 28k separate short entity rows (R-pivot finding). Literature: +9.4% recall on
the LME line (single-author preprint — weak citation).

**Framing (design-review #1):** the literature effect (+9.4%) is *below* the N=40 unpaired MDE
(~15pp), so this run is NOT a "falsification" — it is a **go/no-go SCREEN for a large effect**,
read on the **paired** within-retriever delta (McNemar on discordant pairs — tighter than the
unpaired MDE). **Pre-registered rule:** if the paired `enriched − plain` (FTS, pooled R@10) is
directionally positive and the placebo (§C) does NOT move it, **scale to N=160** to confirm; if
flat/negative, stop. No "confirm/refute" claim at N=40.

## B. Mechanism
`enrich_doc(body, graph) -> str`: append a compact, deterministic block to the original body:
```
<original session body>

[entities] name1; name2; …
[facts] subjA predA objA; subjB predB objB; …
```
- Entities = the session's extracted entity names (deduped, order-preserving). Facts = the
  relation triples ("subject predicate object"). Both from the cached Qwen3.6-27B extraction.
- **One FTS row per doc** (the doc is just longer) — NOT separate entity/edge rows. This DOES
  eliminate the report's avgdl-collapse / Verboseness-Fission mode (28k short rows), which is a
  genuinely different failure path — but it is **not free of length effects** (design-review #3):
  enrichment lengthens docs **non-uniformly** (fact-count varies), so BM25 `b` now *penalizes*
  fact-rich (longer) docs, and appended tokens shift IDF corpus-wide. Net sign is empirical → the
  placebo control (§C) and per-class watch (§F) are how we separate "lexical bridge" from "length
  artifact". Tune `b` only if a length artifact is implicated.
- Empty/missing graph → enrichment is a no-op (body unchanged).

## C. Variants (paired, same questions/gold/seed)
| Variant | Retriever | Docs |
|---|---|---|
| `naive_bm25` | BM25 | plain | (reference baseline — the bar to beat) |
| `naive_bm25_enriched` | BM25 | enriched | (isolates enrichment, retriever-agnostic) |
| `fathomdb_fts_only` | FathomDB FTS | plain | (reference) |
| `fathomdb_fts_enriched` | FathomDB FTS | enriched | (the R6 candidate) |
| `fathomdb_fts_placebo` | FathomDB FTS | placebo-enriched | (**length-matched control**, design-review #4) |

**Placebo:** each doc gets a block of the SAME token budget as its real enrichment but drawn from a
FOREIGN vocab (entity names sampled from OTHER sessions, seeded/deterministic) — wrong content, same
length. **If placebo moves recall ≈ as much as real enrichment, the effect is a length/corpus-stat
artifact, not a lexical bridge.** This is the attribution guard.

**Primary endpoint (pre-registered, design-review #7):** paired `fathomdb_fts_enriched − fathomdb_fts_only`,
pooled R@10. Everything else (BM25 pair, per-class, R@5/20) is **exploratory**. **Harness sanity:**
`naive_bm25` pooled R@10 must reproduce ~0.70 (the report's value) before any delta is trusted.
**Controlled A/B:** `enriched − plain` within each retriever (same retriever/gold; only doc content
differs). recall@K is **LLM-free, $0**; reuses the cached extraction.

## D. Acceptance tests (mechanism-level — NOT a recall threshold; the lift is the experimental outcome)
- **AC-R6-1 enrich-applies + deterministic + dedup:** `enrich_doc(body, {entities,relations})` returns
  a string containing the original body **and** every entity name **and** each relation's tokens;
  **byte-identical across two calls** (determinism); **duplicate entity names collapse** (order-preserving).
- **AC-R6-2 findability + hostile-name safe:** a token that appears ONLY in the extracted facts (not in
  the original body) is present in the enriched body AND retrievable through a real engine build; an
  entity name with FTS/tokenizer-hostile chars (`"`, `:`, `*`, `;`, `[ ]`) ingests + indexes without crash.
- **AC-R6-3 no-op on empty graph:** `enrich_doc(body, {})` == body (no trailing block).
- **AC-R6-4 one-row-per-doc / fairness:** the enriched build writes exactly `len(documents)` doc
  rows (no separate entity/edge rows) — structurally avoids the length-norm-pollution path; and the
  runner emits recall@K for all four variants over the identical gold set.
> The "beats BM25" question is the EXPERIMENT's output (a measured recall delta), deliberately NOT
> an AC — asserting a threshold would presuppose the result.

## E. Measurement
40q (10/class, seed 20260614), recall@K {5,10,20}, per-class + pooled, vs the §3/graph-arm baselines.
Reuse `/tmp/gar_dry/extractions.json` (1,906 cached graphs). Read-only recall loop
(`run_retrieval_loop`). Output `/tmp/r6_n40.json`. MDE ~15pp pooled (N=40) — a clear lift or flat.

## F. Risks / honesty
- **Distractor false-match (design-review #2 — effect is ambiguous in SIGN, not just "lift or flat"):**
  enrichment adds vocabulary to EVERY doc, including distractors. A distractor enriched with a common
  entity now matches queries it didn't, displacing gold from top-K — the SAME mechanism that degraded
  the graph arm's multi_session. So net recall could DROP. Watch per-class (esp. multi_session); the
  placebo (§C) helps separate this from a genuine bridge.
- The answer entity is usually already in the raw session → enrichment may be redundant (lift small).
  The controlled A/B (`enriched − plain`) is the clean read of whether enrichment adds anything.
- Enrichment lengthens docs non-uniformly → BM25 `b` penalizes fact-rich docs / IDF shifts (see §B);
  watch for a regression, not just a lift. The placebo isolates length-artifact from lexical-bridge.
- Extractor quality bounds it (Qwen3.6-27B graphs are rich — not the bottleneck).
- N=40 is noisy; treat as directional; scale only if clearly promising (per §A's pre-registered rule).
- **Follow-up if a lift appears (design-review #5):** entity-only vs fact-only ablation to attribute the
  effect (entity names likely redundant w/ the raw body; relation triples are the novel vocabulary). Cheap.
