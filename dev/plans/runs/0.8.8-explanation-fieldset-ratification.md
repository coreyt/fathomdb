## 2. Aligned field set (ratified) — promote to ADR §A.3

Content identical to landed `src/rust/crates/fathomdb-engine/src/lib.rs:1160-1237`. **Only delta from
landed = `#[non_exhaustive]` on the four structs.**

```rust
#[derive(Clone, Debug, PartialEq)]
#[non_exhaustive]                          // ADD — field-add to all-pub struct is else source-breaking
pub struct SearchResult {
    pub projection_cursor: u64,
    pub soft_fallback: Option<SoftFallback>,
    pub results: Vec<SearchHit>,           // byte-identical; SearchHit UNTOUCHED
    pub explanation: Option<Explanation>,  // None on every explain=false path (zero-cost, verified)
}

#[derive(Clone, Debug, PartialEq)]
#[non_exhaustive]                          // ADD
pub struct Explanation {
    pub trace: QueryTrace,
    pub per_hit: Vec<PerHitExplain>,       // parallel to SearchResult.results, same order
}

#[derive(Clone, Debug, PartialEq)]
#[non_exhaustive]                          // ADD — leaf; absorbs deferred timings_ms (Q3) without a break
pub struct QueryTrace {
    pub query_chars: u32,                  // length only — NEVER the query text (§C)
    pub k: u32,
    pub rerank_depth: u32,
    pub pool_n: u32,
    pub alpha: f64,
    pub use_graph_arm: bool,
    pub recency: bool,
    pub embedder_id: String,               // "name@rev (dim=N)" — benign model identity; "" sentinel, not Option
    pub ce_active: bool,
    pub vector_hits: u32,
    pub text_hits: u32,                    // folds text_edge
    pub graph_hits: u32,
}

#[derive(Clone, Debug, PartialEq)]
#[non_exhaustive]                          // ADD — leaf; absorbs new arms / score components without a break
pub struct PerHitExplain {
    pub id: u64,                           // == SearchHit.id; stable logical_id; NO source_id (leak vector)
    pub arm: SoftFallbackBranch,           // == SearchHit.branch; crosses as lowercase string
    pub vector_rank: Option<u32>,
    pub text_rank: Option<u32>,
    pub graph_rank: Option<u32>,
    pub fused_score: f64,                  // RAW post-recency, pre-CE-blend — NOT min-max normalized
    pub ce_score: Option<f64>,             // == SearchHit.ce_score; sigmoid(ce_logit) ∈ [0,1]
    pub blended: f64,                      // == SearchHit.score
}
```

Carrier (LOCKED, HITL-RATIFIED 2026-06-27): sidecar `explanation: Option<Explanation>` on
`SearchResult`; `SearchHit` byte-identical across all three bindings; `None`/no-allocation on the
`explain=false` path. Entry: `Engine::search_explained` (lib.rs:3016); default `search()` paths pass
`explain=false`.

## 3. ADR amendment text (fold verbatim into `dev/design/0.8.8-explain-and-telemetry-adr.md`)

### 3a. §A.2 carrier (folded from the working-tree edit — see §8)

> ### A.2 Decision: carrier = **sidecar `Explanation` on `SearchResult`** — ✅ HITL-RATIFIED 2026-06-27
>
> `SearchResult` gains `explanation: Option<Explanation>` — `None` whenever `explain=False`.
> `SearchHit` stays **byte-identical** across all three bindings. This is the clean way to satisfy
> **R-OBS-2 (zero-cost when off)**: no per-hit struct growth, no allocation on the hot path, the
> zero-cost bench just proves the `None` path equals baseline.
>
> **HITL ratification (2026-06-27):** the *sidecar* shape is chosen over inline-on-`SearchHit`
> explicitly **because it keeps the non-explanation response shape byte-identical to the current
> `SearchResult`**. Confirmed contract: when `explain=False` (default), explanations are **fully
> suppressed** — `explanation` is `None`, no allocation, and `results` is unchanged.
>
> *Rejected alternative (inline):* embedding `Option<arm_rank/rrf_norm/...>` fields on every
> `SearchHit` changes the result contract on all 3 SDKs even when off and complicates the zero-cost
> proof.

### 3b. §A.3 status change

`### A.3 Proposed shape` → `### A.3 Ratified shape (was Proposed)` — replace the proposed pseudo-shape
with the §2 field set above (the landed shape + `#[non_exhaustive]`). Note the divergences resolved in
favor of the landed code per §1.

### 3c. §A.4 — close Q1–Q3 (was "Open ratification questions")

> ### A.4 Ratified answers (was open questions) — RATIFIED 2026-06-27
>
> - **Q1 (raw vs normalized arm scores): RAW.** `PerHitExplain.fused_score` is the raw post-recency,
>   pre-CE RRF score. Experiments (`rerank_tune_probe.py:105`, `:200`) apply their own min-max; a
>   pre-normalized value would force inversion. Per-arm raw scores are **not** carried (no reader);
>   `*_rank` + the single `fused_score` cover every consumed value.
> - **Q2 (filter-exclusion reporting): RETURNED-SET ONLY.** No `excluded_by_filter`. Filtered eval
>   runs score recall over the returned set (`r2_parity_eval.py:347`); no harness inspects excluded hits.
> - **Q3 (timing granularity): NONE shipped, deferred-safe.** No `timings_ms` in the ratified shape.
>   No harness reads engine timings (whole-run wall-clock is measured caller-side). Timings remain a
>   **known future addition** — `QueryTrace`'s `#[non_exhaustive]` makes adding `timings_ms` later a
>   non-breaking, additive change rather than a contract edit.

### 3d. §B (telemetry/gold) — close Q4 + amend

> - **Q4 (gold-row shape to mirror): align to the ROW SHAPE of `GoldQuery` (`r2_parity_eval.py:118-124`),
>   NOT its id values.** Today's gold is string-keyed over an *external* corpus the engine never sees;
>   telemetry gold lives in the *engine's own id namespace*. These are two corpora in two namespaces —
>   forcing one into the other is a category error. The harness `str(u64)` fallback (`:318`, `:666`)
>   bridges to the row shape with **no shim**.
> - **Id contract (both schemas):** every `u64` id (`TelemetryEvent.result_ids`/`arm_of`/`feedback.*_ids`,
>   `GoldRecord.candidate_ids`/`labels`) is the **stable `logical_id`** — identical to `SearchHit.id`
>   and `PerHitExplain.id`. NEVER the interim `write_cursor`; NEVER an external corpus string id.
> - **`GoldRecord` additive fields:** add `query_id: String` (join key back to `TelemetryEvent.query_id`
>   and to the harness gold-row id; B.2 as written had no id) and `id_space: "engine-logical-id"`
>   (machine-checkable namespace tag).
> - **Privacy:** `source_id` (free-form `Option<String>`, may hold a path/URI/session id) is a leak
>   vector — **never** capture it into telemetry/gold and **never** add it to `PerHitExplain`.
> - **Consumer reality (B.3):** a `GoldRecord` cannot drive the live re-run harness (it re-runs from
>   query *text*, forbidden by §C; and `GoldQuery` has no slot for 0/1 negatives). Its consumer is a
>   **new offline frozen-candidate scorer** (score captured `result_ids` against `labels` over the
>   frozen pool) — scope into Slice 20.
