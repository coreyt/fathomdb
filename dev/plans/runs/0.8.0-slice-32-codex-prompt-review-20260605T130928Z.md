## Graph-model synthesis (codex)
- **For 0.8.x: keep the generic binary property-graph substrate** (nodes + typed directed edges, edge
  identity `logical_id` alone, `kind` as relationship type). It is **enough** for deferred G4 `list`, G5
  `neighbors`, G6 search-expand, G7 history.
- **But document the intended agent-memory model as a DUAL GRAPH built on that substrate:** source/
  episode/chunk nodes + entity nodes + reified fact nodes + provenance edges (`MENTIONS`, `SUPPORTS`,
  `OBSERVED_IN`, `DERIVED_FROM`, `SUBJECT`, `OBJECT`, `ROLE`). This avoids forcing edge rows to carry all
  fact metadata — current edges have `kind`/endpoints/`source_id`/`logical_id`/`superseded_at` but **no
  body/properties/confidence/embedding/valid-time** (schema:111/198/302); Fact **nodes** carry that in
  `body` today. Aligns with GraphRAG (entity/relationship/community-summary), Graphiti/Zep (temporal
  timestamps on facts/edges), Neo4j GraphRAG (vector+FTS+traversal hybrid), HyperGraphRAG (binary-only
  struggles with n-ary).
