# G0 Phase-1 design review — findings (independent, adversarial) — 2026-06-14

> Independent design review of `dev/design/slice-G0-design.md` (the G0 Phase-1 memo)
> + the capability map (`dev/plans/runs/0.8.1-g0-capability-map-20260614T175634Z.json`)
> against the code. **This SUPERSEDES the memo's E0a framing.** Verdict: the memo is
> well-grounded on plumbing, but E0a as written ships *correct-but-inert* code +
> contrived-green tests. Reframe required before Phase-2 TDD.

## 🔴 BLOCK-1 — E0a is necessary-but-NOT-sufficient; the graph arm doesn't seed
The graph arm seeds from the top-10 two-arm hits = **doc nodes**. Doc nodes are
written with **`logical_id = None`** (only the entity/edge ingest arms set a
logical_id), so the seed resolver (`SELECT logical_id … WHERE write_cursor=?`,
`lib.rs:5379`) returns `Some(None)` → the `Some(Some(lid))` guard fails → **seed
dropped → BFS frontier empty → zero graph candidates.** Even if doc nodes had a
logical_id, **edges connect *entity* logical_ids; no doc→entity edge is written** →
a doc seed has no incident edges regardless. So the `source_id` carrier is necessary
but **inert**: graph recall stays 0 because the arm emits no hits.
**The real unblock is a graph-arm *seeding* redesign** (seed from query-matched
*entities* / edge-fact hits, not doc nodes) — a **graph-algorithm change, OUT of G0
scope.** → **Reframe E0a:** ship the `source_id` carrier + the **frontier
instrumentation whose primary deliverable is to PROVE `resolved-seed-rate ≈ 0`**;
defer recall to a new **graph-arm seeding slice**. Do NOT claim G0 fixes graph recall.

## 🔴 BLOCK-2 — `LIMIT 1` source resolution is wrong + non-deterministic
The memo resolves a node's source via one arbitrary incident edge (`… WHERE
(from_id=? OR to_id=?) … LIMIT 1`, no `ORDER BY`). An entity spans many sessions →
arbitrary single source **under-credits recall** (the gold session may be a
*different* incident edge) and is **non-deterministic**. → **Carry the *traversed*
edge's `source_id`** (add `e.source_id` to the BFS `edge_stmt` SELECT already
running — free, deterministic, semantically correct: the traversed edge's source is
the session that stated the connecting fact), or emit **one hit per distinct
incident source** (a set). Keep a `None` fallback (direct-write edges may have NULL).

## 🔴 BLOCK-3 — E0a RED tests are contrived-green (Slice-25 vacuous-green trap)
`test_graph_arm_hit_carries_source_id` and the frontier test only pass on an
**entity-seeded** setup the real eval never produces (per BLOCK-1, doc seeds → no
hit). A green there masks the real 0. → Make the instrumentation test **lock
measurement accuracy**: doc seeds ⇒ assert `resolved_seed_rate == 0`; entity seed ⇒
assert `== 1`. Document the contrived connectivity in the hit-carries-source test
(comment → the seeding slice).

## 🟠 CONCERN-4 — byte-stability framing misdirected; TS/conformance not optional
`SearchHit` has no `Default`/`..` → the Rust compiler **forces** all four two-arm
sites (a missed site is a compile error, not a silent byte change), so the Rust RED
test is near-redundant. The real missable surfaces: **Python SDK dataclass
(`types.py`), `engine.py` passthrough, the TS binding (`src/ts/src/binding.ts`), and
the conformance allowlist.** Don't defer TS — verify the allowlist + mirror
`source_id`, per [[conformance-rewrite-vacuous-green-trap]].

## 🟠 CONCERN-5 — E0b sub-doc→session id mapping = silent recall bug
Chunked sub-docs get suffixed `source_doc_id`s (`sess123#c2`); both the cursor map
(`doc_id_of`) and the E0a source resolution must **canonicalize back to the bare
session id** or every chunked session under-recalls. → Specify
`session_id_of(source_doc_id)` used by both + a RED test that chunked-session hits
resolve to the bare session id.

## 🟠 CONCERN-6 — PRE-1 (b) sound, but the product implication is bigger
Sanctioning the test-hook for eval is right for the slice. BUT
`production_vector_kind_surface=[]` means **no caller can enable dense retrieval on
their ingested kinds — fused/dense is non-productionizable today.** Escalate from
"reserved note" to a **roadmap/product gate** (fused recall is test-only until a
vector-kind registration surface ships).

## 🟡 NIT-7 — PRE-3 precision
"Idempotent re-run" is wrong (`ALTER … ADD COLUMN` errors on re-apply; safe only
because `migrate()` filters `step_id > before`) → state "migrate() second call is a
no-op." Node provenance-stamping is **real plumbing** (`PreparedWrite::Node` has no
provenance fields → new struct fields + write-plan + INSERT columns), not just the
ALTER — keep it in the work estimate.

## 🟡 NIT-8 — PRE-2 reader probes are N=1
`usable_reader_ids` is necessary-not-sufficient (one tiny call ≠ sustained throughput
under PRE-2 concurrency). Re-validate before the E1 at-scale run.

## Confirmed sound (no change)
String `source_id` carrier rationale (`SearchHit.id` is u64); node-source-is-NULL
diagnosis; PRE-3 `ready`-only additive (Memex-confirmed, `FATHOMDB-CONSULT-3`); the
4.7 sess/min throughput escalation; the §7 reader re-anchor escalation.

## Net (the one thing to fix before Phase 2)
Reframe E0a around the **frontier instrumentation as the deliverable**, fix source
resolution to **traversed-edge / source-set** (BLOCK-2), make tests lock
**measurement accuracy** (BLOCK-3), and create a separate **graph-arm seeding slice**
as the real graph-recall unblock. Base-retrieval (E2) is unaffected.
