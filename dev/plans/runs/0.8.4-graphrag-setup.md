# 0.8.4 — Reproducing the running Microsoft GraphRAG stand-up

Full recipe to recreate the Microsoft GraphRAG 3.1.0 used in the literal head-to-head
(`0.8.4-vs-microsoft-graphrag-RESULT.md`). The indexed workspace is preserved (gitignored) at
`data/corpus-data/0.8.4-graphrag-artifacts/microsoft-graphrag-workspace/` — re-queryable directly.

## 1. venv + install (PEP 668 blocks system pip; use a venv)
```sh
python3 -m venv /tmp/gtest-venv
/tmp/gtest-venv/bin/pip install graphrag==3.1.0
# exact pin set: data/corpus-data/0.8.4-graphrag-artifacts/graphrag-venv-pip-freeze.txt
```

## 2. local embeddings shim (airlock + local vLLM serve NO embeddings; global-search barely uses them)
`0.8.4-graphrag-embshim.py` — a stdlib OpenAI-compatible `/v1/embeddings` returning deterministic
hashing-BoW vectors (dim 384). Run it on :8090:
```sh
/tmp/gtest-venv/bin/python dev/plans/runs/0.8.4-graphrag-embshim.py &   # serves 127.0.0.1:8090
```

## 3. workspace + input docs
```sh
mkdir -p /tmp/grag/input
# write N AP-News article bodies as input/doc_*.txt (eval.apnews_corpus.load_articles()[:N])
printf '\n\n' | /tmp/gtest-venv/bin/python -m graphrag init --root /tmp/grag   # accept defaults
```

## 4. settings.yaml — point at the airlock (gpt-5.4) + the shim (NO secret on disk)
```yaml
completion_models:
  default_completion_model:
    model_provider: openai
    model: gpt-5.4
    api_base: http://localhost:4000/v1        # airlock proxy
    concurrent_requests: 4
    api_key: ${GRAPHRAG_API_KEY}              # provided via ENV at runtime, never written to .env
embedding_models:
  default_embedding_model:
    model_provider: openai
    model: local-embed
    api_base: http://127.0.0.1:8090/v1        # the shim
    api_key: ${GRAPHRAG_API_KEY}
```
Leave `.env` free of the secret (the auto-mode classifier blocks writing the master key to a file).

## 5. index + query (key held only in the transient env)
```sh
set -a; . ~/projects/airlock/.env; set +a            # loads AIRLOCK_MASTER_KEY
export GRAPHRAG_API_KEY="$AIRLOCK_MASTER_KEY"         # transient, not on disk
/tmp/gtest-venv/bin/python -m graphrag index --root /tmp/grag      # entities → Leiden communities → 115 reports
/tmp/gtest-venv/bin/python -m graphrag query -r /tmp/grag -m global "What are the main public health themes?"
```

## Notes
- Indexing 15 docs ≈ a few minutes + a few $ on gpt-5.4. Output parquets in `output/` (entities,
  relationships, communities, **community_reports**, text_units) + `lancedb/`.
- The comparison harness: `dev/plans/runs/0.8.4-vs-microsoft-graphrag.py` (calls `graphrag query -m global`
  via subprocess, judges cross-family vs FathomDB arms over the same docs).
- To scale to a registered run: index the full corpus (more $/time), raise N questions for `decide_084`
  mde ≤ 0.05, and add the length/directness control to the verdict.
