# FathomDB 0.8.2 / M1 — Does a knowledge graph beat a strong retrieval baseline on multi-hop QA?

**Verdict: NO-GO (robust).** A lexically-seeded Personalized-PageRank graph arm fused with BM25
(`ppr_fusion`) does **not** materially beat a fused-RRF baseline on multi-hop question answering.
Date: 2026-06-19. Reader: `gpt-5.4`. Status: stage-1 complete, signed off, no stage 2.

---

## 1. Hypothesis

A document **knowledge graph** (entities + edge-facts extracted from the corpus, traversed by Personalized
PageRank and seeded from the lexical top-K) supplies *multi-hop bridging* that a flat passage retriever
cannot — so fusing a graph-derived ranking with BM25 should **improve answer accuracy on ≥3-hop questions**,
where the answer requires chaining facts across multiple documents.

Formally (pre-registered): the **primary endpoint** is the pooled ≥3-hop **ΔF1 = F1(ppr_fusion) −
F1(fused-RRF)**. The hypothesis predicts ΔF1 ≥ **+0.04** (the pre-registered `MATERIAL_F1_LIFT`) with a CI
excluding 0. The null/alternative we actually entertain after 0.8.1: the graph adds ~0 over a strong
lexical+dense baseline ([[graph-arm-doesnt-beat-bm25-pivot]]).

## 2. Experiment

A 5-arm, identical-answerer adjudication on MuSiQue-Answerable (distractor setting). All arms retrieve over
the *same* per-question candidate pool and are read by the *same* LLM under the *same* prompt, so any
difference in answer F1 is attributable to **retrieval ordering**, not reader variance.

| Arm | Retrieval |
|---|---|
| `bm25` | lexical BM25 |
| `passage_dense` | dense bi-encoder (BGE, CLS-pooled) |
| `fused` *(comparator)* | reciprocal-rank fusion of BM25 + dense, k=60 |
| `fused_rerank` | `fused` pool re-ranked by a cross-encoder (TinyBERT-L2) |
| `ppr_fusion` *(treatment)* | Personalized PageRank over the doc graph (α=0.85, lexically top-K seeded), RRF-fused with BM25 |

**Endpoint & statistics.** Primary = pooled ≥3-hop ΔF1 (ppr_fusion − fused), **paired bootstrap** 95% CI
(n_boot=2000, seed=0). Secondary = per-hop ΔF1 + an OLS hop-trend slope, and ΔEM. A frozen, code-defined
`decide()` rule (material / trend / EM / confident-wrong / power gates) renders the mechanical verdict; the
**effect size + CI is the load-bearing scientific read**.

## 3. Setup

- **Corpus / sample:** 300 graph-covered, answerable MuSiQue questions; hop distribution {2-hop: 156,
  3-hop: 94, 4-hop: 50} → **144 ≥3-hop** questions in the primary cell. Dataset pinned by content hash
  (`musique_hash`), graph pre-extracted and cached (`graph-cache/0.8.2-m1-v1/`).
- **Reader:** `gpt-5.4` (frontier), temperature 0, seed 0. Cheap-validate first (15 Q, completeness 1.0),
  then the full priced pass.
- **Comparator decision (HITL, 2026-06-19):** comparator = **fused-RRF** (not dense or bm25), chosen from a
  $0 bridge-vs-answer diagnostic showing fused-RRF best balances bridge coverage and clean composition;
  `MATERIAL_F1_LIFT` set to **0.04**.
- **Run integrity:** completeness **1.0** (1500/1500 cells), **0 errors**, cost **$2.50**. The harness is
  resilient by construction — auto-resume from an atomic incremental checkpoint, 429/5xx backoff, and a
  failure≠abstention distinction (a failed call is re-called on resume, never scored as a 0-F1 abstention).
  A validity guard fails any run below `answer_completeness` 0.97.
- **Distinctness guard:** `ppr_fusion ≠ bm25` on 66/300 questions (0.22) — the graph arm is materially
  distinct, not a vacuous copy of the lexical ranking.

## 4. Findings

**5-arm pooled ≥3-hop F1:** `passage_dense` **0.487** > `fused` **0.450** > `fused_rerank` 0.415 >
**`ppr_fusion` 0.410** > `bm25` 0.370.

**Primary endpoint — ΔF1 (ppr_fusion − fused-RRF), pooled ≥3-hop:**

> **ΔF1 = −0.0405**, 95% CI **[−0.116, +0.031]** (n=144). ΔEM = −0.035, CI [−0.111, +0.042].

- The point estimate is **negative** (graph worse by ~0.04 — exactly the materiality threshold, with the
  wrong sign). The CI includes 0, so it is not *significant* harm.
- **Decisively, the CI upper bound (+0.031) is below the +0.04 materiality threshold.** The best case the
  data supports still does not reach a material graph win. More N cannot rescue a sub-material best case.
- **Per-hop ΔF1 are uniformly negative:** 2-hop −0.016, 3-hop −0.044, 4-hop −0.033 (none individually
  significant). The hop-trend slope is −0.013 (not significantly negative) — no evidence the graph helps
  *more* as hops increase, which is where the hypothesis predicted its advantage.
- **`decide()` = NO_GO** (formally via the power gate, n=144 ≪ the 1165 the whole-rule power-sim wants;
  expected for a stage-1 cell — the scientific read is the effect size above).

**Cross-reader robustness.** The direction is invariant to the reader: the gemini-3.1-pro 81% partial
(ppr 0.227 < fused 0.240), the cheap gpt-5.4 pass (fused 0.395 > ppr 0.379), and this full gpt-5.4 run all
rank the graph arm below the comparator. A local qwen3.6-27b reader (gold-passage F1≈0.53) was evaluated as
a free alternative; capable but weaker, retained only as a direction cross-check.

## 5. Conclusion

The hypothesis is **not supported.** On multi-hop QA, the lexically-seeded PPR graph-fusion arm does not beat
— and point-estimates slightly *below* — a fused-RRF baseline, with the materiality threshold ruled out at
the optimistic edge of the CI and a flat-to-negative hop trend. This corroborates the 0.8.1 finding that the
graph arm adds ≈0 recall over a strong lexical/dense baseline ([[graph-arm-doesnt-beat-bm25-pivot]]).

**Decision:** record **NO-GO**; **no stage 2** (a clear non-win needs no added power). Redirect 0.8.3 away
from graph-traversal retrieval and toward **index-key enrichment**. Secondary observation worth carrying
forward: `passage_dense` was the single strongest arm here (0.487 > fused 0.450) — dense retrieval, not
graph structure, is where the multi-hop signal lived.

**Caveats.** (1) The confident-wrong guard is **UNEVALUATED** — this graph covers only answerable questions,
so there is no unanswerable contrast set; that gate is a placeholder, not a measured result. (2) The CI
includes 0; the NO-GO rests on *point-estimate negative + sub-material CI upper*, not on statistical
significance of harm. (3) Comparator and threshold were HITL-frozen pre-run; `decide()` is frozen as code.
