Binds `dev/plans/plan-0.8.2.md` §4 (Slice 15) and the SIGNED

## What it delivers

`src/python/eval/m1_ppr.py` — the HippoRAG-style graph arm under test (HippoRAG
NeurIPS 2024 arXiv:2405.14831; HippoRAG 2 arXiv:2502.14802), the principled
lightweight replacement for the 0.8.1 raw-BFS arm (the documented anti-pattern;
[[graph-arm-doesnt-beat-bm25-pivot]]). PPR strictly dominates BFS, so if PPR-fusion
cannot beat the fixed `fused-RRF` comparator at Slice 20, BFS never would.

The arm produces a ranked passage-index list over a question's identical ~20-passage
pool (same interface as the baseline arms), via the design's 4 steps:

1. **Build the per-question entity graph** from the Slice-10 preserved extractions
   (union over the question's paragraphs): nodes = entities ∪ relation endpoints
   (normalized name), edges = the extracted relations — **body-less** (predicate text
   is irrelevant to PPR topology). Stable sorted node + edge insertion → byte-
   deterministic PageRank.
2. **Seed lexically (NOT self-seeding):** the entities appearing in the BM25 top-`K`
   passages (reuse `m1_baseline.bm25_rank`), **IDF/specificity weighted** by entity
   passage document-frequency `log((N+1)/(df+0.5))` so hub entities don't dominate
   (the 0.8.1 entity-co-mingling failure). FTS5-free, $0.
3. **Propagate** ONE Personalized-PageRank pass biased to the weighted seeds
   (`networkx.pagerank(personalization=…)`, scipy sparse backend). Deterministic:
   fixed damping `α=0.85` (teleport/restart = 0.15), fixed `max_iter`+`tol`, stable
   node order, no RNG.
4. **Rank + fuse:** passage score = summed PPR mass of the entities it mentions; rank
   passages by that; **RRF-fuse with BM25 at k=60** (reuse `rrf_fuse_scored`).

Because the arm is lexical + graph only (no dense embedder), the CLS-pooling
constraint that governs the dense-derived baseline arms does **not** apply here.

## Design choices worth flagging

- **Passage membership = the `entities` list only** (deterministic). Relation-only
  phrase nodes (names that appear in a relation but no `entities` list — e.g. the
  extractor's `'Tripartite'`) still become graph nodes that carry/relay PPR mass, but
  contribute to no passage's score and can never be a seed. This makes passage
  membership, the seed set, and the IDF document-frequency a single well-defined
  function of the `entities` lists (so the RED fixtures are provable by construction)
  while preserving full relation topology for propagation.
- **Convergence sizing.** Near-bipartite entity graphs make PageRank power-iteration
  decay *oscillate* at rate `α` (the negative second eigenvalue), so reaching `tol`
  needs `log(tol)/log(α)` iterations (~85 for `tol=1e-6` at `α=0.85`). `max_iter=1000`
  is a wide deterministic margin; `tol=1e-6` is the networkx default. A too-strict
  `tol` with too-few iters silently fell back to the seed vector — masking propagation
  (caught in RED→GREEN; the fallback now retries at 10× budget, never silently
  collapses).

## Integration

`eval.m1_ppr` is a **parallel** `retrieve_ppr` arm — it deliberately does NOT touch
`m1_baseline.retrieve_arms` (the four baseline arms stay byte-identical).
`add_ppr_fusion_arm(arm_rankings, question, question_extractions)` is the Slice-20
hook that emits `ppr_fusion` alongside the baseline arms.

## The four load-bearing properties (AC-M1-15a..d), pinned RED→GREEN

`src/python/tests/test_m1_ppr.py`, small constructed fixtures (a tiny graph + a
4-passage pool whose BM25 top-2 is provably `{0,1}`):

- **(a) determinism** — identical input → byte-identical ranking *and* byte-identical
  passage scores / PPR masses (run twice, assert equal).
- **(b) restart→1.0 collapse** — at `α=0.0` (teleport=1.0) PageRank returns the
  personalization vector exactly: the propagated (non-seed) entities have **exactly
  zero** mass and the ppr-only top-2 == the BM25 seed passages; at `α=0.85` those same
  entities gain strictly positive mass. Proves propagation is wired.
- **(c) IDF-weighting is live** — a hub-vs-specific fixture (two symmetric graph
  components) where uniform weighting ties the two non-seed passages (index order) but
  IDF weighting flips them — the two ppr-only rankings differ.
- **(d) fusion no-regression** — when both BM25 and PPR rank the seed passages above
  the rest and agree on their order, RRF (k=60) never drops a BM25-top-K passage below
  its BM25-alone rank (the `pr_g9`-style pin).
