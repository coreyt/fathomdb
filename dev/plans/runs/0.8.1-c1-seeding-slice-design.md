## A. Seeding mechanism — query-matched entity-FTS + edge-fact-FTS (Option i)
Confirmed in code: entity nodes are in `canonical_nodes` with a real
`logical_id = derive_logical_id(kind,name)`, `body=name`, `source_id=source_doc_id`
(`lib.rs:8131, 2631-2636`), projected into node-FTS `search_index` (`:8136`); edges in
`canonical_edges(... from_id,to_id,source_id,body,t_valid,t_invalid,temporal_fallback)`
(`:8220`) projected into `search_index_edges` (`:8244`). The bug is at `lib.rs:5384-5396`
(iterate top-N fused **doc** hits → `write_cursor→logical_id` → `Some(Some(lid))` guard
fails because doc nodes carry `logical_id=NULL`).

**Decision: ship Option (i)** (seed from the graph's OWN text surfaces), not Option (ii)
(re-derive seeds from the doc ranking) — (ii) inherits the very doc-dominance that causes
the bug; fold it in later only if Tier-B shows under-seeding. Thread the compiled FTS
match expression into `bfs_graph_arm_candidates` (caller has `compiled.match_expression`
at `:5320`); signature returns the G0 meter tuple
`(Vec<SearchHit>, GraphFrontierStats)`. Replace Phase-1 (`:5372-5398`) with two seed
sources, dedup into `visited`, push each `(lid, 0)`:

**Seed source A — edge-fact endpoints (primary):**
```sql
SELECT ce.from_id, ce.to_id
FROM search_index_edges sei
JOIN canonical_edges ce ON ce.write_cursor = sei.write_cursor
WHERE search_index_edges MATCH ?1
  AND ce.superseded_at IS NULL
  AND (ce.t_invalid IS NULL OR datetime(ce.t_invalid) > datetime('now'))
  AND (ce.temporal_fallback IS NULL OR ce.temporal_fallback = 0)
ORDER BY bm25(search_index_edges), sei.write_cursor
LIMIT SEED_FTS_N
```
**Seed source B — entity-node FTS (isolated/strongly-named entities):**
```sql
SELECT cn.logical_id
FROM search_index si
JOIN canonical_nodes cn ON cn.write_cursor = si.write_cursor
WHERE search_index MATCH ?1
  AND cn.superseded_at IS NULL
  AND cn.logical_id IS NOT NULL          -- excludes doc nodes (the bug surface)
ORDER BY bm25(search_index), si.write_cursor
LIMIT SEED_FTS_N
```
`logical_id IS NOT NULL` is the one line that structurally cures BLOCK-1. Phase-2 BFS
(`:5400-5474`) is unchanged — it already traverses `canonical_edges` and (per G0) carries
the **traversed edge's `source_id`** onto each emitted hit, with the `fused_hits` dedup.

**Meter (the 0→>0 flip):** repoint G0's `GraphFrontierStats` — `seeds_considered` =
distinct candidate entity logical_ids from A∪B; `seeds_resolved` = those confirmed active
in `canonical_nodes` (dangling endpoints count considered-but-unresolved); rate `0/0→0.0`.

## C. Tier-B (<$1) real-corpus validation — the ONLY LLM spend
1. Extract ~150-200 real LME sessions with **`gemini-flash-lite`** via
   `elps_live_harness.py` (G0-hardened/chunked/backoff path) — full haystacks of **3-4
   entity-connected questions** (1-2 multi_session + 1-2 temporal/KU). Minutes.
2. Run the **LLM-free** recall loop (`r2_parity_eval.py`, `use_graph_arm=true`) reading
   `_graph_frontier_stats_for_test` per query.
3. **Pass bar:** on the real doc-seeded queries (those that read 0 under G0),
   `resolved_seed_rate > 0` for the majority, `graph_candidates_emitted > 0`, and ≥1
   emitted hit whose `source_id` resolves to a gold session id. Seeding-liveness gate,
   NOT recall-lift (lift is E1, priced, gated on G1).
