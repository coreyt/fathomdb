## Numbering — N-mod-5 with reserved gaps (carried HITL directive)

Planned slices are numbered **only in multiples of 5**. The gaps
(`1–4, 6–9, 11–14, 16–19, 21–24, 26–29, 31–34, 36–39, 41–44`) are the **insertion
mechanism** for unplanned follow-on a slice reveals while executing.

**Reserved-gap policy (unchanged from 0.8.0).**
- Follow-on that must land before N+5 becomes Slice **N+1…N+4** — a **fully
  orchestrated slice** (own prompt, own worktree from a fresh baseline, own
  `output.json`, own codex §9 review, own CLOSED block) — NOT an ad-hoc patch.
- **Reserved-gap ≠ fix-N.** fix-N re-enters the **same** slice's worktree to clear
  a BLOCK/CONCERN on that slice's own deliverable; a reserved-gap slice is **new
  adjacent scope** (fresh baseline + worktree).
- **HALT rule:** if a gap band (N+1…N+4) fills and follow-on remains, the slice was
  mis-scoped → **HALT to HITL**. Never overflow into the next mod-5 number.

**Pre-identified reserved-gap candidates** (see per-slice "Reserved follow-on" in
`0.8.1-implementation.md`):
- **eu7 harness perf** (parallel GT re-embed + retained-f32 reuse; measurement-only,
  byte-identical-result + codex measurement-neutral gate) — band 6–9 or 41–44.
- **R4 — whole-doc long-context dense probe** (floor-gated ≥0.90 at the model dim) —
  band 31–34 (after R3) or 36–39.
- **G4↔G10 unification = reserved-gap 37** (NEEDED, not optional; touches the shipped
  G10 `SearchFilter`).
- **BYO-LLM harness conformance** follow-on if Slice 15's ingest API reveals protocol
  gaps against the Memex harness (`dev/plans/prompts/IR-C-byo-llm-extraction-harness-memex.md`).
- **~4-pt vector-stage fidelity-regression diagnosis** (0.937→0.896, engine A/B → bisect
  the vector path) — carried from 0.8.0 (STATUS-0.8.0 §1/§7), **HITL-sequenced AFTER the
  IR/graph tracks**; band 41–44 / Slice-40-adjacent, or a dedicated slice if the bisect
  lands a cause. Measurement-only diagnosis first. See `0.8.1-implementation.md` §"Tracked item".

**Explicitly OUT of 0.8.1 (deferred):**
- **R3b — bundled CPU local-LLM extractor → 0.8.2** (`dev/roadmap/0.8.2.md`). 0.8.1
  graph construction is **BYO-LLM only**.
- **R5 — vector-PRF** — opportunistic/post-R1; not in the committed R0–R4 set.
- **Node-centric PPR (HippoRAG)** — later ranking pass over the same substrate.

## Cross-cutting Definition of Done (X1/X2/X3 — bind EVERY slice)

Carried verbatim from 0.8.0 (`0.8.0-plan.md`); every slice's success criteria **and**
the codex review checklist additionally assert:

- **X1 — SDK parity + functional harnesses.** Any surface/behavior/error/result
  change lands in **both** Python and TypeScript SDKs in the same slice, each with a
  live functional harness (write→search→retrieve→admin + cross-binding equivalence,
  not symbol presence). New graph/rerank verbs extend the harnesses.
- **X2 — `mkdocs build` stays green.** Any `docs/` add/rename updates `mkdocs.yml`
  `nav` in the same slice; the slice self-check runs `mkdocs build` (strict where
  feasible). **Slice 40 enforces it as a release gate.**
- **X3 — docs updated per slice + `dev/DOC-INDEX.md` maintained** in the closing docs
  commit.

`runs/STATUS-0.8.1.md` carries an **X1/X2/X3 per-slice column**.
