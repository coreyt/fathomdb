## Scope — engine surface

### Schema migration

Add migration `008_source_id.sql`:

```sql
ALTER TABLE canonical_nodes ADD COLUMN source_id TEXT;
ALTER TABLE canonical_edges ADD COLUMN source_id TEXT;
CREATE INDEX IF NOT EXISTS canonical_nodes_source_id_idx
    ON canonical_nodes(source_id);
CREATE INDEX IF NOT EXISTS canonical_edges_source_id_idx
    ON canonical_edges(source_id);
```

`source_id` is nullable: writes from callers that do not set it
land as NULL. Bumps `SCHEMA_VERSION` from 7 → 8 in
`fathomdb-schema/src/lib.rs`. Migration row appended to the
`MIGRATIONS` static.

Schema accretion guard (REQ-045): the migration adds tables /
columns. Document the offsetting removal in the migration comment
or in `dev/notes/schema-history.md`. If no candidate, write the
"why removal is impossible" justification inline.

`feedback_no_data_migration` does not apply — Pack B must add a
column, not migrate data. The memory blocks INSERT…SELECT-style
data backfill, which Pack B does not do.

### `PreparedWrite` field add

Per `dev/adr/ADR-0.6.0-prepared-write-shape.md`, `PreparedWrite`
is `#[non_exhaustive]`. Add `source_id: Option<String>` to the
`Node` and `Edge` variants. Bindings + CLI parsing will pick this
up later; the engine accepts None for back-compat with existing
tests.

Update `commit_batch` to write `source_id` into the new column.
Update `validate_batch` if schema-validation cares.

### `Engine::trace_source_ref(source_id: &str) -> Result<TraceReport, EngineError>`

`TraceReport { source_ref: String, events: Vec<TraceEvent> }`.
`TraceEvent { write_cursor: u64, kind: String, table: &'static str }`
where `table` is `"canonical_nodes"` or `"canonical_edges"`.

Per AC-042: returns exactly the canonical-row id set produced by
the given source_id — no extra rows, no missing rows. Excludes
rows with NULL source_id even when called with empty string;
empty string is not a valid source_id.

Implementation: `SELECT write_cursor, kind FROM canonical_nodes
WHERE source_id = ?1 UNION ALL SELECT ... FROM canonical_edges
WHERE source_id = ?1 ORDER BY write_cursor`.

### `Engine::excise_source(source_id: &str) -> Result<ExciseReport, EngineError>`

`ExciseReport { source_ref: String, nodes_excised: u64,
edges_excised: u64, projections_invalidated: u64 }`.

Per AC-028a/b/c semantics:

- Audit trail: append a row to `operational_mutations` collection
  `excise_source_audit` with `record_key = source_id` and
  `payload_json` containing `source_id` + `excised_at` + counts.
  AC-028a checks for ≥ 1 audit row naming the excised source.
- FTS5 + vec0 residue removal: `DELETE FROM search_index WHERE
write_cursor IN (...)`, `DELETE FROM vector_default WHERE rowid
IN (...)`, `DELETE FROM _fathomdb_vector_rows WHERE write_cursor
IN (...)`, `DELETE FROM _fathomdb_projection_terminal WHERE
write_cursor IN (...)`. AC-028b: zero residue from S1 in
  shadow tables.
- `DELETE FROM canonical_nodes WHERE source_id = ?1` + same on
  canonical_edges.
- Non-perturbation: AC-028c — rows from other sources untouched.
  Engine must NOT issue blanket projection rebuild; only invalidate
  rows attributable to the excised source. The test seeds S1 + S2,
  excises S1, asserts S2 result-set unchanged.

Drain projection scheduler before the excise transaction (Pack A
pattern). Bubble drain timeout instead of swallowing.
