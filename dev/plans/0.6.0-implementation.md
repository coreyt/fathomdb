# 0.6.0 Implementation Plan

This is the **single source of truth** for 0.6.0 implementation sequencing and
current execution posture.

This plan starts after the `0.6.0-design-frozen` tag. The frozen contract is
the locked corpus under `dev/`: needs, requirements, acceptance, architecture,
designs, interfaces, test-plan, traceability, security-review, and PSD.

Implementation rule: **TDD is mandatory**. Each implementation slice starts
with a failing test that cites at least one `AC-*` id and one relevant ADR or
design/interface doc. Red -> green -> refactor. Do not change a test to make
the implementation pass; fix the code. If a locked doc is wrong, stop and get
explicit HITL approval before amending it.

Exception: interface-only contract-shape tests (Phase 5) may cite an
interface or design doc alone, with no `AC-*` id. These tests assert the
public surface compiles to the locked spelling (types, error leaves, flag
sets, sentinel constants); the matching `AC-*` behavior coverage lands in
the phase that implements the behavior.

## Phase 5 — Interface Stubs

Goal: make public surfaces compile and fail/pass against contract-shape tests
without pretending the engine behavior is implemented.

Scope:

- Rust facade types, constructors, error enums (full 16-row matrix from
  `dev/design/errors.md`), `OpenStage` enum, `SoftFallback` branch enum,
  engine-attached instrumentation method stubs (`drain`, `counters`,
  `set_profiling`, `set_slow_threshold_ms`, `subscribe`), and public
  docstrings.
- Python and TypeScript five-verb SDK shape, including config knobs
  (`embedder_pool_size` / `embedderPoolSize` etc.), `admin.configure`
  exposure, instrumentation method stubs, and the full single-rooted error
  hierarchy (Python base `EngineError`; TS base `FathomDbError`) with every
  leaf class from `dev/design/errors.md` declared.
- CLI parser-shape pinning via `clap`-derive: subcommand structure for
  `fathomdb recover` + `fathomdb doctor <verb>`, isolation of
  `--accept-data-loss` to `recover` only, exit-code constants for the six
  classes in `dev/interfaces/cli.md`. Stub command bodies print
  `{"status":"not_implemented","verb":"<name>"}` and exit `70`; real
  invocability, JSON shape per verb, and exit-code semantics land in
  Phase 10.
- Wire/on-disk sentinel constants and schema-version exposure.
- Explicit `unimplemented` / typed placeholder errors where behavior is not yet
  real. Empty-query rejection routes through the matrix-canonical
  `WriteValidationError` so the typed-hierarchy invariant is pinned from
  this slice forward.

## Phase 7 — Writer, Typed Writes, and Op-Store

Goal: make canonical writes transactional, cursor-bearing, and typed, with
op-store collections in the same SQLite file.

Scope:

- Single writer thread and typed `PreparedWrite` execution.
- Canonical node/edge/chunk tables and cursor allocation.
- Op-store collection registry, `append_only_log`, `latest_state`, and JSON
  Schema save-time validation.
- Security hardening for bounded validation and external `$ref` rejection.

First failing tests:

- `src/rust/crates/fathomdb-engine/tests/op_store.rs` for AC-061a/b/c,
  AC-062, AC-063a/b.
- Public-surface cursor tests for AC-059a/b.
- Security tests for AC-064 and AC-065.
- Non-AC writer sanity tests may be added here; AC-011a/b remain perf gates.

Exit gate:

- Canonical writes and op-store rows commit in one transaction.
- `WriteReceipt { cursor }` is monotonic and satisfiable by read-side
  `projection_cursor`.
- Validation failures leave no canonical or op-store residue.

## Phase 10 — CLI Recovery and Operator Tooling

Goal: make the operator-only surface real without leaking recovery operations
into SDKs.

ADR-0.6.0-cli-scope locks the full verb set: seven `doctor` verbs
(`check-integrity`, `safe-export`, `verify-embedder`, `trace`, `dump-schema`,
`dump-row-counts`, `dump-profile`) and six `recover` sub-flags
(`--truncate-wal`, `--rebuild-vec0`, `--rebuild-projections`,
`--excise-source`, `--purge-logical-id`, `--restore-logical-id`). Six engine
seams landed in Phase 9 packs A / B; the rest must be built in this phase.

To avoid an unsatisfiable single-step exit gate, Phase 10 is split into
**10a** (facade re-exports + CLI wire-up for the six landed seams) and
**10b** (build the remaining engine seams + extend CLI to cover the full
ADR-locked verb set).

### Wiring decision (2026-05-11)

`fathomdb-cli` depends on the `fathomdb` facade (`Cargo.toml` confirms no
direct `fathomdb-engine` dep). Phase 10a adds **targeted facade re-exports**
for the six landed recovery/reporting types; Phase 10b extends the
re-export list as new engine seams land.

- Rationale: surface is small; 0.6.0 is the rewrite so public-API churn is
  cheapest now; Phase 11 bindings consume the same surface, so diverging
  CLI vs. binding import paths is avoidable debt.
- Re-exports are mechanical. ADR-0.6.0-crate-topology carries a 2026-05-11
  amendment confirming the CLI-via-facade dependency direction. No new
  ADR required.
- Locked symbol spellings live in `dev/interfaces/rust.md`.

### Test fixture strategy

Recovery verbs operate on broken databases. Reuse the corruption-injection
pattern landed in Pack C (`614df2b`, `d7a558f`, `3654fab`) and the
perf-fixture harness pattern from Pack D (`187441c`):

- corruption fixtures for `doctor check-integrity` and `recover --rebuild-*`
  reuse Pack C injection seams
- `safe_export` round-trip fixtures reuse Pack D fixture-build harness
- no new ad-hoc DB-mutating helpers in the CLI test crate; route through
  the engine corruption tool surface Pack C already exposes
