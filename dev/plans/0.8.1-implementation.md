## Slice-orchestrator model (carried from 0.8.0)

Each slice is a fully-orchestrated unit: own prompt → own worktree from a fresh
`main` baseline → **TDD RED→GREEN→refactor** (or, for `[design-adr]`, a falsifiable
acceptance bar + self-check) → SDK parity (X1) + docs (X2/X3) → self-check → merge to
local `main` (the agent lands its own work, **no push**) → **codex §9 review** →
PASS / CONCERN+override / BLOCK→fix-N → orchestrator updates the board in one docs
commit. The **main-thread orchestrates and gates; it does not do slice work**
([[orchestration-execution-traps]]).

**Reviewer:** codex is the **primary §9 reviewer**
(`codex exec review --base <sha> --dangerously-bypass-approvals-and-sandbox`); an
independent adversarial subagent is the fallback only ([[orchestration-execution-traps]]).

## Cross-cutting requirements (every slice's Definition of Done)

- **X1 / X2 / X3** as in `0.8.1-plan.md` (SDK parity + functional harness; mkdocs
  green; docs + `dev/DOC-INDEX.md` per slice).
- **TDD evidence**: the RED sha (failing tests committed first) recorded in the
  slice's `output.json` `tdd_evidence`.
- **Footprint invariant (0.8.1-specific):** every retrieval/graph change stays
  **CPU-only, no-API at the library boundary, 1-bit-binary-safe**. Any path that
  needs a GPU, a network call from FathomDB, or breaks the 1-bit Hamming store is a
  **BLOCK** unless explicitly HITL-waived. The BYO-LLM ingest is the only LLM seam,
  and the **LLM lives in the caller's harness, never in FathomDB**.
- **Determinism pins**: RRF ordering stays byte-deterministic
  (`pr_g9_rrf_fusion.rs`); any new fused arm extends, never weakens, those pins.

## Critical path + parallelization

**Critical path:** `0 → 15 (graph substrate KEYSTONE) → 20 (traversal) → 30 (R3)`,
with `30` additionally gated on `25 (R2)` which is gated on `10 (R1)` gated on
`5 (R0)`. **Slice 40** closes on all of 5/10/15/20/25/30/35.

**Parallelization (REQUIRED — three tracks off Slice 0):**
- **Rerank/eval track:** `5 (R0) → 10 (R1) → 25 (R2)` — read-path + reranker + eval;
  touches `rerank_fused` + a new eval harness, not the schema.
- **Graph track:** `15 (substrate) → 20 (traversal)` — schema + traversal; the
  keystone band.
- **Filter track:** `35 (G4 + G4↔G10 + deferred ADRs)` — after 15 (shares the
  `read.*`/migration surface), otherwise independent.
- **Join:** `30 (R3)` needs substrate (15) + traversal (20) + R2 (25). **Anything
  touching `commit_batch`, the `canonical_edges` schema, or the RRF carrier MUST
  serialize through Slice 15** to avoid carrier/schema churn (the keystone rule).

   (`body`/`t_valid`/`t_invalid`/`confidence`) + edge projectability shape, harmonized

**Acceptance bar (replaces TDD for design-adr).** Each ADR `status: decision-ready`

prepared. **HITL gate:** the three ADRs (eval-measure, BYO-LLM protocol, graph
migration) signed before Slice 15/25 start.
**Reserved follow-on (1–4):** a 4th ADR if R2's answerer/baseline choice needs its own
sign-off; the session-scaffold generation for the parallel tracks.

### Slice 5 — R0: candidate-recall CDF + rerank cost model · `[implementation (measurement)]` · depends-on: 0 · gaps: 6–9
**Objective.** Measure `found@K` (K = 50/100/200/500/1000) per query class on the
frozen corpus + a CPU cross-encoder latency/cost curve. **Resets the ceiling math**
(the depth-50 ~0.62 is depth-conditional; lexical found@1000 ≈ 0.86 — IR-C roadmap
C1/C2). Read-path measurement only; **no engine change**.
**TDD.** RED: a harness test asserting the CDF is computed over the pinned
`corpus_hash` and emits a structured artifact; GREEN: the runner. Pin the artifact
schema, not the (machine-specific) latency numbers. Output → `runs/IR-C-recall-cdf.json`.
**DoD.** X1 n/a (no SDK surface); X3 findings doc + DOC-INDEX. Gates Slice 10's rerank
depth choice.
**Reserved follow-on (6–9):** eu7 harness perf (parallel GT re-embed + retained-f32
reuse) is a natural neighbor here — measurement-only, byte-identical-result +
codex-measurement-neutral gate (`dev/archive/0.8.1-roadmap-direction-20260612.md` §4).

### Slice 10 — R1: CPU cross-encoder reranker in the `rerank_fused` seam · `[implementation]` · depends-on: 5 · gaps: 11–14
**Objective.** Wire a FlashRank-class ONNX cross-encoder (4–34 MB; no Torch/GPU/API)
into the **existing `rerank_fused` identity stub** (`lib.rs:3707`, from 0.8.0 Slice
10/G9) over the fused top-K. Rerank depth from R0. Score-blend (RRF+CE interpolation)
preferred over pure reorder (IR-C roadmap C6).
**TDD.** RED: tests pinning (a) **factoid R@10 ≥ 0.90 no-regress** gate, (b)
deterministic reranked ordering, (c) the soft-fallback (reranker absent/empty → fused
order byte-identical). GREEN: the reranker + the blend. **Footprint:** CPU/ONNX, 1-bit
store untouched.
**DoD.** X1 (Py+TS: optional rerank flag + cross-binding order parity), X2, X3. Codex
§9. Expected exploratory R@10 0.307 → ~0.40–0.47 (report-only; bounded by candidate
recall at the rerank depth).
**Reserved follow-on (11–14):** rerank model-card/license pin; a deeper-pool variant if
R0 shows the depth-100→500 curve pays.

### Slice 15 — Graph substrate KEYSTONE: G11 edge enrichment + edge projectability + BYO-LLM ingest API · `[implementation (schema)]` · depends-on: 0 · gaps: 16–19
**Objective (the keystone).** Activate the H3-reserved fact-on-edge enrichment:
additive `canonical_edges` columns `body` / `t_valid` / `t_invalid` / `confidence`
(schema step + `SCHEMA_VERSION` bump, additive/accretion-exempt); **edge projectability**
(edge `body` → FTS + the node 1-bit vector projection path); and the **BYO-LLM ingest
API** implementing `fathomdb.extract.v1` (engine spawns/▸calls the caller's harness,
maps `entities`→nodes by `logical_id`, `edges`→enriched `canonical_edges`, does the
**invalidate-not-accumulate** bookkeeping). **No LLM in FathomDB.**
**TDD.** RED: migration tests (columns present, additive, legacy rows NULL-safe);
ingest-API tests (a fixture harness over stdio NDJSON → expected edges; superseding
fact sets prior `t_invalid`); projectability tests (edge text searchable). GREEN: the
migration + ingest + projection. **Footprint:** the harness is a subprocess; FathomDB
makes no network call (assert in test).
**DoD.** X1 (Py+TS ingest surface + cross-binding parity), X2, X3 (op-store/graph docs +
the protocol doc). Codex §9. **This slice gates 20 + 30** — merge before either starts.
**Reserved follow-on (16–19):** shadow vec0/FTS5 reconciliation for invalidated edges;
BYO-LLM protocol gaps surfaced vs the Memex harness conformance fixture.

### Slice 20 — G5/G6 graph traversal (neighbors + search-with-expand) · `[implementation]` · depends-on: 15 · gaps: 21–24
**Objective.** Port the v0.5.6 depth-bounded `WITH RECURSIVE` BFS (visited-set cycle
guard, directional joins, **hard cap 50**) re-targeted to `from_id`/`to_id`; SDK
`depth` defaults to **1**, **rejects > 3** (typed error, not silent clamp). **G5** =
neighbors; **G6 = G1 + G4 + G5 + G9** (retrieve + expand). Traversal filter honors
`superseded_at IS NULL` **AND** the G11 valid-time (`t_invalid IS NULL OR t_invalid >
now`). Build G6 before standalone G5.
**TDD.** RED: traversal tests (depth-1/2/3, >3 rejected, cycle-guard, valid-time
filter drops invalidated edges); EXPLAIN gate (hits `canonical_edges(from_id)/(to_id)`,
no SCAN). GREEN: the traversal. **DoD.** X1 (Py+TS traverse/expand verbs), X2, X3.
Codex §9.
**Reserved follow-on (21–24):** depth/perf profiling tuning (impl-time, per the ADR);
node-distance reranking as a later scoring pass.

### Slice 25 — R2: end-to-end Mem0/Zep parity eval harness · `[implementation (eval)]` · depends-on: 10 · gaps: 26–29
**Objective.** Build a LongMemEval-style end-to-end eval: an **identical answerer**
over FathomDB (post-R1 build) vs naive-RAG vs a **local Mem0-OSS baseline**, scored
per-class (factoid / temporal / multi-hop / knowledge-update / multi-session) with
abstention. **Decision ①:** AC-077 (Evidence Recall@K) stays the product GATE; **R2 is
the report-only north-star** for the "as-good-or-better than Mem0/Zep" goal. R2 is the
**only instrument that can see R3's value** → it MUST exist before R3's go/no-go.
**TDD.** RED: harness tests (deterministic answerer; per-class scoring; the parity
claim cites *this*, never vendor numbers). GREEN: the harness + the baseline adapter.
**Footprint:** the answerer/baseline LLM is test-infra (BYO/local), gated behind a
feature/env like the embedder tests; the engine SUT stays no-API.
**DoD.** X3 (eval design + results doc, DOC-INDEX); X1 n/a (eval harness, not SDK).
Codex §9 (measurement-neutrality, like eu7).
**Reserved follow-on (26–29):** the AC for the parity metric (minted at the eval gate,
HITL); additional baseline adapters (Zep-OSS if feasible).

### Slice 30 — R3: graph-retrieval arm (temporal fact-edges as a third RRF arm) · `[implementation]` · depends-on: 15, 20, 25 · gaps: 31–34
**Objective.** Make the graph a **third retrieval arm**: project edge `body` →
FTS+vector candidates + bounded BFS over the substrate, **fused as a third RRF arm**
with dense + lexical, with the **invalidate-not-accumulate** temporal filter. node/edge
pick = **fact-on-EDGE** (Graphiti-shaped); HyperGraphRAG fact-NODES stay the H6 escape
hatch. Payoff is mostly **off the Recall@K axis** → judged on **R2's**
temporal/multi-hop/update classes.
**TDD.** RED: third-arm fusion tests (RRF determinism extended, not weakened; temporal
filter correctness; factoid/exploratory Recall@K no-regress); the R2 memory-class
deltas measured (report-only). GREEN: the arm + fusion. **DoD.** X1 (Py+TS graph-arm
flag + parity), X2, X3. Codex §9. **Go/no-go uses R2** (C3) — if R3a fails on the
corpus, capture and HALT to HITL rather than ship a regressing arm.
**Reserved follow-on (31–34):** **R4 — whole-doc long-context dense probe** (floor-gated
≥0.90); node-centric PPR ranking pass; confidence-threshold weighting on the arm.
