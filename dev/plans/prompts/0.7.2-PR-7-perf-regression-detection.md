### Schema (handoff says)

```json
{
  "commit_sha": "…",
  "ac_id": "AC-013" | "AC-013b" | "AC-019" | "AC-012",
  "n": 10000,
  "p50_ms": 36.0,
  "p99_ms": 49.0,
  "recall": 0.937,
  "timestamp": "2026-06-01T07:23:39Z"
}
```

Append-only. JSON-per-canonical-run (one file per run; bin reads the
directory).

### Thresholds (handoff recommendation; HITL-gated)

- Latency: degradation > **10%** vs rolling median of last 10 → flag.
- Recall: degradation > **0.02 absolute** vs rolling median → flag.
- These are the recommended starting points; HITL gate confirms before they
  land in the bin source.

   - Reads `dev/perf-history/`; groups by `ac_id` + `n`; computes rolling

     block at the top of the bin source — codex review will look for this.
   - Output: human-readable + machine-parseable (JSON-on-stdout-with-
     `--json` flag is the convention).
   - Exit code: 0 = no regression, 1 = regression flagged, 2 = data
     missing / malformed.

     noise; bin MUST NOT flag.
   - **Backfill flag-on-known-bug**: feed the bin the backfilled history
     ending at the 2026-05-27 batch-collapse commit; assert it flags.
   - **No false positives on backfill**: feed the bin the clean backfill
     (excluding the bug); assert it does NOT flag.
6. **CI integration** (`.github/workflows/perf-canonical.yml`): after the
