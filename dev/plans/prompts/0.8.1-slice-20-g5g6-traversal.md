### 3.2 GREEN — implement

#### 3.2.1 BFS CTE in engine

Add to `Engine` in `lib.rs`:

```rust
pub fn graph_neighbors(
    &self,
    root_logical_id: &str,
    depth: u32,
    direction: TraversalDirection,
) -> Result<Vec<NodeRecord>, EngineError>
```

Where `TraversalDirection` is a new enum `{ Outgoing, Incoming, Both }`.

The recursive CTE pattern from v0.5.6 adapted to the current schema:

```sql
WITH RECURSIVE
  traversal(logical_id, depth) AS (
    -- anchor: neighbors of root at depth 1
    SELECT DISTINCT
      CASE WHEN e.from_id = root.logical_id THEN e.to_id ELSE e.from_id END AS logical_id,
      1 AS depth
    FROM canonical_nodes root
    JOIN canonical_edges e ON (
        (direction=Outgoing AND e.from_id = root.logical_id) OR
        (direction=Incoming AND e.to_id   = root.logical_id) OR
        (direction=Both AND (e.from_id = root.logical_id OR e.to_id = root.logical_id))
    )
    WHERE root.logical_id = ?1
      AND root.superseded_at IS NULL
      AND e.superseded_at IS NULL
      AND (e.t_invalid IS NULL OR e.t_invalid > ?2)   -- ?2 = now (ISO-8601 string)
    UNION
    -- recursive: expand from frontier
    SELECT DISTINCT
      CASE WHEN e.from_id = t.logical_id THEN e.to_id ELSE e.from_id END,
      t.depth + 1
    FROM traversal t
    JOIN canonical_edges e ON (
        (direction=Outgoing AND e.from_id = t.logical_id) OR
        (direction=Incoming AND e.to_id   = t.logical_id) OR
        (direction=Both AND (e.from_id = t.logical_id OR e.to_id = t.logical_id))
    )
    WHERE t.depth < ?3   -- ?3 = max_depth (≤ 3)
      AND e.superseded_at IS NULL
      AND (e.t_invalid IS NULL OR e.t_invalid > ?2)
      AND CASE WHEN e.from_id = t.logical_id THEN e.to_id ELSE e.from_id END != ?1  -- don't re-visit root
  )
SELECT DISTINCT n.logical_id, n.kind, n.body, n.write_cursor
FROM traversal tr
JOIN canonical_nodes n ON n.logical_id = tr.logical_id
WHERE n.superseded_at IS NULL
LIMIT ?4   -- ?4 = hard cap 50
```

**Important SQLite notes:**

- SQLite `WITH RECURSIVE` does not support `DISTINCT` inside the recursive term directly — you may need to manage visited-set deduplication at the application level or via a `GROUP BY` in the final SELECT. Read the v0.5.6 implementation for the exact visited-set guard pattern.
- The direction parameter cannot be parameterized in SQL string substitution safely — use a three-way branch: build one of three SQL strings depending on `direction` (or use a `CASE WHEN` style if SQLite allows it in JOIN conditions — verify).
- "now" for temporal filter: use the engine's current write_cursor as a proxy for "now" (or derive an ISO-8601 timestamp; read how the ADR defines "now" for `t_invalid` comparison). The ADR says `t_invalid > now` where now is transaction-time. For a read-only traversal, use `datetime('now')` in the SQL or pass the current ISO-8601 timestamp as a bind parameter.

Reject `depth > 3` before constructing any SQL:

```rust
if depth > 3 {
    return Err(EngineError::InvalidArgument { /* depth > 3 */ });
}
```

Add `InvalidArgument` to `EngineError` if it doesn't exist, or use the closest existing variant and document it.

Hard cap: `LIMIT 50` in the CTE result.

#### 3.2.2 `search_expand` (G6)

```rust
pub fn search_expand(
    &self,
    query: &str,
    filter: Option<SearchFilter>,
    depth: u32,
) -> Result<SearchExpandResult, EngineError>
```

Where `SearchExpandResult` has:

- `search_hits: Vec<SearchHit>` — the original search results (with their RRF scores)
- `expanded: Vec<(NodeRecord, u32)>` — (node, hop_count) for nodes reached by traversal but not in search_hits
- `all_logical_ids: Vec<String>` — union of search hits + expanded, deduplicated

Implementation:

1. Call `self.search_filtered(query, filter)`.
2. For each search hit: resolve its `logical_id` from `canonical_nodes WHERE write_cursor = hit.id`.
3. Call `self.graph_neighbors(logical_id, depth, Both)` for each root.
4. Deduplicate: a node that appears in both search_hits and expansion appears only in `search_hits` (search score takes priority).

Reject `depth > 3` same as `graph_neighbors`.
