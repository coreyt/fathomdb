## Mandate

The 0.6.1 planning slice surfaced **B-001** in
`dev/plans/runs/0.6.1-planning-output.json`:
`scripts/security/check_removal_changelog.py` (called via
`scripts/security/check-removal-changelog.sh` from
`scripts/agent-security.sh` for AC-050c gate) defaults `--base` to
`"0.6.0-rewrite"` — a branch that was closed at 0.6.0 GA (per HITL
2026-05-22) and no longer exists locally. Live `agent-verify`
invocations therefore fail with
`fatal: bad revision '0.6.0-rewrite..HEAD'` → BLOCKER toolchain-
missing. Reproduces on `main` baseline. Pre-existing post-0.6.0-GA
regression.

**Fix:** change the `--base` default from `"0.6.0-rewrite"` to
`"v0.6.0"` — the immutable Git tag at 0.6.0 GA (sha `edf00f7`).
Semantically: AC-050c detects undocumented public-surface removals
since the last released API. The 0.6.0 GA tag is the canonical
"last released" anchor; it never moves; it works regardless of
branch topology. After this fix lands and 0.6.1 ships, future
patch releases will need to retag the default forward (the
0.6.1-BUMP slice should be made aware).

This unblocks the `0.6.1-AXIS-E-PIN` slice which wires a new test
into `scripts/agent-verify.sh`; the verify pipeline must be GREEN
before that slice spawns.
