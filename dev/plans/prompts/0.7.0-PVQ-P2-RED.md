The recall test must:
- Reuse the AC-013 fixture (`seed_ac013_corpus`, `ac013_query_bodies`) so we measure on the same corpus.
- Compute brute-force f32 ground truth top-10 per query by direct SQL `SELECT rowid FROM vector_default WHERE embedding MATCH vec_f32(?) ORDER BY distance LIMIT 10` (this is exactly the current read path — pre-Pack-2 the production engine returns the same set as ground truth, so recall = 1.0).
- Run the production search path (`engine.search(query)`) to obtain its top-10 rowids per query.
- Assert `recall@10 >= 0.90` across the query set (HITL-locked floor; see `dev/plans/0.7.0-HITL-recommendations.md:89` and `dev/adr/ADR-0.7.0-vector-binary-quant.md` § 2 point 4).
- The recall computation: for each query, `|prod_top10 ∩ ground_truth_top10| / 10`; average over queries.
- Print one `RECALL_NUMBERS n=… samples=… recall_at_10=…` line so canonical-CI logs capture the numeric.
