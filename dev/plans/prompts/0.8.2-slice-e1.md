## 3. Mandate (implement the CE; nothing else)
1. **Pick + pin a real TinyBERT-L-2 cross-encoder** (e.g. `cross-encoder/ms-marco-TinyBERT-L-2-v2` —
   ~4 MB, MS-MARCO reranker). Pin the exact weight + tokenizer + config files + **sha256** (mirror how the
   embedder pins bge-small). Document the choice + provenance.
2. **`try_get_loaded()`** — probe `~/.cache/fathomdb/reranker/` (configurable override per the design);
   lazy-load the model once (`OnceLock`/`OnceCell`); on cache miss, download via `ureq` + sha256-verify +
   atomic rename (reuse `loader.rs`); return `None`/`Err` on absence/failure (**never panic**, soft-fallback).
3. **`score(query, passage)`** — tokenize the `(query, passage)` pair (`tokenizers`), run the 2-layer BERT
   (`candle-transformers` bert) + the classification head over CLS, return the raw logit. (`ce_rerank`
   already sigmoids + blends with RRF — don't touch that.)
4. **Cargo feature wiring** — make the candle stack reachable from the engine's reranker under
   `default-reranker` (the embedder owns candle; wire `default-reranker` to pull the needed embedder
   feature, or place the CE model in `fathomdb-embedder` and call it from the engine — your call, document
   it). Keep the **default (feature-off) build candle-free + no-network**.

**Footprint invariant (BLOCK if violated):** zero network when the feature is off, OR when
`rerank_depth==0`, OR when weights are already cached. `rerank_depth==0` stays byte-identical to identity.
