## How to use

- **Step 0 (now, in parallel):** GA-1 (1) ∥ IR-A (6) ∥ COR-1→COR-2 (5). No cross-deps.
- **GA critical path:** 1 → ◆2 → 3 → ◆Slice-40 merge → ◆4 → ★ 0.8.0 GA.
- **IR-eval path:** 6 (now) ∥ 5 (freeze) → 7 (needs ◆2 + ⬛5 + 6 merged; Ph4 needs Slice-40 merged) → 8 → ◆9 → ★ AC-077 LIVE.
- For a **prompted** session the scaffold is a thin launcher — read the authoritative prompt; the scaffold only adds
  the prereq verify lines + the output contract + 2–3 guardrails.
- For an **unprompted** session (GA-1, COR-2) the scaffold is the real starter — **expand it into a full prompt
  before running**.
- For a **◆ HITL gate** the scaffold is the decision-package shape the orchestrator assembles — there is no agent to spawn.

**Discipline (all sessions):** `--release`+isolated for measurement; read the REAL exit/numbers
([[background-exit-masks-real-exit]]); no fabricated numbers (TBD with its decision-owner); no push/tag; the board is
orchestrator-owned. **Never lower the recall floor or set a threshold above the measured ceiling.**
