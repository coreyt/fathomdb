#### 3.2.1 Core types

In `lib.rs` (or a new module, but re-exported from `lib.rs`):

```rust
#[derive(Clone, Debug, PartialEq)]
pub enum ScalarValue {
    Text(String),
    Integer(i64),
    Bool(bool),
}

#[derive(Clone, Debug, PartialEq)]
pub enum ComparisonOp { Gt, Gte, Lt, Lte }

#[derive(Clone, Debug, PartialEq)]
pub enum Predicate {
    JsonPathEq      { path: String, value: ScalarValue },
    JsonPathCompare { path: String, op: ComparisonOp, value: ScalarValue },
}

impl Predicate {
    // Constructor validates path against ALLOWLIST; returns Err if not allowed.
    pub fn json_path_eq(path: impl Into<String>, value: ScalarValue) -> Result<Self, EngineError> { ... }
    pub fn json_path_compare(path: impl Into<String>, op: ComparisonOp, value: ScalarValue) -> Result<Self, EngineError> { ... }
}

/// Allowed JSON paths for read.list predicates.
const PREDICATE_PATH_ALLOWLIST: &[&str] = &[
    "$.status",
    "$.priority",
    "$.tags",
    "$.kind",
    "$.created_at",
    // add more as consumer signal warrants
];
```

No `Fused*`, no `*_unchecked`. The constructors enforce the allowlist.

#### 3.2.2 `Engine::read_list`

```rust
pub fn read_list(
    &self,
    kind: &str,
    predicates: &[Predicate],
    limit: usize,
) -> Result<Vec<NodeRecord>, EngineError>
```

SQL generated:

```sql
SELECT logical_id, kind, body, write_cursor
FROM canonical_nodes
WHERE kind = ?1
  AND superseded_at IS NULL
  [AND json_extract(body, '$.field') <op> ?N ...]
LIMIT ?L
```

**Injection-safe compilation:**

- The `kind` is always a bound parameter.
- Each predicate's value is a bound parameter.
- The `json_extract(body, '$.path')` path is taken from the allowlist — it is a LITERAL string in the SQL (not a parameter, since SQLite's `json_extract` requires a literal path), BUT it is never the caller-supplied string; it is only used if it matches an allowlist entry exactly. The allowlist entries are server-side constants.
- `ComparisonOp` compiles to a literal operator string (`>`, `>=`, `<`, `<=`) — not a parameter, but a server-side constant from a closed enum; no interpolation risk.
- **Never use `format!("json_extract(body, '{}')", path)` with a caller-supplied `path`.** The caller supplies a path string, which is validated against the allowlist, and the ALLOWLIST ENTRY (a server-side constant) is used in the SQL — not the caller's string.
