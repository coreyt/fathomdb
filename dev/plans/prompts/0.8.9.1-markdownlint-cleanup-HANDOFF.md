## 2. The gate, exactly (what "done" means)

`scripts/agent-lint-md.sh` runs three checks under `set -euo pipefail` (aborts at the first failure):

1. `./node_modules/.bin/markdownlint-cli2` — structural lint (`agent-lint-md.sh:11-16`).
2. `./node_modules/.bin/prettier --check '**/*.md' --log-level warn` — format check (`:18-23`).
3. `lychee … '**/*.md'` — link integrity, offline by default (`:25-34`).

Config:

- `.markdownlint.jsonc` — rule config. Notable existing decisions: `MD013` line-length OFF (`:8`); `MD033`
  inline-HTML allowed; `MD024` siblings-only; `MD036` OFF; `MD025` front-matter-title exempt; **`MD049`
  style = `underscore`** and **`MD050` style = `asterisk`** (deliberately aligned to prettier so the two
  formatters don't fight — `.markdownlint.jsonc:31-32`). **Honor this alignment**: the fix for the ~3,266
  MD049 findings is to converge on prettier's `_emphasis_` / `**strong**`, not to flip the rule.
- `.markdownlint-cli2.jsonc` — glob `**/*.md` (`:9`) with ignores: `node_modules`, `target`, `site`,
  `dist`, `.venv`, `.cache`, `.wake`, **`dev/tmp{,2,3}`** (bare-URL research substrate), `dev/memex`,
  **`dev/archive`** (`:32`), **`dev/plans/runs/**`** (`:36`, verbatim agent run-logs), **`.claude/**`**
  (`:39`), and **`docs/**`** (`:42`, gated by `mkdocs build --strict` instead). The in-scope debt
  therefore lives mainly in `dev/plans`, `dev/design`, `dev/adr`, `dev/`, top-level `*.md`,
  `CHANGELOG.md`, etc. **Do not silence debt by widening the ignore list** — every ignore addition is a
  rule-decision requiring the §6 justification + codex review.

Note the **`prettier --check` glob is `**/*.md` with no ignore binding** — prettier is not bound by
`.markdownlint-cli2.jsonc` ignores; it only honors a `.prettierignore` if one exists (check for one). It
sees more files than markdownlint-cli2 does. Both must be clean for the gate to pass.

## 4. Anti-regression — the generator half (do not skip; this is half the mission)

Fixing the debt once is worthless if tools regenerate it. **Every script/tool that emits markdown must
emit gate-compliant markdown** (correct emphasis char per `.markdownlint.jsonc:31`, table style,
blank-line spacing around headings/lists/fences, fenced-code languages, no bare URLs, trailing newline),
**and a guard must exist** so a regenerated doc that violates the gate is caught.

**Guard requirement:** add a test/CI step that runs the generator (or lints a committed sample of its
output) through the **same** `scripts/agent-lint-md.sh` (or directly `markdownlint-cli2` + `prettier
--check` on the generated path) and fails on non-compliant output. A generator whose output lands in a
currently-**ignored** path (e.g. `dev/plans/runs/**`, `.claude/**`) does not trip today's gate, but must
still be made compliant for hygiene and pointed at the lint in its own check — otherwise retargeting it
later silently re-imports debt.

### 4a. The `docs/` generator — CALL THIS OUT, it is the dangerous one

`docs/**` is **IGNORED by `.markdownlint-cli2.jsonc:42`** and gated **only** by `mkdocs build --strict`.
`mkdocs --strict` validates nav/links/build — it does **not** enforce markdownlint/prettier style. So the
docs-generation flow can emit MD049/MD060/MD040-violating markdown into the **published site** unchecked.

- The docs flow: `dev/update-docs.md` is the operational prompt that (re)writes `docs/**` from the `dev/`
  source-of-truth (`dev/update-docs.md:58-107,95`); it is built/verified via `tools/docs/build.sh`
  (`mkdocs build --strict`, root `mkdocs.yml`, `docs_dir: docs`) and served via `tools/docs/serve.sh`.
  `dev/update-docs.md:186` already tells doc authors to run `npm run format:md` + `markdownlint-cli2
  --fix` — but nothing **enforces** it because `docs/**` is ignored.
- **Required:** (a) make the docs-generation path emit markdown compliant with the project
  markdownlint/prettier style (not just mkdocs-strict) — strengthen `dev/update-docs.md` Step 6 to a hard
  gate; (b) **add a guard so regenerated `docs/` stays compliant** — e.g. lint `docs/**` with the project
  config in a CI/check step (a separate `markdownlint-cli2` invocation with its own glob, since the main
  one ignores `docs/`), so a non-compliant page fails before publish. Decide explicitly whether to
  *remove* `docs/**` from the markdownlint ignore (bringing it under the main gate) or keep it
  ignored-by-main + add a dedicated docs-lint step; either is a §6 config decision — record the choice.

## 6. Config-decision discipline (when you change a rule instead of the docs)

Any time the fix is a **config change** (disabling/retuning a rule in `.markdownlint.jsonc`, adding an
ignore in `.markdownlint-cli2.jsonc`, adding a `.prettierignore` entry, or choosing a canonical
table/emphasis style) rather than editing the markdown:

- It must be **justified in a comment in the config file** (matching the existing style — every override in
  `.markdownlint.jsonc` carries a why-comment), and
- It must be **narrow** (prefer fixing content; prefer per-rule retune over a blanket ignore; prefer a
  scoped path-ignore over a global rule-off), and
- It is part of the **codex §9 review surface** (§3 Slice 40). The mechanical auto-format diff is not the
  review target; the **rule-decisions** are.

Default posture: **fix the markdown, not the rule.** Recall the steward already found a pure config-relax
**insufficient** (§0) — config is the exception, not the strategy. Reach for it only where a rule genuinely
fights a deliberate repo convention (the MD049/MD050↔prettier alignment is the canonical precedent).
