## 3. ⭐ NEXT EXPERIMENT — the fused/dense arm (ready to run)
**Question:** does FathomDB's **dense (BGE-small, 1-bit) + FTS fused** arm beat BM25
where **FTS-only did not** (cause C)? FTS-only trails BM25 ~5–7 pp; dense may close or
widen that. This is the central P0-A base question (`fathomdb_fused` was excluded until
now because embedding was 27 h on CPU — it is now **seconds on GPU**).

### 3.1 Run it
The batch runner hard-codes `include_fused=False`; the **sync** runner
(`eval/p0a_base_retrieval.py`) builds `fathomdb_fused` (via the sanctioned
`_configure_vector_kind_for_test("doc")` seam) and runs recall@K. Embed on GPU with
batching:
```bash
cd src/python
FATHOMDB_EMBED_DEVICE=cuda FATHOMDB_PROJECTION_BATCH=1 \
LD_LIBRARY_PATH=/usr/local/cuda/lib64 \
  ../../.venv/bin/python -m eval.p0a_base_retrieval \
    --per-class 40 --db-dir /tmp/p0a_n160_fused --output /tmp/p0a_fused_n160.json
```
(Confirm the sync runner's flags with `--help`; it has `--no-fused`/`--no-e2e`. Keep the
**same `--per-class 40` + default seed** as the nano/gemini runs for a paired compare.)
- **Verify the embed is complete BEFORE trusting fused recall:**
  `../../.venv/bin/python -m eval.verify_embed_db /tmp/p0a_n160_fused/p0a_fused.sqlite
  --expected-docs <N>` → must be `coverage=1.0`. (The build path already gates on this.)
- Recall@K is **$0** — run freely. This answers the dense-vs-BM25 question with no LLM.

### 3.2 Optional strong-reader gate on fused (costs ~$7)
If recall@K shows dense helps, run the e2e answer-accuracy on the fused variant with
`gemini-3.1-pro` (aistudio batch) for the paired comparison — **NEW output file**
(don't overwrite the FTS-only results), log the spend to the ledger, HITL-confirm before
spending. The batch e2e runner currently builds bm25+fts only; enabling fused there is a
small change (`include_fused=not args.no_fused`) — do it if the gate is wanted.

### 3.3 Done-criteria
Per-class recall@K for `naive_bm25` vs `fathomdb_fts_only` vs **`fathomdb_fused`** at
n=160; call **dense-helps-or-hurts vs BM25** (mind the §3.5 MDEs: per-class ~21 pp,
pooled ~11 pp — target the pooled + graded MRR/nDCG, don't over-read per-class). Log any
$ to the ledger.
