## Findings to fix

### [P1] `invalidate` must actually hide the edge from active FTS/vector retrieval — and prune projection state ([P2] #3 is the same fix)

File: `src/rust/crates/fathomdb-engine/src/lib.rs`, `apply_consolidate_verdicts` (~3387).
Problem: `invalidate` sets only `t_invalid`, and `supersede`/`merge` sets only `superseded_at`, but neither
removes the edge's **static projection shadow rows**, so FTS (`search_index_edges`) and vector
(`vector_default`/`_fathomdb_vector_rows`) still surface the consolidated-away edge. Graph *traversal*
already excludes ended facts (`superseded_at IS NULL AND (t_invalid IS NULL OR t_invalid > now)`, ~3969) —
FTS/vector must match that exclusion for consolidated edges.
Fix: when a verdict removes an edge from the active set, ALSO prune its shadow projection rows by the
edge's `write_cursor` (the row is keyed by `logical_id = edge_ref`; look up its active-row `write_cursor`),
mirroring the existing excision pattern at `lib.rs:5182-5208`:
`DELETE FROM search_index_edges / vector_default / _fathomdb_vector_rows / _fathomdb_projection_terminal
WHERE write_cursor = ?` (and `search_index` if a node — here it's edges). Do this inside the SAME
transaction.

- **`supersede`/`merge`:** set `superseded_at` (as now) AND prune the shadow rows (unconditional — a
  tombstoned edge is excluded from active retrieval).
- **`invalidate`:** set `t_invalid` (temporal record, as now); prune the shadow rows ONLY when the edge is
  actually ended as of the engine's current time (i.e. treat it like the traversal filter: an edge with
  `t_invalid > now` is still valid → keep its projection; `t_invalid <= now` → prune). Use whatever
  "now" the engine already uses for the traversal filter so behavior is consistent. Keep the row + body
  (non-destructive; ADR-0.8.12 §2.1).
Still: never DELETE a `canonical_edges` row, never rewrite a body.
Add a test to `tests/consolidate_provider.rs` proving the stale edge is **no longer returned by edge FTS
search** (and, if feasible, vector search) after `invalidate`/`supersede`, while the winning edge is.
