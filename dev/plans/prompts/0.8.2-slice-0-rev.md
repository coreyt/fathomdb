## 3. Mandate — the amended rule (encode EXACTLY this)

**`decide()` consumes already-computed summary statistics** (the Slice-20 harness runs the paired
bootstrap; `decide()` stays pure/deterministic, **no RNG, no I/O**). New signature:

```python
decide(material, em, trend, confident_wrong, power_ok) -> "GO" | "NO_GO"
#   material        = {"f1_delta": float, "f1_ci_low": float}   # pooled >=3-hop ΔF1 vs the fixed comparator + its paired-bootstrap CI lower bound
#   em              = {"ci_high": float}                          # pooled >=3-hop ΔEM CI upper bound
#   trend           = {"neg_significant": bool}                   # is the ΔF1-vs-hop slope SIGNIFICANTLY negative?
#   confident_wrong = {"increase_significant": bool}              # did ppr-fusion SIGNIFICANTLY raise the unanswerable-set confident-answer rate?
#   power_ok        = bool
```

**GO iff ALL** (else NO_GO):

1. `material["f1_delta"] >= MATERIAL_F1_LIFT` **and** `material["f1_ci_low"] > 0`   (material **and** CI excludes 0)
2. `not trend["neg_significant"]`   (**veto only on a significantly NEGATIVE** slope — flat or positive passes; **no strict-monotonic requirement**)
3. `em["ci_high"] >= 0`   (EM **not significantly worse** — CI-banded, not a point-estimate veto)
4. `not confident_wrong["increase_significant"]`   (unanswerable-set carries the confident-wrong role)
5. `power_ok`

Constants: keep `MATERIAL_F1_LIFT = 0.02` (now documented as **at/above** the Slice-5 pooled ≥3-hop MDE).
Drop the old `REQUIRED_HOPS` strict-ordering logic and `EM_MIN_LIFT_3PLUS` point-veto. **Keep the
