# 0.7.1 — Un-Defer the Default Embedder: Implementation Plan

Save as `dev/plans/prompts/0.7.1-EMBEDDER-UNDEFER-HANDOFF.md`. Sources: `dev/adr/ADR-0.6.0-default-embedder.md`, `dev/adr/ADR-0.6.0-embedder-protocol.md`, `dev/adr/ADR-0.6.0-vector-identity-embedder-owned.md`, `dev/adr/ADR-0.6.0-subprocess-bridge-deferral.md`, `dev/design/embedder.md`, `dev/design/orchestration.md`, `dev/needs.md:132`, `dev/requirements.md:270-274`, current scaffold at `src/rust/crates/fathomdb-embedder/src/lib.rs` and `src/rust/crates/fathomdb-engine/src/lib.rs:3499-3608`.

## 0. Baseline + framing

- **Starting state**: `NoopEmbedder` returns `[1, 0, 0, ...]` (`src/rust/crates/fathomdb-embedder/src/lib.rs:19-25`). `Engine.open` already records `default_embedder_identity()` (`fathomdb-engine/src/lib.rs:3507`) using hard-coded `fathomdb-noop / 0.6.0-scaffold / 384`. `_fathomdb_embedder_profiles` profile-pins on first write and fails-closed on mismatch (`lib.rs:3515-3614`). `OpenReport` already carries `default_embedder` + `embedder_warmup_ms` (`lib.rs:548-554`). Bindings already expose `EmbedderIdentity` and the open-report fields (`fathomdb-napi/src/lib.rs:356-388`, `fathomdb-py/src/lib.rs:35`), but neither binding exposes a way to *supply* an embedder yet.
- **Campaign target**: ship a real default embedder so the `default_embedder` identity in `OpenReport` reports a real model, the engine can synthesize query/doc vectors for AC-013/AC-019 against natural-text corpora, and "default embedder = on" is the documented happy path. Caller-supplied embedders remain available; bindings expose only the *default-on/off* toggle in 0.7.1.
- **Pre-flight**: this work is gated on `0.7.0-PERF-VECTOR-QUANT` closing via Option 1 (synthetic-corpus densification). When PVQ closes, identify the mainline SHA; that SHA is the baseline for all 0.7.1 slices.

## 1. Slice sequence

Slices land in order unless explicitly marked parallel-safe. Each slice gets one worktree per `dev/design/orchestration.md` § 2. Slice IDs prefixed `0.7.1-EU-` (Embedder Un-defer).

## 0.5 — EU-0 outcome (resolved 2026-05-28)

EU-0 closed. HITL chose: **`BAAI/bge-small-en-v1.5` (384d), rerank K=128,
mean-centering ON**. Detailed evidence + Pareto analysis in
`dev/notes/0.7.1-default-embedder-research.md` §5.1–§5.3. The EU-1 through
EU-7 sections below were drafted with "the chosen model" as a placeholder;
the resolved values are:

- **Model identity**: `fathomdb-bge-small-en-v1.5` / revision = HF snapshot
  `5c38ec7c405ec4b44b94cc5a9bb96e735b38267a` / dimension = 384.
- **Rerank K (sign-bit candidate set → f32 rerank → top-10)**: **128**.
  (Engine constant change land in EU-5; binary-quant ADR / pack-2 may need
  a matching update.)
- **Mean-centering**: ON. Adds a 1536-byte (384 × f32) corpus-mean blob to
  `_fathomdb_embedder_profiles` per workspace. Subtracted before sign-bit
  quantization on both write and query paths; f32 rerank uses un-centered
  vectors. Recomputation policy is an EU-2 design call (default proposal:
  compute-once-on-first-ingest, re-derive only on explicit reindex; do not
  silently mutate the pinned mean on incremental writes). Pin threshold
  resolved to `MEAN_VEC_PIN_THRESHOLD: u64 = 256` (see
  `dev/design/embedder.md` §0.3 for the statistical justification +
  topic-drift caveat).
- **Accepted risk**: floor headroom is thin (0.907 vs 0.90 floor; CI
  overlaps the floor). EU-7 must re-derive `AC013B_RECALL_FLOOR` against
  real bge-small embeddings (separately from EU-7's own AC validation)
  before the EU-5 lock-flip; if real-corpus recall lands materially below
  0.907, the documented fallback is K=256 (mc) or escalation to bge-base.

### Slice EU-0 — Research: candidate embedder under sign-bit + f32-rerank

**Mandate**: empirically validate (or refute) BGE-small-en-v1.5 as the default-embedder choice under the existing 0.7.0 binary-quant + f32-rerank pipeline at K=64.

**Why first**: ADR-0.6.0-default-embedder.md was written before the 0.7.0 PVQ pipeline existed. The 384-dim, sign-bit + f32-rerank combination at K=64 has not been measured for BGE-small. STATUS-perf-vector-quant cites e5-base-v2 at 74.8% recall@10 (768d, K=64×4) — a 384d model probably degrades further. Implementing the wrong model is a 3-release problem to undo.

**Blast radius (read-only)**: `dev/notes/0.7.1-default-embedder-research.md` (new write artifact, *not* code). Reference fixtures: `data/corpus-data/raw/*.jsonl` (~7,667 docs).

**Method (research agent — Claude in a worktree, no code commits)**:
1. Literature pass — find published MTEB / BEIR retrieval numbers for each candidate, then *binary quantization* recall floors where available (Cohere, HuggingFace blog posts on binary quant, recent 2025 sign-bit-rerank papers).
2. Empirical pass — if literature is thin (likely for 384d models), produce a Python notebook (in a research worktree, not committed to mainline) that: pulls each candidate via `sentence-transformers`, embeds the 7,667 doc corpus + ~100 held-out queries, computes brute-force f32 ground-truth top-10, then computes recall@10 under sign-bit + f32-rerank with K∈{32, 64, 96, 128, 256}. Mean-pool + L2-norm to match candle path.
3. Candidates (all in-process / pure-Rust-feasible via candle / wheel-size-tolerable): **bge-small-en-v1.5 (384d)**, **all-MiniLM-L6-v2 (384d)**, **e5-small-v2 (384d)**, **gte-small (384d)**, **bge-base-en-v1.5 (768d)** as upper-size bound.
4. If no 384d candidate hits ≥0.85 recall@10 at K=64, evaluate: (a) raise K (cost ≈ linear extra f32-rerank), (b) post-quantization centering (subtract corpus mean before sign — one extra f32 vector in `_fathomdb_embedder_profiles`, no learned rotation, cheap), (c) jump to 768d (bge-base, +wheel-size).

**Deliverable**: `dev/notes/0.7.1-default-embedder-research.md` with:
- Recall@10 table per (model, K) with sample sizes + confidence intervals.
- Recommendation block: one candidate model + one K value + optional mean-centering toggle.
- Explicit HITL question (see § 5 below).

**TDD test surface**: none (research, no behavior change).

**Codex review focus**: methodology soundness (corpus split, ground-truth construction, statistical sample size); claim-vs-evidence in the recommendation.

**Success criteria**: HITL has a clear pick-one between (a) BGE-small at the chosen K, (b) a different 384d model at the chosen K, (c) bge-base 768d, (d) accept "BGE-small with documented recall caveat."

**Work units**: medium-large (1–2 days research-agent runtime, codex review pass).

**Dependencies**: PVQ closure (so the rerank pipeline is the canonical version).

### Slice EU-1 — ADR: NEED-017 / REQ-033 exception

**Mandate**: carve a tightly-scoped exception to NEED-017 (`dev/needs.md:132`) and REQ-033 (`dev/requirements.md:270-274`) permitting the engine to fetch default-embedder weights on first use, gated by the caller having explicitly opted into the default embedder.

**Proposed ADR**: `dev/adr/ADR-0.7.1-default-embedder-weight-fetch.md`, status `accepted`. Proposed wording for the exception clause (carry verbatim into both `needs.md` and `requirements.md` cross-cites):

> **Exception (default embedder only).** When the caller opts into the default embedder by passing `use_default_embedder=true` (or by selecting it explicitly at construction), the engine MAY download the single declared default-embedder weight set from a fixed URL set on first use, cache it under the platform user-cache directory, verify by sha256, and load it. This exception is scoped to: (a) the single named default-embedder identity recorded in the workspace; (b) weight files referenced by sha256 in the published `fathomdb-embedder` crate; (c) no other model, no arbitrary URL, no user-controllable model name. Caller-supplied embedders remain caller-owned per the unchanged rule. The first-use download path SHALL emit a structured `default_embedder_download` event in `OpenReport.embedder_events` describing url + bytes + sha256 + cache-path, so the wire/disk activity is visible.

**Blast radius**: `dev/adr/ADR-0.7.1-default-embedder-weight-fetch.md` (new); `dev/needs.md:132` (amend NEED-017 with cross-cite); `dev/requirements.md:270-274` (amend REQ-033 with cross-cite); `dev/adr/ADR-0.6.0-default-embedder.md` (add cross-cite to the new ADR); `dev/adr/ADR-0.6.0-decision-index.md` (register).

**TDD surface**: none yet (docs only).

**Codex review focus**: scope-creep — confirm the exception cannot be read as license to fetch arbitrary user-named models; confirm the visibility requirement (`embedder_events`) is mandatory not optional; confirm caller-supplied embedders are still subject to the original rule.

**Success criteria**: HITL signs the ADR `accepted`. NEED-017 and REQ-033 link to the new ADR. No grep of "implicit download" in the codebase is unaccompanied by the exception reference.

**Work units**: small.

**Dependencies**: EU-0 (so the exception names a concrete model identity).

### Slice EU-2 — Design: EMB-5 loader sub-design in `design/embedder.md`

**Mandate**: write the loader sub-design called out by `ADR-0.6.0-default-embedder.md:106-109` (EMB-5) directly into `dev/design/embedder.md`. Today that file (`dev/design/embedder.md:1-23`) only covers dispatch-pool / warmup; this slice multiplies its size.

**Required sections**:
0. **Mean-centering sub-design (added 2026-05-28 from EU-0 outcome).** Schema:
   one new column on `_fathomdb_embedder_profiles`, `mean_vec BLOB` (1536
   bytes for the chosen 384d model), nullable for backwards-compat with
   pre-mc workspaces. Lifecycle: computed once at first ingest by
   averaging the corpus f32 vectors observed so far in a streaming pass,
   pinned at the workspace's first non-trivial commit; subsequent writes
   leave it unchanged (no silent drift). Application: subtracted from each
   f32 vector before the sign-bit step on both write and query; the f32
   rerank uses the un-centered vector (centering is a sign-quant bias-
   correction, not a geometry change). Explicit reindex path is the only
   way to refresh the pinned mean — document this as a known limitation.
   This is a forward-cite for EU-5's schema work.
1. **Loader scope**: HF resolve URL pattern; files fetched (`config.json`, `tokenizer.json`, `model.safetensors`); per-file sha256 pinned in `fathomdb-embedder` Rust constants for `BAAI/bge-small-en-v1.5` (revision `5c38ec7c405ec4b44b94cc5a9bb96e735b38267a`).
2. **HTTP transport**: `ureq` blocking client; explicit handling of HF 302 → CloudFront; `Range` resume on partial files; configurable connect/read timeouts (defaults 10s connect / 60s read); fixed retry/backoff (3 attempts).
3. **Auth tokens**: read `HF_TOKEN` env var if present, send `Authorization: Bearer`. No fallback to user keychain. Document that public BGE-small does not require it; the support is there for token-gated mirrors only.
4. **Cache layout**: `<dirs::cache_dir>/fathomdb/embedders/<model-sha-prefix>/<file>`. Best-effort compat with `~/.cache/huggingface` layout for users who already have the file (read-only probe; never write into hf-hub's layout).
5. **Atomic write**: download to `<file>.partial`, fsync, rename.
6. **Verification**: sha256 of completed file MUST equal the pinned constant or the file is removed and the load fails with `EmbedderLoadError::ChecksumMismatch`. No "trust on first use."
7. **Cold-load timing**: warmup runs in `Engine.open` (per ADR-0.6.0-async-surface Invariant D); first-use download counted toward `embedder_warmup_ms`; a separate `embedder_download_ms` field is added to `OpenReport`.
8. **Endianness invariant**: pre-converted little-endian assumption per `ADR-0.6.0-default-embedder.md:110-113`.
9. **Failure taxonomy**: `EmbedderLoadError::{NetworkUnavailable, ChecksumMismatch, CacheIoError, ModelDeserialize, TokenizerLoad}`, each mapped to an engine-level open error.
10. **Concurrency**: process-level file lock around the cache directory during download so concurrent `Engine.open` calls do not race (`fs2::FileExt::lock_exclusive` on a `.lock` sibling). Document that this serializes first-use cold-starts.

**Blast radius**: `dev/design/embedder.md` (extend); cross-cites in `dev/adr/ADR-0.6.0-default-embedder.md` and the new EU-1 ADR.

**TDD surface**: none.

**Review gate**: codex review on the design document. **HITL sign-off required before any loader code lands** (EU-3+). This mirrors the PVQ pattern of `0.7.0-PVQ-P1-DESIGN` going through three fix passes before `P1-IMPL` opened.

**Success criteria**: design PASSes codex; HITL approves; the document answers every bullet above with a concrete decision (no "TBD").

**Work units**: medium.

**Dependencies**: EU-0 (need the chosen model's file list + shas), EU-1 (exception ADR established so the design can reference the legal basis).

### Slice EU-3 — Loader implementation (RED + GREEN)

**Mandate**: implement the loader specified by EU-2 inside `fathomdb-embedder`, behind a `default-embedder` Cargo feature so users who never opt in pay zero dependency cost.

**TDD-RED first** (separate commit): tests that fail because the loader does not exist:

### Slice EU-4 — Candle BGE embedder implementation (RED + GREEN)

**Mandate**: implement `CandleBgeEmbedder` (rename if EU-0 picks a different model) implementing the `Embedder` trait from `fathomdb-embedder-api/src/lib.rs:23-27` against `candle-transformers::bert::BertModel`, with mean-pool + L2-norm.

**TDD-RED first**:
- `identity_returns_chosen_model_at_chosen_revision` — name + revision + dimension match constants.
- `embed_returns_unit_norm_vector` — `(‖v‖ - 1.0).abs() < 1e-5` (Invariant 1 of ADR-0.6.0-embedder-protocol).
- `embed_returns_dimension_correct_vector` — `v.len() == identity().dimension`.
- `embed_is_deterministic_for_same_input` — two calls return byte-identical bytes (modulo platform-FP tolerance documented in test).
- `embed_two_similar_inputs_high_cosine` — e.g. cosine("the cat sat on the mat", "a cat is on a mat") > cosine of those vs an unrelated sentence. Anchors the model is doing semantic work, not random.
- `embed_does_not_panic_on_empty_string` — returns a valid unit-norm vector (BERT will produce one from `[CLS][SEP]`).
- `embed_does_not_call_back_into_engine` — Invariant 2 (compile-time: the impl crate does not link `fathomdb-engine`).

**GREEN**: load model via the EU-3 loader; build `BertModel`; tokenize with `tokenizers::Tokenizer`; forward pass; mean-pool across attention-masked tokens; L2-norm; return `Vec<f32>`.

**Cargo deps added (feature-gated)**: `candle-core`, `candle-nn`, `candle-transformers`, `tokenizers`, `safetensors` (via candle re-export per ADR consequence).

**Blast radius**: `src/rust/crates/fathomdb-embedder/src/candle_bge.rs` (new), `Cargo.toml` (deps under `[features] default-embedder = [...]`).

**Codex review focus**: mean-pool uses attention-mask (not unconditional mean across pad tokens), L2-norm is post-pool, dimension constant matches the loaded model's `config.json`, no `pyo3-log` / no `tracing::info!` inside `embed()` (Invariant 3).

**Success criteria**: all RED tests GREEN; codex PASS; wheel-size diff measured + reported (per EMB-3).

**Work units**: medium.

**Dependencies**: EU-3 PASS.

### Slice EU-7 — Real-corpus AC validation + lock-flip

**Mandate**: re-run AC-013/AC-013b/AC-019 against `data/corpus-data/raw/*.jsonl` using the real default embedder (bge-small + mc, K=128); confirm recall@10 against the **re-derived** floor (see below).

**Pre-work — floor re-derivation (added 2026-05-28).** The existing
`AC013B_RECALL_FLOOR = 0.90` (`tests/perf_gates.rs:159`) was calibrated on
the isotropic AC-013b synthetic fixture, which is anomalously friendly to
sign-bit quantization. It cannot be applied unchanged to real bge-small
embeddings without re-derivation. Before the EU-5 lock-flip cherry-picks
to mainline, this slice must:
1. Sample N≥1000 real-corpus queries (synthesized as in EU-0 §1.2, or a
   larger held-out set if available).
2. Measure recall@10 for the chosen config (bge-small + mc, K=128) at
   canonical-CI scale (N=1M docs from `data/corpus-data/raw/` + synthetic
   replicates if needed).
3. Set a new floor at `measured - 3σ` (or another HITL-locked margin

## 4. Risk register

1. **Wheel size**. ADR-0.6.0 estimated +130MB. With the EU-0 choice of
   bge-small (~133MB model weights, plus candle deps), the actual delta
   is meaningfully smaller than the ADR's worst-case estimate, but still
   the dominant per-platform wheel-size driver. If platform CI rejects,
   EU-6 blocks. *Mitigation*: feature flag means users without
   `default-embedder` extra pay nothing; document the
   `pip install fathomdb[default-embedder]` shape. Note: had HITL picked
   bge-base, the model alone would have been ~440MB — the EU-0 choice
   already reflects a wheel-size cost calculation.
2. **CI cannot reach Hugging Face**. First-use download in CI will hang or fail. *Mitigation*: CI pre-warms cache via an explicit `cargo run --bin warm-embedder-cache` step before test runs; documented in EU-3.
3. **CPU-only candle on macOS arm64 is slow at cold-start**. *Mitigation*: warmup duration reported in `OpenReport`; document expected range; do not gate on a warmup latency budget in 0.7.1.
4. **`pyo3-log` re-entrancy regression**. Invariant 3. *Mitigation*: EU-4 has a compile-time guard (no `log` macros in `embed()`); EU-6 has a regression test (`logging.basicConfig(DEBUG)` + 4 concurrent writes).
5. **AC-013b RED on real corpus**. If EU-0's recommendation does not hold up at full corpus scale, EU-7 fails and EU-4 must re-run on the fallback model. *Mitigation*: EU-0 uses the same corpus EU-7 will measure against.
6. **Migration story for 0.6.x/0.7.0 workspaces with `fathomdb-noop` profile-pinned**. Per `ADR-0.8.0-embedder-identity-change-workflow`, no in-place swap. 0.7.1 release notes must document the wipe-and-rewrite story for users who actually deployed a 0.7.0 workspace (likely few-to-zero in practice). *Mitigation*: clear release-note copy.
7. **Endianness**. ADR-0.6.0-default-embedder.md:110-113 calls out LE assumption. CI runs on LE only. *Mitigation*: explicit debug-assert in `CandleBgeEmbedder::new` that `cfg!(target_endian = "little")`; release builds document.

## 5. Definition of done

The default embedder is no longer deferred when:

1. `fathomdb-embedder` ships a real `Embedder` impl (not `NoopEmbedder`) gated by `default-embedder` feature.
2. `default_embedder_identity()` in `fathomdb-engine` returns the chosen model's identity, and `OpenReport.default_embedder` reflects that identity at runtime.
3. First-use weight download is visible in `OpenReport.embedder_events` and verified by sha256.
4. NEED-017 and REQ-033 cite `ADR-0.7.1-default-embedder-weight-fetch.md` and the exception wording is in place.
5. `dev/design/embedder.md` contains the EMB-5 loader sub-design (10 sections per § EU-2), HITL-signed.
6. Python and TypeScript bindings expose `use_default_embedder` / `useDefaultEmbedder` and round-trip the real identity in their open reports.
7. AC-013 / AC-013b / AC-019 GREEN against the real corpus with the default embedder, at canonical-CI scale (N=1M).
8. ADR-0.6.0-default-embedder.md status updated from "accepted but unimplemented" to "implemented in 0.7.1" with the chosen-model + chosen-K + chosen-centering decisions appended.
9. 0.7.1 release notes (`CHANGELOG.md`) call out: default embedder shipped; opt-in flag; first-use download; wheel-size delta; pre-existing workspaces with `fathomdb-noop` profile fail-closed by design.

When those nine boxes are checked, the three-release deferral is over.
