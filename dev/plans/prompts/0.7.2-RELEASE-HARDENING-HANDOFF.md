### PR-2 — AC-013b recall floor re-derivation against real embeddings

**Mandate**: replace the current `AC013B_RECALL_FLOOR = 0.90` constant
in `tests/perf_gates.rs` with a floor derived from the actual EU-7
measurement on real corpus + real embedder, with explicit headroom for
production variance. Lock the new floor in
`ADR-0.7.0-vector-binary-quant.md § 2 point 4` via an ADR amendment.

**Why this is a separate slice from EU-7**: EU-7 measured recall against
real embeddings as part of validating the embedder choice. This slice
takes that measurement and turns it into a permanent gate: which number
is the floor, what headroom does it carry, what does it mean for
regression detection.

**Context to carry**: empirical research on bge-small + mc landed these
numbers at the 7,667-doc corpus, N=100 queries (full table in
`dev/notes/0.7.1-default-embedder-research.md` and synthesized in
`dev/design/embedder-decision.md`):
- K=64 mc: 0.843
- K=96 mc: 0.880
- K=128 mc: 0.907 (clears 0.90, thin 0.7 pp cushion)
- K=256 mc: ~0.945 (mc lift non-significant)

**`TOP_K_BIT_CANDIDATES` is locked at K=192** for the shipping default
(2026-05-28 decision; see embedder-decision.md). EU-7 measures the
real-corpus recall at K=192 / canonical-CI scale; PR-2 takes that as
the anchor for the floor. PVQ chose 0.90 calibrated for isotropic
synthetic — that floor is no longer the right shape and PR-2 derives
the new one.

**Method**:
1. Read EU-7's closure number from `STATUS-embedder-undefer.md`. Call
   it `R_measured`.
2. Compute floor as `R_measured - 2*σ_bootstrap` (one-sided lower
   bound; aim for ~95% confidence that production lands ≥ floor).
   Round down to the nearest 0.01. Document the derivation.
3. Update `tests/perf_gates.rs::AC013B_RECALL_FLOOR` to the new value.
4. ADR amendment: `dev/adr/ADR-0.7.0-vector-binary-quant.md § 2 point 4`
   gets the new floor + a one-sentence note pointing at the derivation
   commit and PR-2.
5. RED test pre-commit: a sentinel test
   (`ac_013b_floor_matches_adr`) that reads the floor constant and
   asserts it equals the value documented in the ADR amendment. Catches
   future drift between the gate and the doc.

**TDD surface**: the floor-vs-ADR sentinel test (above).

**Codex review focus**: that the headroom calculation is honest (no
sleight-of-hand making R_measured = floor); that the ADR amendment
clearly says what the floor is and what it isn't (e.g., "binary-quant
retention on real bge-small embeddings, not absolute retrieval
quality").

**Success criteria**: floor updated; ADR amendment signed by HITL; gate
test passes; canonical-CI rerun (PR-3) confirms the new floor with
≥2σ margin.

**HITL gate**: floor number must be approved before commit. The user
explicitly stated "do not change wording to fit" — if EU-7's number is
below 0.90, the floor must drop, but the floor must also have honest
headroom and not be a number gerrymandered to match the result.

**Work units**: small-medium.

**Dependencies**: **PR-2a** (and PR-2b if GO) — PR-2 anchors `R-2σ` on the
post-investigation number, not the raw 0.828; and PR-1 (so the ADR amendment
lands against the already-cleaned doc).

### PR-5 — Corpus-driven test harness

**Mandate**: build a reusable Rust integration-test harness that takes
any subset of `data/corpus-data/raw/*.jsonl` and runs it through the
engine with assertable invariants. Today's `tests/support/corpus_subset.rs`
is the seed (Pack 4 corpus tests use it). PR-5 extends it into a
first-class harness available to any future perf or behavior test.

**Scope**:
- New module `tests/support/corpus_harness.rs` (or extend
  `corpus_subset.rs`).
- API:
  - `CorpusFixture::small()` / `::medium()` / `::full()` — three
    deterministic subset sizes (e.g., 100 / 1000 / all-7667).
  - `with_real_embedder()` vs `with_synthetic_embedder()` — toggles
    between the shipped default embedder and the dense-isotropic
    VaryingEmbedder. Both code paths must be supported.
  - `ingest_into(engine: &Engine)` — uses batched writes (the
    `4a95cfd` regression test pattern).
  - `query_set(n: usize, seed: u64)` — deterministic held-out query
    bodies, excluding the source doc (per EU-0's
    `target_doc_idx` exclusion pattern).
  - Invariants helpers:
    - `assert_vec0_row_count_matches_ingest(...)`
    - `assert_fts_index_populated(...)`
    - `assert_search_returns_non_empty_for_each(...)`
- Cache real-embedder vectors per (model, corpus-subset) to local disk
  so tests that don't change the model don't re-embed. Crucial for
  dev-loop perf — re-embedding 7,667 docs every test run is the
  inner-loop killer.

**TDD surface**:
- `corpus_harness_invariants` test that exercises every assert helper
  on a tiny fixture.
- Determinism test: two identical fixture constructions produce
  byte-identical embedding caches.

**Codex review focus**: cache invalidation correctness (cache must
invalidate on model identity change OR corpus content change); no test
silently uses a stale cache; deterministic-write invariants on the
cache files.

**Success criteria**: existing Pack 4 tests
(`corpus_vector.rs` / `corpus_fts.rs` / `corpus_graph.rs`) migrate to
the harness with no behavior change; future tests can call
`CorpusFixture::medium().with_real_embedder().ingest_into(&engine)` in
one line.

**Work units**: medium.

**Dependencies**: PR-4 (so the real embedder is shipped and importable
in tests).

**Mandate**: build a fast-running subset of AC-013/AC-013b/AC-019 that
exercises the same code paths at small N (≈1K–10K) using the corpus

**Why this matters**: today's `cargo test ac_013` only runs under
`AGENT_LONG=1` and seeds N=10K-1M corpus per run, taking minutes to
hours. That's appropriate for canonical-CI but useless for inner-loop
work. Dev-loop perf needs to run in <30 seconds per AC.

**Scope**:
- New `perf_gates_devloop.rs` with `ac_013_devloop`,
  `ac_013b_devloop`, `ac_019_devloop`. Each:
  - Uses `CorpusFixture::small()` (~1K rows) via PR-5.
  - Caches embeddings (per PR-5) so seed cost is ~0 after first run.
  - Runs ≥100 queries.
  - Asserts a *dev-loop* budget — looser than canonical-CI but tight
    enough to catch real regressions. Recommended: 2-3× canonical
    budget at the smaller N.
  - Prints `DEVLOOP_NUMBERS` line for trend tracking.
- Doc in `dev/design/perf-gates.md` (new): the two-tier model
  (devloop = fast feedback, canonical = ship verdict).

**TDD surface**: each devloop AC is its own test; collectively they
double as the regression suite.

**Codex review focus**: that the devloop budget is not so loose it
masks real regressions (anti-pattern: budget = current+infinity); that
the cached-embedding path doesn't accidentally bypass the production
read SQL.

**Success criteria**: `cargo test perf_gates_devloop` runs in ≤30
seconds on dev box from a warm cache; catches both the batch-collapse
regression (artificially re-introduce in CI to verify) and the

**Work units**: medium.

### PR-9 — Embedder robustness (concurrent-embed safety + embed watchdog)

**Mandate**: close the two engine-robustness items EU-5f surfaced but did not
resolve (see `dev/plans/runs/0.7.1-EU-7-findings.md`):
1. **Concurrent-embed safety.** `PROJECTION_WORKERS=2` calls the candle embedder
   concurrently on one shared `Arc<dyn Embedder>`; this was only ever exercised
   safely with a harness-side serializing Mutex. Either prove it safe under a
   sustained concurrent real-corpus seed (≥10K docs, unserialized) with a RED
   regression test, or serialize embeds engine-side (e.g. guard the embedder, or
   document that `Embedder` impls must be internally concurrency-safe and make
   `CandleBgeEmbedder` so). This **gates PR-3's N=1M dispatch** (see PR-3 risk).
2. **Embed watchdog (ADR-0.6.0-embedder-protocol Invariant 5).** The 30s
   per-`embed()` timeout is unimplemented on the projection path — a *hung*
   (non-panicking) embed still blocks a worker forever; the EU-5f `catch_unwind`
   only catches panics. Add the watchdog so a hung embed surfaces
   `EmbedderError::Timeout` and the existing retry/failure path engages.

**TDD surface**: a sustained-concurrent-seed regression test (no Mutex); a
hanging-embedder test that asserts `drain` returns via timeout rather than
wedging. Both RED-first.

**Codex review focus**: that the chosen concurrency approach doesn't regress
throughput; that the watchdog can't false-timeout a legitimately-slow cold embed.

**Work units**: medium. **Dependencies**: PR-1. **Sequencing**: should land
before PR-3 (it gates the N=1M seed). If PR-3's ~10K pre-flight seed passes
clean, item (1) may reduce to "add the regression test"; item (2) stands
regardless.
