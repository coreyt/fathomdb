## The decision loop (per slice — orchestration.md §9, applied)
1. Generate the slice prompt from the template + the slice contract. **Verify every
   load-bearing "test X asserts/doesn't assert Y" claim against the file at the baseline
   BEFORE writing it into the prompt** ([[slice-prompt-verify-test-claims]]).
2. Spawn the implementer (own worktree, fresh baseline). It TDDs + merges to local `main`.
3. **Gate from git:** confirm the merge + the `output.json` (incl. `tdd_evidence` RED sha).
4. **Run codex §9** on the slice diff:
   `codex exec review --base <baseline-sha> --dangerously-bypass-approvals-and-sandbox`
   (codex is PRIMARY; an adversarial subagent is fallback only). **Read the real exit/
   findings, not a wrapper echo** ([[background-exit-masks-real-exit]]).
5. PASS → close. CONCERN → override (§7, justify on the board) or fix-N. BLOCK → **fix-N**
   in the same worktree (not a reserved-gap). Re-review the fix diff.
6. On close: update the board (status, X1/X2/X3, the §7 decision entry) in **one docs
   commit**; advance the pointer; remove the worktree (§11).
7. **Reserved-gap follow-on** that must land before N+5 → a **new** slice N+1…N+4 (fresh
   baseline + worktree + prompt + review). If the band fills, **HALT to HITL** (mis-scoped).
