# Slice 0.7.0-WF-FIX-PYPI-SMOKE — PyPI index propagation retry

Carried from 0.6.1 close-out. The 0.6.1 GA cycle observed the
PyPI smoke-test failing on first run inside the index-propagation
window (5-15 minutes between publish step and registry serving
the new wheel). This slice lands a retry-with-backoff helper in
`scripts/release/smoke/smoke-pypi-wheel.sh` so the 0.7.0 RC1 / GA
cycles do not race propagation.

## Hard constraints

- **No release.yml YAML edits outside the smoke step.** This
  slice owns the smoke helper + its workflow wiring; any other
  release.yml change is a separate WF-FIX slice.
- **No silent retries.** Every attempt logs.
- **Non-propagation failures must not be retried.** Version
  mismatch on a 200 response = publish bug = stop immediately.
