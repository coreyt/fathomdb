## Canonical-runner perf gate

This slice runs `perf record` instrumentation **alongside** the
existing perf harness; it does not replace the harness's verdict.
Workflow invocation shape:

```bash
gh workflow run perf-canonical.yml --ref <branch> \
  -f run_full_scale=true \
  -f ac012_corpus_n=1000000 \
  -f perf_record=true   # NEW; harness must learn this flag
```

If the workflow does not yet support `perf_record=true`, the
implementer's first sub-task is the minimal workflow + harness
edits to thread the flag through (record into
`/tmp/perf-{ac012,ac020,ac013}.data`, upload as artifact, run
`perf report --stdio` and capture the top-50 frames). The
workflow edits are diagnostic-instrumentation only; no engine
behavior changes.

## Tasks

1. **Workflow + harness instrumentation** (if not already wired):
   add `perf_record` workflow input; wire `perf record -g
--call-graph dwarf` around the AC-020 + AC-012 + AC-013
   measurement bodies; upload the per-test perf.data + perf
   report stdio output as workflow artifacts.
2. **Dispatch the canonical run.** Capture workflow URL.
3. **Aggregate.** For each AC's `perf report` output, compute:
   - Top 10 frames by cycle share (sequential mode).
   - Top 10 frames by cycle share (concurrent mode).
   - Any frame whose concurrent share > 2× its sequential share
     (the super-linear-grower list).
4. **Classify.** Each super-linear-grower into one of:
   - **H1 — pcache1 / page-cache mutex** (e.g. `pcache1Fetch`,
     `pcache1Pin`, mutex acquires localised in
     `sqlite3PcacheFetch*`).
   - **H2 — WAL atomics / shm protocol** (e.g. `walShmGet`,
     `walReadFrame`, `__aarch64_*` atomics or x86_64 equivalents
     localised in `walXXX` symbols).
   - **H3 — rusqlite / ReaderPool-borrow Mutex** (e.g.
     `std::sync::Mutex::lock` on `ReaderPool::borrow` path;
     rusqlite-internal mutex symbols).
   - **Other** (capture name + share; do not force into a bucket).
5. **Primary hypothesis call.** Single named hypothesis (H1, H2,
   H3, or "none / no super-linear-grower at canonical scale").
   This is the input HITL Q2 consumes.
6. **AC-013 / AC-019 baseline (conditional).** If the canonical
   runner can complete bulk-vec seeding at 1 M corpus inside the
   workflow timeout, also capture AC-013 / AC-019 p50 / p99 +
   the equivalent perf classification. If seeding does not
   complete, record the AC-013 / AC-019 descope verdict in the
   output JSON.
7. **Write output JSON** (schema below).
