## Deliverables

### 1. Eval QA Artifact Contract

Add a small shared helper for eval-only QA JSONL rows. Keep it under
`tests/corpus/scripts/` unless a better local convention already
exists.

Target shape:

```jsonc
{
  "qa_id": "stable deterministic id",
  "source": "qaconv|qasper|enronqa|qmsum|pmc_oa",
  "source_type": "email|meeting|paper|article|note|todo",
  "question": "string",
  "answers": ["gold answer strings"],
  "answer_type": "span|free_form|yes_no_maybe|summary|abstain",
  "evidence_doc_ids": ["doc ids in corpus"],
  "evidence_spans": [
    {"doc_id": "id", "start": 0, "end": 120, "text": "optional"}
  ],
  "negative_doc_ids": ["optional hard negatives"],
  "relation_type": "optional existing relation vocabulary value",
  "metadata": {
    "split": "train|validation|test",
    "user_id": "optional",
    "thread_id": "optional",
    "upstream_id": "optional"
  },
  "license": "SPDX or LicenseRef",
  "provenance": "pinned upstream tag"
}
```

Rules:

- Eval QA rows are not ingested as documents.
- Every row must have deterministic `qa_id`.
- If evidence maps to an ingested corpus doc, include
  `evidence_doc_ids`.
- If evidence is not in the ingested corpus, either omit that row or
  mark it clearly so accuracy tests cannot accidentally count it as
  grounded.
- Record output file sha256 values in `manifest.json` or an adjacent
  eval manifest if that is cleaner.

### 2. Add QAConv

Add a new acquisition script:

- `tests/corpus/scripts/acquire_qaconv.py`

Source:

- GitHub: `salesforce/QAConv`
- License: BSD-3-Clause
- Expected upstream scale: 10,259 selected conversations and 34,204
  QA pairs.

Implementation intent:

- Ingest a deterministic subset of **1,000-1,500 conversation docs**
  as corpus documents.
- Export **3,000-5,000 QA rows** for the selected conversations.
- Map business email conversations to `source_type = "email"`.
- Map meeting/panel/work-channel conversations conservatively to
  `meeting` or `note`, based on upstream metadata.
- Preserve speaker/user metadata where available.

Subsampling:

- Sort by split, upstream source, conversation ID, question ID.
- Prefer balanced source coverage over taking one source in bulk.
- Keep all QA rows for selected conversations until the QA cap is
  reached.

Manifest/docs:

- Add source and eval outputs to the manifest.
- Update `tests/corpus/corpus-card.md`.
- Update `dev/corpus-creation/architecture.md`.

### 3. Add QASPER

Add a new acquisition script:

- `tests/corpus/scripts/acquire_qasper.py`

Source:

- Hugging Face: `allenai/qasper`
- License: CC-BY-4.0
- Expected upstream scale: 1,585 NLP papers and 5,049 QA/evidence
  questions.

Implementation intent:

- Ingest **1,000-1,500 paper docs** with
  `source_type = "paper"`.
- Export all QA/evidence rows for selected papers.
- Represent each paper as one corpus document using title, abstract,
  and section text in `body`.
- Preserve authors, paper ID, title, year, and section/evidence
  metadata where available.

Subsampling:

- If full QASPER is acceptable at about 1,585 docs, ingest all
  papers.
- Otherwise sort by paper ID and cap at 1,250 docs while preserving
  split balance.

Chain update:

- Add `paper` anchors to `generate_chain_corpus.py`.
- Add at least one paper chain shape, such as `PAPER->NOTE->TODO`.
- Use only existing relation vocabulary.
- Use `relation:cites` only when the source actually supports a
  citation/evidence relation; otherwise prefer `summarizes` or
  `mentions`.

Manifest/docs:

- Add source and eval outputs to the manifest.
- Update corpus-card and architecture docs.
