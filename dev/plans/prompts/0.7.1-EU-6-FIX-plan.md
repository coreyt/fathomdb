## Root-cause analysis — why FIX-1 is one slice, not four

P1, #5, #6 all reduce to the same architectural mistake:

- `pyproject.toml [tool.maturin] features = [..., "test-hooks", "default-embedder"]`
  bakes dev-time-only toggles into the release wheel build that `release.yml`'s
  maturin-action reads.
- `package.json` has the inverse asymmetry: `build:native:debug` (used by
  tests) has both features; `build:native` (used by release) has neither.

Net effect: tests pass against a feature surface (`test-hooks` + `default-embedder`)
that diverges in both directions from what gets shipped. Pytest validates a
binary nobody installs; npm ships a binary pytest never sees.

#4 is a smaller variant of the same theme: the size gate is partial coverage.
Adding the missing platforms costs little if done while the gate is already
being touched; deferring to "EU-6b" risks the soft TODO calcifying.

The right fix is to lock the feature axis to **one shipped configuration**
per binding, name it explicitly in workflow `--cargo-extra-args` /
`--features`, and run the gate on what actually ships across all 4 platforms.

## Shipped-feature decision (drives FIX-1 implementation)

**Decision**: ship one wheel/`.node` per binding, **compiled with
`default-embedder`**. The `use_default_embedder` flag is the runtime opt-in;
the Cargo feature must be on at compile time for the opt-in to function.

Rationale:
- The whole point of EU-5b → EU-6 is to give end users an opt-in surface.
  Shipping a no-feature wheel makes `use_default_embedder=True` always raise
  `EmbedderNotConfigured`, which defeats EU-6.
- "Dual-wheel" via PyPI extras can't change native binary contents; would
  require split-package publishing (`fathomdb` + `fathomdb-default-embedder`),
  which is its own multi-slice campaign.
- ADR-0.6.0-default-embedder.md EMB-3's "no-feature wheel size regression"
  budget is preserved as a **CI sanity check** on the no-feature build path
  (does the underlying minimal-deps compile path still hold?), not a shipped
  artifact gate.

Consequence: wheel-size gate baseline is the **with-feature** number; the
no-feature build is recorded informationally to detect dependency creep.

## Test-hooks-leak mitigation (drives #5 fix)

`test-hooks` cargo feature exposes `_write_vector_for_test` /
`writeVectorForTest` on the binding surface. Currently leaks because
`pyproject.toml` lists `test-hooks` unconditionally.

**Decision**: remove `test-hooks` from `pyproject.toml [tool.maturin] features`.
Tests install the binding via an explicit pytest fixture that runs
`maturin develop --features pyo3/extension-module,test-hooks,default-embedder`
before the session. Release workflow's maturin-action passes
`--features pyo3/extension-module,default-embedder` via `args:` —
`test-hooks` is never on the release-build path.

TS side already has this separation (`build:native:debug` vs `build:native`);
only need to add `--features default-embedder` to `build:native` for P1.

## Acceptance criteria (cross-slice)

**FIX-1 (packaging)**:

- AC-FIX1-1: Published-artifact-equivalent build of `fathomdb-py` (i.e.
  `maturin build` run with the exact `args:` from `release.yml`) does NOT
  expose `_write_vector_for_test` or `_configure_vector_kind_for_test` on
  the `Engine` PyO3 class. Asserted by a new test that introspects the
  built wheel.
- AC-FIX1-2: Published-artifact-equivalent build of `fathomdb-napi` does
  NOT expose `writeVectorForTest`. Asserted symmetrically.
- AC-FIX1-3: Published-artifact-equivalent build of `fathomdb-py`
  successfully opens an engine with `use_default_embedder=True` (i.e.
  `default-embedder` feature is compiled in). Asserted by a network-gated
  smoke test that uses the release-shaped wheel.
- AC-FIX1-4: Published-artifact-equivalent build of `fathomdb-napi`
  successfully opens an engine with `useDefaultEmbedder: true`. Symmetric.
- AC-FIX1-5: CI wheel-size gate runs on all four release-target platforms:
  Linux x86_64, Linux aarch64, macOS (universal or x86_64), Windows x86_64.
  Each platform has a recorded baseline; regression > 20 MB blocks.
- AC-FIX1-6: `pyproject.toml` no longer lists `test-hooks` in
  `[tool.maturin] features`. Pytest fixture installs with explicit
  features before running the suite.
- AC-FIX1-7: `package.json` `build:native` includes `--features default-embedder`.
- AC-FIX1-8: `release.yml` build-python step passes explicit
  `args: --release --out dist --features pyo3/extension-module,default-embedder -i python3.10 python3.11 python3.12`
  (or equivalent — explicit feature list, not pyproject discovery).
- AC-FIX1-9: `release.yml` build-napi step passes `--features default-embedder`
  to `npm run build:native` (now that it includes the feature) OR via
  `CARGO_BUILD_FLAGS`; verified by inspecting the built `.node`.
- AC-FIX1-10: `dev/interfaces/python.md` and `dev/interfaces/typescript.md`
  state explicitly: "released wheels/binaries are compiled with
  `default-embedder`; the no-feature build path is preserved as a CI
  sanity check, not a shipped artifact."

**FIX-2 (typing)**:

- AC-FIX2-1: Python `_fathomdb.pyi` declares `EmbedderEvent` as a
  `typing.Union[DefaultEmbedderDownloadEvent, DefaultEmbedderCacheHitEvent, MeanVecPinnedEvent]`
  where each is a `TypedDict` with `kind: Literal["..."]` and
  per-variant required payload keys.
- AC-FIX2-2: Python `fathomdb.types.OpenReport.embedder_events` annotated
  as `list[EmbedderEvent]` (the union), and `fathomdb/types.py` exports
  the variant `TypedDict`s for caller import.
- AC-FIX2-3: Pyright at `typeCheckingMode = "standard"` over
  `src/python/tests/` confirms narrowing: `if event["kind"] == "DefaultEmbedderDownload": reveal_type(event["bytes"])`
  must reveal `int` (not `int | None`). Asserted by a `pyright_narrowing.py`
  test that uses `reveal_type` and is checked in CI.
- AC-FIX2-4: TS `EmbedderEvent` declared in `src/ts/src/index.ts` as a
  discriminated union `DefaultEmbedderDownloadEvent | DefaultEmbedderCacheHitEvent | MeanVecPinnedEvent`,
  each with `kind: "..."` literal and only the relevant payload fields
  (not `Option<T>`-collapsed).
- AC-FIX2-5: TypeScript `tsc --strict` over `src/ts/tests/` confirms
  narrowing: in an `if (event.kind === "MeanVecPinned")` branch, `event.docCount`
  is `number` (not `number | undefined`). Asserted by a compile-time
  narrowing test (using `// @ts-expect-error` on the negative case).
- AC-FIX2-6: The native NAPI layer continues to emit a wide struct
  (napi-rs Rust→TS limitation), but the TS-side declared type in `index.ts`
  is the discriminated union, and a runtime test asserts the field set
  per `kind` matches what the union promises (no `dim` on a
  `DefaultEmbedderDownload`, etc.).
- AC-FIX2-7: `dev/interfaces/python.md` and `dev/interfaces/typescript.md`
  show callers the typed-narrowing pattern as the recommended consumption
  shape, with copy-pasteable code samples.

### Agent B1 — AC + design

Produces `dev/design/0.7.1-EU-6-FIX-2-design.md` with:

- Restatement of the 7 ACs.
- Decision: keep the Rust-side NAPI struct wide (compat with
  napi-rs codegen) but declare the canonical TS shape in `index.ts` as
  a discriminated union. Document the invariant that the Rust emitter
  MUST populate exactly the variant-appropriate fields and leave the
  others null/absent; cite the test that enforces this.
- Decision: Python uses `TypedDict` (not `@dataclass`) so existing
  `dict`-consumption code in user-land doesn't break; `TypedDict` is
  structural and runtime-checkable as `dict`.
- Compatibility note: this is type-stub only; runtime shape is
  unchanged from EU-6 GREEN, so callers who pattern-match on `event["kind"]`
  / `event.kind` keep working. Bumping the user-facing type from "dict"
  to "TypedDict union" is a strict refinement.
