## Tasks (chosen lever drives the surface)

The implementer's surface depends on the lever recorded in the
AC-020 ADR's Decision section. Choose the matching sub-section
below and ignore the others.

### If lever = PCACHE2

1. TDD RED. Update budget; confirm RED.
2. Implement custom `sqlite3_pcache_methods2` allocator under a
   new module `src/rust/crates/fathomdb-engine/src/pcache2.rs`.
   FFI through `rusqlite::ffi::sqlite3_config(SQLITE_CONFIG_PCACHE2,
...)`; install at process start before any
   `Connection::open` (ordering-correct per the B.1 lesson in
   whitepaper § 5).
3. Per-handle (or per-reader-pool-slot) page-cache slabs that do
   not contend on a global mutex.
4. Confirm `sqlite3_config_rc == SQLITE_OK = 0` (B.1 / G.1
   precedent for runtime-config validation). Add an accessor
   under `#[cfg(debug_assertions)]` per F.0 convention.
5. `dev/design/recovery.md` doc delta: none (PCACHE2 does not
   change durability surface).

### If lever = WAL2

1. TDD RED. Update budget; confirm RED.
2. Replace `libsqlite3-sys` with a custom build / fork pinned at
   the WAL2 patchset (`Cargo.toml` change + lock-file update).
3. Update `dev/design/recovery.md § WAL2 layout` section: two
   `*-wal` files alternately maintained; control file shape;
   external-backup-tool implications.
4. Confirm crash-recovery still passes the existing recovery
   test suite; add WAL2-specific recovery tests if the patchset
   surfaces new failure modes.

### If lever = R-W split

1. TDD RED. Update budget; confirm RED.
2. Replace single-database-file reader pool with reader-on-
   snapshot pool: writer on canonical DB file; reader pool on a
   read-only snapshot via `sqlite3_snapshot_*` API or
   checkpointed read-replica.
3. Snapshot lifecycle ADR + `dev/design/recovery.md § Reader
snapshot lifecycle` section: snapshot freshness contract,
   handoff at projection-runtime commit, AC-059b preserved.
4. Touch points in `lib.rs`: `ReaderPool` struct (line 158),
   constructor (line 707), pool fill (line 773); writer-commit
   path that triggers snapshot refresh.

### If lever = vendor-SQLite swap (libSQL)

1. **STOP and surface to HITL** before any source change.
   Vendor-SQLite swap is last-resort; HITL must explicitly
   re-confirm this lever choice before the slice proceeds. Any
   other reading of the ADR is a process error.
2. After re-confirmation: replace `libsqlite3-sys` /
   `rusqlite/bundled` with libSQL crate; confirm vec0 + FTS5
   extensions are available in the chosen libSQL version.
3. Full durability + recovery contract re-validation; document
   the new SQL-engine identity in `dev/design/recovery.md` and
   release notes.

### If lever = NONE (DIAG showed no super-linear-grower)

1. Confirm DIAG output JSON's `primary_hypothesis == "none"`.
2. AC-074 is asserted by re-measurement alone: dispatch the
   canonical-runner workflow against the current `main` tip;
   capture sequential / concurrent / speedup. If all three meet
   the revised envelope, AC-074 closes GREEN without any source
   change. Append AC-074; record the re-measurement verdict in
   the slice closure JSON; close the slice.
