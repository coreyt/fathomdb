## ADR contents (mandatory sections)

Follow the frontmatter / heading style of `dev/adr/ADR-0.7.0-ac020-architectural-lever.md`. Frontmatter must include `title`, `date: 2026-05-27`, `target_release: 0.7.0`, `desc`, `blast_radius` (cite file:line touch sites — see Pack 1/2 sections of the handoff), `status: draft, HITL-required`.

### 1. Problem

State AC-013 canonical-CI measurement (p50=2048ms / p99=2327ms at N=1M, W4.1-stacked-O1 stack) vs the proposed budgets (80 / 300 ms). Cite the canonical-output JSON. Cite the hot SQL at `lib.rs:2317-2323`. Identify root cause: per-query O(N) full scan of 1M × 768 × 4 B = 3 GB at ~1.5 GB/s memory bandwidth.

### 2. Decision

Adopt:

- **Binary quantization** of the f32 embedding into `bit[768]` via `vec_quantize_binary(embedding)`, stored on `vector_default` (sibling column on the existing `vec0` table — see Pack 1.1 in the handoff for the criterion).
- **Two-phase query** (Pack 2): bit-KNN top-K=64 candidates → f32 rerank → top-10.
- **Schema extension on the same migration**: `source_type` (vec0 partition_key, cardinality ~6) + metadata columns (≤16 vec0 limit) per HITL.
- **Recall floor**: ≥ 0.90 @ k=10 (HITL-locked).
- **Sqlite-vec pin**: keep `=0.1.7`. Bump to `=0.1.9` only if needed.
- **Embedder unchanged.** `VaryingEmbedder` + `_fathomdb_embedder_profiles` pinning untouched.

### 3. Alternatives considered (and rejected)

For each: 1-sentence mechanism, 1-sentence reason for rejection, citation into `dev/notes/0.7.0-vector-cost-research.md`.

- **Vectorlite** (DiskANN-on-SQLite fork).
- **sqlite-vec ANN alpha** (HNSW upstream-experimental).
- **Rust-side `usearch` / `instant-distance`** (out-of-process ANN).
- **Embedder swap** (dim reduction 768 → 384 or 256).
- **Partitioning alone** (no quantization).

### 4. Consequences

- Schema change (Pack 1): new `embedding_bin bit[768]` column on `vector_default`, plus `source_type` partition_key and metadata columns. One-pass migration via `INSERT … SELECT vec_quantize_binary(embedding)`.
- Ingest cost (Pack 1): writer double-write inside the same transaction. Single-writer + projection-cursor contract preserved.
- Query-path rewrite (Pack 2) at `lib.rs:2317-2323`. SQL shape documented in the handoff.
- Recall floor enforced by a new perf-gates test (`ac_NNN_recall_at_10`) against brute-force f32 ground truth on the AC-013 fixture.
- Post-0.7.0 follow-ups (graph index / ANN) tracked in `dev/notes/pcache2-followups.md`.

### 5. Architectural-lever accounting

Reaffirm — citing `dev/notes/0.7.0-vector-cost-research.md` — that binary-quant is a **data-encoding change**, not a second architectural lever. The 0.7.0 architectural lever remains `SQLITE_CONFIG_PCACHE2` (locked by `dev/adr/ADR-0.7.0-ac020-architectural-lever.md`). Document the fallback: if a future reviewer counts binary-quant as a second lever, slip Pack 2 to 0.7.1.
