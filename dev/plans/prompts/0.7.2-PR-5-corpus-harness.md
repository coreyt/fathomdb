  measurement from `dev/plans/runs/0.7.2-PR-2c-recall-rootcause.md`. The

- **Seed determinism**: `query_set(n, seed)` and any subset selection must be
  reproducible across runs and machines (handoff "Determinism test"). xorshift
  or `StdRng::seed_from_u64` is fine; `rand::thread_rng` is not.

## Plan

### 1 — API surface (per handoff line 535+)

Land the API in `tests/support/corpus_harness.rs` (preferred) or as an
extension of `corpus_subset.rs`. The handoff is permissive on layout; pick
whichever causes the smaller migration diff and document the choice in the
slice closure.

```rust
pub struct CorpusFixture { /* opaque */ }

impl CorpusFixture {
    pub fn small()  -> Self;  // ~100 docs deterministic
    pub fn medium() -> Self;  // ~1000 docs deterministic
    pub fn full()   -> Self;  // all (~7667) docs deterministic

    pub fn with_real_embedder(self) -> Self;       // cfg(default-embedder)
    pub fn with_synthetic_embedder(self) -> Self;  // always available

    pub fn ingest_into(&self, engine: &Engine) -> IngestReport;
    pub fn query_set(&self, n: usize, seed: u64) -> Vec<HeldOutQuery>;

    pub fn assert_vec0_row_count_matches_ingest(&self, engine: &Engine);
    pub fn assert_fts_index_populated(&self, engine: &Engine);
    pub fn assert_search_returns_non_empty_for_each(&self, engine: &Engine, qs: &[HeldOutQuery]);
}
```

- `small/medium/full` resolve to deterministic doc-id sets (not "first 100" —
  sorted by `doc_id` with a fixed slice — so two builds at different commits
  on the same checkout see the same set, modulo corpus-content changes).
- `with_real_embedder` must SKIP-gracefully when the `default-embedder` feature
  is off and on corpus-absent checkouts. `with_synthetic_embedder` is the
  always-available default.
- `ingest_into` MUST use the batched `PreparedWrite::Node` / `::Edge` path
  (the `4a95cfd` regression-prevention pattern) — do not single-row write.
- `query_set(n, seed)` follows EU-0 §1.2 (exclude-before-dedup).

### 2 — Per-(model, subset) embedding cache

- Cache key: `sha256(EmbedderIdentity || sorted_doc_ids || subset_label)`.
  Both `EmbedderIdentity` field changes AND subset content changes invalidate
  — the codex review focus is exactly this.
- File format: `bincode` or a simple little-endian `[len: u32][dim: u32][f32
  × len*dim]` body, plus a sidecar `.meta.json` with the identity, subset
  label, doc-id manifest sha, dim, and tool version. Atomic write (`.tmp` →
  rename).
- Cache location: gitignored under `data/corpus-data/.cache/` (confirm against
  `tests/corpus/corpus-card.md`); make the path overridable via env var for
  CI runners (e.g., `FATHOMDB_CORPUS_CACHE_DIR`).
- **First load** populates the cache; subsequent loads memory-map or
  read-once. A cache miss must NOT fail the test — fall through to live
  embedding and warn (`CORPUS_CACHE_MISS reason=…`).

## Codex review focus (call out in the review request)

Per handoff:

- **Cache invalidation correctness.** Identity change → invalidate. Subset
  content change → invalidate. No silent staleness path.
- **No test silently uses a stale cache.** A cache-miss path must be loud
  (log line) and the cache-hit path must verify the manifest sha before
  trusting bytes.
- **Deterministic-write invariants on the cache files.** Two equivalent
  constructions produce byte-identical files (the determinism test exists for
  this).
- **No accidental SKIP regressions.** The corpus-absent and
  default-embedder-off paths still SKIP gracefully. (A test that should SKIP
  but instead panics is the worst kind of CI bug.)
- **Batched ingest preserved.** The `4a95cfd` regression pattern is not
  silently single-row'd through any helper.
- **EU-0 §1.2 methodology preserved.** `query_set` excludes before truncate
  and dedups bodies. Do not re-introduce the withdrawn exclude-after path.
- **Single-writer + projection-cursor freshness contracts preserved** (no
  bypass of the projection pool; harness uses the public `Engine` API).
- **`EmbedderIdentity` immutable contract respected.** Tests use a seam to
  vary identity for cache-invalidation testing, not a workaround that mutates
  the production type.
