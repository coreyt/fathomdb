## 4. The signed HITL decisions you must honor (substrate gate, signed 2026-06-03)

- **Decision 4 ‚Äî op-store cascade under supersession (RATIFIED).** Supersession (the
  `UPDATE ‚Ä¶superseded_at` tombstone + the new-active `INSERT`) and any cascading op-store effect commit
  **atomically in the same single `commit_batch` transaction** as the rest of the write batch. A reader
  never sees two active rows for one `(logical_id, kind)`, nor zero mid-supersession (the partial-unique
  index enforces this structurally). `latest_state` op collections update to the new active row in the
  same txn; `append_only_log` collections accrete (supersession is a new appended fact ‚Äî prior entries
  retained, never rewritten). **Preserve the existing op-store cascade contract**; if its current shape
  diverges from this, ADJUST minimally + flag ‚Äî do not silently deviate.
- **Forward-migration policy (SIGNED) ‚Äî in-place additive `ALTER`, no re-open, no data migration.** The
  columns are nullable; legacy (pre-0.8.0) rows read **`logical_id = NULL`**. **You own a documented
  NULL-on-legacy-rows rule** (this is the companion mechanism the sign-off requires): an active row with
  NULL `logical_id` is a legacy/own-identity row ‚Äî it must NOT collide with other NULL rows (SQLite
  treats each NULL as distinct in the partial-unique index ‚Äî this NULL-safety is **load-bearing**, test
  it), and by-id resolution (future G2) finds a legacy row by `logical_id` only once it is rewritten with
  a non-NULL `logical_id`. Do **not** backfill/derive `logical_id` for legacy rows (that would be a data
  migration, explicitly out of scope). State this rule in the design memo + a code comment.
- **`write_cursor`-as-row-id (ACCEPTED for 0.8.0).** Reuse the existing monotonic `write_cursor` as the
  per-row identity, surfaced as the new `WriteReceipt.row_cursors`. Do NOT add a `row_id` or
  `restore_provenance` column. **Still FLAG this deviation in `output.json.blockers_encountered`** (it is
  a recorded, accepted deviation vs the roadmap's `row_id`+`restore_provenance` ‚Äî sign-off ‚âÝ silent
  reconciliation), but land the accepted shape.
