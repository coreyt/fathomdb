## Why this addendum exists (the B-1 finding, in one paragraph)
The Slice-40 Phase-A halt (`recall@10 = 0.8710 < 0.90`) was **not** a corpus move and **not** a confirmed ANN
regression. GA-1 + orchestrator verification proved: (1) the corpus is gitignored and **byte-identical** across
both anchor runs (no OLD≠NEW corpus exists); (2) `eu7`'s `measure_recall` builds ground truth as an **exact-f32
VECTOR top-10** but compares it against `.search()`, which Slice 10 (`d28d204`) made **unconditional RRF-hybrid**
(vector ⊕ FTS5). So recall@10 was measuring **fusion divergence**, conflated with ANN-quantization fidelity —
which is exactly what the 0.90 floor is *defined* to measure ([[fathomdb-recall-fidelity-vs-relevance]];
`ADR-0.7.0-vector-binary-quant.md:151-158`). The 0.937→0.8710 "drop" is dominated by the **shipped, intended**
hybrid-RRF behavior, not a defect. Evidence: `dev/plans/runs/GA-1-corpus-ab-20260608T012503Z.{md,json}`.

## The ruling (HITL, ◆ B-1, Option 1)
**Correct `eu7` to measure ANN-quantization FIDELITY on the vector stage in isolation, then assert the unchanged
0.90 floor.** This **RESTORES eu7's documented semantics** — HITL ruled this a *correction, NOT a weakening* of
the eu7 assert. **Do NOT lower the floor. Do NOT weaken/remove the assert. Do NOT redefine the floor as
end-to-end hybrid fidelity** (that was Option 3, explicitly rejected).

### A. Engine — add a TEST-ONLY seam exposing the pre-fusion vector ranking
- The search pipeline already computes the vector branch separately:
  `crates/fathomdb-engine/src/lib.rs:~3950-3959` —
  `let fused = rerank_fused(apply_recency_reweight(fuse_rrf(vector_results, text_results), …))`.
  **`vector_results`** is the bit-KNN (K=192 Hamming) + f32-rerank vector ranking — the ANN+ fidelity signal —
  *before* `fuse_rrf` folds in the FTS5 (`text_results`) branch.
- Add a **test-only** way to obtain that `vector_results` ranking for a query (the vector-branch top-N, pre-fusion,
  pre-recency-reweight, pre-`rerank_fused`). Mirror the EXISTING seam conventions in this file — e.g.
  `set_search_limit_for_test` / `set_recency_reweight_enabled_for_test` / the `*_for_test` methods (`~:766-2402`);
  follow whichever pattern (a `…_for_test` method returning the vector-branch hits, and/or a flag the search path
  reads) is **least invasive and held out of release builds** the same way the existing seams are.

  output for any non-test caller, **NOT** reintroduce a `fusion_mode` knob (HITL Q3 dropped it — unconditional
  RRF stays), and **NOT** alter `fuse_rrf` / `rerank_fused` / recency behavior. `git diff` on the fusion ranking
