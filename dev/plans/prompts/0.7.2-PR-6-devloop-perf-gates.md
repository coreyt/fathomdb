### What the canonical AC tests look like today

- All three canonical AC tests (`ac_013_vector_retrieval_latency`,
  `ac_013b_recall_at_10_floor`, `ac_019_mixed_retrieval_stress_workload_tail`)
  gate behind `if !long_run_enabled() { return; }` — they are inert by
  default. **Devloop tests must NOT gate behind `AGENT_LONG`** — they are
  the inner-loop signal; they always run.
- Smoke test `ac_013_vector_read_path_smoke` (line 761) is always-on but
  fixture-independent (no corpus); it does NOT replace the devloop tests
  PR-6 owns.
- Tiered budget knobs (`AC013_DEFAULT_N`, `AC013_GATE_N`, `AC013_TIER_INFO`,
  `AC019_REPORT_ONLY`) belong to the canonical path. Devloop is a separate
  knob surface — pick names like `DEVLOOP_*` to avoid bleeding the canonical
  contract.

### What PR-5 gives you

- `tests/support/corpus_harness.rs` — `CorpusFixture::small()` (~100 docs)
  / `::medium()` (~1000) / `::full()` (~7667). The handoff for PR-6 says
  "~1K rows" — `CorpusFixture::small()` is ~100 (insufficient); use
  `::medium()` (~1000) for the devloop tier, OR construct via `::per_source`
  / `::from_docs` for a tuned size between small and medium. Surface the
  choice in the closure.
- `with_real_embedder()` (cfg `default-embedder`) — **gated behind
  `AGENT_LONG=1` in PR-5's gated smoke** because candle CPU inference on 100
  docs took >300 s in a sandbox. Devloop's ≤30 s budget means real-embedder
  is only viable with a **warm cache hit** (PR-5's
  `data/corpus-data/.cache/embeddings/`). Cold-cache devloop is allowed to
  take longer (first run prints the cache-miss reason); WARM-cache must be
  ≤30 s.
- `with_synthetic_embedder()` — always available. The dev-box default
  feature set does NOT include `default-embedder`, so the **always-runs**
  path is synthetic. Devloop should support both: synthetic by default,
  real-embedder behind the feature + warm cache.

### Budget shape (your HITL ask)

- Canonical 10k tier (per PR-3 ADR amend): AC-013 p50 ≤ 80 ms / p99 ≤ 300 ms.
- Handoff recommends devloop budget = **2–3× canonical at smaller N**.
  Concrete starting point to surface: at N=1000, p50 ≤ 50 ms / p99 ≤ 150 ms
  (loose enough that ~10× sample noise on a small N doesn't flap; tight
  enough that the batch-collapse regression — which inflated p50 by orders
  of magnitude — trips). AC-013b floor at small N is dominated by ANN
  fidelity noise — surface a proposed recall floor (0.85 is plausible at
  N=1000; the ADR's 0.90 is N=7667 ANN-fidelity).
- AC-019 devloop: report-only mirrors the canonical path (synthetic
  isotropic can't meet baseline-relative bound; real-corpus devloop with
  warm cache + small N may or may not — surface measured numbers before
  asserting).

### Regressions the devloop MUST catch (handoff Success criteria)

- **Batch-collapse bug `4a95cfd`** — the engine bug where batched
  `engine.write` with vector-indexed nodes collapsed to a single row.
  Symptom: ingest reports correct doc count but vec0 row count is way
  below. The PR-5 harness's `assert_vec0_row_count_matches_ingest`
  triggers on this directly; devloop should call it.
- **Projection-scanner throughput regression `53a270d`** — the projection
  scanner throughput fix. Symptom: AC-013 p50/p99 inflate by orders of
  magnitude at the same N.
- The closure artifact must record a CI experiment that artificially
  re-introduces each regression (a guarded test seam or a temporary code
  change in a throwaway branch) and confirms the devloop tests RED-show.
  This is the handoff's literal Success-criteria bar; do not skip.
