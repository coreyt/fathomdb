# Phase 0.6.1-planning — Patch release scoping + plan-doc authoring

0.6.0 GA shipped 2026-05-20 at `edf00f7`. Four items deferred at
GA close need a patch-release home:

1. **AC-012 canonical-runner re-measurement.** Text-query latency
   gate. No engine code change expected; dev runner aarch64 Tegra
   is ~3× slower than canonical x86_64. Re-measure under
   `AC_FULL_SCALE=1` on canonical tier-1 CI and publish numbers.
2. **`12-TX-OPENREPORT` slice.** Surface `OpenedEngine.report`
   (migration-version-reached / embedder-identity-confirmed /
   open-stage) through both Python (PyO3) and TypeScript (napi)
   bindings. Native Rust populates it; both bindings currently
   drop it. Symmetric, not a parity gap.
3. **TS SDK Python-parity close.** Phase 12-TX delivered
   milestone-1 + counters wrapper fix; the full
   `12-TX-parity-matrix.md` audit still lists open `❌ gap` rows.
   0.6.1 closes the remaining gap rows; `📋` divergence-by-design
   rows stay as documented.
4. **Axis-E independence demo.** Per decision `d-001`, axis-E
   (`fathomdb-embedder-api`) is now structurally independent.
   First post-GA axis-W bump (likely this release) must hold
   axis-E at `0.6.0` unless the trait surface changes. Discipline
   check, not a feature — verify via `scripts/set-version.sh`
   behavior + ADR-0.6.0-embedder-protocol cross-reference.

Additionally, **Dependabot 4 advisories** must be triaged before
slice work opens: `pyo3` low x2, `js-yaml` medium, `idna` medium.

0.6.1 is a **patch release per ADR-0.6.0-no-shims-policy §54**:
no API breaks, bugfix-only. Scope-creep into new features blocks
the release. Verify each candidate slice against that rule before
inclusion.
