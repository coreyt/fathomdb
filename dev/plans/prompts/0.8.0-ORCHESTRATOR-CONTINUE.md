## 2. The loop you run, per slice (this is "keep doing")

For the current pointer slice, act as its **Slice N agent** (slice orchestrator on
the main thread — never spawn an orchestrator subagent, §1):

1. **Orient + plan-adjust.** Read the slice's contract. If reality diverges from the
   plan, ADJUST the plan/board and record it in the closing docs commit (§12.4) —
   never silently deviate.
2. **Write the self-contained slice prompt** (`dev/plans/prompts/0.8.0-slice-<N>.md`)
   carrying every fact by file reference: baseline SHA, worktree path, branch, output
   path, AC ids, touch-points, the per-slice discipline, and X1/X2/X3 (§12.3). The
   subagent never sees this conversation.
3. **Create the worktree YOU own** (impl slices): `git worktree add /tmp/fdb-slice-<N>-<ts>
   -b slice-<N>-<ts> <baseline_sha>` from the current `main` HEAD (§1; never Agent-native
   isolation). Re-check `git rev-parse main` right before — siblings may have advanced it
   (trap 4).
4. **Spawn ≤1 level of subagents** (N.a, N.b) via the Agent tool, `subagent_type:
   implementer` (it IS registered — re-verify in the agent list at session start;
   fall back to `general-purpose` with Agent/Task omitted only if a spawn errors —
   trap 3). Subagents never spawn subagents (§10 anti-chain). The implementer applies
   **design-first + self-check → TDD RED→GREEN→refactor**; writes `output.json` (§8) LAST.
5. **Gate the transition** (§1.5 inv. 2): after return, confirm `dev/plans/runs/<id>-output.json`
   exists AND the branch head advanced past baseline. If absent or a blocker is
   reported → triage (fix-N or abandon), don't cherry-pick a non-state.
6. **Cherry-pick to `main`** (§5, **never merge**) — one dependent git op per Bash call,
   verify each from git before the next (trap 1).
7. **Review.** Spawn the reviewer. **Codex is unrunnable in this environment** (trap 5,
   no netns) — use an **independent adversarial review subagent** (a fresh
   `general-purpose`/`Explore` agent read-only over the diff) as the documented
   substitute; promote its verdict to `dev/plans/runs/<id>-review-<ts>.md` with a
   `## Verdict:` line.
8. **Decide (§9):** PASS → close. Structural/prompt-induced CONCERN → orchestrator
   override (§7, never override BLOCK). Substantive CONCERN / BLOCK → **fix-N** (fresh
   implementer into the *existing* worktree/branch, §6); ~1 pass per BLOCK; unclearable
   BLOCK or fix-N past a small bound → **HALT to HITL**.
9. **Close in ONE docs commit:** add the `Slice <N> CLOSED` block to the plan, advance
   the pointer, update the board (incl. X1/X2/X3 columns) + `dev/DOC-INDEX.md` + per-AC
   scoreboard — all in the same commit (§9 step 7). Commit message:
   `docs(0.8.0): close Slice <N> — <summary>; advance pointer to Slice <next>`.
10. **Worktree cleanup** after the slice family closes (§11): one `git worktree remove
    --force` per Bash call, one `git branch -D` per call; verify `git worktree list` clean.
    Never `find -delete`.
11. **Advance** to the next pointer slice and repeat from step 1 — unless a STOP
    condition below fires.

## 4. Reserved-gap discipline (the N-mod-5 mechanism)

Planned slices are mod-5 only (0,5,10,…). If a slice reveals adjacent work that must
land before the next planned slice, that becomes a **new fully-orchestrated slice** in
the reserved band (Slice N+1..N+4: e.g. shadow vec0/FTS5 reconciliation is the
pre-named **Slice 16**; G10 `status` population a **Slice 13** candidate). A reserved-gap
slice ≠ a fix-N: fix-N re-enters the same slice's worktree for its own BLOCK; a
reserved-gap slice is new scope with a fresh baseline + worktree + review + CLOSED
block. Band fills and more remains → the slice was mis-scoped → HALT to HITL.
