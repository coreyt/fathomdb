## HITL correction (2026-05-30) -- read this first

An earlier draft of this slice said "reservoir over the pre-pin vectors, pin at
the 256 threshold." **That is wrong and must not be implemented as written**: the
pin fires at the 256th embedded doc, so at that instant only the first 256 docs
exist; ANY sample of them is still the first-256 (all `bahmutov_dailylogs` for the
benchmark corpus). You cannot sample sources you have not ingested yet.

HITL directive: **the initial 256 must be a random selection from across the
corpus.** The only way a streaming engine can honor that is to **maintain a
uniform reservoir (Algorithm R, size 256, seeded) over the pre-pin stream and
DEFER the pin past 256** so the reservoir actually spans multiple sources before
it freezes. This trades a longer un-centered cold-start window (EU-0: the
un-centered path is ~-5pp recall) for a representative pin -- make that tradeoff
explicit and measure it. PR-2b's drift recompute then refines the mean further as
the rest of the corpus arrives.

### Design to implement

Keep K=192, the at-pin re-quantize, and crash-atomicity fixed. Change WHICH
vectors determine the pinned mean and WHEN the pin fires:

1. **Uniform reservoir (Algorithm R), size `MEAN_VEC_PIN_THRESHOLD` (256), seeded**
   from a fixed engine constant for determinism. Feed every pre-pin embedded
   vector through `reservoir.observe(v)` in BOTH pin sites. The reservoir holds a
   uniform random 256-sample of all pre-pin vectors seen so far.
2. **Deferred pin.** Introduce `MEAN_PIN_SETTLE_THRESHOLD` (propose 2048; justify
   and surface for HITL ratification). The pin fires when the embedded-doc count
   reaches the settle threshold (NOT at 256), pinning the mean of the reservoir's
   256 samples -- a uniform sample across the first `settle` docs (multiple
   sources). Until the pin, the write/query paths run un-centered (the existing
   pre-pin branch), exactly as the current 0..256 window does -- the window is just
   longer now. The at-pin re-quantize then back-centers all rows as today.
   - If a workspace never reaches the settle threshold (small corpus), pin the
     reservoir mean at end-of-ingest quiescence OR keep the existing 256 fallback
     -- pick one, justify it, and make small corpora behave sanely (a <=settle
     workspace is searched mostly by f32 rerank where centering barely matters,
     per embedder.md §0.3).
3. **Compose with PR-2b.** After this representative pin, PR-2b's drift detector
   continues to refine the mean as later sources arrive. Confirm the two interact
   correctly (the drift detector reads the pinned mean; the reservoir is only used
   pre-pin). Reset/curtail reservoir state after pin.
4. **Determinism.** Same corpus + seed -> identical pinned mean.
