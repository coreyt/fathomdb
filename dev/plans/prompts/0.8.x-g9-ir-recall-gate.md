# IR-Eval `g9` — Analyze `g8` outputs → recommend the IR/evidence-recall gate to HITL

> **Initiative & naming.** This is **`g9`** of the **IR-eval initiative** (post-0.8.0-GA / 0.8.1) — the gate
> that turns `g8`'s defined measure + experiment data into a **HITL recommendation** for the binding
> product-value recall gate. **`g8`/`g9` are initiative labels, NOT the shipped G8 (dangling-edge) / G9 (RRF)
> gaps.** **Do not run `g9` until `g8` is merged** (`dev/plans/prompts/0.8.x-g8-ir-recall-measure.md`): `g9`
> consumes `g8`'s consensus measure + AC structure + experiment outputs. `g9` **analyzes and recommends — it
> does NOT decide** (HITL sets the binding thresholds) and does **not** weaken the eu7/AC-075 fidelity gate.

## 0. What this gate is
A `[analysis / decision-support]` work item, adapted-loop (the analysis report is the deliverable; a
falsifiable acceptance bar replaces TDD; codex reviews the report; HITL signs the gate). It reads `g8`'s
decisions + measured data, performs the analysis `g8` deliberately did not (it was forbidden to recommend
thresholds), and produces a **HITL gate recommendation**: proposed binding thresholds for the IR/evidence-recall
AC(s), per-class tiers, gate-now-vs-track-only, and regression tolerance — each **grounded in `g8`'s measured
numbers**, not the framework's arbitrary defaults. On HITL sign-off, the AC thresholds (left `TBD` by `g8`) are
filled.

## 2. The analysis to perform (over `g8`'s measured data)
1. **Achievability vs aspiration.** For each candidate threshold (overall Evidence Recall@20/@50, the
   must-not-miss class tiers, MRR@20, nDCG, negative-query FP), compare the framework's proposed level against
   `g8`'s *measured* numbers + the embedder-bound ceiling. Flag every threshold that is **above the achievable
   ceiling with today's embedder/no-reranker/no-graph** → those are *aspirational/0.8.1+*, not bindable now.
2. **Per-class + per-mode breakdown.** Which query classes (commitment / exact-fact / semantic / negative /
   exploratory) clear which levels, in which retrieval mode (FTS / vector / RRF-hybrid / +reranker-if-built /
   +graph-future)? Identify where FTS already wins (exact strings/numbers) vs where vector/relevance is weak —
   the asymmetric-failure-cost classes (commitments/tasks/exact facts) get the most scrutiny.
3. **Gate-now vs track-only.** For each AC, recommend whether it should be a **binding release gate now**, a
   **tracked signal** (report-only, like eu8 is today), or **deferred** (blocked on the reranker/graph/a better
   embedder). The default for anything embedder-ceiling-bound is *track, don't gate* (per the assessment).
4. **Regression tolerance + corpus pinning.** Recommend a per-class regression tolerance (the framework
   suggests "no class drops > 2 points") and reaffirm the **pinned versioned eval set** (so the IR gate can't
   drift under corpus expansion the way the fidelity number did — the lesson behind the 0.8.0 GA halt).
5. **Fidelity vs relevance — keep them separate.** Explicitly state that this gate is the *product-value* axis
   and does **not** replace or relax eu7/AC-075 (fidelity / system-health). Note any interaction.
6. **Residuals from `g8`.** Resolve or escalate any Claude↔codex residual disagreements `g8` left for HITL.

## 3. Adversarial verification before recommending (codex)
Run **codex** over your analysis report (`codex exec` / `review`) hunting for: a threshold recommended above
`g8`'s measured ceiling, a class gate unsupported by the data, an arbitrary number imported from the framework
without grounding, conflation of fidelity with relevance, a silently-unpinned eval set, or a "gate now" that
should be "track". Capture to `dev/plans/runs/g9-codex-review-<ts>.md`; reconcile to PASS or list residuals.

## 4. Deliverable + the HITL recommendation
Write `dev/plans/runs/g9-ir-recall-gate-recommendation-<ts>.md`:
- A per-AC, per-class **recommended gate** (threshold | tracked | deferred), each with the **measured number
  it's grounded in** and a one-line rationale; the gate-now-vs-track call; the regression tolerance; the
  pinned-snapshot reaffirmation; the fidelity-separation statement.
- An explicit **HITL fork** (e.g. ship the gate at the recommended levels / track-only this release + revisit
  after reranker+graph / a stricter-vs-looser variant) with a recommendation. **You recommend; HITL decides.**
- On HITL sign-off: the orchestrator (or a tiny follow-up) fills the `TBD` thresholds in the `g8`-minted
  AC(s), flips the relevant ones from tracked → gated, and records the decision on the board.

## 5. Guardrails + DoD
- **Do not invent numbers beyond what `g8` measured** (the framework's tiers are arbitrary — anchor every
  recommendation in data or label it aspirational). **Do not make the GA/gate decision** (HITL does). **Do not
  weaken eu7/AC-075.** **Do not entangle with the 0.8.0-GA B-1 ruling** (separate track; though both turn on
  the same corpus-pinning hygiene). Reaffirm the pinned eval set. No fabricated green; read real exits.
- **DoD:** every recommended threshold traces to a `g8` measured number or is explicitly flagged
  aspirational/deferred; per-class + per-mode breakdown present; gate-now-vs-track ruled per AC; fidelity kept
  separate; codex-reviewed to PASS (or residuals listed); a clean HITL fork + recommendation; no binding number
  set without HITL sign-off. Adapted-loop: no feature worktree required (analysis + recommendation doc);
  follow the no-worktree adaptation. Board updated at hand-off.

> Output feeds HITL. On sign-off, the IR/evidence-recall AC(s) become real (gated or tracked per the ruling),
> closing the product-value-recall gap the recall-eval assessment identified — complementary to, and separate
> from, the eu7 fidelity gate.
