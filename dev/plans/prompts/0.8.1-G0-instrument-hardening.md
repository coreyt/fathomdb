## 1. Mandate
Make the instrument trustworthy and the signal path proven, in one slice. Two parts:

### Part A — Capability tracer (`--tracer` mode on the P0-A runner). Cost: ~$0.
Each check is one tiny operation; emit a **capability map** JSON +
human-readable summary.
- **T1 dense/fused reachable:** open an engine WITH the embedder, write 2 tiny docs,
  run 1 query, assert a dense/fused hit returns. Record the path used (production
  binding vs test-hook). *This determines PRE-1.*
- **T2 airlock readers reachable:** one tiny call each to the candidate ids
  (`claude-haiku`, `gpt-5`, plus any Sonnet/Opus/Gemini ids you'll use as
  reader/extractor), assert HTTP 200 + parseable; record the working ids + any
  429/rate-limit behavior. *This determines PRE-2.*
- **T3 data+scorer:** load LME `s_cleaned`, run the scorer on 1–2 questions with a
  stub retriever; assert gold ids parse and per-class / abstention / multi_session-
  full-set scoring runs.
- **T4 embedding throughput:** embed ~20 sessions, measure **sessions/min**; write it
  to the capability map (feeds the plan §4.3 forecast).

### Part B — Resolve the three prerequisites
- **PRE-1 vector-kind binding.** Based on T1: either (a) add a **minimal production
  Python/SDK surface to register a node vector kind** so `fathomdb_fused` works
  without test-hooks, or (b) if a production binding is out of scope here, wire a
  **sanctioned eval-only path** and record the decision + rationale in `output.json`
  as a `forced_deviation`. Prefer (a) if small. Add a test that the fused arm returns
  a hit through the chosen path.
- **PRE-2 airlock reader reliability.** In the answerer path: **exponential backoff +
  retry on 429**, **bounded concurrency** (configurable; default conservative), and
  use the **confirmed reader ids** from T2. Add a test (mock a 429 → assert retry).
- **PRE-3 extraction provenance (TDD; schema migration — see §2 schema-gate).**
  - **Harness (`elps_live_harness.py`):** capture the **resolved** model from the
    airlock/litellm API **response** (`response.model`), not the requested alias;
    attach a `provenance` object to **each result** (per-result granularity): 
    `{model_requested, model_resolved, effort, temperature, prompt_version,
    protocol, schema_version, extracted_at}`.
  - **Protocol (`extract.v1`):** extend the `result` payload to carry per-entity /
    per-edge `provenance` (and keep `ready.model` as the session default). Update the
    ADR in the same slice.
  - **Schema (SCHEMA_VERSION 15→16, step 16):** add additive
    **`extractor_provenance TEXT`** (JSON) to **BOTH** `canonical_nodes` and
    `canonical_edges`; keep `extractor_model_id` populated with the **resolved**
    model. Migration is additive/nullable; pre-16 rows read NULL.
  - **Ingest (`lib.rs`):** write per-result `extractor_provenance` to every node and
    edge row; set `extractor_model_id` = resolved model.
  - **Tests:** RED→GREEN — a round-trip asserting an ingested entity AND edge each
    carry `extractor_provenance` with `model_resolved` + `effort` + `temperature`;
    legacy rows read NULL.
