## 1. Mandate
Make the instrument trustworthy and the signal path proven, in one slice. Two parts:

### Part A — Capability tracer (`--tracer` mode on the P0-A runner). Cost: ~$0.
Each check is one tiny operation; emit a **capability map** JSON +
human-readable summary.
- **T1 dense/fused reachable:** open an engine WITH the embedder, write 2 tiny docs,
  run 1 query, assert a dense/fused hit returns. Record the path used (production
  binding vs test-hook). *This determines PRE-1.*
- **T2 airlock readers reachable:** one tiny call each to the candidate ids
  (`claude-haiku`, `gpt-5`, plus any Sonnet/Opus/Gemini ids you'll use as
  reader/extractor), assert HTTP 200 + parseable; record the working ids + any
  429/rate-limit behavior. *This determines PRE-2.*
- **T3 data+scorer:** load LME `s_cleaned`, run the scorer on 1–2 questions with a
  stub retriever; assert gold ids parse and per-class / abstention / multi_session-
  full-set scoring runs.
- **T4 embedding throughput:** embed ~20 sessions, measure **sessions/min**; write it
  to the capability map (feeds the plan §4.3 forecast).

### Part B — Resolve E0a + E0b + PRE-1/2/3 (all ELPS-touching work in one pass)

> **CROSS-REPO GUARD — keep everything FathomDB-local.** E0b is caller-side (no
> protocol change). PRE-3 rides an **additive `ready` extension** only — **do NOT add
> per-result protocol fields** (that would be a heavier `extract.v1` change needing a
> Memex re-ratification of the 5 pins). **Do NOT touch the `extract.v1` result
> schema or the Slice-25 golden.** If you believe a protocol field is truly required,
> STOP and escalate as a forced-deviation — don't do it inline.

- **E0a (= Slice 30b) graph-arm `source_id` fix (engine, Rust).** Add an **additive
  `source_id: Option<String>`** field to `SearchHit` (and `PySearchHit`), **`None`
  for vector/text hits → two-arm output stays byte-stable.** A u64 `write_cursor`
  cannot carry a session-id *string* and doc nodes have no stored key, so the **string
  field is the only viable carrier** (the prompt's earlier "source-doc cursor" was
  loose — Phase-1 design finding). **Resolution nuance (verified):** entity **nodes**
  carry `source_id = NULL` today (the ELPS entity payload is `{name,type,aliases}` —
  no `source_doc_id`); **edges DO** carry `source_id`. So resolve each reached node's
  source via an **incident `canonical_edges` row** (`from_id=lid OR to_id=lid AND
  superseded_at IS NULL`). Add BFS **frontier instrumentation** (seeds by branch,
  resolved-seed rate, non-empty-frontier rate). **Correctness fix** (the arm scores 0
  because hits never match gold session ids) — does **not** flip `use_graph_arm`
  (that default stays HITL-blocked). Test: a graph-arm hit carries a `source_id`
  string that can match a gold session id; two-arm results byte-unchanged.
- **E0b extraction robustness (caller-side; NOT the protocol).** Truncation lives in
  the runner's document builder (`_ELPS_MAX_CHARS=2000`, commit `d949a10`). Change the
  **runner/harness orchestration** to **chunk a long session into N sub-documents**
  and call the extractor N times (the protocol already takes a batch of docs); raise
  / repair `max_tokens`; **retry on truncated-JSON**. Target the G1 AC: **≥90% of each
  session's content fed, <2% truncation.** Test: a long session → multiple sub-docs,
  full content covered; a truncated-JSON response → retried, not dropped.
- **PRE-1 vector-kind binding.** Based on T1: either (a) add a **minimal production
  Python/SDK surface to register a node vector kind** so `fathomdb_fused` works
  without test-hooks, or (b) if a production binding is out of scope here, wire a
  **sanctioned eval-only path** and record the decision + rationale in `output.json`
  as a `forced_deviation`. Prefer (a) if small. Test that the fused arm returns a hit.
- **PRE-2 airlock reliability — answerer AND extractor.** ELPS has **zero resilience
  today** (no retry/backoff/concurrency cap), so the at-scale E1 run would 429 and
  bleed. Add **exponential backoff + retry on 429 + bounded concurrency** (configurable,
  conservative default) to **both** the answerer path **and** the ELPS extractor; use
  the **confirmed ids** from T2. Tests: mock a 429 on each path → assert retry.
- **PRE-3 extraction provenance (local-first; TDD; schema migration — see §2).**
  - **Harness (`elps_live_harness.py`):** capture the **resolved** model from the
    airlock/litellm API **response** (`response.model`), not the requested alias; put
    it + `effort`/`reasoning_effort`, `temperature`, `prompt_version`, protocol/schema
    version into an **additive `ready` `provenance` object** (backward-compatible;
    `ready.model` already exists).
  - **Protocol:** **`ready`-only additive extension.** **No per-result/result-payload
    provenance field** (see the cross-repo guard). Update the BYO-LLM ADR to note the
    additive `ready` provenance.
  - **Schema (SCHEMA_VERSION 15→16, step 16; draft SQL — Phase-1 finding):**
    ```sql
    -- MIGRATION-ACCRETION-EXEMPTION: PRE-3 extraction provenance (additive nullable)
    ALTER TABLE canonical_nodes ADD COLUMN extractor_provenance TEXT;
    ALTER TABLE canonical_edges ADD COLUMN extractor_provenance TEXT;
    ALTER TABLE canonical_nodes ADD COLUMN extractor_model_id TEXT;  -- edges already have it (step 14)
    ```
    Populate `extractor_provenance` (JSON) + `extractor_model_id` (resolved model) on
    both tables; additive/nullable; pre-16 rows read NULL.
  - **Ingest (`lib.rs`):** **stamp every node and edge row** from the session/`ready`
    provenance (this yields per-result *storage* granularity without a per-result
    protocol field, since one ELPS process = one config); set `extractor_model_id` =
    resolved model.
  - **Tests:** RED→GREEN — round-trip asserting an ingested entity AND edge each carry
    `extractor_provenance` with `model_resolved` + `effort` + `temperature`; legacy
    rows read NULL.
