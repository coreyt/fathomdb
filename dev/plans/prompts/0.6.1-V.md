## Tasks

1. Provision (or identify) a fresh non-CI host: ideally a machine
   that has never installed fathomdb. Capture: OS + version,
   package-manager versions (cargo, pip/uv, npm/pnpm).
2. Per binding, install from the canonical registry:
   - Python: `pip install fathomdb==0.6.1rc1` (or uv equivalent).
   - TypeScript: `npm install fathomdb@0.6.1-rc.1`.
   - Rust: `cargo add fathomdb@=0.6.1-rc.1` in a scratch crate.
3. Smoke-test the 5-verb canonical surface (Engine.open, write,
   search, close, admin.configure) per binding. Use the snippets
   from `docs/release-notes/0.6.1.md` or `docs/quickstart.md`.
4. **OpenReport-specific check** — `Engine.open(...)` returns the
   new shape; the three report fields are populated and read-able.
   This is the canonical verification that 12-TX-OPENREPORT closed
   end-to-end.
5. Capture a transcript (stdout/stderr) of each install + smoke
   under `dev/progress/0.6.1.md` or
   `dev/plans/runs/0.6.1-V-transcript.txt`.
6. If anything is RED on a fresh host, surface — RC1 may need
   an RC2 spin.

## Hard constraints

- **No CI runner.** The whole point is to exercise the published
  artifacts against a host that has never seen the source tree.
- **No source-tree dev install.** Install from registries only.
- **No release-notes edits during verification.** If gaps surface,
  log them; they land in a subsequent slice or in GA prose.
