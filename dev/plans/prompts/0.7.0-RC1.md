## Hard constraints

- **HITL approval required before tag push.** RC1 = first
  irreversible action.
- **Push `main` FIRST, then tag.** The 0.6.1 GA cycle failed its
  first release.yml run in 11 s because the tag was pushed
  before `main` (verify-release-gates references the wrong ref).
- **No tag re-use.** If RC1 partial-publishes, next attempt is
  `v0.7.0-rc.2`.
- **No retry on `verify-release-gates` failure referencing the
  ref shape** — surface immediately; that's `082321e`-class.
- **No workflow edits at tag time.** If release.yml needs a fix,
  STOP, land a WF-FIX slice, then re-tag as `rc.2`.
- **No edits to `docs/release-notes/0.7.0.md`** at tag time. If
  gaps surface, log them; they land in V slice or GA prose.
