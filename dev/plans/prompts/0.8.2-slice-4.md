## 3. Mandate (what this slice delivers — and not)

1. **`tests/corpus/scripts/acquire_musique.py`** — deterministic acquisition of **MuSiQue-Ans, distractor
   setting** (CC-BY-4.0, `StonyBrookNLP/musique`). Prefer HF `datasets` (v5.0.0 is installed); if the HF
   dataset isn't directly loadable, fall back to the official MuSiQue release zip and **document which**.
   Write raw to `data/corpus-data/raw/musique_*.jsonl` (gitignored). Acquire **both** the answerable dev
   set **and** the unanswerable contrast set (MuSiQue-Full's unanswerable items, or the dataset's
   `answerable` flag — document the source).
2. **Materialize the per-question paragraph corpus** → a stable on-disk form both Slice 5 and 10 read:
   per question `{id, question, hop_count (2|3|4), answer + aliases, answerable: bool, paragraphs: [{idx,
   title, text, is_supporting}]}`. **Hop count is parseable from the MuSiQue question id** (`2hop__…`,
   `3hop1__…`, `4hop3__…`) — use that (construction-defined, reliable); assert it matches the count of
   supporting paragraphs where applicable.
3. **Pin `musique_hash`** via the `freeze_corpus.py` hashing convention; add a `musique` entry to
   `tests/corpus/scripts/manifest.json` (or a sibling manifest) with sha256 + counts + snapshot id.
4. **Manifest artifact** → `dev/plans/runs/0.8.2-m1-corpus-manifest.json`: `musique_hash`, per-hop counts
   (2/3/4), answerable vs unanswerable counts, the materialized-corpus path, and the acquire command.

**Out of scope (do NOT build):** FTS5 index, embeddings, any retrieval/baseline arm (Slice 5); any
graph/extractor (Slice 10); any answerer call. This slice is **acquire + materialize + pin** only.
