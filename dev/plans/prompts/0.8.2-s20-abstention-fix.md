## 1. The bug (codex §9 [P2]) — `src/python/eval/m1_baseline.py:~774` + `m1_verdict.py::prior_answers_from_artifact`

The checkpoint distinguishes **abstention** (`answers[arm] = None`, key PRESENT) from **failure** (key
ABSENT). But the resume-reuse path reuses only **non-None** values → a persisted abstention is **re-called**
on every relaunch (re-spend + can flip a scored abstention into a missing failure if the endpoint blips).

## 2. Mandate

Make resume reuse **any PERSISTED cell**, abstentions included; re-call **only ABSENT** cells:

- `prior_answers_from_artifact` must return a map whose **keys are exactly the persisted (qid,arm) cells**
  (a key-present `None` = a kept abstention; an ABSENT key = a failure to re-call). Membership, not
  value-non-None, is the signal.
- The reuse logic (run_baseline / `_do`) must reuse a cell **iff `(qid,arm) in prior_answers`** (using the
  stored value as-is, including `None`=abstention), and (re)call only cells NOT in the map.
Keep the failure path: a retry-exhausted FAILURE is still left ABSENT (not persisted) → re-called + counted
against `answer_completeness`. **Out of scope:** decide()/comparator/threshold, the arms, stage 2.
