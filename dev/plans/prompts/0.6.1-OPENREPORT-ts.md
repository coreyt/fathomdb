## Hard constraints

- **TDD red-green-refactor.**
- **No native (`fathomdb-engine`) code changes.** Native shape is
  authoritative.
- **No change to `Engine.open` signature.** Engine.open(path,
  options?) keeps returning `Promise<Engine>` — exactly the 0.6.0
  shape. Existing `const engine = await Engine.open(path)`
  callers are unaffected.
- **Native field names are authoritative (camelCase per
  `dev/interfaces/typescript.md` convention).** Surface
  `schemaVersionBefore`, `schemaVersionAfter`, `migrationSteps`,
  `embedderWarmupMs`, `queryBackend`, `defaultEmbedder`. Do not
  invent paraphrases; do not omit the extended native fields.
- **Idempotent sync accessor.** `engine.openReport()` is **sync**
  (no Promise) — the data was captured at open time and lives on
  the napi engine struct. Mirrors `engine.counters()` (also sync
  in ts per parity matrix § 2). Must return identical data on
  every call.
- **Mirror py slice Shape D verbatim.** Both bindings expose an
  engine-attached `open_report` / `openReport` accessor — same
  taxonomy bucket, same idempotency contract, same six fields.
  Idiomatic differences (camelCase vs snake_case) per
  `dev/interfaces/*.md`; structural shape (accessor on Engine,
  not return-tuple) must match.
- **No new verb.** `Engine.openWithReport` is rejected by HITL
  Q-002 2026-05-24; do not reintroduce. `engine.openReport()` is
  an engine-attached accessor, NOT a new top-level verb.
