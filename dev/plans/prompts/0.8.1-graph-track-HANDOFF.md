## 6. The gated workflow + governance (how to run slices here)
- **Every slice: plan → design review → TDD → code review**, with HITL gates for
  schema changes and any cross-repo (Memex) protocol change.
- **Worktrees:** orchestrator pre-creates from `$(git rev-parse main)` + verifies;
  agent runs a STEP-0 preflight; never auto-`isolation:"worktree"`; never
  `maturin develop` from a worktree in a non-Rust phase.
- **Two metrics / two loops:** Recall@K (LLM-free, fast iteration) vs end-to-end
  answer accuracy (airlock reader + judge, gates only).
- **Tracer-first:** prove the signal path (~$0) before sized runs.
- **Reviewers:** independent design review (a fresh agent / codex) before TDD; codex
  §9 (or Opus `/code-review`) on the diff before close.
- **Cross-repo guard:** keep changes FathomDB-local; the only thing that re-ratifies
  `extract.v1` is E4 grounding-as-a-protocol-input-field (deferred, behind proof).
- **Commits on local main only on HITL request; push/PR are HITL-only.**
