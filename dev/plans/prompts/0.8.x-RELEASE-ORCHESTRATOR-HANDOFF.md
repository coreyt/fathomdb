## 0. Hard preflight checks (CURRENT — apply every session, regardless of the 0.8.3/0.8.4 body below)

These two checks are not optional. The 0.8.x working tree is **shared** across the Steward, multiple
release orchestrators, and implementer sessions; a wrong branch or a stale worktree base silently corrupts
work and loses slices.

1. **Branch check — before EVERY commit or push.** Run `git rev-parse --abbrev-ref HEAD` and confirm you
   are on the branch you intend — your **slice branch** for slice work, or `main` for a docs/closure
   commit. **Never assume `main`:** another session may have left the working tree on a feature branch you
   did not switch to. If your commit belongs on a different branch than HEAD, switch or cherry-pick it
   there explicitly (a throwaway worktree cut from the target branch is the clean way); never commit to the
   wrong branch. This is "trust git, not narration" applied to your own position in the tree.

2. **Worktree-base check — before spawning any implementer.** Every slice worktree MUST be cut from the
   **live tip of `main`**, never a stale base (`agent-worktree-stale-base-trap`: a stale-base worktree once
   lost two slices and broke the shared `.venv` binding). Do, and verify:

   ```bash
   git fetch origin && BASE=$(git rev-parse origin/main)      # or $(git rev-parse main) if known-current
   git worktree add <dir> "$BASE"
   git -C <dir> merge-base --is-ancestor "$BASE" HEAD && echo "base OK"   # MUST print base OK
   git -C <dir> rev-parse --abbrev-ref HEAD                                # confirm the slice branch
   scripts/preflight.sh --worktree                                        # MUST print: preflight: pass
   ```

   If `base OK` or `preflight: pass` does not print, **stop and fix the worktree** — do not let the
   implementer start. Forbid `maturin develop` / `pip install -e` from a worktree; the GPU/maturin build
   runs on the **MAIN tree only**.

## 5. How to steward — operating principles (the load-bearing disciplines)

- **Resolution-driven.** Gate on the **external competitor delta**; report the **internal** lever-vs-
  corrected-fused delta only for *attribution*. Order levers by expected gap-closure × cost; **stop at
  parity** (or pursue surpass on HITL choice). A lever not needed to reach parity is not run "for completeness."
- **Pre-register, then measure.** Endpoints + decision rule frozen as code before data; **power-size N** so
  per-class MDE ≤ the parity tolerance (the n=160 / 21pp-MDE trap is real). No claim on an under-powered class.
- **Cheap-validate-before-spend AND -before-engine.** A $0 LLM-free probe gates every priced run and every
  engine/schema migration (embedder probe 15a before the re-embed; D2 proxy 15b before fielded-FTS). A
  failed gate ⇒ the engine slice does not run; record the negative.
- **Footprint invariant (hard).** Library query path = CPU-only, no network/API, 1-bit binary (Hamming),
  deterministic. The **only** LLM seam is **BYO-LLM in the caller**; no LLM in the library, ever. GPU +
  priced frontier models are OFFLINE-BUILD / EVAL-ONLY. Tag every technique IN-LIBRARY / CALLER-SIDE
  BYO-LLM / OFFLINE-BUILD / EVAL-ONLY.
- **Hard BLOCK→HITL gates:** **eu7 ANN-fidelity ≥0.90** (any embedder/pooling change rewrites vectors →
  re-clear once on the chosen embedder; an unbisected 0.937→0.896 regression lurks — bisect it) and
  **latency** (AC012/AC013/AC020 re-measured on read-path slices). Never ship a floor/budget breach silently.
- **Budget discipline + resilient priced runs.** Cheap-validate (`gemini-2.5-flash-lite`) → bounded pilot →
  HITL spend gate → priced pass (`gpt-5.4`, temp 0/seed 0). Always use the resilient harness (auto-resume,
  atomic checkpoint, 429/5xx backoff, failure≠abstention, completeness validity guard). Keep a $ ledger.
- **Honesty prior.** Across 0.8.1+0.8.2 *no cheap retrieval lever has yet won*. Treat every null as a clean,
  pre-registered negative — never move the goalpost. Report ties as ties (inside the MDE), not wins.

## 6. Orchestration mechanics

- **Three roles.** HITL signs/decides; **you** gate + review + keep state; `implementer` subagents do one
  slice each in a pre-created worktree (commit on the slice branch, write `output.json`, never spawn agents).
- **codex §9 is the PRIMARY reviewer:** `codex exec review --base <sha> --dangerously-bypass-approvals-and-sandbox`
  (an adversarial subagent is fallback only). For independent *consulting* confirmation, use
  `codex exec --dangerously-bypass-approvals-and-sandbox "<prompt>"` with data+goals and **no leaning**.
- **Verify state from git, not from narration.** Cross-check every "green" claim against printed output
  (PIPESTATUS / real exit), and a collection/import error ≠ a code defect — check the build flags first.
- **Worktrees:** pre-create + verify from `$(git rev-parse main)` (the stale-base trap lost two slices once);
  give agents a fail-fast STEP-0 preflight; forbid `maturin develop` / `pip install -e` from a worktree.
  The GPU maturin build happens on the MAIN tree only.
- **Don't parallel-batch dependent git ops** (they cascade-cancel). Land docs in one closing commit per
  slice transition.
