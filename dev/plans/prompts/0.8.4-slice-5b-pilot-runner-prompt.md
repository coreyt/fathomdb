## Deliverables (TDD, RED ‚Üí GREEN; pure-Python; FAKE LLMs in tests; no native rebuild)

Create `eval/autoe_pilot_run.py` (+ `tests/test_autoe_pilot_run.py`):

1. **`LLMJudge`** ‚Äî a real `Judge` impl POSTing `/chat/completions` to a **judge** endpoint,
   from env `R2_JUDGE_BASE_URL` / `R2_JUDGE_MODEL` / `R2_JUDGE_API_KEY` (SEPARATE from the
   answerer env, because the judge must be cross-family). temp 0. Empty/None/HTTP-fail ‚Üí
   return None (the harness maps None ‚Üí ABSENT, re-judged on resume). Mirror `LLMAnswerer`'s
   urllib pattern; do NOT add an SDK dep.
2. **`run_pilot(...)`** ‚Äî the orchestration: sample AutoQ questions (a balanced spread across
   buckets, `--limit` of them), build `{vector_rag, long_context}` adapters over a passed-in
   document map, `build_arm_answers` via the shared answerer, `run_autoe` for the pair
   (default `("vector_rag","long_context")`) over `n_runs` √ó both orders, `compute_winrates`,
   assemble `BiasControls`/`LengthCorroboration`, and (for the premise read) call
   `strong_baseline_clears` on the treatment-vs-long_context win-rate. Returns a report dict.
3. **Resilience (mandatory):** atomic checkpoint after each judged key (reuse `_atomic_write_json`);
   `--resume` re-derives the same `JudgmentKey`s and skips decided ones (the harness already
   re-judges fully-ABSENT cells); wrap LLM calls in the `_is_retryable`/`_retry_after_seconds`
   backoff (honor `Retry-After`); a `BudgetLedger` tracks $ and supports a `--max-usd` guard
   that raises before exceeding it. Empty/None completion ‚Üí ABSENT, never a fabricated score.
4. **Cost projection:** from the measured per-call prompt/completion tokens in the run, call
   `project_autoe_cost` to emit the **projected powered-run USD** for a given target question
   count √ó n_runs √ó 2 orders √ó n_pairs. This is the number the HITL approves the top-up against.
5. **Cheap-validate mode (`--cheap-validate`):** a tiny-N path (e.g. 2-3 questions, 1 run)
   intended for `gemini-2.5-flash-lite`, whose only job is to (a) prove the pipeline end-to-end
   and (b) measure per-call cost. It writes the projection.
6. **CLI `main()`** gated by `R2_RUN=1` for any real call; with `R2_RUN` unset it must still
   import and run its unit tests (fakes). Args: `--limit`, `--n-runs`, `--pair`, `--cheap-validate`,
   `--checkpoint`, `--resume`, `--out`, `--max-usd`, `--target-questions` (for the projection).
