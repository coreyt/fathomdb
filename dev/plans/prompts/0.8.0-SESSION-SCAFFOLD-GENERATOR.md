> **You are a scaffolding agent.** Your job is to **write one starter scaffold file per session** in the
> GA→IR-eval campaign, so each session begins with its **prerequisites**, its **output-to-orchestrator
> contract**, and its **work to-do** already laid out. You **do not do any of the sessions' work** — you only
> generate the scaffolds. You own a worktree and merge the scaffolds to local `main` (no push).

1. `dev/plans/0.8.0-GA-and-IR-eval-roadmap.md` — the sequence + Visual map + dependencies (the authority for
   prereqs/order).
2. `dev/plans/prompts/0.8.0-MASTER-ORCHESTRATOR-HANDOFF.md` — the spawn table (session → track → prompt → gate).

   scaffold for an *unprompted* session must carry a real starter to-do):
   - `0.8.x-IR-1-phase1-measure-consensus.md` (IR-A), `0.8.x-IR-1-recall-measure.md` (IR-B/C/D),
     `0.8.x-IR-2-recall-gate.md` (IR-E), `0.8.0-slice-40.md` (GA-2/Slice-40),
     `0.8.0-ORCHESTRATOR-CONTINUE-GA-RECALL.md` (GA-track orchestration).
4. `dev/plans/runs/STATUS-0.8.0.md` §1/§7 (current state, so prereq "verify" lines are accurate).

## Output

Write to `dev/plans/prompts/scaffolds/` (create the dir). **One file per session below**, named
`<order>-<id>.md` (e.g. `1-GA-1-corpus-ab.md`). Add a `scaffolds/README.md` index. Add DOC-INDEX rows.

## The scaffold template (use verbatim for every session)

```
# <SESSION ID> — <one-line title> · track:<GA|Corpus|IR-eval> · type:<work|HITL-gate>

## Purpose (1–2 sentences)
<why this session exists; what it produces>

## Prerequisites (verify ALL before starting — do not start if any is unmet)
- [ ] <gate/merge/freeze that must be true>  — verify: `<concrete command or file check>`
- [ ] ...
(If type:HITL-gate, prereq = the upstream session's output exists; this scaffold is a DECISION PACKAGE the
 orchestrator assembles, not an agent to spawn.)

## Work to-do (the steps)
1. <step>  2. <step>  ...
(For a PROMPTED session: "Follow the authoritative prompt `<path>` — this scaffold is the launcher.
 Key reminders: <2–3 critical guardrails>.")
(For an UNPROMPTED session — GA-1, COR-2: a real, ordered checklist.)

## Output to the orchestrator (how this session reports back)
- Artifact(s): `<path>` (e.g. `dev/plans/runs/<id>-output.json` / a findings doc / the merge).
- Schema/contract: <output.json keys, or the merge convention, or the decision-package contents>.
- Hand-off line: what the orchestrator gates on + what it unblocks next (per the roadmap).
- Discipline: `--release`+isolated for any measurement; read the REAL exit/numbers
  ([[background-exit-masks-real-exit]]); no fabricated numbers (TBD where unknown); no push/tag; board is
  orchestrator-owned.

## Full prompt / next
- Authoritative prompt: `<path>` (or "THIS scaffold is the starter — expand into a full prompt before running").
- On completion → <next session / gate>.
```

## Guardrails

- Scaffolds are **starters, not the work** — do not run any A/B, experiment, or merge any session's deliverable.

  current board state.

- Stay-in-scope / stray-edit tripwire / recover-out-loud (template §0/§6/§7). Console-log `[SCAFFOLD][...]`.
