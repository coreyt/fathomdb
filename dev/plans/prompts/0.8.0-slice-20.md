## 2. Mandate (what this slice delivers — and does not)

Add an **additive, default-non-rejecting** referential check at write time so callers
learn how many edge endpoints point at non-existent **or superseded** canonical nodes,
surfaced as a new **`WriteReceipt.dangling_edge_endpoints: u64`** field with **Python +
TypeScript parity**. This is **F10/G8** for graph consumers. It hard-depends on G0
(landed in Slice 15): edge endpoints are node **`logical_id`s**, and "exists" means an
**active** node (`superseded_at IS NULL`) carries that `logical_id`.

The check is **cross-row** — bulk loaders legitimately insert an edge *before* its target
node in the **same batch**, so a same-batch later-inserted node must **NOT** be flagged.
It therefore lives as a **post-row-insert pass inside `commit_batch`'s open transaction**
(after the batch loop, before `enforce_provenance_retention`), **NOT** in `validate_write`
(single-row, pre-insert — it cannot see same-batch siblings). Default behavior is
**FLAG-AND-COUNT (commit anyway)**; an **optional strict mode** rolls back before commit.

**Probe predicate — SETTLED by construction (do not re-litigate; confirm in your design
memo and cite the migration):** count an edge endpoint as dangling when
`edge.superseded_at IS NULL` (the edge itself is active) **AND** `NOT EXISTS (SELECT 1
FROM canonical_nodes WHERE logical_id = <endpoint> AND superseded_at IS NULL)`. Probe
**`from_id` and `to_id` independently** (an edge can contribute 0, 1, or 2 to the count).
The probe is **`logical_id`-alone** (NOT `(logical_id, kind)`): `canonical_edges` stores
only the *edge's own* `kind`, **not the endpoint node's kind**, so the "edges-carry-kind"
variant the contract mentions is ruled out by the landed schema. A `logical_id = ? AND
superseded_at IS NULL` probe **hits the step-12 partial index
`canonical_nodes_logical_active_idx ON canonical_nodes(logical_id, kind) WHERE
superseded_at IS NULL`** via its leading column — verify with EXPLAIN QUERY PLAN (no SCAN).

**Document this consequence in the design memo (it is intended, per "G8 rides on G0"):**
only nodes written with a **non-NULL `logical_id`** are valid (non-dangling) endpoints. A
legacy / own-identity node (`logical_id = NULL`, the byte-identical 0.7.x path) is not
matchable by `logical_id`, so an edge pointing at one counts as dangling. That is correct:
G8 validates the **logical_id-keyed graph**; the count is **informational** (flag-and-count
default, non-rejecting), and **no current consumer sets `logical_id`**, so this changes no
existing behavior — it only adds a signal.

**Deferred / out of scope (do NOT build here):**
- **Shadow vec0/FTS5 reconciliation** (reserved Slice 16) — if the probe surfaces that a
  superseded node still has live projection shadows, that is **not** yours to fix; flag it.
- **Read verbs / `read.*` surface** (Slice 30, gated by Slice 25) — G8 adds no SDK verb.
- **A filter DSL / query grammar** (Slice 35).
- **`restore_validated_edges` / any recovery verb on the SDK** — stays CLI-only; the
  recovery-unreachability suites must stay **byte-unchanged and green**.
- Do not change the meaning of `from_id`/`to_id` for existing rows, and do not add a hard
  reject by default.
