## 0. What this work item is (and is not)

**Is:** define *how FathomDB measures IR / agentic-relevance recall* — the **product-value** axis ("when the
agent needs a memory to act, is the required evidence retrieved"), distinct from the ANN-fidelity axis
(eu7/AC-075) already gated. Reach a **documented Claude↔codex consensus on the measure** *before* any
experiment. Then mint the new AC(s) (measure + methodology; thresholds left for `g9`/HITL), build the eval
infrastructure, run the experiments, and emit clean outputs for `g9`.

**Is NOT:** changing or weakening the eu7/AC-075 fidelity gate; setting the binding thresholds (that is
`g9`+HITL); shipping a release gate (this produces a *tracked signal* + the AC scaffolding); 0.8.0-GA work
(GA is gated on B-1, separately — do not entangle). No engine retrieval-behavior change beyond landing the
**rerank seam** if the consensus deems it in-scope (the current `rerank_fused` is an identity stub).

## 2. Phase 1 — DEFINE the measure, then reach Claude↔codex CONSENSUS (gate; before any experiment)

This is the load-bearing step. **No experiment runs until the measure is consensus-signed.**

1. **Draft the measure** in `dev/design/ir-recall-measure.md` (under `$WT`). It must answer, grounded in
   FathomDB's actual architecture (no chunking → fact-vs-document recall framing; reranker is a stub; graph is
   0.8.1; eu8 exists): (a) **what is measured** — the primary measure (candidate: *Evidence Recall@K* = the
   retrieved context contains all evidence required to act) and its relationship to plain retrieval Recall@K,
   MRR/nDCG, and to the fidelity gate; (b) **the unit of relevance** — document vs **atomic-fact** label
   (`fact_id`-style), and how the gold set encodes "all evidence required"; (c) **the K-ladder** to report
   (@5/@20/@50/…); (d) **per-class structure** (commitment / action / exact-fact / preference / exploratory /
   negative-"not-found") — the *structure*, not the numbers; (e) **which retrieval modes** are compared
   (FTS / vector / RRF-hybrid / +reranker / +graph — flagging which exist today vs are 0.8.1); (f) **the eval
   set composition + qrels/pooling methodology** + the **pinned, versioned corpus snapshot** it runs against
   (so the gate can't drift the way the fidelity number did under corpus expansion); (g) **explicitly: how
   this stays separate from and complementary to the eu7 fidelity gate**, and how eu8 is promoted/extended.
2. **Run the consensus loop (orchestrator-style, documented):**
   - Claude (you) write the draft + an explicit rationale.
   - Run **codex** as the independent reviewer over the draft:
     `codex exec review --base <pre-draft sha> --dangerously-bypass-approvals-and-sandbox` (or
     `codex exec` with the draft as input if `review --base` doesn't fit a docs-only diff). Capture its critique
     to `dev/plans/runs/g8-measure-codex-consult-<ts>.md`.
   - **Reconcile to consensus:** address every codex point; iterate (re-consult) until Claude and codex
     **converge** on the measure definition (or the residual disagreements are explicitly listed for HITL).
     Record the converged decision + any HITL-escalated residuals in `ir-recall-measure.md` § "Consensus".
   - **Bar:** the measure is *settle-able now* (definition + methodology + structure), even though the
     *thresholds* are TBD. If consensus genuinely cannot be reached on a definitional point, STOP and escalate
     that point to HITL — do not paper over it.
3. `echo "[g8][CONSENSUS] measure signed: <one line>; residuals-to-HITL: <none|list>"`.

## 4. Phase 3 — build the eval infrastructure (TDD where it's code)
Per the assessment's §5, build the minimum to *measure* (not to gate):
- **Promote/extend `eu8_ir_validation.rs`** into the tracked IR-recall signal: add the **K-ladder** (@5/@20/@50);
  keep it report-only (no assert — gating is `g9`/HITL). Pin its corpus + qrels to the versioned snapshot.
- **A fact/atomic-fact gold set** over a *pinned subset* (the single highest-leverage asset; the assessment
  calls it the prerequisite for evidence recall — prioritize the long-doc cases where fact-burial is worst:
  enron/qmsum/cnn). Document the labeling protocol; keep it small but real (no synthetic-only).
- **A retrieval-mode pooling harness** that runs FTS-only / vector-only / RRF-hybrid as separate modes and
  pools+labels the union (per the assessment — primitives exist; pooling does not).
- **The rerank seam** (`rerank_fused`, `lib.rs:3658` is an identity stub) — land it *for real* only if the
  consensus put "+reranker" in `g8` scope; otherwise flag it as a `g9`/0.8.1 dependency and measure without it.
- Graded-relevance qrels for full nDCG only if cheap; else note binary-relevance limitation.
