## Who does what (roles)
- **HITL** — spawns each **work session** (pastes its work prompt into a fresh agent) and makes the **◆ gate
  decisions** (B-1, GA sign-off, IR-gate). You never spawn work agents yourself; you *prepare* the gate
  decision packages for HITL.
- **Corpus owner** — drives the Corpus track (out-of-band); you consume its frozen snapshot, you don't run it.
- **You (orchestrator, main thread)** — between sessions: **gate from git** (verify merges/closes), run **codex
  §9** reviews, update the **board** in one docs commit per transition, prepare each ◆ gate package, and tell
  HITL which session is unblocked next. **Orchestrate, don't do the work** (the A/B, eu7, experiments are
  delegated/`perf-canonical`, never main-thread).
- **Work agents** — own a worktree, do the work, merge to local `main` themselves (no push), report via
  `output.json` + console `[SUMMARY]`.

## The three tracks (purpose)
- **GA track** — finish 0.8.0: rule the recall corpus-basis (B-1), land the Slice-40 gate restructure, GA
  sign-off. *Serial critical path to the release.*
- **Corpus track** — owner-managed: finish Version B (~7.6K→~10K) → refine → test → **freeze a versioned
  SHA-256 snapshot**. Parallel; **does not gate GA** (GA pins a snapshot at B-1); **gates IR-1's code phase**.
- **IR-eval track** — the **product-value** recall gate FathomDB lacks (fidelity ≠ relevance;
  [[fathomdb-recall-fidelity-vs-relevance]]). Order **MEASURE→CODE→EXPERIMENTS→AC**. Post-GA / 0.8.1.

## Per-gate decision packages you (orchestrator) prepare for HITL
- **◆ B-1** — from GA-1: the old-vs-new recall numbers + the harder-corpus-vs-regression verdict; the
  three options (pin-old / adopt-expanded → engine work / pin-snapshot) + recommendation (pin-snapshot,
  per the assessment); **never lower the floor**.
- **◆ GA sign-off** — the all-green per-AC scoreboard (incl. AC-075/076), the 3 documented behavior-compat
  events, the real-embedder recall verdict on the *ruled* corpus basis.
- **◆ IR-gate** — IR-2's per-class/per-mode recommendation; achievability vs the 0.571 ceiling; gate-now-vs-track.

## Standing rules (carry from the loop + traps)
1. Verify every merge/close **from git** before narrating ([[orchestration-execution-traps]]). 2. **codex is the
primary §9 reviewer** (`codex exec review --base <sha> --dangerously-bypass-approvals-and-sandbox`). 3. **Read
the real exit/numbers** of long gates, not a wrapper echo ([[background-exit-masks-real-exit]]). 4. Board updates
in one docs commit per transition; the slice agent never edits the board. 5. **Never** push/tag, lower the
recall floor, weaken the eu7 assert, or merge a B-1-pending Slice-40. 6. The eu7 **fidelity** gate and the
IR-eval **relevance** gate are different axes — keep them separate. 7. HITL gates are natural
session/compaction boundaries.

## STOP → HITL
- Before each ◆ gate (B-1, GA sign-off, IR-gate): present the package + recommendation, wait.
- Recall-floor breach handling: do not ship below 0.90 fidelity on the *ruled* basis without a HITL ruling.
- Any unclearable codex BLOCK, a corpus that won't freeze, or an IR threshold proposed above the measured
  ceiling → HALT, capture on the board, escalate.
