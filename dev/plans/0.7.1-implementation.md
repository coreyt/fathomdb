(`dev/plans/runs/STATUS-embedder-undefer.md`), the master handoff

## Outcome

The default embedder is no longer deferred. fathomdb ships a real, opt-in,
in-process default embedder — `BAAI/bge-small-en-v1.5` (384d) via
`candle-transformers` — feeding the 0.7.0 sign-bit + f32-rerank pipeline with
mean-centering at K=192. The three-release deferral that began in 0.6.0 is over,
**with two release-hardening items explicitly carried to 0.7.2** (recall-floor
re-derivation + canonical N=1M validation, and the `v0.7.1` tag/push).

## Known limitations (carried forward)

- **Recall floor not met on real data / not re-derived.** Dev-box recall@10
  ~0.83 (EU-7; `dev/plans/runs/0.7.1-EU-7-output.json`) is below the synthetic
  0.90 gate. The synthetic AC-013b fixture still passes; the real floor is
  re-derived (anchor `R - 2σ` ≈ 0.79) and canonical N=1M validation run in
  0.7.2 PR-2/PR-3. No real-corpus recall claim is made for 0.7.1.
- **Topic-drift mean.** The corpus mean is pinned on the first 256 ingested
  docs and never silently recomputed; unrepresentative first-256 sets
  under-center. Reindex is the remedy (deferred to a later campaign).
- **Custom Python/TS embedders** deferred to 0.8.x (binding surface is binary
  default-on/off in 0.7.1).
- **Migration.** `fathomdb-noop`-pinned pre-0.7.1 workspaces fail closed on
  re-open with the default embedder; wipe-and-rewrite, no in-place swap.
