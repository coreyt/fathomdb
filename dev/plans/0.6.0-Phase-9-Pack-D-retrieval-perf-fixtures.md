## Scope

### AC-012 — 1M text-query latency on FTS5 path

- Build a 1M-row corpus seeder (per `dev/acceptance.md` AC-012 fixture
  spec). Lives under `tests/support/` or as a `#[cfg(test)]` helper in
  the engine crate.
- Corpus shape: documented in AC-012 — synthetic Markov-ish text or the
  documented seeded RNG, NOT random per-run text (must be reproducible
  for measurement).
- Measurement protocol: per AC-012's documented latency budget +
  percentile (P50 / P99). Use the same `Instant::now()` / wall-clock
  pattern as Pack 4's read-mix harness.
- Long-run gated. Add to `scripts/check.sh AGENT_LONG=1` and document
  in `dev/test-plan.md` alongside AC-020 / AC-021 / AC-059b.
- Remove `#[ignore = "protocol-incomplete: ..."]` on
  `ac_012_text_query_latency_on_fts5_path`. If GREEN, leave green; if
  RED, the budget needs an ADR before Pack D can close.

### AC-013 — vector-retrieval latency

- Seed N vec0 rows with a deterministic embedder (re-use
  `DeterministicEmbedder` pattern from `tests/excise_source.rs`).
- Measurement protocol: per AC-013 budget. Same percentile + wall-clock
  pattern as AC-012.
- Long-run gated.
- Remove `#[ignore]` on `ac_013_vector_retrieval_latency`.

### AC-019 — mixed-retrieval stress workload tail

- Workload: documented mix of FTS5 + vector + canonical reads under N
  concurrent reader threads. Per AC-019 spec, exercises a non-synthetic
  second branch (current ignore reason).
- Tail metric: P99 (or whatever AC-019 documents). Capture via
  `Histogram::record` style; do NOT compute from a `Vec<Duration>` of
  unbounded size.
- Long-run gated.
- Remove `#[ignore]` on `ac_019_mixed_retrieval_stress_workload_tail`.

### Out of scope

- AC-020. Stays RED in CI per the deferral.
- AC-014 / AC-015 / AC-016 / AC-017 / AC-018. Already green or

  fix in a separate pack, NOT Pack D.
- `ac_a3_*` / `g3_5_*` diagnostics. Owned by perf packs.
- New perf-gate ACs. Pack D closes existing gates only.
