### Gate-2 — oracle-routing upper bound (Slice 5) · $0 EVAL-ONLY

- **Hypothesis.** There exists a measurable best-plan-per-query ceiling that bounds the maximum
  value dynamic routing can buy, reconcilable with FathomDB's existing oracle bounds
  (recall@K_deep gold-in-pool; per-feature CE; measured oracle **+0.39 over Mem0**, 0.8.3 ledger).
- **Method.** Exhaustively run candidate plans per query, pick the best; couple to the
  **asymmetric mis-route cost matrix** (EXP-Fr-acc) rather than a single Recall@K margin; feed
  **measured per-arm cost tiers** (from EXP-A/B′) so the ceiling is cost-aware.
- **KILL.** If the oracle ceiling is within the noise band of the static fused-RRF baseline for
  every intent class (no headroom any router could capture) → register "routing buys ≈0; do not
  build a dynamic router" and short-circuit the downstream value argument.
- **Cost ceiling.** $0 (local exhaustive plan runs).
- **Script.** `src/python/eval/gate2_oracle_run.py`. Output: per-class oracle ceiling + per-arm
  cost tiers → `runs/gate2-oracle-output.json` + ledger row.

### EXP-B′ — 3-stage joint tuning (Slice 15, KEYSTONE) · $0 + $ judge (≤ $6)

- **Hypothesis.** The optimal `(candidate_k × pool_n × α × final_K)` config **differs per intent
  class**, and a single global config is dominated. The crux (PSD §II.C, 0.8.3 ledger): **α=1.0 @
  pool_n=50 *drops* r@10 0.548→0.498** (CE-confident distractors displace base-favored gold),
  while α=1.0 @ pool_n=10 gives MRR 0.587 / r@10 0.540 — i.e. it is a **constrained joint
  optimization, not additive**. Knobs: `alpha`∈[0,1] and `pool_n` are caller-exposed
  (`engine.py`, `lib.rs`; EXP-0, landed 2026-06-25); `candidate_k`/`final_K` are internal sweep
  parameters.
- **Output (the keystone artifact).** A **per-intent config tuple** (format §3) for each of the
  5 classes + the **EXP-B′.5 joint-regression guard** result: a config chosen for feature X must
  not regress feature Y (the §II.B coupling hazard) — report a forbidden-composition matrix.
- **KILL.** If the per-intent optima collapse to one global config (no class diverges beyond
  noise) → register "stacks do not diverge per intent"; this weakens the EXP-Fr routing-value
  case (the 0.8.15 dispatcher gate notes this dependency) but the L2 prototype still ships pinned
  to the global tuple (DP-A hedge).
- **Priced judge.** The relevance/quality judge for the tuning objective is the only priced arm
  (LLM-as-judge over routed samples). **Cheap-validate first** (gemini-flash-lite field check),
  then run on the resilient harness. **≤ $6.**
- **Script.** `src/python/eval/expb_joint_tune_run.py`. Output: `runs/expb-joint-tune-output.json`
  (per-intent tuples + B′.5 guard) + ledger row.

### EXP-Fr-acc base — classifier accuracy + mis-route cost (Slice 20) · $0 + small $ (≤ $3)

- **Hypothesis.** A 5-class intent classifier reaches usable accuracy over `{needle | multi_session
  | temporal | global | multi_hop}`, and the **mis-route cost is asymmetric** — routing a needle to
  `C` (map-reduce/QFS) summarizes the needle away: measured **−0.362 + an LLM call** (PSD §II.D /
  §III.D).
- **Method.** Label a held-out set across the 5 classes (intent label preference order: agent
  label → provider-callback → internal classifier fallback, PSD §II.A); measure per-class accuracy
  with CI; build the **per (intent, route) mis-route cost matrix** (confirm needle→C −0.362).
- **KILL.** If classifier accuracy is at chance for ≥2 classes → register the accuracy floor; the
  prototype leans on the agent-passed label (preference #1) and treats the internal classifier as
  a low-confidence fallback only.
- **Cost ceiling.** ≤ **$3** (LLM calls for the mis-route map-reduce arm + any classifier labels;
  cheap-validate first).
- **Script.** `src/python/eval/fracc_classifier_run.py` (base mode). Output:
  `runs/fracc-base-output.json` + ledger row.

### EXP-Fr-acc / VoI finalize (Slice 25) · $0 + small $ (≤ $3)

- **Extends the Slice-20 base** with PSD §III.C's three additions:
  - **(a) value-of-signal** — does an agent relevance signal beat `ce_score`-alone on routed
    queries? (CI-bounded lift.)
  - **(b) ask-or-not VoI policy** — numeric `(ce_score, route-margin)` break-even points where the
    expected mis-route cost saved exceeds the round-trip cost.
  - **(c) asymmetric weighting** — does the policy preferentially suppress the high-cost cross-wire
    (needle→global/`C`) over cheap same-tier misses?
- **Inputs.** EXP-OBS telemetry (I-1, satisfied @0.8.8: `Explanation`/`QueryTrace`/`PerHitExplain`
  on main); the Slice-20 mis-route matrix.
- **KILL.** If no `(ce_score, route-margin)` region shows a positive VoI → register "ask-or-not
  buys nothing; route on internal `ce_score` only"; the prototype omits the escalation policy.
- **Cost ceiling.** ≤ **$3**.
- **Script.** `src/python/eval/fracc_classifier_run.py` (voi mode). Output:
  `runs/fracc-voi-output.json` + STATUS row + sequencing §6 finding.

### EXP-AF — agent-feedback value test (Slice 30, KILL/GO → HITL) · $0 + small $ (≤ $5)

- **Hypothesis.** An agent relevance signal beats `ce_score`-only **net of round-trip cost** on the
  existing substrate (no fresh 50–100-Q rebuild), within the 1–2 re-plan depth bound.
- **Method.** Using the `record_feedback` telemetry pipeline (OPP-9, 0.8.8 Slice 15): one-shot vs
  iterative; measure lift vs `ce_score`-only + realized VoI break-even + mis-route reduction.
- **KILL/GO (HITL #3).** **GO** = agent signal beats `ce_score` net of round-trip → the L2

  (§5). **KILL** = it does not → the prototype drops the agent-signal loop (router stays on internal
  `ce_score`) **and** `record_feedback` stays instrumentation (overrides any F-8b promote).
- **Cost ceiling.** ≤ **$5** (agent round-trip eval; resilient harness; cheap-validate first).
- **Script.** `src/python/eval/expaf_value_run.py`. Output: `runs/expaf-value-output.json` +
  ledger row + one-shot-vs-iterative + depth-bound (1 vs 2) decision.

## 3. Per-intent config tuple — output format (EXP-B′ → L2 prototype → 0.8.15 pre-stage)

The keystone artifact (DP-B). One tuple per intent class, machine-readable, committed to
`runs/expb-joint-tune-output.json` and carried by the L2 prototype's registry:

```json
{
  "intent": "needle",
  "stack": ["fts_bm25", "vector_ann", "rrf", "ce_rerank"],
  "index": "vector_default",
  "retrieval": { "candidate_k": 200, "final_K": 10 },
  "alpha": 0.3,
  "pool_n": 10,
  "mmr": { "enabled": false, "lambda": null },
  "recency": { "enabled": false, "half_life_days": null },
  "forbidden_ops": ["map_reduce_qfs", "community_summary"],
  "source_exp": "EXP-B'",
  "ci": { "metric": "r@10", "point": 0.540, "lo": 0.51, "hi": 0.57 },
  "provisional": false
}
```

- `forbidden_ops` encodes the §II.B router-isolation rule (map-reduce/QFS + community summary are
  **valid only for `global`**, forbidden on `needle`/`multi_session`/factoid) — this is the
  EXP-B′.5 forbidden-composition output the 0.8.15 validator consumes.
- `provisional: true` marks tuples pinned from **EXP-0** (α=0.3/pool_n=10 prod, α=1.0/pool_n=10
  opt-in) pending EXP-B′ confirmation; `false` once EXP-B′ measures the class.
- Classes with no per-class divergence (EXP-B′ KILL) carry the **global** tuple, flagged `source_exp: "EXP-0-global"`.

## 4. Agent-side L2 router prototype — API contract (Slice 35, CALLER-SIDE)

**Footprint: CALLER-SIDE-BYO-LLM. Zero changes to `fathomdb-engine` / `fathomdb-py` / `fathomdb-ts`**
(R-L2-4; codex §9 verifies). The prototype lives **outside the library** (e.g. `eval/router_prototype/`
or `dev/prototypes/l2-router/`), is the **DP-A hedge** (a working agent-side router ships regardless
of EXP-S's 0.8.12 locus verdict), and **recommends without executing**.

```python
# recommend(query, *, agent_hint=None) -> Recommendation
@dataclass(frozen=True)
class Recommendation:
    intent: str            # one of the 5 classes (from agent_hint if given, else classifier)
    stack: list[str]       # operator chain from the intent's EXP-B' tuple
    config: dict           # the (index, retrieval, alpha, pool_n, mmr, recency) tuple (§3)
    confidence: float      # classifier confidence (1.0 if agent_hint overrides)
    cost_tier: str         # "low" | "medium" | "high" from Gate-2 per-arm cost tiers
    rationale: str         # human-readable: why this stack, runner-up, route margin
    feedback_arm: bool     # True only if EXP-AF GO (else the agent-signal loop is omitted)
```

- **R-L2-1**: smoke test routes all 5 classes; **R-L2-2**: each class carries a registered tuple
  (the registry is the 0.8.15 pre-stage artifact); **R-L2-3**: `agent_hint` override is respected
  without fallback-to-internal-route; **R-L2-4**: no engine changes.
- The per-intent tuple **registry** is committed as the dispatcher pre-stage artifact (DP-B):
  the 0.8.15 EXP-Fr **integrates** these rather than inventing them.

## 6. Filter-grammar unification contract (Slice 40, Track G) — pointer

Full contract → **`dev/adr/ADR-0.8.11-filter-grammar-unification.md`** (Slice 0 deliverable).
Decision: **Option A — one unified typed `Filter`, two compilation backends** (json-extract over
allowlisted body paths for G4 `read.list`; indexed vec0-metadata pre-KNN for G10 `search_filtered`).
Hard requirements (anti-laziness mandate): (1) field-availability matrix; (2) **total**
backend-dispatch with **typed rejection** (no silent fallback / no demotion of an indexed-metadata
predicate to slow post-KNN json_extract); (3) confirm the 4 G10 fields resolve in both stores;
(4) `search_filtered` SDK shape stays accepted (sugar lowering to the unified type), `filter=None →
byte-identical 0.7.2 SQL` pin holds; (5) RED→GREEN G10 parity + Py↔TS X1. Reserved-gap fallback is
legitimate **only** if a field genuinely cannot resolve in a required store (named) — never to skip
dispatch work.
