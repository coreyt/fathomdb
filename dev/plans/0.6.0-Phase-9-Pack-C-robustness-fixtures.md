## Scope

### AC-006 — corruption injection harness

- Add a documented page-corruption tool (per AC-006 fixture spec). Likely
  shape: `corrupt_page(path: &Path, page_index: u32, byte_offset: u16,
xor_mask: u8)` that opens the file directly (NOT via SQLite), seeks to
  `page_index * page_size + byte_offset`, XORs the byte, fsyncs. Tool
  lives under `src/rust/crates/fathomdb-engine/tests/support/` or a new
  `fathomdb-engine/src/test_support/corruption.rs` (`#[cfg(any(test,
feature = "test-support"))]`).
- Two flavors:
  - **Page-1 header bit-flip** → `SQLITE_NOTADB` on reopen.
  - **Interior-page row bit-flip** → `SQLITE_CORRUPT` on the next
    statement that touches the page (force via `PRAGMA integrity_check`
    or a known-row read).
- Test path: open → write canonical row(s) → close → corrupt → reopen
  expecting `EngineOpenError::Corruption` + capture
  `(SqliteInternal, Corruption, code)` event via test subscriber.
- Remove `#[ignore]` on `ac_006_*` behavioral tests in
  `tests/lifecycle_observability.rs`. Keep the existing shape-only test
  intact (handoff: do not delete the compile-time shape lock).

### AC-009 — one-thread-poison fixture

- Stress runner: spawn N reader threads + 1 writer thread; one specific
  thread (the "poison thread") performs an op variant chosen to fail
  deterministically (e.g., write with a body that violates a known
  constraint, or a search with an oversized term). Other threads make
  forward progress.
- The failure event MUST populate ALL four fields: `thread_group_id`,
  `op_kind`, `last_error_chain`, `projection_state`. `last_error_chain`
  must include the underlying `EngineError::stable_code` plus any
  causal-chain string segments documented in `dev/design/lifecycle.md`.
- Add behavioral test `ac_009_poison_thread_emits_full_stress_failure_context`
  in `tests/lifecycle_observability.rs`. Existing pure-type test
  `ac_009_stress_failure_context_constructs` stays intact.
