# 0.7.0 Implementation Plan

This is the **single source of truth** for 0.7.0 sequencing and
execution posture. 0.7.0 is a **perf-only single-thrust minor
release** (HITL 2026-05-24). One goal: bring the AC-012 / AC-013 /
AC-019 / AC-020 canonical-runner numbers into a defensible,
empirically-pinned budget by (a) revising the budgets against real
measurements and (b) tuning SQLite + FathomDB engine-side levers
to meet them.

0.6.1 GA closed 2026-05-25 at `2ef1009`
(`dev/plans/0.6.1-implementation.md` § Slice sequence; final tag
`v0.6.1`). 0.7.0 work begins at the descendant of that SHA.

0.7.x release-line goal (locked HITL 2026-05-24): **improve
performance.** Knowledge-store / retrieval features for Memex are
the 0.8.0 anchor (see `dev/roadmap/0.8.0.md`); the canonical-
identity substrate + logical-id verbs (originally a 0.7.x
candidate) moved to 0.8.0 with the Memex anchor.

## Scope (locked, do not re-scope)

1. **Revise the budgets empirically.** Land
   `ADR-0.7.0-text-query-latency-gates-revised` and (if the
   measurement evidence justifies it) parallel successor ADRs for
   AC-013 / AC-019 / AC-020. Budgets pin against canonical-runner
   measurements at 1 M corpus on tier-1 x86_64 (`ubuntu-latest`,
   per `dev/adr/ADR-0.6.0-tier1-ci-platforms.md`), **not**
   theoretical "warm-cache" reasoning.
2. **Tune SQLite + FathomDB engine-side levers** to bring the
   measured numbers down to the revised budgets. Lever menu:
   - top-K `LIMIT` cap on the search verb (caller-spec change;
     bounded result-set traversal vs full materialization).
   - `PRAGMA page_cache_size`, `mmap_size`, `temp_store=MEMORY`.
   - Tokenizer / FTS5 config sweep.
   - `SQLITE_CONFIG_PCACHE2` for AC-020 (ADR-class; see
     `ADR-0.7.0-ac020-architectural-lever`).
   - WAL2 / reader-writer pool split (ADR-class).
   - Vendor-SQLite swap (libSQL / sqlite3mc; last-resort;
     ADR-class).

The **reverted-experiments ledger**
(`dev/notes/performance-whitepaper-notes.md` § 5) is the
do-not-retry list. No 0.7.0 slice may propose any lever on that
ledger without an explicit HITL override.

## Out of scope (locked, see `dev/roadmap/0.8.0.md`)

- Canonical-identity substrate (`row_id` / `logical_id` /
  `superseded_at` columns; writer supersession step; partial unique
  index on `(logical_id, kind)`).
- Logical-id verbs (`purge_logical_id`, `restore_logical_id`) and
  their CLI / SDK surfaces.
- Knowledge-store / retrieval features for Memex (entity tables,
  relation tables, top-K rank fusion).
- Forward-migration tooling unless the perf work forces a schema
  change (it should not).
- SR-005 / SR-011 security hardening from `dev/security-review.md`.
- Trait-surface changes to `fathomdb-embedder-api` (axis-E moves
  only on trait-surface change per Wake `d-001`; perf-only does
  not move it).

If during the perf work a 0.6.0-class reliability issue surfaces,
the orchestrator hot-fixes it into 0.6.x rather than carrying it
into 0.7.0. Perf-only stays perf-only.

### Ordering rules

- **DIAG before any structural lever.** Per
  `dev/notes/performance-whitepaper-notes.md` § 7.1, the
  measurement-first diagnostic pack runs before any engine
  change. PRAGMA + AC020 + AC013-AC019 all block on DIAG output.
- **TOPK-LIMIT and PRAGMA may parallelise.** Separate surfaces
  (caller-spec change vs engine PRAGMA wiring); both touch the
  search verb path but neither touches the other's contract.
- **AC-020 architectural lever is sequential.** Do not stack with
  PRAGMA or TOPK-LIMIT. The chosen lever (PCACHE2 / WAL2 / R-W
  split / vendor swap per Q2) lands alone for clean attribution.
  A second lever fires only if HITL re-opens
  `ADR-0.7.0-ac020-architectural-lever` after the first
  underdelivers.
- **Revised-budgets ADR lock is a prerequisite** for any
  perf-gate RED-test author-step. PERF-PRAGMA / PERF-AC020 /
  PERF-AC013-AC019 / PERF-TOPK-LIMIT cannot author their RED
  tests until the ADR reads `status: locked`.
- **WF-FIX lands before RC1.** The 0.6.1 GA cycle raced PyPI
  propagation; the retry helper must be in `main` before the
  0.7.0 RC1 tag fires.
- **BUMP → RC1 → V → GA strictly serial.** Same posture as 0.6.0
  / 0.6.1.
- **No Pack 5 / 6 / 6.G ledger lever may be re-proposed** without
  an explicit HITL override
  (`dev/notes/performance-whitepaper-notes.md` § 5).

## Inherited 0.6.1 close-out lessons

- **Tag-vs-main push order.** Push `main` BEFORE the tag. The
  0.6.1 GA cycle failed its first release.yml run in 11 s because
  the tag was pushed before `main` (verify-release-gates referenced
  the wrong ref). RC1 + GA slice prompts encode this explicitly.
- **PyPI post-publish-smoke race.** PyPI index propagation can
  lag the publish step by 5-15 minutes
  (`dev/plans/0.6.1-implementation.md` § "Pre-RC1 checkpoint"
  L367-371). WF-FIX-PYPI-SMOKE lands the retry helper before RC1
  so the GA cycle does not race propagation.
- **In-slice rc-bump / GA-bump before tagging.**
  `verify-release-gates.sh:74` rejects any tag whose bare version
  ≠ `[workspace.package].version`. RC1 lands the
  `0.7.0` → `0.7.0-rc.1` bump in-slice before the tag push; GA
  lands the `0.7.0-rc.1` → `0.7.0` bump in-slice. The procedure
  is captured in `dev/design/release.md § Pre-tag procedure`.
- **Single annotated tag at the bump SHA.** Do not chain bumps;
  the tag points at the bump commit's SHA.
- **Prettier-fixup + markdownlint before every prompt-authoring
  commit** (per 0.6.0 GA prettier-rebound lesson, inherited in
  the 0.6.1 plan).

Per `dev/design/orchestration.md § 5`: each slice implements on

## TDD discipline

Every behavior slice (PERF-DIAG, PERF-TOPK-LIMIT, PERF-PRAGMA,
PERF-AC020, PERF-AC013-AC019) follows red-green-refactor per
`feedback_tdd`:

1. **RED:** add the perf-gate test (or behavioral test for
   caller-spec changes) asserting the revised budget. Confirm
   RED at the current baseline on the canonical runner.
2. **GREEN:** implement the engine-side lever; confirm the test
   flips to GREEN on the canonical runner.
3. **REFACTOR:** rename, factor, delete dead code; keep GREEN.
   Net-negative-LoC where possible per
   `feedback_reliability_principles`.

Mechanical slices (WF-FIX-PYPI-SMOKE, DOCS, BUMP, RC1, V, GA)
use the existing test suite as the gate (no new RED test
required).

The budget the RED test asserts is the **revised number, not the
legacy 0.6.0-era number**. Per-slice prompts encode this
explicitly.
