# Roadmap

## Purpose

Durable implementation sequencing, milestone planning, and transition plans.

## Keep here

- phased implementation plans
- migration sequencing notes
- execution roadmaps that another engineer could follow

## Do not keep here

- accepted ADRs
- public release notes
- throwaway planning fragments

## Canonicality

Canonical for current internal execution plans, subject to replacement when a
new plan is adopted.

## Lifecycle

Revise as milestones change; retire stale plans deliberately.
