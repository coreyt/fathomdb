# Roadmap

## Purpose

Future-release planning only: **0.7.0 and later**.

## Keep here

- post-0.6.0 release planning
- deferred work that has been explicitly moved out of the 0.6.0 release line
- target-release roadmap files such as `0.8.0.md`

## Do not keep here

- active 0.6.0 implementation plans
- 0.6.0 packet plans, prompts, or run artifacts
- accepted ADRs
- public release notes
- throwaway planning fragments

## Canonicality

Canonical for **future-release** internal planning only.

For 0.6.0:

- implementation source of truth: `../plans/0.6.0-implementation.md`
- chronological work log: `../progress/0.6.0.md`

## Lifecycle

Revise as milestones change; retire stale plans deliberately.
