# FathomDB 0.2.0 — Current State Assessment

Assessment date: 2026-04-08

---

## Overview

FathomDB is a local datastore for persistent AI agents — a graph-backed database
with vector search (sqlite-vec) and full-text search (FTS5), built on SQLite as
the single durable file. Single-writer, multi-reader architecture with SDKs in
Rust, Python, and TypeScript, plus a Go operator CLI (`fathom-integrity`).

---

## What Changed Since 0.1.1

### New Features

- **TypeScript/Node.js SDK** with full Python parity (napi-rs bindings)
- **Cross-language SDK consistency harness** — proves Python and TypeScript
  produce identical database state across 6 deterministic scenarios
- **Progress callback / feedback support** in TypeScript SDK
- **User-facing documentation** with MkDocs and auto-generated API reference
- **Configurable timeouts** for bridge and recovery operations
- **WriterTimedOut error variant** — distinguishes "write MAY commit" from
  "write WILL NOT commit"
- **InvalidConfig error** — `read_pool_size=0` now rejected cleanly instead
  of panicking
- **SQLITE_OPEN_READONLY** for reader pool connections (defense in depth)

### Breaking Changes

- **TypeScript `toJsonString()` behavior**: Now JSON.stringify's all values
  including strings. Code passing pre-serialized JSON strings must wrap them
  in `new PreserializedJson(jsonString)`.

### Bug Fixes

- Fix TypeScript SDK package exports and native binding discovery
- Fix `describeOperationalCollection` JSON parsing in Go bridge
- Fix string/JSON conflation in write builder
- Add 6 missing fields to Go `bridgeSemanticReport` to match Rust `SemanticReport`
- Include stderr in bridge errors and bound subprocess output buffers
- Tighten vec0 error matching and mark `raw_pragma` as doc-hidden
- Add `callNative` error wrapper for better TypeScript error messages

## Current Gaps

### Blocking for Broader Adoption

| Gap | Detail | Mitigation |
|-----|--------|------------|
| Pre-1.0 version | API may change without deprecation period | Acceptable for first-party use |
| No published packages | Not on crates.io, PyPI, or npm | Source-build only; publish when API stabilizes |
| No CHANGELOG | Breaking changes discoverable only via git | Now addressed in this release |
| No MSRV | Rust edition 2024 requires 1.94+; no backward compat guarantee | Pin in CI when stabilizing |

### Non-Blocking

| Gap | Detail | Priority |
|-----|--------|----------|
| No code coverage metrics | No tarpaulin, coverage.py, or vitest --coverage | Medium |
| Retention not automatic | Operator must schedule `run_operational_retention()` | Low |
| No encryption at rest | Design doc exists, implementation deferred | Low (embedded = process trust) |
| Scale testing | No documented 10K+ node stress tests | Medium |
| `synchronous=NORMAL` | Safe for WAL but not power-loss-proof; `FULL` is safer | Document trade-off |
| mmap 3GB default | May be aggressive on constrained systems | Make configurable |

## Architecture Summary

```
                   +-----------+   +-----------+   +-----------+
                   | Python    |   | TypeScript|   | Go CLI    |
                   | SDK (PyO3)|   | SDK (napi)|   | (bridge)  |
                   +-----+-----+   +-----+-----+   +-----+-----+
                         |               |               |
                   +-----+---------------+---------------+-----+
                   |              Rust Engine                   |
                   |  fathomdb / fathomdb-engine / query / schema |
                   +-----+-------------------------------------+
                         |
                   +-----+-----+
                   |  SQLite   |
                   | WAL + FTS5|
                   | + vec     |
                   +-----------+
```

- **4 Rust crates**: fathomdb (bindings), fathomdb-engine (core), fathomdb-query (AST/compiler), fathomdb-schema (migrations)
- **Single-writer**: Dedicated background thread, channel-based (capacity 256)
- **Multi-reader**: Pool of Mutex-wrapped READONLY connections
- **Inside-out query compiler**: Narrowest indexed set first, resolve through chunks, join to canonical state

## Unsafe Code Audit

6 `unsafe` blocks in production code, all justified:

1. `sqlite3_trace_v2` callback registration (SQLite FFI, Once-guarded)
2. `sqlite3_auto_extension` for vec extension (idempotent)
3. `sqlite3_db_status` for cache telemetry (read-only)
4. Windows DACL/ACL verification (error-checked)
5. `ManuallyDrop::drop()` for channel shutdown (called exactly once)
6. Environment variable mutation (test-only)

## Recommendation

FathomDB is production-grade for its intended use case: embedded local storage
for AI agent systems where the operator controls the upgrade path. The 0.2.0
release marks the addition of the TypeScript SDK, cross-language consistency
testing, and improved error handling.

**Suitable for**: Internal/first-party agent systems, prototypes, early products.
**Not yet suitable for**: External consumers needing long-term API stability,
environments requiring encryption at rest, or deployments without source access.
