# Release

## Purpose

Internal release process, checklists, and shipping coordination notes.

## Keep here

- internal release sequencing
- packaging coordination notes
- publish and verification checklists

## Do not keep here

- public release notes
- dependency manifests
- implementation design documents unrelated to release flow

## Canonicality

## Lifecycle
