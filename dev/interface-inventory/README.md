# Interface Inventory

## Purpose

Audit and inventory artifacts that map internal/public surface ownership and
cross-document consistency.

## Keep here

- inventory runs
- option comparisons
- ownership matrices
- contract-audit findings

## Do not keep here

- final public API reference
- ADRs
- rough scratch work without durable value

## Canonicality

Canonical for inventory outputs, not for final product behavior.

## Lifecycle

Keep while useful for design alignment; regenerate or replace when stale.
