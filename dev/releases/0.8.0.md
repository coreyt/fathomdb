(`dev/design/0.8.0-agent-memory-fit.md`): canonical identity substrate (G0),
governed `read.*` surface (G2/G3), structured search hits (G1), hybrid RRF

## Three deliberate behavior-compatibility events

1. **RRF ranking order — no knob.** `search` returns the unconditional RRF
   fusion of the vector and FTS5 branches (Slice 10, `d28d204`). No `fusion_mode`
   toggle (HITL Q3). Pinned by `pr_g9_rrf_fusion.rs` (determinism, not legacy
   reproducibility).
2. **`SearchHit` reshape.** `Vec<String>` → `Vec<SearchHit{id,kind,body,score,
   branch}>`; `Eq` dropped (carries `f64`). Pinned by `search_result_shape.rs` /
   `pr_g1_search_hits.rs`.
3. **Governed-surface supersession (AC-057a → AC-074).** Operator/recovery seam
   gated behind `--features operator`; default facade is recovery-clean. Slice 27
   (Q5=BIND-RUST). Conformance: `test_surface.py` / `surface.test.ts`.

Plus: additive `WriteReceipt` fields (`row_cursors`, `dangling_edge_endpoints`);
zero-surface FTS5 tokenizer upgrade (`porter unicode61 remove_diacritics 2`).

## Gate restructure (this slice — AC-075 / AC-076)

- **AC-075 (recall verdict, ◆ B-1).** The asserting recall@10 gate is the
  real-embedder `eu7_real_corpus_ac.rs`, repointed to measure the **pre-fusion
  vector stage** (ANN+ bit-KNN K=192 + f32 rerank) vs the exact-f32 vector top-10
  GT — an ANN-quantization-FIDELITY gate in isolation, NOT the RRF-fused
  `search()` output. ≥ 0.90 (measured 0.937, CI 0.913–0.957, σ 0.0116). The
  synthetic `perf_gates::ac_013b_recall_at_10_floor` is demoted to report-only
  (`RECALL_FIDELITY_INFO`); the `AC013B_RECALL_FLOOR = 0.90` constant + its
  `ac_013b_floor_matches_adr` sentinel are retained. A test-only engine seam
  (`set_vector_stage_only_for_test`) exposes the vector stage; production
  `search()` is byte-unchanged (the seam is additive, off by default, never a
  `fusion_mode` knob). Amends `ADR-0.7.0-vector-binary-quant.md` § 2 point 4.
  - **Why the correction (B-1).** The eu7 GT is a vector-only top-10; comparing
    it against the now-hybrid `search()` conflated quantization fidelity with
    intended fusion divergence — the cause of the Slice-40 Phase-A HALT (0.8710
    < 0.90). GA-1 (`dev/plans/runs/GA-1-corpus-ab-20260608T012503Z.md`) proved
    the corpus was byte-identical (no regression, no corpus move); HITL ◆ B-1
    (Option 1) corrected the SUT. Fused-`search()` recall (~0.871) is reported
    as the `EU7_RECALL_FUSED` delta, not gated.
  - Complementary to, NOT a substitute for, the IR/relevance axis (eu8 / IR-1).
- **AC-076 (ac_012 tier).** `ac_012_text_query_latency_on_fts5_path` is tiered:
  binding at `AC012_GATE_N = 10_000` (p50 ≤ 20 / p99 ≤ 150 ms), 100k/1M reported
  (`AC012_TIER_INFO`), mirroring `ac_013`. Tokenizer + migration 011 unchanged
  (Slice 6: porter ≈ unicode61, the cost is O(N) FTS scan).

## Verification posture (GA-2 environment note)

The real bge-small weights are not in the local HF cache and the gitignored
corpus is not materialized in this worktree, so the ~1.5 h real-embedder eu7 run
is the LOCAL/perf-canonical-dispatch verdict (as AC-072/073 already document),
not executed in this worktree. The fused-vs-vector delta is evidenced on-record:
0.8710 fused (prior slice-40 `output.json`) vs 0.937 vector-stage (the 0.7.1
anchor measured pre-unconditional-RRF). The corrected harness structurally
encodes the RED→GREEN demonstration; the `ga2_vector_stage_seam.rs` unit test
verifies the seam returns the pre-fusion vector branch with the FTS-only hit
excluded (passes in this worktree).
