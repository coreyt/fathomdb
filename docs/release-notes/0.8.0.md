# Release notes — 0.8.0

**Status: release-candidate (pre-GA).** The GA tag is HITL-gated; this page is
authored at the 0.8.0 verification + release-readiness slice (Slice 40) and
documents what 0.8.0 ships. Install commands are added when the GA tag fires.

0.8.0 turns FathomDB toward **agent-memory** workloads: a governed read surface,
hybrid (vector ⊕ keyword) retrieval, structured search hits, and a canonical
identity substrate — closing the G0–G12 agent-memory gap ladder
(`dev/design/0.8.0-agent-memory-fit.md`).

## What 0.8.0 delivers

### Hybrid retrieval (G9/G10/G12)

- **`search` is now an unconditional Reciprocal Rank Fusion (RRF) of the vector
  and FTS5 branches.** The two ranked branches are fused by `Σ 1/(RRF_K + rank)`
  keyed on the hit body, with a vector-first tiebreak and dedup-on-body. There
  is **no `fusion_mode` knob** — the fused ranking is the single ranking
  (HITL Q3). See the behavior-compat note below.
- **Filtered search (G10).** `search_filtered` accepts an optional closed
  metadata filter; the unfiltered path's phase-1 SQL is byte-identical to 0.7.2.
- **Recency reweight (G12)** is present behind an off-by-default seam; it is not
  a production toggle in 0.8.0.

### Structured search hits (G1)

- **`search` returns `Vec<SearchHit>`** where `SearchHit { id, kind, body,
  score, branch }`, replacing the prior `Vec<String>` of bodies. See the
  behavior-compat note below.

### Governed read surface (G2/G3)

- **New governed `read.*` verbs:** `read.get` / `read.get_many` (active-only
  point lookup by `logical_id`, request-order, partial), and `read.collection` /
  `read.mutations` (paginated op-store read-back, `ORDER BY id`, mandatory
  limit clamped to a ~1M ceiling, exclusive `after_id` cursor). Reads ride the
  reader-worker DEFERRED-tx path and never take the writer lock.

### Canonical identity substrate (G0)

- **Active-uniqueness is `logical_id` alone** on both `canonical_nodes` and
  `canonical_edges` (correcting a v0.5.x compound-key regression). One active
  row per `logical_id`; superseded versions are retained but never returned by
  the active read path.

### Write receipts

- **Additive `WriteReceipt` fields:** `row_cursors` and
  `dangling_edge_endpoints`. Existing fields are unchanged.

### FTS5 tokenizer upgrade

- **Zero-surface FTS5 tokenizer upgrade** (`porter unicode61 remove_diacritics 2`)
  for morphological recall (G1). It is **latency-neutral**: the Slice 6 engine
  A/B (`dev/plans/runs/0.8.0-slice-6-tokenizer-experiment-20260607T003001Z.md`)
  showed porter ≈ unicode61 within measurement noise; text-query latency is O(N)
  FTS-scan cost, not a tokenizer cost.

## Deliberate behavior-compatibility events

0.8.0 is pre-1.0 and carries no backwards-compatibility shims. Three deliberate
surface changes:

1. **RRF ranking order.** `search` now returns the RRF-fused hybrid ranking, not
   the prior vector-only / union order. There is **no knob** to restore the old
   order; the pinned contract is determinism (repeated identical queries return
   byte-identical order + scores), not legacy reproducibility.
2. **`SearchHit` reshape.** `search` returns `Vec<SearchHit{id,kind,body,score,
   branch}>` instead of `Vec<String>`. `SearchHit` does **not** derive `Eq`
   (it carries an `f64` score). Callers that compared result vectors for
   equality or indexed `String` bodies must migrate to the struct fields.
3. **Governed-surface supersession (AC-057a → AC-074).** The Rust facade's
   operator/recovery seam (12 `Engine` methods + their re-exported report types)
   is gated behind the `--features operator` flag; the **default** `fathomdb`
   facade is recovery-clean and raw-SQL-free at the method level. This is
   feature-gating, not deletion — engine behavior is identical with the feature
   on; `fathomdb-cli` enables it. The Python/TypeScript SDK surface is unchanged.

## Quality gates (release verification)

- **Recall fidelity (AC-075).** The asserting recall verdict is the
  real-embedder `eu7_real_corpus_ac.rs`, measured on the **pre-fusion vector
  stage** (1-bit sign-quant K=192 Hamming + f32 rerank) vs the exact-f32 vector
  top-10 of the same model — an ANN-**quantization-fidelity** gate, recall@10 ≥
  0.90 (measured 0.937 on bge-small over the real corpus, N≈7,667). This is the
  system-health axis (does the quantized index preserve the f32 vector order); it
  is complementary to, not a substitute for, IR/relevance. The fused-`search()`
  recall is reported as a delta, not gated (a hybrid result legitimately diverges
  from a vector-only ground truth).
- **Text-query latency (AC-076).** Tiered: p50 ≤ 20 ms / p99 ≤ 150 ms is the
  **binding** gate at the 10k-row tier; the 100k/1M tiers are tracked post-1.0
  targets (the FTS5 MATCH scan is O(N)).
- **Security battery** (AC-036/037/038/050a/050c) runs per-push on a
  userns-permissive runner so AC-037 (netns-deny-egress) genuinely exercises.
